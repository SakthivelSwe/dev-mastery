## WHY
Deploying an application to production is not the end of your job. You then need to continuously monitor: Is the app healthy? Is the database connection pool exhausted? Is the memory about to run out? How many HTTP requests are being served per second?

**Spring Boot Actuator** is a production-ready sub-system built into Spring Boot that exposes all of this diagnostic information via HTTP endpoints, completely automatically, with zero custom code.

## THEORY
### The Actuator Endpoint System
Actuator endpoints are automatically generated and registered at application startup. Key endpoints include:

- `/actuator/health`: Reports the application's health status (UP/DOWN).
- `/actuator/info`: Shows application info (version, git commit).
- `/actuator/metrics`: Exposes hundreds of JVM and application metrics.
- `/actuator/env`: Shows all environment variables and config properties.
- `/actuator/loggers`: Lets you change log levels at runtime WITHOUT a restart!
- `/actuator/httptrace`: Shows the last 100 HTTP requests/responses.

## CODE

### Level 1 — Beginner (Setup)
```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

```yaml
# application.yml
management:
  endpoints:
    web:
      exposure:
        # Expose all endpoints (RESTRICT this in production!)
        include: "health,info,metrics,loggers"
      base-path: /actuator
  endpoint:
    health:
      # Show full health details (DB connection, disk space, Redis, etc.)
      show-details: always
```

### Level 3 — Intermediate (Custom Health Indicators)
```java
import org.springframework.boot.actuate.health.*;
import org.springframework.stereotype.Component;

// Spring auto-registers this as a health check!
@Component
public class ExternalApiHealthIndicator implements HealthIndicator {

    private final RestTemplate restTemplate;

    @Override
    public Health health() {
        try {
            // Call the external payment API gateway's ping endpoint
            restTemplate.getForObject("https://api.stripe.com/v1/ping", String.class);
            
            return Health.up()
                    .withDetail("service", "Stripe Payment Gateway")
                    .withDetail("status", "Reachable")
                    .build();
        } catch (Exception ex) {
            return Health.down()
                    .withDetail("service", "Stripe Payment Gateway")
                    .withException(ex)
                    .build();
        }
    }
}
```

### Level 4 — Advanced (Custom Metrics with Micrometer)
Spring Boot uses **Micrometer** as its metrics library (analogous to SLF4J for metrics). You can record custom business metrics (like "number of orders placed") that flow directly into Prometheus and are visualized in Grafana!

```java
import io.micrometer.core.instrument.*;
import org.springframework.stereotype.Service;

@Service
public class OrderService {

    private final Counter ordersPlacedCounter;
    private final Timer orderProcessingTimer;

    // Inject the MeterRegistry (the Micrometer hub)
    public OrderService(MeterRegistry registry) {
        // 1. A Counter that increments with every successful order
        this.ordersPlacedCounter = Counter.builder("orders.placed.total")
                .description("Total number of successfully placed orders")
                .tag("region", "us-east-1")
                .register(registry);

        // 2. A Timer to measure order processing latency
        this.orderProcessingTimer = Timer.builder("orders.processing.time")
                .description("Time to process an order end-to-end")
                .register(registry);
    }

    public Order placeOrder(OrderRequest req) {
        // Record the order processing time automatically
        return orderProcessingTimer.record(() -> {
            Order order = /* ... processing logic ... */ new Order();
            ordersPlacedCounter.increment(); // Increment the business metric
            return order;
        });
    }
}
```

## REAL_WORLD
**The Modern Observability Stack (The "Goldilocks" Setup):**
1. **Spring Boot Actuator** — Exposes metrics, health, and info endpoints.
2. **Prometheus** — Scrapes the `/actuator/prometheus` endpoint every 15 seconds and stores the time-series metrics data.
3. **Grafana** — Provides beautiful real-time dashboards built on top of Prometheus data.
4. **Loki** — Aggregates log streams from all service instances.
5. **Tempo** — Collects distributed traces (with Micrometer Tracing / OpenTelemetry).

This stack is completely open-source and universally used in the Enterprise.

## INTERVIEW
**Q: How do you secure Actuator endpoints in production?**
A: You should never expose `/actuator/env` (which shows all environment variables including secrets!) to the public internet. Best practices:
1. Move actuator to a different, internal-only port: `management.server.port=8081`.
2. Restrict endpoint exposure to only `health` and `info` for external access.
3. Secure the admin actuator endpoints with Spring Security, requiring `ROLE_ADMIN`.

**Q: What is the difference between Micrometer and Spring Boot Actuator?**
A: **Micrometer** is the standalone metrics facade library (analogous to SLF4J for logging). It abstracts the metrics API from specific monitoring systems (Prometheus, Datadog, Cloudwatch). **Spring Boot Actuator** is the Spring Boot integration that auto-configures Micrometer, wires it to JVM metrics, and exposes the HTTP endpoints. Actuator depends on Micrometer; they work together.
