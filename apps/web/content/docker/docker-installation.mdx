## WHY
Before you can run any Docker commands, you need Docker installed on your system. Docker Desktop provides a complete development environment for Mac and Windows, while Linux installations use the Docker Engine directly.
Setting up Docker correctly from the start — including permissions and configuration — saves hours of debugging later.

## THEORY
### Docker Desktop vs Docker Engine
- **Docker Desktop** (Mac, Windows, Linux GUI): A complete GUI application that includes Docker Engine, Docker CLI, Docker Compose, and Kubernetes. Easiest setup for developers.
- **Docker Engine** (Linux servers): Just the Docker daemon without a GUI. Used on production servers and CI/CD systems.

## CODE

### Linux (Ubuntu/Debian) Installation
```bash
# 1. Update package index
sudo apt-get update

# 2. Install prerequisites
sudo apt-get install -y ca-certificates curl gnupg

# 3. Add Docker's GPG key
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 4. Add Docker repository
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 5. Install Docker Engine and Compose
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 6. Add your user to the docker group (no sudo needed for docker commands!)
sudo usermod -aG docker $USER
newgrp docker  # Apply group change immediately

# 7. Verify
docker --version
docker compose version
docker run hello-world  # Should work without sudo!
```

### Verify Installation
```bash
# Check Docker is running
docker info

# Check versions
docker --version        # Docker version 25.0.3
docker compose version  # Docker Compose version v2.24.5

# Run the classic verification
docker run hello-world
# This pulls 'hello-world' image and runs it — confirms everything works!
```

### Configure Docker Daemon (daemon.json)
```json
// /etc/docker/daemon.json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  },
  "default-address-pools": [
    {"base": "172.20.0.0/16", "size": 24}
  ]
}
```

## REAL_WORLD
**Setting Up Docker for CI/CD (GitHub Actions)**
GitHub Actions provides Docker pre-installed. But for self-hosted runners, you need the installation above. Always pin Docker versions in production CI to avoid unexpected updates.

## INTERVIEW
**Q: After installing Docker on Linux, why do you need to add your user to the `docker` group? What's the security implication?**
A: The Docker daemon (`dockerd`) runs as root and listens on a Unix socket at `/var/run/docker.sock`. By default, only root can interact with this socket.

Adding your user to the `docker` group grants access to the socket — meaning you can run `docker` commands without `sudo`.

**The security implication:** Docker group membership is essentially equivalent to root access. A user in the docker group can run `docker run -v /:/host alpine chroot /host` to get a root shell on the host system. For this reason, only trusted developers should be in the docker group. In production servers, application processes should use dedicated service accounts with minimal permissions rather than running docker commands directly.
