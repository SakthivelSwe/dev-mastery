## WHY

Navigation components (Tabs, Breadcrumbs, Sidebars, Navbars) dictate how users traverse your application architecture. If a user cannot figure out where they are or how to get back, your app fails immediately.

From an engineering perspective, building Navigation components correctly requires solving three complex problems:
1. **State Synchronization:** The UI must perfectly reflect the browser's URL. If the URL says `?tab=billing`, the "Billing" tab must be active.
2. **Accessibility (ARIA):** Tabs are not just buttons. They require strict `role="tab"`, `role="tablist"`, keyboard arrow navigation, and `aria-selected` attributes.
3. **Responsive Collapse:** Navbars and Sidebars must gracefully downgrade to Hamburger menus or bottom-navigation bars on mobile devices.

---

## THEORY

### The W3C Tablist Pattern
To build accessible Tabs, you cannot just render a list of buttons. You must follow the W3C ARIA Authoring Practices:
- The container gets `role="tablist"`.
- The tabs get `role="tab"` and `aria-selected="true/false"`.
- The content panels get `role="tabpanel"`.
- **Keyboard Navigation:** When a tab is focused, pressing the Right Arrow should focus the next tab, not the page content. 

### Controlled vs Uncontrolled Components
A robust Navigation component should support both modes:
- **Uncontrolled:** The component manages its own internal state (e.g., clicking a tab just switches the active tab internally).
- **Controlled:** The parent passes an `activeTab` prop and an `onChange` handler, allowing the parent to sync the tab state with the URL router (e.g., Next.js `useRouter`).

## IMPLEMENTATION

Here is an advanced, accessible Tab system that supports keyboard navigation and controlled/uncontrolled state.

```tsx
import React, { useState, KeyboardEvent, useRef } from 'react';
import styled from 'styled-components';

// --- STYLED COMPONENTS ---

const TabListContainer = styled.div`
  display: flex;
  border-bottom: 1px solid var(--border-subtle);
  gap: 16px;
`;

const TabButton = styled.button<{ $isActive: boolean }>`
  background: transparent;
  border: none;
  border-bottom: 2px solid ${({ $isActive }) => ($isActive ? 'var(--color-primary-500)' : 'transparent')};
  color: ${({ $isActive }) => ($isActive ? 'var(--color-primary-600)' : 'var(--text-secondary)')};
  padding: 12px 4px;
  font-size: 16px;
  font-weight: 500;
  cursor: pointer;
  transition: all 0.2s ease;

  &:hover {
    color: var(--color-primary-500);
  }

  &:focus-visible {
    outline: 2px solid var(--color-primary-300);
    outline-offset: 2px;
  }
`;

const TabPanelContainer = styled.div`
  padding: 16px 0;
`;

// --- COMPONENT API ---

interface Tab {
  id: string;
  label: string;
  content: React.ReactNode;
}

interface TabsProps {
  tabs: Tab[];
  defaultActiveId?: string;
  activeId?: string; // For controlled mode
  onChange?: (id: string) => void;
}

export const Tabs: React.FC<TabsProps> = ({ tabs, defaultActiveId, activeId, onChange }) => {
  // Support both Controlled and Uncontrolled modes
  const isControlled = activeId !== undefined;
  const [internalId, setInternalId] = useState(defaultActiveId || tabs[0].id);
  
  const currentId = isControlled ? activeId : internalId;
  const tabRefs = useRef<(HTMLButtonElement | null)[]>([]);

  const handleSelect = (id: string) => {
    if (!isControlled) setInternalId(id);
    if (onChange) onChange(id);
  };

  // Implement W3C Keyboard Arrow Navigation
  const handleKeyDown = (e: KeyboardEvent, index: number) => {
    let newIndex = index;
    if (e.key === 'ArrowRight') {
      newIndex = index === tabs.length - 1 ? 0 : index + 1;
    } else if (e.key === 'ArrowLeft') {
      newIndex = index === 0 ? tabs.length - 1 : index - 1;
    }

    if (newIndex !== index) {
      tabRefs.current[newIndex]?.focus();
      handleSelect(tabs[newIndex].id);
    }
  };

  const activeTabContent = tabs.find(t => t.id === currentId)?.content;

  return (
    <div>
      <TabListContainer role="tablist" aria-label="Content Navigation">
        {tabs.map((tab, index) => (
          <TabButton
            key={tab.id}
            ref={el => (tabRefs.current[index] = el)}
            role="tab"
            aria-selected={currentId === tab.id}
            aria-controls={`panel-${tab.id}`}
            id={`tab-${tab.id}`}
            tabIndex={currentId === tab.id ? 0 : -1} // Only active tab is in tab sequence
            $isActive={currentId === tab.id}
            onClick={() => handleSelect(tab.id)}
            onKeyDown={(e) => handleKeyDown(e, index)}
          >
            {tab.label}
          </TabButton>
        ))}
      </TabListContainer>

      <TabPanelContainer
        role="tabpanel"
        id={`panel-${currentId}`}
        aria-labelledby={`tab-${currentId}`}
        tabIndex={0} // Allows panel content to be scrolled via keyboard
      >
        {activeTabContent}
      </TabPanelContainer>
    </div>
  );
};
```

## SYSTEM_DESIGN

### Integrating with Next.js App Router
A true production system synchronizes tabs with the URL so users can share links to specific tabs.

```tsx
'use client';
import { useRouter, useSearchParams, usePathname } from 'next/navigation';
import { Tabs } from '@my-system/components';

export function SettingsPage() {
  const router = useRouter();
  const pathname = usePathname();
  const searchParams = useSearchParams();
  
  const currentTab = searchParams.get('tab') || 'profile';

  const handleTabChange = (newTab: string) => {
    // Update the URL without a full page reload
    const params = new URLSearchParams(searchParams);
    params.set('tab', newTab);
    router.replace(`${pathname}?${params.toString()}`);
  };

  const tabData = [
    { id: 'profile', label: 'Profile', content: <ProfileSettings /> },
    { id: 'billing', label: 'Billing', content: <BillingSettings /> },
  ];

  return (
    <Tabs 
      tabs={tabData} 
      activeId={currentTab} 
      onChange={handleTabChange} 
    />
  );
}
```
