## WHY
If your website looks amateurish, the first culprit is usually spacing; the second is typography. A mature application doesn't just arbitrarily pick `font-size: 23px` for one title and `font-size: 19px` for another. 

A Typography System establishes a strict mathematical scale for font sizes, line heights, and weights. This guarantees vertical rhythm, visual hierarchy, and ensures that text remains readable across mobile and desktop devices without writing hundreds of custom media queries.

## THEORY
### The Core Elements of Web Typography
1. **Font Family (Typeface):** The actual font (e.g., Inter, Roboto, Times New Roman).
2. **Font Size:** The height of the characters. Should be defined on a mathematical scale.
3. **Line Height (Leading):** The vertical space between lines of text. (Crucial for readability).
4. **Font Weight:** The thickness of the characters (e.g., 400 for Normal, 700 for Bold).
5. **Tracking (Letter Spacing):** The horizontal space between characters.

## CODE

### Level 1 — Beginner (The Typographic Scale)
A typographic scale limits the available font sizes to a predefined set, usually based on a mathematical ratio (like the Golden Ratio or a Major Third). Most modern systems use a "Rem-based" scale anchored to the browser default of `16px`.

```css
/* A standard Modular Scale using REMs */
:root {
  /* 1rem = 16px (Browser Default) */
  --text-xs:   0.75rem; /* 12px - Small captions */
  --text-sm:   0.875rem; /* 14px - UI text, labels */
  --text-base: 1rem;     /* 16px - Standard body text */
  --text-lg:   1.125rem; /* 18px - Large body text, sub-headers */
  --text-xl:   1.25rem;  /* 20px - H4 */
  --text-2xl:  1.5rem;   /* 24px - H3 */
  --text-3xl:  1.875rem; /* 30px - H2 */
  --text-4xl:  2.25rem;  /* 36px - H1 */
}

/* Usage */
h1 { font-size: var(--text-4xl); }
p  { font-size: var(--text-base); }
```

### Level 2 — Building Blocks (Line Height & Readability)
Font size is useless without proper Line Height. If lines of text are too close together, the reader's eye gets lost when jumping to the next line. If they are too far apart, the text feels disjointed.

**The Golden Rules of Line Height:**
- **Body text** should have a generous line height (usually `1.5` to `1.75`).
- **Headings** (large text) should have a *tighter* line height (usually `1.1` to `1.25`).

```css
:root {
  --leading-none: 1;
  --leading-tight: 1.25; /* Good for H1, H2 */
  --leading-snug: 1.375;
  --leading-normal: 1.5; /* Good for Body Text */
  --leading-loose: 2;
}

h1 {
  font-size: var(--text-4xl);
  line-height: var(--leading-tight); /* Tighter line height for large text */
}

p {
  font-size: var(--text-base);
  line-height: var(--leading-normal); /* Generous line height for reading */
}
```

### Level 3 — Intermediate (Letter Spacing and Weights)
A subtle but professional touch is adjusting Letter Spacing based on Font Size and Font Weight.

- **Large/Bold Text (Headings):** Needs *tighter* (negative) letter spacing to look cohesive.
- **Small/Uppercase Text (Labels):** Needs *wider* (positive) letter spacing to remain legible.

```css
.heading-display {
  font-size: var(--text-4xl);
  font-weight: 800; /* Extra Bold */
  letter-spacing: -0.025em; /* Pull characters slightly closer together */
}

.caption-uppercase {
  font-size: var(--text-xs);
  font-weight: 600; /* Semi-Bold */
  text-transform: uppercase;
  letter-spacing: 0.05em; /* Push characters apart for legibility! */
}
```

### Level 4 — Advanced (Fluid Typography)
In standard responsive design, you use media queries to change font sizes on mobile:
```css
h1 { font-size: 2rem; }
@media (min-width: 768px) { h1 { font-size: 3rem; } }
```
This is clunky because it "snaps" abruptly at the breakpoint.

**Fluid Typography** uses the CSS `clamp()` function to smoothly scale the font size based on the width of the viewport (`vw`), ensuring perfect proportions on every single device!

```css
h1 {
  /* clamp(MIN_SIZE, PREFERRED_SIZE (Viewport based), MAX_SIZE) */
  font-size: clamp(2rem, 5vw + 1rem, 4rem);
}
/* 
On mobile (320px): It will not go below 2rem.
On desktop (1920px): It will smoothly scale up, but will not exceed 4rem.
In between (Tablet): It calculates fluidly based on viewport width (vw)!
*/
```

## REAL_WORLD
**System Fonts vs Web Fonts**
Loading custom web fonts (like Google Fonts) incurs a performance penalty (Network requests + FOUT: Flash of Unstyled Text). Many massive applications (GitHub, Bootstrap, Medium) rely on the **System Font Stack**. 

```css
/* The ultimate System Font Stack */
body {
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, 
               Helvetica, Arial, sans-serif, "Apple Color Emoji", "Segoe UI Emoji";
}
```
This code tells the browser: "Use San Francisco on Macs, Segoe UI on Windows, and Roboto on Android." It results in zero network requests, instant loading, and the UI feels perfectly native to the user's operating system!

## INTERVIEW
**Q: Why should we use `rem` for font sizes instead of `px`?**
A: `px` is an absolute unit. If you hardcode `font-size: 16px`, you override the user's browser settings. Visually impaired users often change their default browser font size from 16px to 24px. If you use `px`, your site ignores their preference and stays tiny, ruining accessibility. `rem` stands for "Root Em". `1rem` equals the user's base browser font size. If their base is 24px, `1rem` = 24px, and your entire application's typography scales up perfectly and accessible to fit their needs!
