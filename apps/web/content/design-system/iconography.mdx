## WHY

Icons are the visual language of your application. In a Design System, managing icons haphazardly (e.g., mixing PNGs, inline SVGs, and font icons) leads to inconsistent sizing, broken alignments, massive bundle sizes, and poor accessibility.

A robust iconography system ensures that every icon is strictly bound by design tokens (sizing, coloring, stroke width). By treating SVGs as dynamic React components (or sprite maps), you guarantee that they instantly adapt to Dark Mode, user themes, and CSS color inheritance, while keeping the DOM clean and performant.

---

## THEORY

### The Anatomy of an SVG Icon
Scalable Vector Graphics (SVG) are XML-based images. In a design system, SVGs must be stripped of hardcoded colors (like `fill="#FF0000"`) and dimensions (like `width="24px"`). Instead, they should inherit these properties via `fill="currentColor"` and `width="1em"`.

### Delivery Mechanisms
There are two primary ways to deliver icons in a modern web application:
1. **SVG Sprite Maps:** All SVGs are combined into a single `<svg>` element injected into the DOM (or loaded via an external file). Individual icons are referenced using `<use href="#icon-name" />`. This is incredibly performant for caching and reduces DOM node counts.
2. **React/Vue Components:** Each SVG is transformed into a framework component (e.g., using `SVGR`). This allows for dynamic prop injection (e.g., `<Icon name="home" size="lg" color="primary" />`) and tree-shaking, ensuring only the used icons are bundled.

### Sizing and Alignment
Icons must visually align perfectly with text. This is achieved by:
- Wrapping the icon in a container with exactly equal `width` and `height`.
- Applying `display: inline-flex` and `align-items: center` to the parent container.
- Using `vertical-align: middle` or `em` based sizing to scale perfectly with the surrounding font size.

## IMPLEMENTATION

Let's build a highly flexible, tree-shakeable Icon component in React. We will use a central SVG map to ensure all SVGs use `currentColor` for seamless theming.

```tsx
import React from 'react';
import styled from 'styled-components';

// 1. Define strict, token-based sizing
const ICON_SIZES = {
  sm: '16px',
  md: '24px',
  lg: '32px',
  xl: '48px'
} as const;

export type IconName = 'home' | 'settings' | 'user' | 'bell';
export type IconSize = keyof typeof ICON_SIZES;

interface IconProps extends React.SVGProps<SVGSVGElement> {
  name: IconName;
  size?: IconSize;
  color?: string; // Hex, rgb, or CSS variable
}

// 2. The raw SVG paths (stripped of hardcoded fills)
const IconPaths: Record<IconName, React.ReactNode> = {
  home: <path d="M10 20v-6h4v6h5v-8h3L12 3 2 12h3v8z" />,
  settings: <path d="M19.14,12.94c0.04-0.3,0.06-0.61,0.06-0.94c0-0.32-0.02-0.64-0.06-0.94l2.03-1.58..."/>,
  user: <path d="M12 12c2.21 0 4-1.79 4-4s-1.79-4-4-4-4 1.79-4 4 1.79 4 4 4zm0 2c-2.67 0-8 1.34-8 4v2h16v-2c0-2.66-5.33-4-8-4z"/>,
  bell: <path d="M12 22c1.1 0 2-.9 2-2h-4c0 1.1.9 2 2 2zm6-6v-5c0-3.07-1.63-5.64-4.5-6.32V4c0-.83-.67-1.5-1.5-1.5s-1.5.67-1.5 1.5v.68C7.64 5.36 6 7.92 6 11v5l-2 2v1h16v-1l-2-2zm-2 1H8v-6c0-2.48 1.51-4.5 4-4.5s4 2.02 4 4.5v6z"/>
};

// 3. The Styled SVG container
const StyledSvg = styled.svg<{ $size: string; $color: string }>`
  width: ${({ $size }) => $size};
  height: ${({ $size }) => $size};
  fill: ${({ $color }) => $color};
  transition: fill 0.2s ease-in-out;
  flex-shrink: 0; /* Prevents squishing in flex containers */
  vertical-align: middle;
`;

export const Icon = ({ 
  name, 
  size = 'md', 
  color = 'currentColor', 
  ...rest 
}: IconProps) => {
  return (
    <StyledSvg
      viewBox="0 0 24 24"
      $size={ICON_SIZES[size]}
      $color={color}
      aria-hidden="true" // Hides from screen readers unless explicitly given an aria-label
      focusable="false" // Prevents IE/Edge bugs where SVGs receive focus
      {...rest}
    >
      {IconPaths[name]}
    </StyledSvg>
  );
};
```

### Usage in the Application
```tsx
// Adapts automatically to the surrounding text color
<button style={{ color: 'var(--brand-primary)' }}>
  <Icon name="settings" size="sm" /> 
  Settings
</button>
```

## SYSTEM_DESIGN

When building an enterprise Design System, distributing icons as raw React components via NPM can bloat the consumer's bundle if they only use 2 out of 500 icons. 

To solve this, implement an **Icon Build Pipeline**:
1. Designers upload raw `.svg` files to Figma.
2. A GitHub Action runs `Figma API` to download the SVGs.
3. The pipeline runs `SVGO` to compress and strip out hardcoded `fill` properties.
4. The pipeline uses `SVGR` to compile them into individual React components (`HomeIcon.tsx`, `BellIcon.tsx`).
5. The pipeline generates an `index.ts` file that exports them all, allowing consumers to tree-shake imports: `import { HomeIcon } from '@my-system/icons'`.
