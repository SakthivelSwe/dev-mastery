## WHY
In JavaScript, Template Literals (the backticks `` `Hello ${name}` ``) revolutionized string concatenation, making it dynamic and readable.
TypeScript took this exact syntax and applied it to the **Type System**. 
Instead of writing 50 different literal string types manually (`"padding-top" | "padding-bottom" | "padding-left" ...`), you can mathematically generate permutations of strings. This allows you to strongly type massive API endpoints, CSS classes, and Event systems with just a few lines of code.

## THEORY
### The Cartesian Product
When you embed a Union type inside a Template Literal Type, TypeScript doesn't just create one string. It mathematically computes the **Cartesian Product**—combining every possible option in the first union with every possible option in the second union to generate a massive, exhaustive new Union type!

## CODE

### Level 1 — Basic String Generation
Let's generate the CSS spacing classes for Tailwind CSS!

```typescript
type Direction = "top" | "bottom" | "left" | "right";
type Amount = "sm" | "md" | "lg";

// Generate all combinations!
type MarginClass = `margin-${Direction}-${Amount}`;

// TypeScript automatically expands this into a union of 12 strings:
// "margin-top-sm" | "margin-top-md" | "margin-top-lg" | "margin-bottom-sm" ...

let validClass: MarginClass = "margin-left-md"; // Safe!
// let invalid: MarginClass = "margin-center-sm"; // ERROR!
```

### Level 2 — Intrinsic String Manipulation
TypeScript provides global utility types to manipulate the casing of strings *during* the generation process:
- `Uppercase<StringType>`
- `Lowercase<StringType>`
- `Capitalize<StringType>`
- `Uncapitalize<StringType>`

```typescript
type Entity = "user" | "post" | "comment";

// We want to generate Redux action types: "FETCH_USER", "FETCH_POST", etc.
type FetchAction = `FETCH_${Uppercase<Entity>}`;
// Result: "FETCH_USER" | "FETCH_POST" | "FETCH_COMMENT"

// Generate getter methods: "getUser", "getPost", etc.
type GetterMethod = `get${Capitalize<Entity>}`;
// Result: "getUser" | "getPost" | "getComment"
```

### Level 3 — Extracting Substrings with `infer`
You can combine Template Literal Types with Conditional Types and `infer` to literally parse and extract substrings out of a literal type!

```typescript
// A type that extracts the domain from an email address!
type ExtractDomain<Email> = Email extends `${string}@${infer Domain}` ? Domain : never;

type MyDomain = ExtractDomain<"alice@google.com">; 
// Result: "google.com"

type InvalidDomain = ExtractDomain<"justastring">;
// Result: never
```
*How it works:* The condition `${string}@${infer Domain}` asks: "Does this type start with *any string*, followed by an `@` symbol, followed by *some other string*?" If yes, capture the second string into `Domain`!

### Level 4 — Recursive Routing (Typing Express/Next.js)
You can recursively parse a URL string to extract all the path parameters automatically!

```typescript
// Extract parameters like ":userId" or ":postId" from a URL route!
type ExtractRouteParams<Route extends string> = 
    // Does the route contain a parameter? (e.g., "/users/:id/posts")
    Route extends `${string}:${infer Param}/${infer Rest}`
        ? Param | ExtractRouteParams<`/${Rest}`> // Recursive call!
    // Does it end with a parameter? (e.g., "/users/:id")
    : Route extends `${string}:${infer Param}`
        ? Param
    // No parameters found
    : never;

type Route = "/api/v1/users/:userId/posts/:postId";
type Params = ExtractRouteParams<Route>; 
// Result: "userId" | "postId"
```
You can then use this union to force the user to pass an object containing those exact keys when calling the API!

## REAL_WORLD
**Strongly Typed Event Emitters**
If you have an object, you often want to listen to changes on its properties (e.g., `user.on('ageChanged', callback)`).
Using Template Literals and Mapped Types, you can automatically generate the correct event names and callback signatures for ANY object!

```typescript
type User = {
    name: string;
    age: number;
};

// Map over the User object to generate the "on" method signatures
type UserEvents = {
    // For 'name', generate 'nameChanged'. The callback must receive a string!
    [K in keyof User as `${K}Changed`]: (newValue: User[K]) => void;
};

/* Result:
{
    nameChanged: (newValue: string) => void;
    ageChanged: (newValue: number) => void;
}
*/
```

## INTERVIEW
**Q: You want to create a type that forces a string to be an absolute URL (it must start with `http://` or `https://`). How do you define this using Template Literal Types?**
A: **Use the primitive `string` type inside the template.**
```typescript
type AbsoluteUrl = `http://${string}` | `https://${string}`;

const url1: AbsoluteUrl = "https://google.com"; // Valid
const url2: AbsoluteUrl = "http://localhost"; // Valid
// const url3: AbsoluteUrl = "www.google.com"; // ERROR: Type '...' is not assignable.
```
By embedding the generic `string` keyword inside the template, you are telling TypeScript that the prefix must be exact, but the suffix can be absolutely any string. This is incredibly useful for validating API inputs or database schema types at compile time!
