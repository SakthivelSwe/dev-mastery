## WHY
Imagine you have an interface `User` with 20 properties. Now, you need to create a `UpdateUser` interface where all 20 properties are optional, so the API can accept partial updates. 
In a naive world, you would copy-paste the entire interface and manually add `?` to every single property. When you add a 21st property to `User`, you have to remember to add it to `UpdateUser`. This violates DRY (Don't Repeat Yourself).

**Mapped Types** allow you to programmatically loop over an existing type and generate a brand new type. It is essentially an array `map()` function, but for TypeScript Types!

## THEORY
### The Mapping Syntax
Mapped Types rely on the `in` operator combined with `keyof`.
`[Key in keyof Type]` means: "Get all the keys of `Type`, loop through them one by one, assign the current key to the variable `Key`, and do something with it."

## CODE

### Level 1 — The Basic Map (Clone)
Let's create a mapped type that simply loops over a type and creates an exact clone.

```typescript
interface User {
    id: number;
    name: string;
    isActive: boolean;
}

// 1. 'keyof User' extracts a union of literal strings: "id" | "name" | "isActive"
// 2. '[K in keyof T]' loops over those strings. K becomes "id", then "name"...
// 3. 'T[K]' looks up the original type (e.g., User["id"] is number)
type Clone<T> = {
    [K in keyof T]: T[K];
};

type ClonedUser = Clone<User>; 
// Result is exactly the same as the User interface!
```

### Level 2 — Mapping Modifiers (Optional and Readonly)
The power of mapped types is adding or removing modifiers during the loop.
You can add `?` (optional) or `readonly` to every property automatically!

```typescript
// Add 'readonly' to every key, and make every key optional '?'
type ReadonlyAndPartial<T> = {
    readonly [K in keyof T]?: T[K];
};

type SecureUserUpdate = ReadonlyAndPartial<User>;
/* Result:
{
    readonly id?: number;
    readonly name?: string;
    readonly isActive?: boolean;
}
*/
```

### Level 3 — Removing Modifiers
What if the original type had optional properties, and you want to mathematically generate a type where everything is strictly REQUIRED?
You can use the minus sign (`-`) in front of the modifier to strip it away during the map!

```typescript
interface PartialConfig {
    host?: string;
    port?: number;
    readonly secure?: boolean;
}

// -? removes optionality. -readonly removes readonly!
type StrictConfig<T> = {
    -readonly [K in keyof T]-?: T[K];
};

type FinalConfig = StrictConfig<PartialConfig>;
/* Result:
{
    host: string;
    port: number;
    secure: boolean;
}
*/
```

### Level 4 — Key Remapping (The `as` Clause)
Introduced in TypeScript 4.1, you can actually *change the names of the keys* while you are mapping over them using the `as` keyword and Template Literal Types!

Let's generate a type full of "Getter" methods for a User!

```typescript
interface User {
    name: string;
    age: number;
}

// 1. Loop over the keys (name, age)
// 2. AS `get${Capitalize<string & K>}` renames "name" to "getName"
// 3. The value type becomes a function: () => T[K]
type Getters<T> = {
    [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K]
};

type UserGetters = Getters<User>;
/* Result:
{
    getName: () => string;
    getAge: () => number;
}
*/
```
This is pure magic. You just generated an entire API layer's type definition programmatically!

## REAL_WORLD
**Built-in Utility Types**
You rarely have to write `[K in keyof T]?: T[K]` yourself. 
TypeScript provides built-in global Utility Types that use Mapped Types under the hood.
- `Partial<T>` makes everything optional.
- `Required<T>` removes all optional modifiers.
- `Readonly<T>` makes everything readonly.
- `Record<K, V>` maps over a union of keys to create a dictionary type.

Whenever you use `Partial<User>`, you are executing a Mapped Type deep inside the TypeScript compiler.

## INTERVIEW
**Q: You want to create a Mapped Type that takes an interface and completely removes any property that is typed as a `Function`. How do you filter out keys during a Mapped Type loop?**
A: **Remap the Key to `never`.**
If you use the `as` clause to rename a key to `never`, TypeScript completely deletes that property from the resulting object type!
Combining this with Conditional Types allows you to filter objects:
```typescript
type RemoveFunctions<T> = {
    // If the value is a Function, remap the key to 'never' (delete it).
    // Otherwise, keep the key 'K'.
    [K in keyof T as T[K] extends Function ? never : K]: T[K]
};

interface UserObj {
    name: string;
    save: () => void;
}

type CleanUser = RemoveFunctions<UserObj>; // Results in { name: string; }
```

## FEYNMAN CHECK

### Explain Mapped Types Like I'm 10 Years Old
> A mapped type is a TYPE-LEVEL FOR LOOP over the keys of another type. `{ [K in keyof T]: T[K] }` says "for every key K in T, produce a property K with value T[K]" — that's just a copy. Add modifiers: `{ [K in keyof T]?: T[K] }` makes every prop optional (this IS `Partial<T>`). `{ readonly [K in keyof T]: T[K] }` makes them readonly. `{ -readonly [K in keyof T]: T[K] }` REMOVES readonly. `{ [K in keyof T]: T[K] | null }` makes every value nullable. KEY REMAPPING (TS 4.1+): `{ [K in keyof T as `get${Capitalize<K & string>}`]: () => T[K] }` transforms `{ name: string }` into `{ getName: () => string }`. This is how Vue's typed reactivity and Mongoose's getter/setter types are built.

### 5 Deep Conceptual Questions

**Q1: What problem do mapped types solve?**
> **A:** Deriving NEW types from EXISTING types by transforming each property. Without them, you'd write `Partial<User>` by hand: `{ id?: number; name?: string; email?: string; ... }` — tedious and brittle. With mapped types, ONE definition (`Partial<T> = { [K in keyof T]?: T[K] }`) works for every type forever.

**Q2: Mental model for mapped types?**
> **A:** Iteration over keys. `{ [K in Keys]: ValueExpr }` is "for each K in Keys, produce a property K with type ValueExpr." Keys is usually `keyof T`. The value expression can use K (to reference the original) or T[K] (to reference the original's value type). Add `?` and `readonly` (with `-` to remove). Add `as NewKey` to rename.

**Q3: Most dangerous misconception? Show with code.**
> **A:** Forgetting that mapped types preserve modifiers:
> ```ts
> // ❌ Doesn't actually remove optional flag
> type StripOptional<T> = { [K in keyof T]: T[K] };
> type Original = { a?: string; b: number };
> type Result = StripOptional<Original>;   // { a?: string; b: number } — still optional!
>
> // ✅ Use - modifier
> type StripOptional<T> = { [K in keyof T]-?: T[K] };
> // Note: this is essentially Required<T>
> ```

**Q4: How does key remapping interact with template literals?**
> **A:** Combined with template literals, key remapping enables "transform keys by string pattern" — `as `on${Capitalize<K & string>}`` turns `{ click }` into `{ onClick }`. This is how event-handler types in React are typed (`{ onClick, onMouseOver, ... }` derived from a dom-events object). The TS 4.1+ `as` clause runs after the K binding, so you can compute the new key from K's value.

**Q5: FAANG-grade definition?**
> **A:** "Mapped types are type-level iterations over the keys of an object type, with optional/readonly modifiers (addable via no prefix, removable via -), and key remapping via the `as` clause combined with template literal types — collectively enabling Partial, Required, Readonly, Record, and the entire utility-type family, plus framework-level typing for reactive systems, event handlers, and ORM-derived shapes."

#### Step 2 — Mapped Type Magic
```ts
// src/form.ts
type Capitalize_<S extends string> = S extends `${infer F}${infer R}` ? `${Uppercase<F>}${R}` : S;

// For each key K, add { K: T[K]; setK: (v: T[K]) => void }
export type FormState<T> = {
  [K in keyof T]: T[K];
} & {
  [K in keyof T as `set${Capitalize_<K & string>}`]: (value: T[K]) => void;
};

export function createForm<T extends Record<string, unknown>>(initial: T): FormState<T> {
  const state = { ...initial } as Record<string, unknown>;
  const setters: Record<string, (v: unknown) => void> = {};
  for (const key of Object.keys(initial)) {
    const setter = 'set' + key.charAt(0).toUpperCase() + key.slice(1);
    setters[setter] = (v) => { state[key] = v; };
  }
  return { ...state, ...setters } as FormState<T>;
}
```

#### Step 3 — Use The Form Builder
```ts
// src/main.ts
import { createForm } from './form.js';

interface User { name: string; age: number; email: string }

const form = createForm<User>({ name: 'Ana', age: 30, email: 'a@b.io' });

console.log(form.name);             // 'Ana'
form.setName('Anna');               // typed: setName(value: string) => void
form.setAge(31);                    // typed: setAge(value: number) => void
console.log(form.name, form.age);   // 'Anna' 31

// ❌ TS catches misuse
// form.setName(99);                // string expected
// form.setEmail = (v) => {};       // can't reassign — fine, runtime mutability is a different concern
```

#### Step 4 — Error Handling: ReadOnly + Strip Modifiers
```ts
// Make every prop readonly
type DeepReadonly<T> = { readonly [K in keyof T]: T[K] extends object ? DeepReadonly<T[K]> : T[K] };

// Strip readonly + optional
type Mutable<T> = { -readonly [K in keyof T]-?: T[K] };

interface Frozen { readonly name?: string }
type Thawed = Mutable<Frozen>;   // { name: string }
```
