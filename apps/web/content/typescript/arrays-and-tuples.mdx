## WHY
In JavaScript, an array is a chaotic bucket. You can create an array like `["Apple", 42, true, { name: "Bob" }]`. While flexible, this makes iterating over the array terrifying because you have no idea what data type you will encounter on the next iteration.

TypeScript fixes this by enforcing **Homogeneous Arrays** (arrays where every item must be the exact same type). Furthermore, TypeScript introduces a completely new concept that doesn't exist in JavaScript: **Tuples**, which are fixed-length arrays where each specific index has a specific type.

## THEORY
### Arrays vs. Tuples
- **Array (`Type[]`):** A list of items. The list can grow or shrink infinitely, but *all items must be of the specified type*.
- **Tuple (`[TypeA, TypeB]`):** A list with a *fixed, predetermined length*. The type of the item at index `0` might be different from the type at index `1`.

## CODE

### Level 1 — Standard Arrays
There are two ways to annotate an array in TypeScript. Both do exactly the same thing.

```typescript
// Syntax 1: The bracket syntax (Most common)
let names: string[] = ["Alice", "Bob", "Charlie"];

// Syntax 2: The Generic syntax
let scores: Array<number> = [95, 80, 100];

// ERROR: Type 'number' is not assignable to type 'string'.
names.push(42); 

// TypeScript knows that extracting an item from a string[] yields a string!
let first = names[0]; 
console.log(first.toUpperCase()); // Perfectly safe
```

### Level 2 — Multi-Dimensional Arrays
If you are building a grid (like a chessboard or a spreadsheet), you need an array of arrays. You simply stack the brackets.

```typescript
// An array where every item is itself an array of numbers
let matrix: number[][] = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
];

// ERROR: Type 'string' is not assignable to type 'number'.
matrix[0].push("Hello"); 
```

### Level 3 — Tuples (Fixed Length & Mixed Types)
Sometimes you need to return exactly two different things from a function (like a coordinate `[x, y]` or a React `[state, setState]`).
You use a Tuple. You define the types inside brackets.

```typescript
// A Tuple: Exactly 2 items. Index 0 MUST be a string, Index 1 MUST be a number.
let userRecord: [string, number] = ["Alice", 28];

// ERROR: Type 'number' is not assignable to type 'string' at index 0.
userRecord = [28, "Alice"]; 

// ERROR: Source has 3 element(s) but target allows only 2.
userRecord = ["Alice", 28, true]; 

// TypeScript knows the exact type at each index!
console.log(userRecord[0].toUpperCase()); // Safe: It's a string
console.log(userRecord[1].toFixed(2));    // Safe: It's a number
```

### Level 4 — Advanced Tuples (Optional and Rest Elements)
Tuples can have optional elements, or "Rest" elements to capture an unknown number of trailing items.

```typescript
// The '?' makes the Z coordinate optional. Length can be 2 or 3.
let coordinate3D: [number, number, number?] = [10, 20];
coordinate3D = [10, 20, 30];

// A tuple where the first item is a string, and the rest are ANY number of booleans!
let testResults: [string, ...boolean[]] = ["John", true, false, true, true];
```

## REAL_WORLD
**React `useState` is a Tuple!**
If you use React, you use Tuples every single day without realizing it.
```typescript
const [count, setCount] = useState(0);
```
Why does `useState` return an array instead of an object like `{ state: 0, setState: () => {} }`?
Because with an object, you are forced to use the exact keys `state` and `setState`. 
By returning a Tuple `[number, Function]`, you can instantly destructure the array and name the variables whatever you want (`count` and `setCount`). TypeScript's Tuple typing ensures that it mathematically knows that index 0 is the value, and index 1 is the setter function!

## INTERVIEW
**Q: I created a Tuple: `let person: [string, number] = ["Alice", 30];`. I then wrote `person.push(40);`. What happens? Does TypeScript throw an error?**
A: **TypeScript does NOT throw an error (The Tuple Push Loophole).**
This is one of the most famous quirks in TypeScript. A Tuple enforces length and type strictly upon *assignment* (`person = ["Alice", 30, 40]` will fail). 
However, at runtime, a Tuple is just a standard JavaScript array. Because JavaScript arrays have a `.push()` method, TypeScript technically allows you to call `.push()` on a Tuple, breaking the fixed-length contract!
To completely prevent this and lock the Tuple, you must use the `readonly` modifier:
`let person: readonly [string, number] = ["Alice", 30];`
Now, `person.push()` will throw a TypeScript error because the `.push` method doesn't exist on readonly arrays.
