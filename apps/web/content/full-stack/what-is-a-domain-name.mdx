## WHY
If you want to launch a product, you can't just tell users "Go to 142.250.190.69." You need a domain name. Understanding the anatomy of a domain name, how subdomains work, and how the Domain Name System hierarchy is structured is the first step to deploying web applications to the public internet.

## THEORY
### The Anatomy of a Domain
A domain name is read from **right to left**. The most important part of the domain is at the very end.

Take the domain: `blog.devmastery.com`

1. **Top-Level Domain (TLD):** `.com`
   This is the highest level. TLDs are managed by global organizations (like ICANN). Other examples include `.org`, `.io`, `.net`, or country codes like `.co.uk`.
2. **Second-Level Domain (SLD):** `devmastery`
   This is the specific name you purchase from a registrar (like Namecheap or GoDaddy). You own the exclusive right to this name under the `.com` TLD.
3. **Subdomain:** `blog`
   Once you own the SLD, you can create infinite subdomains for free. You could create `api.devmastery.com`, `admin.devmastery.com`, etc.

## CODE

### Level 1 — Beginner (Buying a Domain)
You do not "buy" a domain permanently. You lease it for a period of time (usually 1 to 10 years).
1. Go to a **Domain Registrar** (Google Domains, Namecheap, Route53, GoDaddy).
2. Search for availability. If someone else owns it, you cannot buy it unless they let it expire.
3. Pay the annual fee (usually $10 - $15 for a `.com`).

Once purchased, the registrar updates the global `.com` registry to say: "If anyone asks for devmastery.com, send them to this specific Name Server."

### Level 2 — Building Blocks (Name Servers)
A domain name is useless on its own. It needs to be connected to a server. This is done using **Name Servers**.

When you buy a domain on Namecheap, it defaults to Namecheap's Name Servers. However, if you are hosting your application on AWS, AWS will tell you: "Please point your domain to our Name Servers: `ns1.awsdns.com`". 

You log into Namecheap, update the Name Server settings, and from that point forward, Namecheap steps out of the way. AWS Route53 now has total control over routing traffic for your domain.

### Level 3 — Intermediate (Subdomains and Environments)
In professional software development, you never push code straight to the public website. You test it first. Subdomains are the perfect tool for this.

Because subdomains are free and managed entirely by your own Name Servers, a standard architecture looks like this:
- **Production:** `www.myapp.com` (Points to the live, stable servers)
- **Staging/QA:** `staging.myapp.com` (Points to identical test servers)
- **API:** `api.myapp.com` (Points directly to the backend microservices)
- **Admin:** `internal.myapp.com` (Points to an internal dashboard, often protected by a VPN)

### Level 4 — Advanced (The WWW Subdomain)
Is there a difference between `devmastery.com` (the naked/apex domain) and `www.devmastery.com`?

Historically, yes. In the 1990s, companies had different servers for different tasks. They used `ftp.company.com` for file transfers, `mail.company.com` for email, and `www.company.com` for the World Wide Web server.

Today, `www` is just an arbitrary subdomain. It has no special technical meaning. However, most users still type `www.`, so standard practice is to configure a redirect: If a user types `devmastery.com`, your server instantly returns an HTTP 301 Redirect to `www.devmastery.com` (or vice versa), ensuring there is only one canonical URL for your application.

## REAL_WORLD
**Premium TLDs and Hacks**
Not all domains cost $10. TLDs are managed by different organizations that can charge whatever they want.
- `.io` (British Indian Ocean Territory) became massively popular in tech, often costing $50+/year.
- `.ai` (Anguilla) is currently booming for AI startups, often costing $100+/year.

**Domain Hacks:** Using a country code to form a word. 
- Example: The URL shortener `bit.ly` uses the `.ly` (Libya) extension.
- Example: `notion.so` uses the `.so` (Somalia) extension.

## INTERVIEW
**Q: I built a React app and deployed it to Vercel (myapp.vercel.app). I just bought `myapp.com`. How do I link them?**
A: You log into your Domain Registrar and configure your DNS Records. Specifically, because you are using a PaaS (Vercel), you don't use a static IP Address (an A Record). Instead, you create a **CNAME Record** for the `www` subdomain that points to `myapp.vercel.app`. Vercel's servers will intercept the traffic, look at the host header, recognize `myapp.com`, and serve your specific React application.
