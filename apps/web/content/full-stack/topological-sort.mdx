# Topological Sort

Topological Sort is a linear ordering of vertices in a Directed Acyclic Graph (DAG) such that for every directed edge $u \to v$, vertex $u$ comes before $v$ in the ordering.

## 1. Core Mechanics & Theoretical Foundation

### The DAG Requirement
A topological sort is only possible if and only if the graph is a **Directed Acyclic Graph (DAG)**. If a cycle exists (e.g., $A \to B \to C \to A$), no topological ordering can exist because $A$ must precede $B$, which must precede $C$, which must precede $A$, creating an impossible ordering.

### Kahn's Algorithm (BFS Approach)
Kahn's algorithm approaches the problem by managing vertex **in-degrees** (the number of incoming edges pointing to a node):
1. Compute the in-degree of every vertex.
2. Initialize a queue with all vertices that have an in-degree of `0` (no dependencies).
3. While the queue is not empty:
   * Dequeue a vertex $u$ and append it to the topological order.
   * For each outgoing edge $u \to v$, decrement the in-degree of $v$.
   * If $v$'s in-degree drops to `0`, enqueue $v$.

### DFS Approach (Reverse Post-Order)
The DFS approach relies on finish times:
1. Run DFS on the graph.
2. When a node's exploration is complete (it is about to be colored `BLACK`), push it onto a stack.
3. Once all nodes are visited, pop elements from the stack to obtain the topological sort.
* *Why it works*: For any edge $u \to v$, the finish time of $u$ must be greater than the finish time of $v$ ($f[u] > f[v]$). Thus, reversing this order places $u$ before $v$.

### Cycle Detection
* **Kahn's**: If the size of the final sorted list is less than $V$, a cycle exists (nodes in the cycle never have their in-degrees reduced to `0`).
* **DFS**: If we encounter a node currently in the recursion stack (a `GRAY` node), a cycle is detected, and no topological sort is possible.

## 2. Mathematical Proofs & Complexity Derivations

### Time Complexity: $O(V + E)$
* **Kahn's**:
  1. Calculating in-degrees requires traversing all edges: $O(V + E)$.
  2. Each vertex is placed in the queue exactly once: $O(V)$.
  3. Every edge is processed and decremented exactly once: $O(E)$.
  4. Total Time: $O(V + E)$.

### Space Complexity: $O(V)$
* Auxiliary space is needed for the in-degree array ($O(V)$) and the queue/stack ($O(V)$). Thus, the space complexity is bounded by $O(V)$.

## 5. Implementation (Java & Python)

### Java 21 Implementation
```java
package dsa.topological;

import java.util.*;

public final class TopologicalSort {

    public static List<Integer> sortKahn(int numVertices, Map<Integer, List<Integer>> adjList) {
        Objects.requireNonNull(adjList, "Adjacency list cannot be null");
        
        int[] inDegree = new int[numVertices];
        for (var entry : adjList.entrySet()) {
            for (int neighbor : entry.getValue()) {
                inDegree[neighbor]++;
            }
        }

        Queue<Integer> queue = new ArrayDeque<>();
        for (int i = 0; i < numVertices; i++) {
            if (inDegree[i] == 0) {
                queue.add(i);
            }
        }

        List<Integer> order = new ArrayList<>();
        while (!queue.isEmpty()) {
            int u = queue.poll();
            order.add(u);

            for (int v : adjList.getOrDefault(u, Collections.emptyList())) {
                inDegree[v]--;
                if (inDegree[v] == 0) {
                    queue.add(v);
                }
            }
        }

        if (order.size() != numVertices) {
            throw new IllegalArgumentException("Graph contains a cycle; topological sort impossible.");
        }

        return order;
    }
}
```

## 6. Interview Deep Dive

### Probing Questions & Expert Answers

1. **How does a build system like Bazel or Gradle use Topological Sort?**
   * *Answer*: Build systems represent tasks as vertices and dependencies as directed edges. A topological sort determines a valid compilation order. When running builds in parallel, nodes with an in-degree of `0` can be processed simultaneously across different threads or worker machines.

2. **Can there be more than one valid topological sort for a given DAG?**
   * *Answer*: Yes. A DAG has a unique topological sort if and only if it contains a **Hamiltonian path** (a path visiting every vertex exactly once). If no such path exists, multiple valid topological orderings can be found.
