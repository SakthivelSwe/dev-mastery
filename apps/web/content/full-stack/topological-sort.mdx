# Topological Sort

Topological Sort is a linear ordering of vertices in a Directed Acyclic Graph (DAG) such that for every directed edge $u \to v$, vertex $u$ comes before $v$ in the ordering.

## 1. Core Mechanics & Theoretical Foundation

### The DAG Requirement
A topological sort is only possible if and only if the graph is a **Directed Acyclic Graph (DAG)**. If a cycle exists (e.g., $A \to B \to C \to A$), no topological ordering can exist because $A$ must precede $B$, which must precede $C$, which must precede $A$, creating an impossible ordering.

### Kahn's Algorithm (BFS Approach)
Kahn's algorithm approaches the problem by managing vertex **in-degrees** (the number of incoming edges pointing to a node):
1. Compute the in-degree of every vertex.
2. Initialize a queue with all vertices that have an in-degree of `0` (no dependencies).
3. While the queue is not empty:
   * Dequeue a vertex $u$ and append it to the topological order.
   * For each outgoing edge $u \to v$, decrement the in-degree of $v$.
   * If $v$'s in-degree drops to `0`, enqueue $v$.

### DFS Approach (Reverse Post-Order)
The DFS approach relies on finish times:
1. Run DFS on the graph.
2. When a node's exploration is complete (it is about to be colored `BLACK`), push it onto a stack.
3. Once all nodes are visited, pop elements from the stack to obtain the topological sort.
* *Why it works*: For any edge $u \to v$, the finish time of $u$ must be greater than the finish time of $v$ ($f[u] > f[v]$). Thus, reversing this order places $u$ before $v$.

### Cycle Detection
* **Kahn's**: If the size of the final sorted list is less than $V$, a cycle exists (nodes in the cycle never have their in-degrees reduced to `0`).
* **DFS**: If we encounter a node currently in the recursion stack (a `GRAY` node), a cycle is detected, and no topological sort is possible.

## 2. Mathematical Proofs & Complexity Derivations

### Time Complexity: $O(V + E)$
* **Kahn's**:
  1. Calculating in-degrees requires traversing all edges: $O(V + E)$.
  2. Each vertex is placed in the queue exactly once: $O(V)$.
  3. Every edge is processed and decremented exactly once: $O(E)$.
  4. Total Time: $O(V + E)$.

### Space Complexity: $O(V)$
* Auxiliary space is needed for the in-degree array ($O(V)$) and the queue/stack ($O(V)$). Thus, the space complexity is bounded by $O(V)$.

## 5. Implementation (Java & Python)

### Java 21 Implementation
```java
package dsa.topological;

import java.util.*;

public final class TopologicalSort {

    public static List<Integer> sortKahn(int numVertices, Map<Integer, List<Integer>> adjList) {
        Objects.requireNonNull(adjList, "Adjacency list cannot be null");
        
        int[] inDegree = new int[numVertices];
        for (var entry : adjList.entrySet()) {
            for (int neighbor : entry.getValue()) {
                inDegree[neighbor]++;
            }
        }

        Queue<Integer> queue = new ArrayDeque<>();
        for (int i = 0; i < numVertices; i++) {
            if (inDegree[i] == 0) {
                queue.add(i);
            }
        }

        List<Integer> order = new ArrayList<>();
        while (!queue.isEmpty()) {
            int u = queue.poll();
            order.add(u);

            for (int v : adjList.getOrDefault(u, Collections.emptyList())) {
                inDegree[v]--;
                if (inDegree[v] == 0) {
                    queue.add(v);
                }
            }
        }

        if (order.size() != numVertices) {
            throw new IllegalArgumentException("Graph contains a cycle; topological sort impossible.");
        }

        return order;
    }
}
```

## 6. Interview Deep Dive

### Probing Questions & Expert Answers

1. **How does a build system like Bazel or Gradle use Topological Sort?**
   * *Answer*: Build systems represent tasks as vertices and dependencies as directed edges. A topological sort determines a valid compilation order. When running builds in parallel, nodes with an in-degree of `0` can be processed simultaneously across different threads or worker machines.

2. **Can there be more than one valid topological sort for a given DAG?**
   * *Answer*: Yes. A DAG has a unique topological sort if and only if it contains a **Hamiltonian path** (a path visiting every vertex exactly once). If no such path exists, multiple valid topological orderings can be found.

## WHY
Imagine you are building a university scheduling app. A student wants to take "Machine Learning". But to take Machine Learning, they must first pass "Calculus 2". To take Calculus 2, they must pass "Calculus 1". 
How do you write an algorithm that takes 500 classes with complex prerequisites and outputs a perfectly valid chronological order for the student to take them?
This is **Topological Sorting**. It takes a Directed Acyclic Graph (DAG) and flattens it into a linear 1D Array, guaranteeing that if Node A points to Node B, Node A mathematically appears *before* Node B in the final array!

## THEORY
### Directed Acyclic Graph (DAG)
Topological Sort ONLY works on a DAG.
- **Directed:** The edges have arrows (Prerequisite A -> Class B).
- **Acyclic:** There are no cycles! If A requires B, and B requires A, it is an infinite paradox. A Topological Sort is mathematically impossible!

### Kahn's Algorithm (In-Degree Concept)
The most elegant way to solve this is Kahn's Algorithm. It focuses on the concept of **In-Degree** (how many arrows are pointing *at* you).
1. Calculate the In-Degree of every node.
2. If a node has an In-Degree of `0`, it means it has NO prerequisites! It is safe to process immediately. Put it in a Queue.
3. While the Queue is not empty:
   - Pop a node. Add it to our final sorted array.
   - Look at all the neighbors this node points to, and reduce their In-Degree by 1 (since we just completed their prerequisite!).
   - If a neighbor's In-Degree hits `0`, it is now safe! Push it into the Queue!

## CODE

### Level 1 — Kahn's Algorithm (BFS Approach)
Let's implement Kahn's Algorithm to solve the classic "Course Schedule" problem. We are given `numCourses` and a 2D array of `prerequisites` where `[a, b]` means "to take course A, you must first take course B". (Therefore B -> A).

```java
import java.util.*;

public class TopologicalSort {
    
    public int[] findOrder(int numCourses, int[][] prerequisites) {
        // 1. Build the Adjacency List and the In-Degree array!
        List<List<Integer>> adjList = new ArrayList<>();
        int[] inDegree = new int[numCourses];

        for (int i = 0; i < numCourses; i++) {
            adjList.add(new ArrayList<>());
        }

        // Populate the graph
        for (int[] pre : prerequisites) {
            int course = pre[0];
            int prereq = pre[1];
            adjList.get(prereq).add(course); // Arrow goes Prereq -> Course
            inDegree[course]++;              // The course has an incoming arrow!
        }

        // 2. Find all courses with NO prerequisites (In-Degree == 0)
        Queue<Integer> queue = new LinkedList<>();
        for (int i = 0; i < numCourses; i++) {
            if (inDegree[i] == 0) {
                queue.add(i);
            }
        }

        // 3. Process the Queue
        int[] topologicalOrder = new int[numCourses];
        int index = 0;

        while (!queue.isEmpty()) {
            int current = queue.poll();
            topologicalOrder[index++] = current; // Add to final result

            // For every course that depended on 'current'...
            for (int neighbor : adjList.get(current)) {
                // Reduce its prerequisite count by 1!
                inDegree[neighbor]--;
                
                // If it has no more prerequisites, it's safe to take!
                if (inDegree[neighbor] == 0) {
                    queue.add(neighbor);
                }
            }
        }

        // 4. Cycle Detection Check!
        // If we didn't process all courses, there MUST be a cycle (a paradox)!
        if (index == numCourses) {
            return topologicalOrder;
        } else {
            return new int[0]; // Impossible to complete!
        }
    }
}
```

### Level 2 — DFS Approach
You can also perform a Topological Sort using standard DFS!
Instead of tracking In-Degrees, you run a massive DFS. The genius trick is: **When a DFS reaches a dead end, that node has no outgoing edges.** Therefore, it must be the *last* item in the topological order!
When a DFS finishes exploring a node completely, we push that node onto a Stack. When the entire graph is explored, we pop everything off the Stack, and it is in perfect topological order!

```java
public int[] findOrderDFS(int numCourses, int[][] prerequisites) {
    List<List<Integer>> adjList = new ArrayList<>();
    for (int i = 0; i < numCourses; i++) adjList.add(new ArrayList<>());
    for (int[] pre : prerequisites) adjList.get(pre[1]).add(pre[0]);

    int[] state = new int[numCourses]; // 0=Unvisited, 1=Visiting, 2=Visited
    Stack<Integer> stack = new Stack<>();

    // Run DFS on every unvisited node
    for (int i = 0; i < numCourses; i++) {
        if (state[i] == 0) {
            if (dfs(i, adjList, state, stack)) return new int[0]; // Cycle detected!
        }
    }

    // Unload the stack to get the final chronological order
    int[] order = new int[numCourses];
    for (int i = 0; i < numCourses; i++) {
        order[i] = stack.pop();
    }
    return order;
}

private boolean dfs(int current, List<List<Integer>> adj, int[] state, Stack<Integer> stack) {
    state[current] = 1; // VISITING

    for (int neighbor : adj.get(current)) {
        if (state[neighbor] == 1) return true; // Cycle detected!
        if (state[neighbor] == 0) {
            if (dfs(neighbor, adj, state, stack)) return true;
        }
    }

    state[current] = 2; // VISITED
    // THE MAGIC: We finished this node. Push it to the Stack!
    stack.push(current); 
    return false;
}
```

## INTERVIEW
**Q: Is there only one valid Topological Sort for a given DAG?**
A: **No, there can be many valid topological sorts!**
Imagine you need to take "Math 101" and "English 101" in your first semester. Neither course has prerequisites. They both have an In-Degree of `0`.
When Kahn's Algorithm runs, it puts both Math and English into the Queue. If you pop Math first, your final array starts with `[Math, English...]`. If you pop English first, your array starts with `[English, Math...]`.
Both arrays are mathematically valid Topological Sorts because neither violated a dependency rule! Topological Sort guarantees that dependencies are respected, but it does NOT guarantee a strict order among independent parallel tasks.

### 5 Deep Conceptual Questions

**Q1: What problem does Topological Sort fundamentally solve?**
> **A:** It solves the problem of ordering tasks with dependencies — given a set of nodes where some must come before others (a DAG), produce a linear ordering that respects every directed edge. The key constraint is the graph must be acyclic; if there is a cycle (A depends on B, B depends on A), no valid ordering exists and the algorithm detects this. Kahn's Algorithm detects cycles when the output ordering contains fewer nodes than the total — the remainder are stuck in a cycle with non-zero in-degree.

**Q2: What is the ONE mental model that makes Topological Sort click?**
> **A:** Think of in-degree as a "waiting count". Every node is waiting for its predecessors to finish. When a node's waiting count reaches zero, it is free to execute. Topological Sort is the process of repeatedly picking any free node, executing it, and decrementing the waiting count of all its dependents. This model makes both Kahn's Algorithm (BFS-based) and DFS-based topological sort immediately intuitive.

**Q3: What is the most dangerous misconception? Show it with code.**
> **A:** The misconception is that topological sort works on any directed graph. It only works on DAGs — graphs with no cycles.
> ```java
> // ❌ Misconception: calling topo sort on a graph with a cycle
> // A → B → C → A (cycle!) — Kahn's returns partial order, misses A, B, C
> int[] inDegree = {0, 1, 1, 1}; // node 0 has no predecessors
> // After processing node 0, nodes 1,2,3 never reach in-degree 0
> // Output has fewer than n nodes → CYCLE DETECTED
>
> // ✅ Always check: if (result.size() != numNodes) throw new CycleException();
> if (result.size() != numCourses) return new int[]{}; // LeetCode Course Schedule II pattern
> ```

**Q4: How does Topological Sort relate to DFS at the runtime level?**
> **A:** The DFS-based topological sort uses the post-order finish time of DFS. When DFS finishes visiting a node (all descendants processed), push it onto a stack. The stack in reverse gives topological order — because a node finishes AFTER all its dependents finish, so it belongs BEFORE them in the ordering. The key insight: a node's finish time in DFS is inversely proportional to its topological position. This is why Kosaraju's algorithm for SCCs runs two DFS passes using this exact property.

**Q5: One-sentence senior definition.**
> **A:** "Topological Sort is a linear ordering of vertices in a DAG such that for every directed edge u→v, vertex u appears before v — computed in O(V+E) via either BFS in-degree processing (Kahn's) or DFS post-order reversal — which is why it is the foundation of build dependency resolution, course scheduling, and package manager installation ordering."

#### Step 3 — Core Implementation (Kahn's Algorithm)
```javascript
// index.js
const config = require('./tasks.json');

function topologicalSort(tasks) {
  const inDegree = {};
  const adjList  = {};

  // Initialize
  for (const task of Object.keys(tasks)) {
    inDegree[task] = 0;
    adjList[task]  = [];
  }

  // Build adjacency list and compute in-degrees
  for (const [task, { deps }] of Object.entries(tasks)) {
    for (const dep of deps) {
      adjList[dep].push(task);   // dep must run before task
      inDegree[task]++;
    }
  }

  // Kahn's BFS
  const queue  = Object.keys(tasks).filter(t => inDegree[t] === 0);
  const result = [];

  while (queue.length > 0) {
    const current = queue.shift();
    result.push(current);

    for (const neighbor of adjList[current]) {
      inDegree[neighbor]--;
      if (inDegree[neighbor] === 0) queue.push(neighbor);
    }
  }

  // Cycle detection
  if (result.length !== Object.keys(tasks).length) {
    const cycled = Object.keys(tasks).filter(t => !result.includes(t));
    throw new Error('Cycle detected in tasks: ' + cycled.join(', '));
  }

  return result;
}

async function runTasks(tasks) {
  const order = topologicalSort(tasks);
  console.log('Execution order:', order.join(' → '));
  for (const task of order) {
    console.log('\nRunning:', task);
    // In real implementation: execSync(tasks[task].cmd)
    console.log(' ', tasks[task].cmd);
  }
}

runTasks(config.tasks).catch(console.error);
```
