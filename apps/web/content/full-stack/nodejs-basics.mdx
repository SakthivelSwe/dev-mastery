## WHY
For the first 15 years of its existence, JavaScript was trapped inside the web browser. It could manipulate HTML and CSS, but it could not read files on your hard drive, it could not connect to a PostgreSQL database, and it could not listen to network ports. It was strictly a "Frontend" language.
If you wanted to build a Backend server, you had to learn Java, PHP, or Python.
In 2009, Ryan Dahl created **Node.js**. He took the V8 JavaScript Engine out of Google Chrome, bolted it onto a C++ foundation, and allowed JavaScript to run natively on computer servers. 
Suddenly, developers could use the *exact same language* (JavaScript) on both the Frontend and the Backend, sparking the "Full Stack" revolution!

## THEORY
### Single-Threaded Non-Blocking I/O
Java handles 1,000 users by spinning up 1,000 separate "Threads" (mini-processes) in the CPU. This consumes massive amounts of RAM.
Node.js is radically different. It uses a **Single Thread**. There is only one chef in the kitchen!
How does one chef cook for 10,000 customers? **Non-Blocking Asynchronous I/O**.
When User 1 asks for a 5-second database query, the Chef does NOT stand there staring at the oven waiting for it to finish. The Chef throws the query into the oven, registers a "Callback", and immediately pivots to serve User 2, User 3, and User 4! When the oven dings 5 seconds later, the **Event Loop** notifies the Chef, who picks up the data and hands it to User 1. This architecture makes Node.js unbelievably fast for data-heavy network applications!

## CODE

### 1. Running JS Outside the Browser
Create a file named `app.js` on your laptop. You can now execute it using your terminal, completely bypassing the browser!

```javascript
// app.js
console.log("Hello from the Node.js Server!");

// We have access to the underlying OS!
const os = require('os');
console.log(`You are running ${os.platform()} with ${os.cpus().length} CPU cores!`);
```
**Terminal:**
```bash
node app.js
# Output: Hello from the Node.js Server!
# Output: You are running darwin with 8 CPU cores!
```

### 2. Building a Basic API Server (Express.js)
Node.js can listen to internet traffic. While you can use raw Node, 99% of developers use a framework called **Express.js** to handle the complex HTTP routing.

```bash
# Initialize a project and install Express!
npm init -y
npm install express
```

```javascript
// server.js
const express = require('express'); // Import the framework
const app = express();              // Create the application
const PORT = 3000;

// Enable the server to automatically parse JSON payloads in POST requests!
app.use(express.json());

// 1. A GET Route (Read Data)
app.get('/api/users', (req, res) => {
    // We send JSON data back to whoever called this URL!
    res.json({ message: "Here is the list of users!", users: ["Alice", "Bob"] });
});

// 2. A POST Route (Create Data)
app.post('/api/users', (req, res) => {
    const newUserData = req.body; // Extract the JSON sent by the frontend!
    console.log("Saving user to database:", newUserData);
    
    // Reply with a 201 Created status!
    res.status(201).json({ message: "User successfully created!" });
});

// Start the engine! The server will run infinitely, listening for requests.
app.listen(PORT, () => {
    console.log(`Server is LIVE on http://localhost:${PORT}`);
});
```

### 3. File System Operations
Node allows you to physically read and write files on the server's hard drive!
```javascript
const fs = require('fs');

// Asynchronously write text to a file
fs.writeFile('log.txt', 'User logged in at 5:00PM', (err) => {
    if (err) throw err;
    console.log('The file has been saved!');
});
```

## REAL_WORLD
**When NOT to use Node.js**
Node.js is arguably the best technology in the world for "I/O Bound" tasks (chat applications, REST APIs, streaming Netflix video). It handles millions of network connections effortlessly because the Event Loop just delegates tasks to the background and moves on.
However, Node.js is terrible at **CPU Bound** tasks. 
If you need to compress a 4K video, encrypt a massive hard drive, or train an AI model, the single-threaded CPU is forced to do complex, heavy math for 30 straight seconds. Because the Chef is stuck doing math, he physically cannot process any incoming network requests! Every single user on the server will be frozen, waiting for the math to finish. (This is called "Blocking the Event Loop"). For CPU-heavy tasks, you must use Multi-Threaded languages like Java, Go, or Rust!

## INTERVIEW
**Q: An interviewer asks: "In Node.js, `require()` is synchronous, but `fs.readFile()` is asynchronous. What happens if we write a massive 10,000-line `while(true)` infinite loop right in the middle of our Express server code?"**
A: **The entire server instantly dies for every single user.**
Because Node.js is fundamentally single-threaded, all JavaScript code executes sequentially on that single thread. 
If you accidentally write an infinite `while` loop (or perform an incredibly heavy synchronous calculation), the Event Loop becomes physically trapped. It will never reach the next line of code. It will never return to the event queue to process incoming HTTP requests. 
A single user triggering that infinite loop will cause the entire Node.js server to hang indefinitely, blocking thousands of other innocent users from accessing the API until the server process is forcefully restarted by the system administrator!
