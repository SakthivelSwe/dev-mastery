# CSS Selectors — Combinators, Pseudo-classes, :has() | Level: 1

## 1. Advanced Selector Specifications
Selectors identify the DOM nodes to which style rules apply. Modern CSS engines parse selectors from right to left (the key selector is evaluated first, then the engine climbs up the DOM tree to verify matches).

```
   [Parent] ──(Descendant: " ")──> [Child]
   [Parent] ──(Child: ">")───────> [Direct Child]
   [Node A] ──(Adjacent: "+")────> [Immediate Next Sibling]
   [Node A] ──(General: "~")─────> [Any Next Sibling]
```

### Combinator Mechanics
* **Descendant Selector (`A B`)**: Matches any element `B` that is a descendant of `A`, regardless of depth.
* **Child Selector (`A > B`)**: Matches only elements `B` that are direct children of `A`.
* **Adjacent Sibling Selector (`A + B`)**: Matches element `B` only if it is the immediate sibling following `A`.
* **General Sibling Selector (`A ~ B`)**: Matches all elements `B` that follow `A` as siblings, regardless of whether they are immediately adjacent.

## 2. Attribute Matchers & Pseudo-classes
Attribute selectors match elements based on the presence or value of attributes.

* `[attr]`: Presence of attribute.
* `[attr="val"]`: Exact equality.
* `[attr^="start"]`: Starts with prefix.
* `[attr$="end"]`: Ends with suffix.
* `[attr*="contains"]`: Substring match anywhere within the value.

### Pseudo-classes vs Pseudo-elements
* **Pseudo-classes** (single colon `:`): Represent state or structural positions of elements (e.g., `:hover`, `:focus-visible`, `:nth-child(2n)`).
* **Pseudo-elements** (double colon `::`): Generate virtual elements that are not part of the standard DOM tree (e.g., `::before`, `::after`, `::placeholder`, `::selection`). The `content` property is required for `::before` and `::after` to render.

## 3. Modern Functional Selectors: `:is()`, `:where()`, and `:has()`

### Specificity Differences: `:is()` vs `:where()`
Both selectors allow you to write compact selector groups, but they handle specificity differently:
* **`:is()`**: The specificity of the entire `:is()` functional selector is determined by the **most specific** selector in its argument list.
* **`:where()`**: The specificity of `:where()` is **always zero**, regardless of the arguments passed to it.

```css
/* Specificity: (0, 1, 0, 0) because #hero is an ID selector */
:is(.card, #hero) p {
  color: oklch(0.2 0.01 240);
}

/* Specificity: (0, 0, 0, 1) because :where() has zero specificity and p is a type selector */
:where(.card, #hero) p {
  color: oklch(0.2 0.01 240);
}
```

### The Parent Selector: `:has()`
The `:has()` relational pseudo-class allows styling an element based on its descendants or subsequent siblings. This acts as a parent selector.

```css
/* Style a form wrapper only if it contains a focused input */
.form-group:has(input:focus) {
  border-color: oklch(0.6 0.25 140);
  background-color: oklch(0.98 0.01 240);
}

/* Style an H2 element only if it is immediately followed by a paragraph */
h2:has(+ p) {
  margin-bottom: 0.5rem;
}
```

## 4. Implementation Sandbox & Live Code

```html
<section class="card-list">
  <div class="card" data-status="featured">
    <h3 class="card-title">Database Scalability</h3>
    <p>Sharding and replication strategies for modern architectures.</p>
    <div class="input-group">
      <input type="text" class="tag-input" placeholder="Add tag..." />
    </div>
  </div>
  <div class="card" data-status="archived">
    <h3 class="card-title">Legacy Systems</h3>
    <p>Maintenance windows and deprecation timelines.</p>
    <div class="input-group">
      <input type="text" class="tag-input" placeholder="Disabled..." disabled />
    </div>
  </div>
</section>
```

```css
/* Base styles using logical properties */
.card-list {
  display: grid;
  gap: 1.5rem;
  padding-block: 2rem;
}

/* Match attributes starting with "feat" */
.card[data-status^="feat"] {
  border-inline-start: 4px solid oklch(0.6 0.25 140);
}

/* Match attributes with exact value "archived" */
.card[data-status="archived"] {
  opacity: 0.6;
}

/* Modern parent selector style block */
/* If the card contains an input that is focused, style the card container */
.card:has(.tag-input:focus) {
  box-shadow: 0 10px 15px -3px rgba(0, 0, 0, 0.1);
  transform: translateY(-2px);
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}

/* Pseudo-element styling */
.card-title::before {
  content: "⚡ ";
  color: oklch(0.6 0.25 140);
}

/* Forgiving selector list with zero specificity fallback */
:where(.card-list) :is(h3, p) {
  margin-block-end: 0.75rem;
}
```

## 5. DevTools Diagnostics & Profiling

1. Open DevTools and inspect a `.card` element.
2. In the **Styles** pane, verify if the `:has()` or `:is()` selector is successfully matching. 
3. If an invalid selector is part of a standard selector list (e.g., `h1, h2, :invalid-pseudo`), the entire rule is discarded. However, if wrapped inside `:is()` or `:where()`, the selector list is **forgiving**—the browser ignores the invalid selector and still applies styles to the valid ones.
4. To debug dynamic pseudo-classes like `:focus` or `:hover`, click the **`:hov`** button in the Styles pane to force element states.

## 6. Performance & Compatibility

### Selector Performance
The `:has()` selector was historically avoided due to performance concerns in rendering engines. Modern browsers utilize invalidation filters to fast-path matching. This ensures that updates to the DOM only trigger style recalculations on elements that could match. 
* Avoid deep, complex combinations inside `:has()`, such as `*:has(div span a)`. This can degrade scrolling performance and increase style recalculation times.

### Browser Compatibility
* **`:is()` and `:where()`**: Supported in all modern browsers (since early 2021). ~97% global support.
* **`:has()`**: Supported in all evergreen browsers (Chrome 105+, Safari 15.4+, Firefox 121+). Global support is ~93%.
* **caniuse reference**: [caniuse.com/#feat=css-has](https://caniuse.com/#feat=css-has)
