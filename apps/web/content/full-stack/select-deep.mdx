# SELECT Deep Dive — DISTINCT, ORDER BY, LIMIT | Level: 1

Retrieving data efficiently requires understanding how SQL evaluates queries vs how they are written. We explore execution ordering, sorting costs, and the physical performance impact of pagination strategies.

---

### Logical Processing Order of a SELECT Statement

Although written in a different order, the database engine executes parts of a `SELECT` statement in a strict logical sequence:

$$\text{FROM} \longrightarrow \text{JOIN} \longrightarrow \text{WHERE} \longrightarrow \text{GROUP BY} \longrightarrow \text{HAVING} \longrightarrow \text{SELECT} \longrightarrow \text{DISTINCT} \longrightarrow \text{ORDER BY} \longrightarrow \text{LIMIT / OFFSET}$$
