## WHY
If a website is a house, HTML is the concrete foundation and wooden framing. 
It doesn't care about paint colors (CSS) or electricity (JavaScript). Its only job is to define the **structure** and **content** of the page.
Before a browser can make a button look pretty or make a menu interactive, it needs to know that a button and a menu physically exist. **HyperText Markup Language (HTML)** provides this structural blueprint using a standard vocabulary of "tags" that every browser on Earth understands.

## THEORY
### Elements and Tags
HTML is not a programming language; it has no logic, variables, or loops. It is a *markup* language. You write standard text, and you wrap that text in HTML **Tags** to give it semantic meaning.
- `<p>` means Paragraph.
- `<h1>` means Heading 1 (The most important title).
- `<button>` means Button.

An **Element** consists of the Opening Tag, the Content, and the Closing Tag (which has a forward slash `/`).
Example: `<h1>Welcome to my Website!</h1>`

### The DOM Tree (Nesting)
HTML elements are designed to be nested inside one another, creating a Parent-Child hierarchy. This hierarchy forms a massive upside-down tree structure known as the **Document Object Model (DOM)**. When you write CSS or JavaScript later, you will use this DOM tree to precisely target and manipulate specific HTML elements!

## CODE
### 1. The Standard HTML Boilerplate
Every single HTML file in existence requires this exact foundational shell.

```html
<!-- Tells the browser this is modern HTML5, not ancient HTML from 1999 -->
<!DOCTYPE html> 

<!-- The root element that wraps the entire document -->
<html lang="en"> 

    <!-- The HEAD contains metadata. Nothing here is visible on the actual web page! -->
    <head>
        <meta charset="UTF-8">
        <title>My First Website</title> <!-- The text in the browser tab! -->
    </head>

    <!-- The BODY contains the actual visible content of the page! -->
    <body>
        
        <!-- We write our tags in here! -->

    </body>
</html>
```

### 2. Core Text Elements
```html
<h1>This is the Main Title (Only use one per page for SEO!)</h1>
<h2>This is a Subtitle</h2>
<h3>This is a smaller Sub-subtitle</h3>

<p>This is a standard paragraph of text. Browsers automatically add spacing around paragraphs.</p>

<!-- Some elements are Self-Closing (They have no content inside them!) -->
<br /> <!-- Line Break (Forces a return carriage) -->
<hr /> <!-- Horizontal Rule (Draws a line across the page) -->
```

### 3. Links and Images (Attributes)
Tags can have **Attributes** injected into their opening tag. Attributes provide extra configuration data that the tag needs to function.

```html
<!-- The Anchor tag <a> creates a hyperlink. 
     The 'href' attribute tells it WHERE to go when clicked! -->
<a href="https://google.com">Click here to search</a>

<!-- The Image tag <img> is self-closing. 
     The 'src' attribute points to the image file. 
     The 'alt' attribute provides text for screen readers (Accessibility)! -->
<img src="cute-dog.jpg" alt="A fluffy golden retriever" />
```

### 4. Layout Elements (The Div)
To structure a complex page, we group related elements together into invisible boxes. The most common box is the `<div>` (Division).

```html
<!-- A Div acts as an invisible container holding the profile data together -->
<div>
    <h2>User Profile</h2>
    <img src="avatar.png" alt="User Avatar" />
    <p>Name: John Doe</p>
    
    <!-- A button element for user interaction -->
    <button>Send Message</button> 
</div>
```

## REAL_WORLD
**Semantic HTML and Accessibility**
In the early days of web development, developers used `<div>` tags for absolutely everything. They built navigation bars with `<div>`, footers with `<div>`, and articles with `<div>`.
Visually, this looked fine. But structurally, it was a disaster. 
Search engines (like Google) and Screen Readers (used by blind users) could not look at a massive pile of 500 `<div>` tags and understand what the page was about.
Modern HTML5 introduced **Semantic Tags**. Instead of using a generic `<div>`, you should use `<nav>`, `<header>`, `<main>`, `<article>`, and `<footer>`. These tags function exactly like `<div>` boxes, but they carry deep structural meaning. When a blind user's Screen Reader encounters a `<nav>` tag, it explicitly announces "Navigation Menu", drastically improving Web Accessibility (a legal requirement for modern applications!).

## INTERVIEW
**Q: An interviewer looks at your code: `<img src="logo.png">`. They ask: "Why is it absolutely critical to include the `alt` attribute on every image tag, even if the image is just a decorative background swoosh?"**
A: **Accessibility and SEO.**
1. **Accessibility (A11y):** Visually impaired users rely on screen readers. If an image lacks an `alt` attribute, the screen reader might confusingly read the raw file name (e.g., "Image: logo underscore final version 2 dot png"). If the image is purely decorative, you MUST include an empty alt attribute (`alt=""`). This explicitly commands the screen reader to completely ignore the image and skip it!
2. **SEO (Search Engine Optimization):** Google's web crawlers cannot "see" images. They rely entirely on the `alt` text to understand the context of the image. If you omit the `alt` text, your images will never appear in Google Image Search, harming your website's organic traffic.
