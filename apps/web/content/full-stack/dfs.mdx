# Depth-First Search (DFS)

Depth-First Search is a graph traversal technique that progresses as deep as possible along each branch before backtracking. It is highly suited for pathfinding, connectivity analysis, and topological sorting.

## 1. Core Mechanics & Theoretical Foundation

### The Stack as a Deep-First Driver
DFS uses a **Last-In, First-Out (LIFO)** strategy. Whether utilizing the system call stack via recursion or managing an explicit `java.util.Deque` implementation, the stack structure ensures that the most recently discovered vertex is prioritized for exploration.

```
[Stack Frame: Node A] -> [Stack Frame: Node B] -> [Stack Frame: Node C] (Deepest)
```

### Tree Traversal Variations
On trees, DFS can be categorized by when the root node is processed relative to its children:
* **Pre-order**: Node $\to$ Left $\to$ Right (Used for copying/cloning trees).
* **In-order**: Left $\to$ Node $\to$ Right (Used for binary search trees to retrieve sorted order).
* **Post-order**: Left $\to$ Right $\to$ Node (Used for bottom-up computations, such as calculating tree height or deleting a tree).

### Graph Traversal Attributes: Discovery and Finish Times
When running DFS on a graph, we track two key lifecycle timestamps for each vertex $u$:
1. **Discovery Time ($d[u]$)**: The step at which the node is first marked visited.
2. **Finish Time ($f[u]$)**: The step at which all of $u$'s descendants have been fully explored.

These intervals satisfy the **Parenthesis Theorem**: for any two vertices $u$ and $v$, either the intervals $[d[u], f[u]]$ and $[d[v], f[v]]$ are entirely disjoint, or one is completely nested within the other.

### Classification of Graph Edges
DFS classifies the edges of a graph into four categories:
1. **Tree Edges**: Edges in the depth-first forest. Edge $(u, v)$ is a tree edge if $v$ was first discovered by exploring $(u, v)$.
2. **Back Edges**: Edges pointing from a descendant to an ancestor in the DFS tree. **The presence of a back edge indicates a cycle in the graph.**
3. **Forward Edges**: Non-tree edges $(u, v)$ connecting a vertex $u$ to a descendant $v$ in a DFS tree.
4. **Cross Edges**: All other edges. They go between vertices in different trees or between vertices in the same tree that are not ancestors/descendants.

```
       [Ancestor]
        /      ^
  Tree /        \ Back Edge (Cycle!)
      v          \
   [Descendant] --+
```

### Recursive Risks: Stack Overflow
Recursive DFS relies on the JVM's call stack. In deep graphs (such as a linked list graph with $> 10,000$ nodes), recursive DFS will throw a `StackOverflowError`. For production-grade applications, an **iterative implementation with an explicit heap-allocated stack** is preferred.

## 2. Mathematical Proofs & Complexity Derivations

### Time Complexity: $O(V + E)$
The proof mirrors BFS:
1. Every vertex is transitioned from `WHITE` (unvisited) to `GRAY` (discovered) to `BLACK` (finished) exactly once. This takes $O(V)$ state transitions.
2. During the lifetime of a vertex, its adjacency list is scanned exactly once. The sum of the lengths of all adjacency lists is $E$ (for directed graphs) or $2E$ (for undirected graphs).
3. Thus, the total execution time is:
$$\text{Total Time} = O(V + E)$$

### Space Complexity: $O(V)$
In the worst-case scenario (a completely linear graph, such as a single line of $V$ vertices), the recursion stack or explicit stack will store $V$ frames. Thus, the worst-case space complexity is $O(V)$.

## 4. Concrete Trace

Let's trace DFS on a directed graph:
* Vertices: `0, 1, 2, 3`
* Edges: `(0->1), (1->2), (2->3), (0->2)`

| Step | Node | Stack (Call Stack) | Visited | Action / State Change |
| :--- | :--- | :--- | :--- | :--- |
| **0** | — | `[]` | `{}` | Start DFS at source `0`. |
| **1** | `0` | `[0]` | `{0}` | Discover `0`. Move to neighbor `1`. |
| **2** | `1` | `[0, 1]` | `{0, 1}` | Discover `1`. Move to neighbor `2`. |
| **3** | `2` | `[0, 1, 2]` | `{0, 1, 2}` | Discover `2`. Move to neighbor `3`. |
| **4** | `3` | `[0, 1, 2, 3]` | `{0, 1, 2, 3}`| Discover `3`. No neighbors. Finish `3`. |
| **5** | `2` | `[0, 1, 2]` | `{0, 1, 2, 3}`| Backtrack to `2`. No other unvisited neighbors. Finish `2`. |
| **6** | `1` | `[0, 1]` | `{0, 1, 2, 3}`| Backtrack to `1`. No other unvisited neighbors. Finish `1`. |
| **7** | `0` | `[0]` | `{0, 1, 2, 3}`| Backtrack to `0`. Neighbor `2` is visited. Finish `0`. |

## 5. Implementation (Java & Python)

### Java 21 Implementation
```java
package dsa.dfs;

import java.util.*;

public final class DepthFirstSearch {

    public enum NodeColor { WHITE, GRAY, BLACK }

    public static List<Integer> traverseIterative(Map<Integer, List<Integer>> graph, int source) {
        Objects.requireNonNull(graph, "Graph cannot be null");
        
        List<Integer> traversalOrder = new ArrayList<>();
        Deque<Integer> stack = new ArrayDeque<>();
        Set<Integer> visited = new HashSet<>();

        stack.push(source);

        while (!stack.isEmpty()) {
            int current = stack.pop();

            if (!visited.contains(current)) {
                visited.add(current);
                traversalOrder.add(current);

                List<Integer> neighbors = graph.getOrDefault(current, Collections.emptyList());
                // Push neighbors in reverse order to preserve left-to-right exploration
                for (int i = neighbors.size() - 1; i >= 0; i--) {
                    int neighbor = neighbors.get(i);
                    if (!visited.contains(neighbor)) {
                        stack.push(neighbor);
                    }
                }
            }
        }
        return traversalOrder;
    }
}
```

## 6. Interview Deep Dive

### Probing Questions & Expert Answers

1. **How do you detect cycles in an undirected graph using DFS vs. a directed graph?**
   * *Answer*: 
     * In an **undirected graph**, we pass the `parent` node to the recursive DFS call. If we encounter a visited node that is not the parent of the current node, a cycle exists.
     * In a **directed graph**, a simple visited set is insufficient. We must track nodes in the current recursion stack (the "active path") using a state array (e.g., `WHITE` = unvisited, `GRAY` = visiting, `BLACK` = finished). A cycle is detected if we hit a `GRAY` node.

2. **How can you find the strongly connected components (SCCs) of a directed graph using DFS?**
   * *Answer*: Use **Kosaraju's Algorithm** or **Tarjan's Algorithm**. 
     * Kosaraju's runs DFS on the original graph to get ordering by finish times, transposes the graph, and runs DFS on the transposed graph in reverse order of finish times.
     * Tarjan's uses a single DFS pass tracking `low-link` values to identify SCCs on the fly.
