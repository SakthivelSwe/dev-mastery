## WHY
Every database in the world fundamentally does four things: Create, Read, Update, and Delete (CRUD). While relational databases use standard SQL (like `INSERT` and `SELECT`), MongoDB uses a JavaScript-like syntax. Mastering these four basic operations in MongoDB is the foundational step to building any backend application that interacts with a NoSQL database.

## THEORY
### Documents and Collections
In a relational database (SQL), you have **Tables** and **Rows**.
In MongoDB, you have **Collections** and **Documents**.

A Document is simply a JSON object (technically BSON - Binary JSON). A Collection is a group of Documents. Unlike SQL, a Collection does not enforce a rigid schema. You can insert a document with `{name: "Alice"}` and immediately insert another document with `{name: "Bob", age: 30, skills: ["Java"]}` into the exact same collection.

## CODE

### 1. Create (Insert)
You can insert a single document, or insert an array of documents for better performance. Every document automatically receives a unique `_id` field if you do not provide one.

```javascript
// Insert a single document
db.users.insertOne({
    name: "Alice",
    age: 28,
    status: "active"
});

// Insert multiple documents efficiently
db.users.insertMany([
    { name: "Bob", age: 35, status: "pending" },
    { name: "Charlie", age: 22, status: "active" }
]);
```

### 2. Read (Find)
The `find()` method is the equivalent of a SQL `SELECT` statement.

```javascript
// SELECT * FROM users
// Returns a cursor to all documents
db.users.find({});

// SELECT * FROM users WHERE status = 'active'
// Returns only active users
db.users.find({ status: "active" });

// Find exactly ONE document (returns the actual object, not a cursor)
db.users.findOne({ name: "Alice" });
```

### 3. Update 
Updating in MongoDB requires two objects: The **Filter** (to find the document), and the **Update Operator** (to dictate *how* to change it). 

**Warning:** If you do not use the `$set` operator, MongoDB will completely overwrite the entire document with your new payload!

```javascript
// UPDATE users SET age = 29 WHERE name = 'Alice'
db.users.updateOne(
    { name: "Alice" },          // The Filter
    { $set: { age: 29 } }       // The Update Operator
);

// Update ALL documents that match the filter
db.users.updateMany(
    { status: "pending" }, 
    { $set: { status: "active" } }
);
```

### 4. Delete
Deleting works exactly like finding; you pass in a filter.

```javascript
// DELETE FROM users WHERE name = 'Charlie' LIMIT 1
db.users.deleteOne({ name: "Charlie" });

// DELETE FROM users WHERE status = 'pending'
db.users.deleteMany({ status: "pending" });
```

## REAL_WORLD
**Upsert (Update or Insert)**
Often, you want to update a document if it exists, but create it if it doesn't. Doing this manually requires two database queries (a `findOne`, followed by an `insertOne` or `updateOne`), which creates race conditions.

MongoDB solves this elegantly with the `upsert` option:
```javascript
db.users.updateOne(
    { name: "David" }, 
    { $set: { age: 40, status: "active" } },
    { upsert: true } // If David doesn't exist, create him!
);
```

## INTERVIEW
**Q: What is the difference between `find()` and `findOne()`?**
A: `findOne()` queries the database and immediately returns the actual JSON document object (or `null` if not found). `find()` does **not** return the documents; it returns a **Cursor**. A cursor is a pointer to the result set. This prevents MongoDB from crashing your Node.js server by returning 1 million documents at once. You must iterate over the cursor using `.toArray()` or `.next()` to fetch the actual data in batches.
