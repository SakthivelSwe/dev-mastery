# CSS Box Model — box-sizing, margin collapse | Level: 1

## 1. Core Architecture of the Box Model
Every element in a document is represented as a rectangular box. The rendering engine calculates the layout of these boxes based on their dimensions, margins, paddings, and borders.

```
┌───────────────────────────────────────────┐
│  MARGIN (Collapsible, Background-parent)   │
│  ┌─────────────────────────────────────┐  │
│  │  BORDER                             │  │
│  │  ┌───────────────────────────────┐  │  │
│  │  │  PADDING                      │  │  │
│  │  │  ┌─────────────────────────┐  │  │  │
│  │  │  │  CONTENT                │  │  │  │
│  │  │  │  (min-content/max-     │  │  │  │
│  │  │  │   content bounds)       │  │  │  │
│  │  │  └─────────────────────────┘  │  │  │
│  │  └───────────────────────────────┘  │  │
│  └─────────────────────────────────────┘  │
└───────────────────────────────────────────┘
```

### `box-sizing` Mechanics
* **`content-box` (Default)**: The specified `width` and `height` apply only to the content area. Any padding and border are added outside this area, increasing the element's actual rendered size.
  $$\text{Rendered Width} = \text{width} + \text{padding-left} + \text{padding-right} + \text{border-left} + \text{border-right}$$
* **`border-box`**: The specified `width` and `height` represent the outer boundary of the border. Padding and border subtract from the content area, keeping the total rendered width consistent.
  $$\text{Content Width} = \text{width} - (\text{padding-left} + \text{padding-right} + \text{border-left} + \text{border-right})$$

### Best Practice
```css
*, ::before, ::after {
  box-sizing: border-box;
}
```
This ensures consistent sizing calculations across all elements in your layout.

## 2. Margin Collapse Mechanics
Margin collapse occurs when the vertical margins of adjacent block-level boxes combine into a single margin. The resulting margin is determined as follows:
* If both margins are positive, the value is the **maximum** of the two.
* If one margin is negative, the negative value is subtracted from the positive value.
* If both margins are negative, the value is the **minimum** (most negative) value.

### Rules and Exceptions
* Margin collapse occurs **only on vertical margins** of block-level boxes in the normal flow.
* Horizontal margins never collapse.
* Margin collapse does **not** occur on:
  * Flex items and Grid items.
  * Absolutely positioned elements.
  * Elements with `display: inline-block`.
  * Elements where a Block Formatting Context (BFC) is established between them (e.g., `overflow: hidden`).
  * Parent and child elements if there is a border, padding, or inline content separating their margins.

## 3. Intrinsic vs Extrinsic Sizing
* **Extrinsic Sizing**: Dimensions are explicitly set using absolute or relative units (e.g., `width: 300px` or `width: 50vw`).
* **Intrinsic Sizing**: Dimensions are determined by the element's content.
  * `min-content`: The minimum size an element can take without overflowing its boundaries. For text, this is the length of the longest word.
  * `max-content`: The size the element needs to display all its content on a single line without wrapping.
  * `fit-content`: Uses the available space up to `max-content`, but shrinks down to `min-content` if the container is smaller than the content.

## 4. Implementation Sandbox & Live Code

```html
<div class="box-wrapper">
  <div class="intrinsic-box">
    <h3>Intrinsic Box Sizing</h3>
    <p>This box sizes itself dynamically based on content constraints.</p>
  </div>
  
  <div class="extrinsic-box">
    <h3>Extrinsic Box Sizing</h3>
    <p>This box has a fixed width of 320px regardless of content.</p>
  </div>
</div>
```

```css
/* Global reset to border-box */
*, ::before, ::after {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

.box-wrapper {
  background-color: oklch(0.95 0.01 240);
  padding: 2rem;
  display: flex;
  flex-direction: column;
  gap: 2rem; /* Avoids margin collapse by using flex gaps */
}

/* Intrinsic Sizing Example */
.intrinsic-box {
  width: fit-content;
  max-width: 400px;
  min-width: min-content;
  padding: 1.5rem;
  border: 4px solid oklch(0.6 0.25 140);
  background-color: oklch(1 0 0);
}

/* Extrinsic Sizing Example */
.extrinsic-box {
  width: 320px; /* Fixed width */
  padding: 1.5rem;
  border: 4px solid oklch(0.4 0.15 20);
  background-color: oklch(1 0 0);
  
  /* Negative margin to create overlapping visual effect */
  margin-block-start: -1rem;
}
```

## 5. DevTools Diagnostics & Box Profiling

To inspect box model dimensions:
1. Open DevTools and select an element.
2. Go to the **Computed** tab. The top of the pane displays a visual representation of the box model.
3. Hover over the **Content**, **Padding**, **Border**, and **Margin** zones in the diagram. The browser will highlight the corresponding areas on the page in different colors:
   * **Blue**: Content area
   * **Green**: Padding area
   * **Yellow**: Border area
   * **Orange**: Margin area
4. Use this diagram to verify if vertical margins are collapsing. If the computed margin is smaller than the declared margin, look for sibling elements or parent paddings that may be causing a collapse.

## 6. Performance & Pipeline Mechanics

### Layout vs Paint
Modifying box-model properties that affect geometry (e.g., `width`, `height`, `padding`, `margin`, `border`) forces the rendering engine to run the **Layout** (or Reflow) phase. This recalculates the geometry of the entire affected DOM subtree.
* **Performance Tip**: Avoid animating `width`, `height`, or `margin`. Use `transform: scale()` or `transform: translate()` instead. These are processed during the **Composite** phase and offloaded to the GPU, avoiding expensive layout recalculations.

### Compatibility
* `box-sizing: border-box` is fully supported in all modern browsers (100% support).
* Intrinsic sizing keywords (`min-content`, `max-content`, `fit-content`) are supported across all modern engines.
