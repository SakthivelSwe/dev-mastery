### 1. Figma Workflow & Token Sync Pipeline
The color system utilizes a semantic architecture mapping primitives (raw hues) to semantic tokens (roles) and component tokens (scoped targets).

```
[ Primitive Token ] ──> [ Semantic Token ] ──> [ Component Token ]
(e.g., Slate-900)       (e.g., bg-surface)      (e.g., button-primary-bg)
```

In Figma, this is managed across light and dark collections. The JSON output defines theme-specific semantic layers that reference global primitives.

### 2. Token Definitions

#### CSS Custom Properties
```css
/* colors.css */
:root {
  /* Primitives */
  --primitive-slate-50: #f8fafc;
  --primitive-slate-900: #0f172a;
  --primitive-blue-500: #3b82f6;
  --primitive-blue-400: #60a5fa;
  --primitive-red-500: #ef4444;

  /* Light Theme Semantics */
  --color-bg-canvas: var(--primitive-slate-50);
  --color-text-primary: var(--primitive-slate-900);
  --color-accent: var(--primitive-blue-500);
  --color-error: var(--primitive-red-500);
}

[data-theme='dark'] {
  /* Dark Theme Semantics Override */
  --color-bg-canvas: var(--primitive-slate-900);
  --color-text-primary: var(--primitive-slate-50);
  --color-accent: var(--primitive-blue-400);
  --color-error: var(--primitive-red-500);
}
```

#### Tailwind Configuration
```javascript
// tailwind.config.js
module.exports = {
  darkMode: ["class", '[data-theme="dark"]'],
  theme: {
    extend: {
      colors: {
        canvas: "var(--color-bg-canvas)",
        primary: "var(--color-text-primary)",
        accent: "var(--color-accent)",
        error: "var(--color-error)",
      },
    },
  },
};
```

### 3. Accessibility Specifications (WCAG 2.1 AA)
- **Contrast Ratios**: Body text on canvas must meet **4.5:1** contrast. Large headings must meet **3:1**.
- **Interactive States**: Accent borders or active icons must adhere to **3:1** minimum contrast against adjacent backgrounds.
- **Color Blindness**: Critical semantic states (errors, warnings, success) must never rely solely on color. They must include text labels, visual shapes, or patterns.

### 4. React + TypeScript Implementation

```tsx
// ColorCard.tsx
import React, { useState, useEffect } from "react";

export interface ColorCardProps {
  title: string;
  description: string;
}

export const ColorCard: React.FC<ColorCardProps> = ({ title, description }) => {
  const [theme, setTheme] = useState<"light" | "dark">("light");

  useEffect(() => {
    document.documentElement.setAttribute("data-theme", theme);
  }, [theme]);

  const toggleTheme = () => {
    setTheme((prev) => (prev === "light" ? "dark" : "light"));
  };

  return (
    <div className="p-8 bg-canvas text-primary border rounded-xl transition-colors duration-200 max-w-md shadow-md">
      <div className="flex justify-between items-center mb-6">
        <span className="text-xs font-bold tracking-wider uppercase opacity-75">Theme Sandbox</span>
        <button
          onClick={toggleTheme}
          className="px-3 py-1 text-xs font-semibold rounded-full border border-accent text-accent hover:opacity-80 focus:outline-none focus:ring-2 focus:ring-accent focus:ring-offset-2"
          aria-label={`Switch to ${theme === "light" ? "dark" : "light"} mode`}
        >
          {theme === "light" ? "Dark Mode" : "Light Mode"}
        </button>
      </div>
      <h3 className="text-2xl font-bold mb-2">{title}</h3>
      <p className="text-sm leading-relaxed mb-4 opacity-90">{description}</p>
      <div className="flex gap-2 items-center text-xs text-error font-semibold" role="alert">
        <svg className="w-4 h-4 fill-current" viewBox="0 0 20 20" aria-hidden="true">
          <path fillRule="evenodd" d="M18 10a8 8 0 11-16 0 8 8 0 0116 0zm-7 4a1 1 0 11-2 0 1 1 0 012 0zm-1-9a1 1 0 00-1 1v4a1 1 0 102 0V6a1 1 0 00-1-1z" clipRule="evenodd"/>
        </svg>
        <span>Error indicator: Must be accompanied by an icon.</span>
      </div>
    </div>
  );
};
```
