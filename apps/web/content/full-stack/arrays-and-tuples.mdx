# Arrays and Tuples

TypeScript provides specific structures for collections of elements: dynamic-length **Arrays** and fixed-length, positionally-typed **Tuples**.

---

## 1. Typed Arrays

Arrays can be declared using square brackets (`T[]`) or the generic array wrapper (`Array<T>`). Both syntax variations are equivalent.

### TypeScript Code
```typescript
const scores: number[] = [95, 82, 100];
const names: Array<string> = ["Alice", "Bob"];

// Error: TS2322: Type 'string' is not assignable to type 'number'.
scores.push("99"); 
```

### Compiled JavaScript (tsc output)
```javascript
"use strict";
const scores = [95, 82, 100];
const names = ["Alice", "Bob"];
```

---

## 2. Readonly Arrays

Readonly arrays prevent mutation operations like `push()`, `pop()`, or direct element reassignment.

### TypeScript Code
```typescript
const readOnlyNumbers: readonly number[] = [1, 2, 3];
const alternativeFormat: ReadonlyArray<string> = ["A", "B"];

// Error: TS2339: Property 'push' does not exist on type 'readonly number[]'.
readOnlyNumbers.push(4); 

// Error: TS2542: Index signature in type 'readonly number[]' only permits reading.
readOnlyNumbers[0] = 99; 
```

### Compiled JavaScript (tsc output)
```javascript
"use strict";
const readOnlyNumbers = [1, 2, 3];
const alternativeFormat = ["A", "B"];
```

---

## 3. Tuples

A tuple is an array with a fixed number of elements where each position has a specific type.

### TypeScript Code
```typescript
// A tuple containing a string, a number, and a boolean
let response: [string, number, boolean] = ["Success", 200, true];

// Error: TS2322: Type 'number' is not assignable to type 'string'.
response[0] = 500; 
```

### Compiled JavaScript (tsc output)
```javascript
"use strict";
let response = ["Success", 200, true];
```

---

## 4. Optional and Rest Elements in Tuples

Tuples can contain optional elements marked with a `?` (which must be placed at the end), or rest elements mapping to open-ended elements.

### TypeScript Code
```typescript
// Optional element: third element is optional
type Connection = [host: string, port: number, useSsl?: boolean];
const localConn: Connection = ["localhost", 5432];
const remoteConn: Connection = ["api.com", 443, true];

// Rest elements: first element is string, followed by any number of booleans
type FlagSet = [name: string, ...flags: boolean[]];
const userFlags: FlagSet = ["admin", true, false, true, true];
```

### Compiled JavaScript (tsc output)
```javascript
"use strict";
const localConn = ["localhost", 5432];
const remoteConn = ["api.com", 443, true];
const userFlags = ["admin", true, false, true, true];
```

---

## 5. Labeled Tuples

Introduced in TypeScript 4.0, labeled tuples provide semantic names to tuple positions. These labels exist solely for documentation and IDE tooltips.

```typescript
type Coordinates = [latitude: number, longitude: number];

function locate(coords: Coordinates) {
  const [lat, lng] = coords; // Destructured normally
  console.log(`Lat: ${lat}, Lng: ${lng}`);
}
```

## 6. Practical Use Cases

### Function Returning Multiple Values (e.g., React Hooks)
```typescript
function useToggle(initialValue: boolean): [boolean, (val: boolean) => void] {
  let state = initialValue;
  const setState = (val: boolean) => { state = val; };
  return [state, setState];
}
```

### CSV or Database Row Representation
```typescript
type UserRow = [id: number, email: string, isActive: boolean];
const databaseRows: UserRow[] = [
  [1, "admin@domain.local", true],
  [2, "guest@domain.local", false]
];
```
