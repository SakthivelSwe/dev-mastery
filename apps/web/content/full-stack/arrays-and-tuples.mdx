# Arrays and Tuples

TypeScript provides explicit ways to type collections of data, allowing you to enforce length, type, and order constraints on arrays and tuples.

---

## 1. Typed Arrays

You can define array types in two syntactically equivalent ways: the bracket syntax (`T[]`) or the generic array type (`Array<T>`).

```typescript
const numbers: number[] = [1, 2, 3];
const strings: Array<string> = ["a", "b", "c"];
```

---

## 2. Readonly Arrays

To prevent mutations (such as `push()`, `pop()`, or index-based reassignment), use `ReadonlyArray<T>` or the `readonly T[]` prefix.

```typescript
const mutable: number[] = [1, 2, 3];
mutable.push(4); // Allowed

const immutable: readonly number[] = [1, 2, 3];
// immutable.push(4); 
// Compiler Error: TS2339: Property 'push' does not exist on type 'readonly number[]'.

// immutable[0] = 10;
// Compiler Error: TS2542: Index signature in type 'readonly number[]' only permits reading.
```

---

## 3. Tuples

A tuple is an array with a fixed number of elements whose types are known at specific positions.

```typescript
const userSession: [string, number, boolean] = ["session_id_123", 3600, true];

// Accessing by index retains the exact type for that position
const sessionId: string = userSession[0];
const timeout: number = userSession[1];
```

---

## 4. Advanced Tuple Patterns

### Optional Tuple Elements
Tuple elements can be marked as optional using the `?` modifier. These must always be placed at the end of the tuple definition.

```typescript
type ConnectionConfig = [host: string, port?: number];
const localConn: ConnectionConfig = ["localhost"];
const remoteConn: ConnectionConfig = ["10.0.0.1", 5432];
```

### Rest Elements in Tuples
Tuples can use the rest parameter syntax to accept zero or more elements of a specific type.

```typescript
type StringAndNumbers = [string, ...number[]];
const data: StringAndNumbers = ["scores", 98, 87, 93, 100];
```

### Labeled Tuples
Introduced in TypeScript 4.0, labeled tuples provide semantic names to positions within the tuple. This is solely for documentation and editor tooling.

```typescript
type Coordinate = [latitude: number, longitude: number];
const location: Coordinate = [37.7749, -122.4194];
```

---

## 5. Practical Use Cases

### Return Value of a Custom React Hook
Tuples are ideal for returning multiple values from a function where ordering is crucial (e.g., state and a state setter).

```typescript
function useToggle(initialValue: boolean): [boolean, () => void] {
  let state = initialValue;
  const toggle = () => { state = !state; };
  return [state, toggle];
}
```

### CSV or Tabular Data Rows
```typescript
type CSVRow = [id: number, name: string, email: string, isActive: boolean];
const row: CSVRow = [101, "John Doe", "john@example.com", true];
```

## 6. Type Erasure: Tuples and Arrays

### Side-by-Side Comparison

```typescript
// --- TypeScript Source (collections.ts) ---
const tuple: [string, number] = ["admin", 1];
const readonlyArray: readonly string[] = ["a", "b"];
```

```javascript
// --- Compiled JavaScript (tsc Output - collections.js) ---
"use strict";
const tuple = ["admin", 1];
const readonlyArray = ["a", "b"];
```
