# Angular Services & Dependency Injection: Hierarchies and Resolution Mechanics

Angular features a hierarchical Dependency Injection (DI) system. It is a powerful design pattern that decouples class creation from consumer usage, enabling clean, testable architectures.

---

## 1. Injector Hierarchy

Angular resolves dependencies by traversing an injector tree. This tree contains two main branches: the **Module/Environment Injector Tree** and the **Element/Node Injector Tree**.

```
             NullInjector (Throws error if not @Optional)
                   │
            PlatformInjector (Contains platform-wide services)
                   │
          EnvironmentInjector (Root injector, configured via provideStore, bootstrap)
                   │
            ElementInjector (Component-level, NodeInjector)
```

1.  **`NullInjector`**: The absolute top of the tree. If a dependency lookup reaches this point, Angular throws an error unless the dependency is marked with `@Optional()`.
2.  **`PlatformInjector`**: Configured when bootstrapping the platform (e.g., platform-browser, platform-server).
3.  **`EnvironmentInjector`**: Replaces the old root `ModuleInjector` in standalone applications. Configured in `bootstrapApplication` or within route configurations using `providers: [...]`.
4.  **`ElementInjector` / `NodeInjector`**: Created on every DOM element that hosts a component or directive. Configured via the `providers` or `viewProviders` arrays inside the `@Component` decorator.

---

## 2. Resolution Modifiers

By default, Angular starts searching for a dependency at the current component's `ElementInjector`. If not found, it traverses upward through parent element injectors, then moves to the `EnvironmentInjector`, and finally up to the `NullInjector`. This path can be customized using resolution modifiers:

*   **`@Self()`**: Instructs Angular to look for the dependency *only* in the local component/directive's `ElementInjector`.
*   **`@SkipSelf()`**: Instructs Angular to bypass the local `ElementInjector` and begin the search at the parent injector.
*   **`@Optional()`**: Prevents Angular from throwing an error if the dependency is not found, returning `null` instead.
*   **`@Host()`**: Restricts the search to the current template host. The search will not ascend beyond the component's host element.

```typescript
import { Component, Injectable, Optional, Self, SkipSelf, Host } from '@angular/core';

@Injectable()
export class LoggerService {
  log(msg: string): void { console.log(`[Logger]: ${msg}`); }
}

@Component({
  selector: 'app-di-resolution',
  standalone: true,
  providers: [LoggerService], // Configured on ElementInjector
  template: `<div>Check console for DI resolution outputs.</div>`
})
export class DiResolutionComponent {
  constructor(
    // Looks only at this component's ElementInjector
    @Self() private selfLogger: LoggerService,

    // Skips this component's ElementInjector and looks in parent elements/root
    @SkipSelf() @Optional() private parentLogger: LoggerService | null,

    // Looks up to the host element boundary
    @Host() private hostLogger: LoggerService
  ) {
    this.selfLogger.log('Resolved from local ElementInjector.');
    if (this.parentLogger) {
      this.parentLogger.log('Resolved from parent.');
    } else {
      console.log('Parent logger not found (expected behavior).');
    }
  }
}
```
