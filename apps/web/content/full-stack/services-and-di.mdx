# Architectural Deep Dive: Angular Dependency Injection (DI)

## 1. Architectural Foundations & Core Mechanics

Angular relies on a hierarchical Dependency Injection (DI) framework. This framework is not a simple map of instances, but rather a tree of injectors that mirrors the DOM and component tree.

### The Hierarchical Injector Tree

Angular resolves dependencies by climbing up two parallel injector trees:

```
                  [ModuleInjector Tree]
                  (AppConfig / Root Injector)
                               │
                               ▼
                  [ElementInjector Tree]
                  (Dynamic Component Tree)
                               │
         ┌─────────────────────┴─────────────────────┐
         ▼                                           ▼
[ParentComponent ElementInjector]       [ChildComponent ElementInjector]
(Declares local providers)              (Resolves via NodeInjector lookup)
```

1.  **`ElementInjector` Tree**: Built dynamically alongside the component/directive tree. Each DOM node can host an `ElementInjector`. Components declare local providers here via the `providers` or `viewProviders` arrays.
2.  **`ModuleInjector` Tree**: Configured globally. In modern standalone applications, this is registered via `provideStore`, `provideRouter`, or within the application configuration inside `bootstrapApplication()`.

### The Resolution Algorithm & NodeInjector Lookup

When a class requests a dependency:
1.  Angular starts at the local `ElementInjector` (the `NodeInjector` associated with the current component's view).
2.  If the token is not found, it traverses upward through the parent elements in the template.
3.  If the root of the `ElementInjector` tree is reached without locating the token, Angular switches to the `ModuleInjector` tree, starting at the current environment's injector up to the root injector.
4.  If still not found, it falls back to the `NullInjector`. Unless the dependency is marked optional, the `NullInjector` throws a `StaticInjectorError`.

## 2. Deep-Dive API & Declarative Synthesis

### Injection Token Types

Dependencies are identified by lookup tokens. While classes can act as their own tokens, interfaces cannot exist at runtime in JavaScript. For interfaces, configuration configurations, or primitives, we use `InjectionToken`.

```typescript
import { InjectionToken } from '@angular/core';

export interface AppConfig {
  apiUrl: string;
  timeoutMs: number;
}

export const APP_CONFIG = new InjectionToken<AppConfig>('app.config');
```

### Provider Configuration Strategies

```typescript
import { bootstrapApplication } from '@angular/platform-browser';
import { AppComponent } from './app.component';
import { APP_CONFIG } from './config.token';

bootstrapApplication(AppComponent, {
  providers: [
    // 1. Value Provider
    {
      provide: APP_CONFIG,
      useValue: { apiUrl: 'https://api.enterprise.com', timeoutMs: 5000 }
    },
    // 2. Class Provider (useful for swapping implementations)
    {
      provide: LoggerService,
      useClass: EnterpriseLoggerService
    },
    // 3. Factory Provider (dynamic resolution)
    {
      provide: API_SERVICE,
      useFactory: (config: AppConfig) => new ApiService(config),
      deps: [APP_CONFIG]
    }
  ]
});
```

### Resolution Modifiers

Angular provides parameter decorators to control how the DI engine searches for dependencies:

```typescript
import { Injectable, Inject, Optional, Self, SkipSelf, Host } from '@angular/core';
import { APP_CONFIG, AppConfig } from './config.token';

@Injectable()
export class ConsumerService {
  constructor(
    @Inject(APP_CONFIG) private config: AppConfig,
    @Optional() private logger?: LoggerService, // Returns null instead of throwing error if missing
    @Self() private localService?: LocalService, // Searches ONLY in this element's injector
    @SkipSelf() private parentService?: ParentService, // Skips current element, starts lookup at parent
    @Host() private hostService?: HostService // Restricts lookup to the host component of the template
  ) {}
}
```
