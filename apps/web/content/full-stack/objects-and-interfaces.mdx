# Objects, Interfaces, and Type Aliases

Objects in TypeScript can be typed using inline type annotations, **Interfaces**, or **Type Aliases**.

---

## 1. Interfaces vs. Type Aliases

*   **Interfaces**: Primarily used to define the shape of objects. They support inheritance via `extends` and are **open** (can be extended through declaration merging).
*   **Type Aliases**: Can represent any type, including primitives, unions, intersections, tuples, and functions. They are **closed** and cannot be redeclared to merge properties.

### Declaration Merging (Interfaces Only)
```typescript
interface WindowConfig {
  title: string;
}
interface WindowConfig {
  width: number; // Merged with above
}

const config: WindowConfig = {
  title: "Editor",
  width: 1920
};
```

### Type Alias Composition (Via Intersections)
```typescript
type Position = { x: number; y: number };
type Color = { hex: string };
type ColoredPosition = Position & Color;

const point: ColoredPosition = {
  x: 10,
  y: 20,
  hex: "#FF0000"
};
```

## 2. Optional and Readonly Properties

*   **`?`**: Marks a property as optional (implicitly allowing `undefined`).
*   **`readonly`**: Prevents reassignment of the property after instantiation.

### TypeScript Code
```typescript
interface DatabaseConfig {
  readonly host: string;
  port?: number; // Optional
}

const db: DatabaseConfig = { host: "127.0.0.1" };

// Error: TS2540: Cannot assign to 'host' because it is a read-only property.
db.host = "localhost"; 
```

### Compiled JavaScript (tsc output)
```javascript
"use strict";
const db = { host: "127.0.0.1" };
```

## 3. Index Signatures

Index signatures are used when you do not know the exact property names in advance, but you know the types of keys and values.

### TypeScript Code
```typescript
interface Inventory {
  [itemId: string]: number; // String keys, number values
}

const stock: Inventory = {
  "sku-100": 50,
  "sku-200": 12
};

stock["sku-300"] = 5;
```

### Compiled JavaScript (tsc output)
```javascript
"use strict";
const stock = {
    "sku-100": 50,
    "sku-200": 12
};
stock["sku-300"] = 5;
```

## 4. Excess Property Checking

When assigning an object literal directly to a typed variable, TypeScript performs **excess property checking** to prevent bugs caused by misspelled or unused options. This check is bypassed when assigning through an intermediate variable.

### TypeScript Code
```typescript
interface Options {
  color?: string;
  width?: number;
}

function configure(opts: Options) { /* ... */ }

// Error: TS2353: Object literal may only specify known properties, and 'colour' does not exist in type 'Options'.
configure({ colour: "red" }); 

// Bypassing via intermediate variable
const localOpts = { colour: "red", width: 100 };
configure(localOpts); // Compiles! Shape matches (has optional width, extra properties ignored structurally)
```

### Compiled JavaScript (tsc output)
```javascript
"use strict";
function configure(opts) { }
const localOpts = { colour: "red", width: 100 };
configure(localOpts);
```
