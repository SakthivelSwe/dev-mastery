## WHY
If your database returns a Date object like `2024-12-25T14:30:00Z`, you cannot display that raw string to a user. You must transform it into something human-readable, like `"December 25, 2024"`. 

Historically, developers wrote massive formatting functions in every single component to handle this. If you needed to format currency, dates, or uppercase text, you duplicated logic constantly.

Angular solves this with **Pipes** (`|`). A Pipe is an incredibly lightweight mechanism that takes raw data, passes it through a formatting function directly inside the HTML template, and renders the beautifully formatted output to the screen without altering the original data in the TypeScript class.

## THEORY
### How Pipes Work
Pipes follow the Unix command-line philosophy. You take an input value, and you "pipe" it through a transformer.
`{{ rawData | transformRule }}`

### Pure vs Impure Pipes
This is the most critical concept in Angular performance:
1. **Pure Pipes (The Default)**: Angular only executes the Pipe's formatting logic if the *primitive value* (or object reference) physically changes. If nothing changes, Angular caches the result. This makes them insanely fast.
2. **Impure Pipes**: Angular executes the formatting logic on *every single change detection cycle* (potentially hundreds of times a second). This is a massive performance killer and should be avoided unless absolutely necessary (like the `async` pipe).

## CODE

### Level 1 — Beginner (Built-in Formatting Pipes)
Angular provides a suite of common formatting pipes out-of-the-box. *(In Angular 17+ Standalone Components, you must import `CommonModule` to use them).*

```html
<!-- UPPERCASE & LOWERCASE -->
<h1>{{ 'Welcome to DevMastery' | uppercase }}</h1> 
<!-- Output: WELCOME TO DEVMASTERY -->

<!-- TITLECASE (Capitalizes first letter of every word) -->
<p>{{ 'john doe smith' | titlecase }}</p> 
<!-- Output: John Doe Smith -->

<!-- SLICE (Extracts a portion of a string or array) -->
<p>First 5 chars: {{ 'Angular Framework' | slice:0:5 }}</p>
<!-- Output: Angul -->
```

### Level 2 — Building Blocks (Dates and Currency)
The Date and Currency pipes are incredibly powerful because they support internationalization (i18n).

```typescript
export class BillingComponent {
  transactionDate = new Date();
  accountBalance = 42500.5;
}
```

```html
<!-- THE DATE PIPE -->
<!-- You can pass parameters to a pipe using the colon (:) -->
<p>Default Date: {{ transactionDate | date }}</p> 
<!-- Output: Dec 25, 2024 -->

<p>Short Date: {{ transactionDate | date:'short' }}</p> 
<!-- Output: 12/25/24, 2:30 PM -->

<p>Custom Format: {{ transactionDate | date:'yyyy-MM-dd' }}</p> 
<!-- Output: 2024-12-25 -->


<!-- THE CURRENCY PIPE -->
<p>Default: {{ accountBalance | currency }}</p> 
<!-- Output: $42,500.50 (Assumes USD by default) -->

<p>Euros: {{ accountBalance | currency:'EUR':'symbol' }}</p> 
<!-- Output: €42,500.50 -->
```

### Level 3 — Intermediate (JSON and Async Pipes)
The two most important utility pipes in Angular.

**The JSON Pipe (For Debugging):**
If you try to print a JavaScript object `{{ user }}`, Angular will literally print `[object Object]`. To view the data, use the JSON pipe!
```html
<pre>{{ userObject | json }}</pre>
<!-- Output:
{
  "id": 1,
  "name": "Alice"
}
-->
```

**The Async Pipe (For Observables):**
When fetching data from an API, Angular uses RxJS Observables. Instead of manually subscribing to the Observable in TypeScript and managing memory leaks, you let the HTML template do it for you using the `async` pipe!
```html
<!-- The async pipe subscribes to the user$ stream, extracts the data, 
     renders it, and automatically UNSUBSCRIBES when the component is destroyed! -->
<div *ngIf="userProfile$ | async as profile">
  <h2>{{ profile.name }}</h2>
</div>
```

### Level 4 — Advanced (Custom Pipes)
If you have custom formatting rules (e.g., masking a credit card number), you must generate your own Pipe.

```bash
ng generate pipe mask-card
```

```typescript
import { Pipe, PipeTransform } from '@angular/core';

// 1. The @Pipe decorator registers the name used in HTML
@Pipe({
  name: 'maskCard',
  standalone: true
})
export class MaskCardPipe implements PipeTransform {
  
  // 2. The transform method executes the logic!
  // 'value' is the raw data on the left side of the pipe.
  // 'visibleDigits' is the parameter passed on the right side.
  transform(value: string, visibleDigits: number = 4): string {
    if (!value || value.length < 12) return value;
    
    const maskedSection = '*'.repeat(value.length - visibleDigits);
    const visibleSection = value.slice(-visibleDigits);
    
    return maskedSection + visibleSection;
  }
}
```

```html
<!-- Usage in HTML -->
<p>Card: {{ '1234567812345678' | maskCard:4 }}</p>
<!-- Output: Card: ************5678 -->
```

## REAL_WORLD
One of the most dangerous anti-patterns in Angular is performing data filtering using Custom Pipes (e.g., `*ngFor="let user of users | filterUsers:searchTerm"`). 

Historically, developers loved this because it looked clean. However, the Angular team officially deprecated and removed the built-in `filter` and `orderBy` pipes that existed in AngularJS. Why? Because filtering a massive array is computationally expensive. If you use a pipe to filter an array, Angular will re-run the filtering algorithm every single time the user clicks the mouse anywhere on the screen (if it's an impure pipe), causing the entire browser to freeze.

**Enterprise Standard**: Never use Pipes to filter or sort data arrays. Always filter data inside the TypeScript Class (or RxJS Stream) and expose the final, pre-filtered array to the HTML template.

## INTERVIEW
**Q: Can you chain multiple pipes together?**
A: Yes! Pipes execute from left to right. `{{ transactionDate | date:'short' | uppercase }}` will first format the date, and then convert the resulting string to fully uppercase.

**Q: What makes a Pipe "Pure"?**
A: A Pure Pipe guarantees that if the input value hasn't changed, the output will not change. Angular optimizes this by caching the output. Note: If you pass an Array into a Pure Pipe, and you `array.push()` a new item into it, the Pure Pipe *will not execute!* Why? Because `push()` mutates the array, but the array's memory reference remains identical. The Pure Pipe looks at the memory reference, sees it hasn't changed, and does nothing.

**Q: Why is the `async` pipe considered impure?**
A: Because an Observable stream doesn't change its memory reference, but it continuously emits new data over time. If the `async` pipe were pure, it would never detect the new data arriving from the API. It must be impure so it can evaluate on every change detection cycle to see if the stream emitted a new value.

## FEYNMAN CHECK

### Explain Angular Pipes Like I'm 10 Years Old
> Pipes are template FILTERS — they take a value, transform it, and display the result WITHOUT changing the original data. {{ price | currency:'USD' }} takes a number and displays "$42.00". The template-only nature is deliberate: pipes encourage pure transformations (no side effects, no mutation). Built-in pipes: DatePipe, CurrencyPipe, DecimalPipe, PercentPipe, UpperCasePipe, LowerCasePipe, TitleCasePipe, JsonPipe, AsyncPipe, KeyValuePipe, SlicePipe. The non-obvious performance detail: PURE pipes (the default) are memoised — Angular only re-runs them when the input reference changes. IMPURE pipes run on every change detection cycle. AsyncPipe is technically impure (it subscribes and re-runs when the Observable emits) but Angular handles it specially to avoid memory leaks.

### 5 Deep Conceptual Questions

**Q1: What is the difference between pure and impure pipes?**
> **A:** PURE pipes run only when the input value or reference changes — Angular caches the result and reuses it until the input changes. They must be pure functions (same input → same output, no side effects). IMPURE pipes run on EVERY change detection cycle regardless of input change. Use impure only when the pipe's output depends on external state that changes outside the input (e.g., a translate pipe reading a locale service, a filter pipe that must re-run when the filter criteria change). Impure pipes can tank performance — Angular runs them 60+ times per second in active apps.

**Q2: Mental model for AsyncPipe?**
> **A:** AsyncPipe subscribes to an Observable or Promise and returns the latest emitted value. When the component is destroyed, AsyncPipe AUTOMATICALLY unsubscribes — no manual ngOnDestroy needed. It also calls markForCheck() to trigger change detection in OnPush components when a new value arrives. The pattern "observable$ | async as value" in a template replaces the manual subscribe-in-ngOnInit + ngOnDestroy pattern entirely, and is the recommended way to handle async data in templates.

**Q3: Most dangerous misconception about pipes?**
> **A:** Piping an array filter makes it reactive:
> ```typescript
> // WRONG: pure pipe with array input only re-runs when the array REFERENCE changes
> @Pipe({ name: 'filter', pure: true })
> export class FilterPipe implements PipeTransform {
>   transform(items: Item[], query: string): Item[] {
>     return items.filter(i => i.name.includes(query));
>   }
> }
> // Template: {{ items | filter:searchTerm }}
> // If you push to 'items' without creating a new reference,
> // the pipe does NOT re-run even though the contents changed!
>
> // CORRECT: either make the pipe impure (performance cost)
> // or replace the array reference: this.items = [...this.items, newItem];
> ```

**Q4: When should you create a custom pipe vs a component method?**
> **A:** Create a PIPE when: the transformation is PURE (same input → same output), used in MULTIPLE templates, and conceptually a view transformation (formatting, filtering, mapping). Use a COMPONENT METHOD when: the transformation requires component state, is used only in that component, or has side effects. Pipes are also unit-testable without a TestBed — just instantiate and call transform(). This makes them ideal for formatting logic shared across the app.

**Q5: FAANG-grade definition of Angular pipes.**
> **A:** "Angular pipes are pure (by default) or impure template transformation functions implementing PipeTransform — applied in templates as value | pipeName:arg1:arg2 — with pure pipes memoised per input reference (safe to use in any template, zero re-run cost when inputs don't change), impure pipes running every CD cycle (expensive, use sparingly), and AsyncPipe as the reactive bridge between Observables/Promises and template rendering — automatically subscribing, unwrapping values, triggering markForCheck() on OnPush components, and unsubscribing on destroy."

#### Step 2 — TimeAgo Pipe
```typescript
// time-ago.pipe.ts
import { Pipe, PipeTransform } from '@angular/core';

@Pipe({ name: 'timeAgo', standalone: true, pure: false })  // impure — time changes
export class TimeAgoPipe implements PipeTransform {
  transform(value: Date | string | number): string {
    const date = new Date(value);
    const now = Date.now();
    const diff = Math.floor((now - date.getTime()) / 1000);

    if (diff < 60)  return diff + ' seconds ago';
    if (diff < 3600) return Math.floor(diff / 60) + ' minutes ago';
    if (diff < 86400) return Math.floor(diff / 3600) + ' hours ago';
    if (diff < 604800) return Math.floor(diff / 86400) + ' days ago';
    return date.toLocaleDateString();
  }
}
```

#### Step 3 — Highlight Pipe
```typescript
// highlight.pipe.ts
import { Pipe, PipeTransform } from '@angular/core';
import { DomSanitizer, SafeHtml } from '@angular/platform-browser';

@Pipe({ name: 'highlight', standalone: true, pure: true })
export class HighlightPipe implements PipeTransform {
  constructor(private sanitizer: DomSanitizer) {}

  transform(text: string, query: string): SafeHtml {
    if (!query || !text) return text;
    const escaped = query.split('').map(c => /[a-zA-Z0-9]/.test(c) ? c : '\' + c).join('');
    const re = new RegExp('(' + escaped + ')', 'gi');
    const highlighted = text.replace(re, '<mark>$1</mark>');
    return this.sanitizer.bypassSecurityTrustHtml(highlighted);
  }
}
```

#### Step 4 — SafeHtml Pipe
```typescript
// safe-html.pipe.ts
import { Pipe, PipeTransform } from '@angular/core';
import { DomSanitizer, SafeHtml } from '@angular/platform-browser';

@Pipe({ name: 'safeHtml', standalone: true, pure: true })
export class SafeHtmlPipe implements PipeTransform {
  constructor(private sanitizer: DomSanitizer) {}
  transform(value: string): SafeHtml {
    return this.sanitizer.bypassSecurityTrustHtml(value);
  }
}

// Usage: <div [innerHTML]="htmlContent | safeHtml"></div>
// Note: only use with trusted content — not for user input
```

#### Step 5 — Tests
```typescript
describe('TimeAgoPipe', () => {
  let pipe: TimeAgoPipe;
  beforeEach(() => { pipe = new TimeAgoPipe(); });

  it('returns seconds ago for recent times', () => {
    const d = new Date(Date.now() - 30000);  // 30 seconds ago
    expect(pipe.transform(d)).toBe('30 seconds ago');
  });
  it('returns minutes ago', () => {
    const d = new Date(Date.now() - 3 * 60 * 1000);  // 3 minutes ago
    expect(pipe.transform(d)).toBe('3 minutes ago');
  });
  it('returns hours ago', () => {
    const d = new Date(Date.now() - 2 * 3600 * 1000);  // 2 hours ago
    expect(pipe.transform(d)).toBe('2 hours ago');
  });
});
```
