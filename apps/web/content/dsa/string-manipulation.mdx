## WHY
String manipulation is the bread and butter of technical interviews. 
Because strings represent human-readable data (names, emails, JSON payloads), parsing and validating them is a daily requirement in software engineering. Common interview questions involve checking for Palindromes, validating Anagrams, or parsing string representations of integers (`parseInt()`). 
If you can fluently navigate strings using pointers and frequency maps, you demonstrate a solid grasp of fundamental data manipulation.

## THEORY
### Palindromes & Two Pointers
A palindrome is a string that reads the exact same forwards and backwards (e.g., "racecar"). The most efficient way to verify this is NOT to reverse the string and compare it. Creating a new reversed string costs `O(n)` extra memory. Instead, use the Two-Pointer technique on the original string, checking `left` and `right` indices until they meet in the middle.

### Character Frequencies (The ASCII Array)
If a problem asks: "Are these two strings Anagrams?" (e.g., "listen" and "silent"), you need to count how many times each character appears.
While you *could* use a `HashMap<Character, Integer>`, it is heavy and slow. If you know the string only contains lowercase English letters (a-z), you can use a simple `int[26]` array! The character `'a'` has an ASCII integer value of `97`. By doing `char - 'a'`, you perfectly map `'a'` to index `0` and `'z'` to index `25`. This is a classic, highly performant interview trick.

## CODE

### Level 1 — Beginner (Valid Palindrome)
```java
public class PalindromeCheck {
    // Check if a string is a palindrome.
    // Time Complexity: O(n), Space Complexity: O(1)
    public static boolean isPalindrome(String s) {
        int left = 0;
        int right = s.length() - 1;
        
        while (left < right) {
            if (s.charAt(left) != s.charAt(right)) {
                return false; // Mismatch found!
            }
            left++;
            right--;
        }
        return true; // Pointers met in the middle flawlessly
    }

    public static void main(String[] args) {
        System.out.println(isPalindrome("racecar")); // true
        System.out.println(isPalindrome("hello"));   // false
    }
}
```

### Level 2 — Building Blocks (Valid Anagram)
```java
public class AnagramCheck {
    // Problem: Check if string t is an anagram of string s.
    // Time Complexity: O(n), Space Complexity: O(1) 
    
    public static boolean isAnagram(String s, String t) {
        if (s.length() != t.length()) return false;
        
        // The genius 26-slot integer array instead of a HashMap!
        int[] charCounts = new int[26];
        
        for (int i = 0; i < s.length(); i++) {
            // Increment the count for string s
            charCounts[s.charAt(i) - 'a']++;
            
            // Decrement the count for string t
            charCounts[t.charAt(i) - 'a']--;
        }
        
        // If they are perfect anagrams, every single slot in the array must be exactly 0
        for (int count : charCounts) {
            if (count != 0) return false;
        }
        
        return true;
    }
}
```

### Level 3 — Intermediate (String to Integer - atoi)
```java
public class CustomParseInt {
    // Problem: Convert a String like "123" into the integer 123 from scratch.
    
    public static int myAtoi(String s) {
        int result = 0;
        
        for (int i = 0; i < s.length(); i++) {
            char c = s.charAt(i);
            
            // Convert the character digit into its actual integer value
            // e.g., '3' (ASCII 51) - '0' (ASCII 48) = integer 3
            int digit = c - '0';
            
            // Shift the existing result left by one decimal place, then add the new digit
            // Example for "123":
            // Loop 1: 0 * 10 + 1 = 1
            // Loop 2: 1 * 10 + 2 = 12
            // Loop 3: 12 * 10 + 3 = 123
            result = (result * 10) + digit; 
        }
        
        return result;
    }
}
```

### Level 4 — Advanced (Longest Palindromic Substring)
```java
public class ExpandAroundCenter {
    // Problem: Find the longest substring that is a palindrome.
    // Example: "babad" -> "bab" or "aba"
    
    // Instead of checking every substring (O(n^3)), we treat every character
    // as a potential "center" and expand outwards. Time: O(n^2), Space: O(1)
    public static String longestPalindrome(String s) {
        if (s == null || s.length() < 1) return "";
        
        int start = 0, end = 0;
        
        for (int i = 0; i < s.length(); i++) {
            // Case 1: Odd length palindromes (center is 1 char) e.g., "aba"
            int len1 = expand(s, i, i);
            
            // Case 2: Even length palindromes (center is 2 chars) e.g., "abba"
            int len2 = expand(s, i, i + 1);
            
            int len = Math.max(len1, len2);
            
            // If we found a longer palindrome, update our global boundaries
            if (len > end - start) {
                start = i - (len - 1) / 2;
                end = i + len / 2;
            }
        }
        return s.substring(start, end + 1);
    }
    
    private static int expand(String s, int left, int right) {
        // Keep expanding as long as the characters match!
        while (left >= 0 && right < s.length() && s.charAt(left) == s.charAt(right)) {
            left--;
            right++;
        }
        // Return the length of the palindrome we found
        return right - left - 1; 
    }
}
```

## REAL_WORLD
Parsing Strings to Integers (`Integer.parseInt()`) is one of the most frequently executed pieces of code in the world. When a browser sends a URL parameter (e.g., `?userId=55`), it arrives at your Java server as a pure String `"55"`. The server must run the `atoi` algorithm to convert it to actual mathematical bits.

Additionally, the concept of character mapping (like our `int[26]` frequency map) is the foundational logic used in **Compression Algorithms**. Huffman Coding, the algorithm used inside ZIP files and JPEG images, starts by calculating the frequency of every single byte in the file. It then assigns short binary codes to the most frequent characters and long binary codes to rare characters, drastically shrinking the total file size!

## INTERVIEW
**Q: Why do we use `s.charAt(i) - 'a'` in the Anagram problem?**
A: Characters in Java are fundamentally just 16-bit integers representing Unicode mappings. The lowercase letters 'a' through 'z' are perfectly sequential in memory (a=97, b=98, c=99, etc.). By mathematically subtracting 'a' (97) from any lowercase character, we normalize the range to 0-25. This allows us to use a lightning-fast `O(1)` array lookup instead of a slow, memory-heavy `HashMap`.

**Q: Can the `int[26]` trick be used for all string problems?**
A: No! It only works if the problem explicitly guarantees the string consists *only* of lowercase English letters. If the string contains uppercase letters, numbers, punctuation, or emojis, the math `c - 'a'` will produce negative numbers or massive index numbers, instantly crashing the program with an `ArrayIndexOutOfBoundsException`. For complex character sets, you must fallback to a `HashMap` or a `int[256]` ASCII array.

**Q: What is the time complexity of the "Expand Around Center" palindrome algorithm?**
A: It is `O(n^2)` quadratic time. There are `N` possible centers, and for each center, we might expand up to `N` times in the worst-case scenario (e.g., the string is "aaaaaaa"). While `O(n^2)` sounds slow, it is perfectly optimal for interviews. (There is a hyper-complex `O(n)` algorithm called Manacher's Algorithm, but interviewers will never expect you to know it).

### Explain String Manipulation Like I'm 10 Years Old
> Editing a string is like cutting and taping paper: every cut makes a new sheet. Replace, substring, split — each hands you a fresh copy. The smart move is plan all cuts, then make one new sheet. With two pointers you sweep once, writing kept characters forward, so removing spaces from a million-char string is one pass, not a million little copies.

### 5 Deep Conceptual Questions
**Q1: Why does substring sometimes leak memory?** > **A:** In old JVMs substring shared the parent char[], so a 2-char substring pinned a 1MB string. Modern JDK copies, fixing the leak but adding O(k). Know your runtime.
**Q2: ONE model?** > **A:** "Each op allocates; batch with a builder or two-pointer sweep."
**Q3: Misconception+code.** > **A:** ```java
// ❌ replace in loop reallocates
// ✅ char[] write index sweep
```
**Q4: Interplay with regex?** > **A:** Regex compiles to an NFA/DFA; precompile Pattern once, reuse — compiling per call is O(pattern) waste.
**Q5: Senior one-liner.** > **A:** "String manipulation is repeated allocation hidden behind tidy methods; experts collapse it to one builder pass."
