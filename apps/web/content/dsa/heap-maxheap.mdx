## WHY
You just learned about Min-Heaps, which guarantee the *smallest* element is always at the root.
But what if your application needs to constantly fetch the *largest* element? For example, an operating system needs to schedule the CPU process with the Highest Priority score, or a gaming leaderboard needs to instantly fetch the player with the Top Score.
You use a **Max-Heap**. The logic is identical to a Min-Heap, except the math is reversed: every parent must be GREATER THAN OR EQUAL TO its children!

## THEORY
### The Structure
Like the Min-Heap, the Max-Heap is a Complete Binary Tree stored flat inside an array.
**The Max-Heap Property:** $Parent \ge Children$.
Because of this property, the largest number in the array naturally bubbles up and settles at index `0`.

## CODE

### Level 1 — Max-Heap Implementation
Notice how the logic is identical to the Min-Heap, except we swap `<` for `>`.

```java
import java.util.ArrayList;

public class MaxHeap {
    private ArrayList<Integer> heap;

    public MaxHeap() {
        heap = new ArrayList<>();
    }

    private int parent(int i) { return (i - 1) / 2; }
    private int leftChild(int i) { return 2 * i + 1; }
    private int rightChild(int i) { return 2 * i + 2; }

    private void swap(int i, int j) {
        int temp = heap.get(i);
        heap.set(i, heap.get(j));
        heap.set(j, temp);
    }

    // --- INSERTION (O(log N)) ---
    public void insert(int value) {
        heap.add(value);
        heapifyUp(heap.size() - 1);
    }

    private void heapifyUp(int index) {
        // DIFFERENCE: While we are LARGER than our parent, swap us UP!
        while (index > 0 && heap.get(index) > heap.get(parent(index))) {
            swap(index, parent(index));
            index = parent(index);
        }
    }

    // --- EXTRACTION (O(log N)) ---
    public int extractMax() {
        if (heap.isEmpty()) throw new RuntimeException("Heap is empty!");
        
        int max = heap.get(0);
        int lastElement = heap.remove(heap.size() - 1);
        
        if (!heap.isEmpty()) {
            heap.set(0, lastElement);
            heapifyDown(0);
        }
        return max;
    }

    private void heapifyDown(int index) {
        int largest = index;
        int left = leftChild(index);
        int right = rightChild(index);

        // DIFFERENCE: Find the LARGEST child!
        if (left < heap.size() && heap.get(left) > heap.get(largest)) {
            largest = left;
        }
        if (right < heap.size() && heap.get(right) > heap.get(largest)) {
            largest = right;
        }

        if (largest != index) {
            swap(index, largest);
            heapifyDown(largest);
        }
    }
}
```

### Level 2 — Standard Library Integration (Java PriorityQueue)
You almost never write the 50 lines of Heap math manually in an interview or on the job. Every modern language has Heaps built-in!
In Java, the built-in Heap is called `PriorityQueue`.
**By default, Java's `PriorityQueue` is a Min-Heap.**

```java
import java.util.PriorityQueue;

// This is a Min-Heap!
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.add(10);
minHeap.add(5);
System.out.println(minHeap.poll()); // Prints 5
```

To turn it into a **Max-Heap**, you pass a custom `Comparator` into the constructor that reverses the sorting logic!

```java
import java.util.PriorityQueue;
import java.util.Collections;

// MAGIC: Pass Collections.reverseOrder() to make it a Max-Heap!
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());

maxHeap.add(10);
maxHeap.add(50);
maxHeap.add(20);

// It automatically extracts the largest element!
System.out.println(maxHeap.poll()); // Prints 50
System.out.println(maxHeap.poll()); // Prints 20
```

### Level 3 — Custom Object Max-Heaps
What if you are tracking `Patient` objects instead of simple Integers? You must write a custom Comparator using Java Lambdas to tell the PriorityQueue *exactly* how to calculate priority.

```java
class Patient {
    String name;
    int urgencyScore; // 100 = Heart Attack, 10 = Sprained Ankle
    
    public Patient(String name, int score) {
        this.name = name;
        this.urgencyScore = score;
    }
}

// We want a Max-Heap based on Urgency Score!
// We compare p2's score to p1's score to create descending order (Max-Heap).
PriorityQueue<Patient> erQueue = new PriorityQueue<>(
    (p1, p2) -> Integer.compare(p2.urgencyScore, p1.urgencyScore)
);

erQueue.add(new Patient("Alice", 20));
erQueue.add(new Patient("Bob", 95));

// Bob is extracted first because 95 is the maximum score!
System.out.println(erQueue.poll().name); // "Bob"
```

## REAL_WORLD
**The "Two Heaps" Pattern (Streaming Median)**
A classic FAANG interview question is: *"You are receiving a constant stream of numbers. At any moment, I can ask you for the Median value of all numbers seen so far. Return it in $O(1)$ time."*
If you store the numbers in an array and sort it every time, it takes $O(N \log N)$.
The genius solution uses **Two Heaps**:
1. A **Max-Heap** to store the smaller half of the numbers.
2. A **Min-Heap** to store the larger half of the numbers.

You balance the two heaps so their sizes differ by at most 1. 
Because the largest number of the small half is at the top of the Max-Heap, and the smallest number of the large half is at the top of the Min-Heap, the Median is ALWAYS sitting at the top of one (or both) of the heaps! You can find the median in $O(1)$ time, and insert new numbers in $O(\log N)$ time!

## INTERVIEW
**Q: You use `PriorityQueue` to manage massive tasks. A task in the middle of the queue is cancelled by the user. You call `queue.remove(cancelledTask)`. What is the Time Complexity of this removal operation?**
A: **$O(N)$.**
This is a massive trap in Java. While extracting the root node via `poll()` takes $O(\log N)$ time, calling `.remove(Object)` takes $O(N)$ time!
Because a Heap is an unsorted array, Java has no mathematical way to know where the `cancelledTask` is located. It is forced to perform a linear scan ($O(N)$) across the entire underlying array to find the object. Once found, it replaces it with the last element and bubbles it down ($O(\log N)$). 
If you need to rapidly delete arbitrary elements from the middle of a priority queue, standard Heaps are insufficient. You must use an advanced data structure like a **TreeSet** (which can find and remove arbitrary elements in $O(\log N)$ time using a Red-Black Tree), or augment the Heap with a HashMap that tracks the exact array index of every object!
