## WHY
Once you have an array, what do you do with it? 
In competitive programming and technical interviews, you are rarely asked to simply "create an array". Instead, you are asked to manipulate them: reverse the elements, rotate the array to the right by K steps, remove duplicates, or find pairs that sum to a target.

Mastering fundamental array operations is the absolute prerequisite to passing any coding interview. If you do not intimately understand how to swap variables, use two pointers, or manage boundaries without throwing an `ArrayIndexOutOfBoundsException`, you will fail on the very first question.

## THEORY
### The Swapping Mechanism
If you want to swap the values of `a` and `b`, you cannot simply write `a = b; b = a;`. Once you assign `b` to `a`, the original value of `a` is instantly overwritten and destroyed. You must use a `temp` (temporary) variable to hold the value of `a` safely before overwriting it.

### The Two-Pointer Technique
Many array operations require iterating through the array. The naive approach uses a single `for` loop from index 0 to N.
The **Two-Pointer Technique** uses two variables (pointers) simultaneously. Typically, `left` starts at index 0, and `right` starts at the last index. In a `while(left < right)` loop, the pointers move towards the center, allowing you to process the array from both ends simultaneously in `O(n)` time, halving the actual number of iterations!

## CODE

### Level 1 — Beginner (Swapping and Finding Min/Max)
```java
public class BasicOps {
    public static void main(String[] args) {
        int[] arr = {15, 2, 8, 45, 10};
        
        // 1. Swapping Elements
        int temp = arr[1]; // Hold '2' safely
        arr[1] = arr[2];   // Overwrite '2' with '8'
        arr[2] = temp;     // Overwrite '8' with '2'
        
        // 2. Finding the Maximum Value
        int max = arr[0]; // Assume first element is max
        for (int num : arr) {
            if (num > max) {
                max = num; // We found a new champion!
            }
        }
        System.out.println("Maximum: " + max); // 45
    }
}
```

### Level 2 — Building Blocks (Array Reversal)
```java
import java.util.Arrays;

public class Reversal {
    // Reversing an array IN-PLACE using Two Pointers
    // Space Complexity: O(1)
    // Time Complexity: O(n)
    public static void reverse(int[] nums) {
        int left = 0;
        int right = nums.length - 1;
        
        while (left < right) {
            // Swap the outer elements
            int temp = nums[left];
            nums[left] = nums[right];
            nums[right] = temp;
            
            // Move pointers inward!
            left++;
            right--;
        }
    }

    public static void main(String[] args) {
        int[] data = {1, 2, 3, 4, 5};
        reverse(data);
        System.out.println(Arrays.toString(data)); // [5, 4, 3, 2, 1]
    }
}
```

### Level 3 — Intermediate (Two Sum - Sorted)
```java
public class SortedTwoSum {
    // Given a SORTED array, find two numbers that add up to a target value.
    // If we used nested loops, it would be O(n^2).
    // Using Two Pointers, we solve it in O(n)!
    public static boolean hasTargetSum(int[] sortedArr, int target) {
        int left = 0;
        int right = sortedArr.length - 1;
        
        while (left < right) {
            int currentSum = sortedArr[left] + sortedArr[right];
            
            if (currentSum == target) return true; // Found it!
            
            // If the sum is too small, we need a bigger number. Move left pointer right.
            if (currentSum < target) {
                left++; 
            } 
            // If the sum is too big, we need a smaller number. Move right pointer left.
            else {
                right--; 
            }
        }
        return false;
    }
}
```

### Level 4 — Advanced (Array Rotation)
```java
import java.util.Arrays;

public class Rotation {
    // Problem: Rotate an array to the right by K steps.
    // Example: [1,2,3,4,5,6,7], k=3 -> [5,6,7,1,2,3,4]
    
    // The Genius "Three Reversal" Algorithm (O(n) time, O(1) space)
    public static void rotate(int[] nums, int k) {
        k = k % nums.length; // If k is larger than array length, wrap it around
        
        // Step 1: Reverse the ENTIRE array
        // [7, 6, 5, 4, 3, 2, 1]
        reverseRange(nums, 0, nums.length - 1);
        
        // Step 2: Reverse the first K elements
        // [5, 6, 7, 4, 3, 2, 1]
        reverseRange(nums, 0, k - 1);
        
        // Step 3: Reverse the remaining N - K elements
        // [5, 6, 7, 1, 2, 3, 4]
        reverseRange(nums, k, nums.length - 1);
    }
    
    private static void reverseRange(int[] nums, int start, int end) {
        while (start < end) {
            int temp = nums[start];
            nums[start] = nums[end];
            nums[end] = temp;
            start++; end--;
        }
    }
}
```

## REAL_WORLD
While LeetCode makes you manually reverse arrays and find maximums, in the real world of Java Enterprise development, you virtually never write these loops from scratch.

Why? Because manual `for` loops are highly susceptible to "Off-By-One" bugs (writing `i <= length` instead of `i < length`, causing an instant `ArrayIndexOutOfBoundsException`).
Instead, Java engineers rely on the `java.util.Arrays` utility class or the Streams API.

If you need to sort an array: `Arrays.sort(arr);`
If you need to find a value: `Arrays.binarySearch(arr, target);`
If you need the maximum value: `Arrays.stream(arr).max().getAsInt();`

However, **interviews forbid you from using these utilities.** In an interview, if the prompt is "Sort an array," writing `Arrays.sort(arr)` will instantly fail you because the interviewer wants to see you write QuickSort or MergeSort from scratch to prove you understand the underlying pointer mathematics and memory management.

## INTERVIEW
**Q: What does "In-Place" mean?**
A: An algorithm is "in-place" if it modifies the input data structure directly without requiring any extra auxiliary data structures (like creating a brand new array to hold the result). This achieves `O(1)` Space Complexity. The Two-Pointer reversal is an in-place algorithm. 

**Q: Why do we use `k % nums.length` in the Array Rotation algorithm?**
A: If an array has 5 elements, and you rotate it to the right by exactly 5 steps, the array ends up looking exactly the same as it started. Rotating by 6 steps is mathematically identical to rotating by 1 step. Using the modulo operator (`%`) calculates the remainder, stripping away useless full-rotations and preventing an `ArrayIndexOutOfBoundsException` when `k` is larger than the array.

**Q: In the Two Sum (Sorted) problem, why does moving the left pointer increase the sum?**
A: Because the array is strictly sorted in ascending order (e.g., `[2, 5, 8, 12]`). The `left` pointer starts at the smallest number (`2`), and the `right` pointer starts at the largest (`12`). If their sum (`14`) is less than our target (`17`), we must replace the `2` with a larger number. The only way to get a larger number is to move the `left` pointer one step to the right (to `5`).
