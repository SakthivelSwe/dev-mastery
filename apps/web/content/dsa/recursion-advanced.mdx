## WHY
Basic recursion is used to traverse a straight line, like peeling a string one character at a time. 
But what if the problem involves making choices? What if you need to generate all possible permutations of a password, or explore every possible branch of a maze to find the exit? 
Advanced Recursion allows a function to call itself **multiple times** from the same block of code, effectively splitting the universe into parallel timelines. This branching execution creates the foundation for backtracking, tree traversal, and the most powerful algorithms in computer science.

## THEORY
### Branching and the Execution Tree
If a recursive function calls itself twice, it creates an "Execution Tree".
Level 0: 1 function call.
Level 1: 2 function calls.
Level 2: 4 function calls.
Level 3: 8 function calls.
The execution expands exponentially. Understanding *when* and *in what order* these branches execute is critical. The JVM does NOT execute them in parallel. It executes them strictly using **Depth-First Search (DFS)**. It will follow the very first branch all the way down to the absolute bottom Base Case before it ever begins executing the second branch.

### Returning and Unwinding
When the deepest branch hits a Base Case, it returns its result up to its parent. The parent function, which had been frozen in memory waiting for that result, finally receives it. It can then use that result, perhaps add it to the result of its second branch, and return that combined answer up to *its* parent. This process of bubbling the answers back up to the top is called "Unwinding".

## CODE

### Level 1 — Beginner (Multiple Branching)
```java
public class Branching {
    // Problem: Print all binary combinations of length N
    // If N = 2, print: 00, 01, 10, 11
    
    public static void generateBinary(int n, String prefix) {
        // Base Case: When the prefix reaches the desired length, print it!
        if (n == 0) {
            System.out.println(prefix);
            return;
        }
        
        // Branch 1: What if the next digit is a 0?
        generateBinary(n - 1, prefix + "0");
        
        // Branch 2: What if the next digit is a 1?
        generateBinary(n - 1, prefix + "1");
        
        // The JVM will follow the '0' branch all the way to the bottom 
        // before it ever executes this '1' branch!
    }

    public static void main(String[] args) {
        generateBinary(2, ""); 
    }
}
```

### Level 2 — Building Blocks (Generating Permutations)
```java
import java.util.List;
import java.util.ArrayList;

public class Permutations {
    // Problem: Given "abc", generate all possible arrangements.
    // (abc, acb, bac, bca, cab, cba)
    
    public static void getPermutations(String str, String current, List<String> result) {
        // Base Case: If we've used all letters, save the current arrangement!
        if (str.isEmpty()) {
            result.add(current);
            return;
        }
        
        // We must try making EVERY character in the remaining string the "next" character
        for (int i = 0; i < str.length(); i++) {
            
            // Choose the character at index i
            char chosenChar = str.charAt(i);
            
            // Extract the REST of the string (everything except the chosen character)
            String remainingString = str.substring(0, i) + str.substring(i + 1);
            
            // Recurse with the chosen character added to our current arrangement!
            getPermutations(remainingString, current + chosenChar, result);
        }
    }
}
```

### Level 3 — Intermediate (Subset Generation / Combinations)
```java
import java.util.List;
import java.util.ArrayList;

public class Subsets {
    // Problem: Given an array [1, 2, 3], generate all possible subsets.
    // A subset can be empty [], or the full set [1,2,3]. Order does not matter.
    // Total subsets = 2^N.
    
    public static void generateSubsets(int[] nums, int index, List<Integer> currentSubset) {
        // Base Case: We have made a decision for every single number in the array.
        // We MUST print/save the subset!
        if (index == nums.length) {
            System.out.println(currentSubset);
            return;
        }
        
        // At every number, we have exactly TWO choices:
        
        // Choice 1: EXCLUDE the current number from our subset
        generateSubsets(nums, index + 1, new ArrayList<>(currentSubset));
        
        // Choice 2: INCLUDE the current number in our subset
        currentSubset.add(nums[index]);
        generateSubsets(nums, index + 1, new ArrayList<>(currentSubset));
    }
}
```

### Level 4 — Advanced (Merge Sort Full Algorithm)
```java
import java.util.Arrays;

public class MergeSortImpl {
    // The most famous application of Advanced Recursion (Divide & Conquer)
    // Time: O(N log N) | Space: O(N)
    
    public static void mergeSort(int[] arr) {
        int n = arr.length;
        
        // Base Case: An array of 1 element is already sorted!
        if (n < 2) return;
        
        // 1. DIVIDE
        int mid = n / 2;
        int[] left = Arrays.copyOfRange(arr, 0, mid);
        int[] right = Arrays.copyOfRange(arr, mid, n);
        
        // 2. RECURSIVELY SORT BOTH HALVES
        // This splits the array down to 1 element, then bubbles back up!
        mergeSort(left);
        mergeSort(right);
        
        // 3. CONQUER (Merge the two sorted halves back into the original array)
        merge(arr, left, right);
    }
    
    private static void merge(int[] original, int[] left, int[] right) {
        int i = 0, j = 0, k = 0;
        
        while (i < left.length && j < right.length) {
            if (left[i] <= right[j]) {
                original[k++] = left[i++];
            } else {
                original[k++] = right[j++];
            }
        }
        while (i < left.length) original[k++] = left[i++];
        while (j < right.length) original[k++] = right[j++];
    }
}
```

## REAL_WORLD
Advanced recursion is heavily utilized in Artificial Intelligence and Game Engines.

If you write an AI for Chess or Tic-Tac-Toe, you use a recursive algorithm called **Minimax**. The current state of the board is passed into a recursive function. The function generates a `for` loop for every possible valid move you can make. Inside that loop, it calls the recursive function again, which then generates a loop for every possible move the *opponent* can make! 
The execution tree branches out millions of times into the future, scoring the final boards, and bubbling the best mathematical score all the way back up to the top to decide the next move.

Without the JVM's Call Stack automatically keeping track of the local variables for every single branch of the game, writing a Minimax AI would require thousands of lines of incredibly complex state-management code.

## INTERVIEW
**Q: What is the Time Complexity of generating all Permutations of a string?**
A: It is `O(N!)` (O of N-Factorial). For a string of length 3, there are 3 * 2 * 1 = 6 permutations. For a string of length 10, there are 3.6 million permutations. For a string of length 20, there are 2.4 quintillion permutations! Algorithms that branch in this manner are utterly impossible to run on large datasets, and interviewers will expect you to explicitly state that the inputs for these algorithms must be very small.

**Q: In the Subset Generation code, why do we use `new ArrayList<>(currentSubset)` when recursing?**
A: In Java, Objects (like `ArrayList`) are passed by Reference. If you simply pass `currentSubset` down into the recursive branch, every single branch in the entire execution tree will be adding and deleting from the exact same physical list in memory! By the time the tree finishes executing, your list will be a chaotic mess. By creating a `new` ArrayList, you create a perfect snapshot of the data, isolating the parallel branches from each other. (An alternative to this is Backtracking, which manually undoes the changes).

**Q: In Merge Sort, why does it divide in half (`n / 2`) instead of just peeling off 1 element?**
A: If a recursive function peels off 1 element at a time, the Call Stack depth becomes `N`. If you sort 100,000 items, the Call Stack hits 100,000 frames and throws a `StackOverflowError`. By dividing the array in half, the stack depth is logarithmic (`log N`). Sorting 100,000 items only creates a Call Stack depth of 17! This guarantees the recursion will never crash the JVM.
