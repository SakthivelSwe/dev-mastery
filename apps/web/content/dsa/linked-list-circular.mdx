## WHY
In both Singly and Doubly Linked Lists, the final node always points to `null`. This represents the physical "end" of the data structure. If you are traversing the list and you hit `null`, you must stop.

But what if the data you are modeling doesn't have an end? 
Imagine a multiplayer board game (like Monopoly) with 4 players. After Player 4 takes their turn, who goes next? It instantly loops back to Player 1! Or imagine the Operating System's CPU Task Scheduler allocating processing time to 5 running applications. After App 5 gets its millisecond of CPU time, it loops back to App 1.
A **Circular Linked List** models infinite, repeating sequences perfectly. The final node simply points its `next` reference back to the Head node, forming an unbreakable, infinite ring.

## THEORY
### The Infinite Ring
Because there is no `null` terminator, traversing a Circular Linked List is inherently dangerous. If you write a standard `while (current != null)` loop, your application will freeze and loop infinitely until it crashes. 
To safely traverse a Circular Linked List, you must keep a reference to the starting node, and stop the loop the moment you reach the start node again.

### The Tail Optimization
In a standard Singly Linked List, inserting a new node at the very end requires an `O(N)` traversal from the Head. 
In a Circular Linked List, if you maintain a single pointer to the **Tail** node (instead of the Head), you can access BOTH ends of the list instantly in `O(1)` time! Why? Because `tail.next` is mathematically guaranteed to be the Head! This is a massive architectural optimization.

## CODE

### Level 1 — Beginner (Creating the Ring)
```java
class Node {
    int val;
    Node next;
    Node(int val) { this.val = val; }
}

public class BasicCircular {
    public static void main(String[] args) {
        Node node1 = new Node(10);
        Node node2 = new Node(20);
        Node node3 = new Node(30);
        
        node1.next = node2;
        node2.next = node3;
        
        // The magic step! Point the last node back to the first!
        node3.next = node1;
        
        // We have successfully created an infinite loop!
    }
}
```

### Level 2 — Building Blocks (Safe Traversal)
```java
public class SafeTraversal {
    public static void printCircular(Node head) {
        if (head == null) return;
        
        Node current = head;
        
        // Use a DO-WHILE loop. 
        // This guarantees we process the Head node first, and then stop 
        // the exact moment we wrap around and see the Head again.
        do {
            System.out.print(current.val + " -> ");
            current = current.next;
        } while (current != head);
        
        System.out.println("(Back to Start)");
    }
}
```

### Level 3 — Intermediate (O(1) Head & Tail Insertion)
```java
public class OptimizedCircularList {
    // We ONLY track the Tail pointer!
    Node tail = null;
    
    // Time: O(1) Instant.
    public void addFirst(int val) {
        Node newNode = new Node(val);
        
        if (tail == null) {
            // First node ever created. It points to itself!
            tail = newNode;
            tail.next = tail; 
        } else {
            // Insert between Tail and old Head
            newNode.next = tail.next; // New node points to old Head
            tail.next = newNode;      // Tail points to New node
        }
    }
    
    // Time: O(1) Instant.
    public void addLast(int val) {
        // Adding to the end is exactly the same as adding to the beginning...
        addFirst(val);
        // ...except we move the Tail pointer forward by 1!
        tail = tail.next;
    }
}
```

### Level 4 — Advanced (Josephus Problem)
```java
public class Josephus {
    // The Josephus Problem is a famous theoretical computer science problem.
    // N people stand in a circle. Every K-th person is eliminated.
    // Given N and K, find the last person standing.
    
    public static int findSurvivor(int n, int k) {
        // Step 1: Build a Circular Linked List of N people
        Node head = new Node(1);
        Node prev = head;
        for (int i = 2; i <= n; i++) {
            prev.next = new Node(i);
            prev = prev.next;
        }
        prev.next = head; // Close the circle
        
        // Step 2: Eliminate every K-th person!
        Node current = head;
        Node before = prev; // Keeps track of the node BEFORE current
        
        // Loop until there is only 1 node left pointing to itself
        while (current.next != current) {
            
            // Move K-1 steps forward
            for (int i = 0; i < k - 1; i++) {
                before = current;
                current = current.next;
            }
            
            // Eliminate 'current' by wiring the 'before' node to skip it!
            System.out.println("Eliminated Person: " + current.val);
            before.next = current.next; 
            current = before.next; // Move to the next person
        }
        
        return current.val; // The sole survivor!
    }
}
```

## REAL_WORLD
Circular Linked Lists are the cornerstone of OS architecture and low-level system design.

In Operating Systems, **Round-Robin Task Scheduling** relies entirely on a Circular Linked List. The OS places all active software applications into a ring. The CPU grants Application A exactly 10 milliseconds of compute time. When time expires, the CPU simply moves the pointer to `current = current.next`, instantly pausing App A and launching App B. When it reaches the end of the list, it loops infinitely back to App A. This ensures every single application gets a fair share of the CPU without ever hitting a "null" wall.

Similarly, in multiplayer video games, turn-based systems use Circular Lists to manage the Turn Order. When a player dies or disconnects from the server, their Node is deleted from the ring in `O(1)` time, and the game continues infinitely for the remaining players.

## INTERVIEW
**Q: How do you avoid infinite loops when traversing a Circular Linked List?**
A: You must always retain a reference to the starting node (usually the Head) before the loop begins. Instead of checking for `null`, your loop condition must check if the `current` pointer has wrapped around and become equal to the starting node reference again (`current == head`). Using a `do-while` loop is the cleanest syntax for this.

**Q: Can a Circular Linked List be Doubly Linked?**
A: Yes, absolutely. A Circular Doubly Linked List is incredibly powerful. The Head node's `prev` pointer points to the Tail, and the Tail node's `next` pointer points to the Head. This allows infinite, seamless wrapping traversal in *both* directions.

**Q: Why does tracking ONLY the Tail pointer make both `addFirst()` and `addLast()` execute in O(1) time?**
A: Because in a circular list, the Head is always exactly one pointer away from the Tail (`tail.next = head`). If you tracked the Head, adding to the end would require an `O(N)` traversal to find the Tail. By tracking the Tail, you get instant access to the Tail, AND instant access to the Head!
