## WHY
Imagine you are browsing the internet. You visit Google, then Reddit, then YouTube. You click the "Back" button in your browser. Where does it take you? It takes you to Reddit (the most recent site). You click "Back" again, and it takes you to Google.

This exact behavior—where the **Last** thing you added is the **First** thing you remove—is called **LIFO** (Last-In, First-Out). A **Stack** is a data structure specifically designed to enforce this exact limitation. By mathematically restricting how data is added and removed, Stacks provide incredibly fast `O(1)` performance and are the backbone of recursion, undo buttons, and parsing algorithms.

## THEORY
### LIFO Restriction
A Stack is not a new physical way to store memory. Under the hood, a Stack is just an Array or a Linked List! 
However, it is an "Abstract Data Type". It wraps the array and strictly forbids you from inserting or deleting elements from the middle. You are only allowed to interact with the absolute top of the Stack.
1. **Push**: Place a new item on top of the Stack.
2. **Pop**: Remove the item currently on top of the Stack.
3. **Peek**: Look at the item on top, without removing it.

### The Call Stack
You use a Stack every single day without realizing it. Whenever you call a method in Java, the JVM creates a "Stack Frame" (containing local variables) and **Pushes** it onto the Thread's Call Stack. If that method calls another method, a new frame is Pushed on top. When a method finishes executing with a `return` statement, its frame is **Popped** off, allowing the JVM to resume the previous method exactly where it left off!

## CODE

### Level 1 — Beginner (Using the Java Stack)
```java
import java.util.Stack;

public class BasicStack {
    public static void main(String[] args) {
        // Java provides a built-in Stack class
        Stack<String> browserHistory = new Stack<>();
        
        // PUSH: Add to the top
        browserHistory.push("Google.com");
        browserHistory.push("Reddit.com");
        browserHistory.push("YouTube.com");
        
        // PEEK: Look at the top without removing
        System.out.println("Current Site: " + browserHistory.peek()); // YouTube
        
        // POP: Remove the top element
        String lastSite = browserHistory.pop(); // Removes YouTube
        System.out.println("Went back to: " + browserHistory.peek()); // Reddit
    }
}
```

### Level 2 — Building Blocks (Array Implementation)
```java
public class CustomArrayStack {
    // Implementing a Stack from scratch using an Array!
    private int[] data;
    private int topIndex; // Points to the top element
    
    public CustomArrayStack(int capacity) {
        data = new int[capacity];
        topIndex = -1; // -1 means the stack is totally empty!
    }
    
    // Time: O(1)
    public void push(int val) {
        if (topIndex == data.length - 1) {
            throw new StackOverflowError("Stack is Full!");
        }
        // Increment the pointer, then insert the value
        topIndex++;
        data[topIndex] = val;
    }
    
    // Time: O(1)
    public int pop() {
        if (topIndex == -1) {
            throw new RuntimeException("Stack is Empty!");
        }
        // Grab the value, then decrement the pointer!
        // We don't even need to delete the old data; it will be overwritten later.
        int val = data[topIndex];
        topIndex--;
        return val;
    }
}
```

### Level 3 — Intermediate (Valid Parentheses)
```java
import java.util.Stack;

public class ValidParentheses {
    // Problem: Given a string containing '(', ')', '{', '}', '[', ']',
    // determine if the input string is mathematically valid.
    // Examples: "()" is valid. "([)]" is invalid. "{[]}" is valid.
    
    public static boolean isValid(String s) {
        Stack<Character> stack = new Stack<>();
        
        for (char c : s.toCharArray()) {
            // If it's an OPENING bracket, Push it onto the stack!
            if (c == '(' || c == '{' || c == '[') {
                stack.push(c);
            } 
            // If it's a CLOSING bracket...
            else {
                // If stack is empty, there is no matching opening bracket!
                if (stack.isEmpty()) return false;
                
                // Pop the top opening bracket and check if they match!
                char top = stack.pop();
                if (c == ')' && top != '(') return false;
                if (c == '}' && top != '{') return false;
                if (c == ']' && top != '[') return false;
            }
        }
        
        // If the stack is perfectly empty at the end, it is valid!
        return stack.isEmpty();
    }
}
```

### Level 4 — Advanced (Min Stack)
```java
import java.util.Stack;

public class MinStack {
    // Problem: Design a Stack that supports push, pop, top, 
    // AND retrieving the minimum element in strictly O(1) time.
    
    // The genius solution: Use TWO stacks!
    private Stack<Integer> mainStack;
    private Stack<Integer> minTracker; // ONLY tracks minimums!
    
    public MinStack() {
        mainStack = new Stack<>();
        minTracker = new Stack<>();
    }
    
    public void push(int val) {
        mainStack.push(val);
        
        // If the minTracker is empty, OR this value is <= the current minimum,
        // we push it onto the minTracker as the new reigning champion!
        if (minTracker.isEmpty() || val <= minTracker.peek()) {
            minTracker.push(val);
        }
    }
    
    public void pop() {
        int poppedValue = mainStack.pop();
        
        // If the value we just popped was the reigning champion minimum,
        // we MUST pop it from the minTracker too, revealing the previous champion!
        if (poppedValue == minTracker.peek()) {
            minTracker.pop();
        }
    }
    
    public int getMin() {
        return minTracker.peek(); // Instant O(1) lookup!
    }
}
```

## REAL_WORLD
Why use a `java.util.Stack` when you can just use an `ArrayList`? 

In legacy Java, `java.util.Stack` actually extends `Vector`. Every single method in `Vector` is wrapped in a `synchronized` lock for thread safety. This means every time you call `.push()`, the JVM must acquire and release a heavy OS lock, causing a massive performance penalty. Because of this fatal design flaw, **the official Java Documentation explicitly states you should NEVER use `java.util.Stack` in modern production code.**

Instead, modern enterprise Java engineers use **`ArrayDeque`** (Double Ended Queue) as a Stack!
```java
// The modern, ultra-fast, lock-free way to use a Stack in Java!
Deque<String> fastStack = new ArrayDeque<>();
fastStack.push("Hello");
fastStack.pop();
```
`ArrayDeque` is backed by a resizable array, has zero synchronization overhead, and is significantly faster than both `Stack` and `LinkedList`. 

## INTERVIEW
**Q: What is a Stack Overflow?**
A: It is a fatal error that occurs when you try to push data onto a Stack that has reached its maximum memory capacity. In software engineering, it almost always refers to the JVM's Call Stack. If you write a recursive method that lacks a base case, the JVM will push millions of method frames onto the stack in an infinite loop, eventually exhausting the strict 1MB Stack memory limit and violently crashing the application with a `StackOverflowError`.

**Q: Can you implement a Stack using a Singly Linked List?**
A: Yes, easily! You simply maintain a `head` pointer. To `push()`, you create a new node and insert it at the Head (`newNode.next = head`). To `pop()`, you grab the value at the Head, and then sever it by shifting the Head pointer forward (`head = head.next`). Both operations execute perfectly in `O(1)` time.

**Q: In the Valid Parentheses algorithm, why does it fail if the Stack isn't empty at the very end?**
A: If the string is `"({["`, the algorithm pushes all three characters onto the stack. Because there are no closing brackets, the loop finishes. If the method simply returned `true` at the end, it would incorrectly validate an unclosed string. If the stack is not empty, it means there are dangling opening brackets that were never resolved by a closing bracket.
