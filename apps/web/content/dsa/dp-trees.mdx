## WHY
Dynamic Programming is usually taught on 1D Arrays or 2D Grids. But what if the data structure is a Tree?
Imagine a corporate hierarchy (a Tree) where each node is an employee with a "fun" rating. You want to invite employees to a party to maximize the total "fun" score, but with one strict rule: **You cannot invite an employee AND their direct boss.**
If you try to solve this with a greedy approach, it fails. You must use DP. But there is no 1D array to store `dp[i]`!
**DP on Trees** merges Tree Traversal (Post-Order DFS) with Dynamic Programming. Because every node in a tree is the root of its own isolated sub-tree, a node can perfectly aggregate the optimal DP answers from its children and pass the final optimal answer up to its parent!

## THEORY
### The Post-Order Advantage
In a 1D Array, we usually build answers from Left to Right (Bottom-Up).
In a Tree, "Bottom-Up" means starting at the Leaves and working our way up to the Root.
This perfectly matches **Post-Order Traversal** (Left, Right, Root).
When the DFS is evaluating a parent node, it mathematically guarantees that the left child and right child have *already completely finished* their DP calculations. The parent simply takes the final answers from the children, runs a quick `Math.max()` calculation, and returns!

## CODE

### Pattern 1 — House Robber III (The Classic Tree DP)
*Problem:* You are robbing houses arranged in a Binary Tree. If you rob a node, you CANNOT rob its direct children. Return the maximum amount of money you can rob.

**1. State Definition:** For every single node, we must track exactly two possible realities:
- `ans[0]`: The maximum money if we **DO NOT** rob this node.
- `ans[1]`: The maximum money if we **DO** rob this node.

Because we need to return *two* numbers to the parent, our DFS function will return an `int[]` of size 2!

**2. Recurrence Relation:**
- If we **DO NOT** rob the current node, we are free to rob or skip the children. So we just take the absolute maximum of whatever the children return:
  `ans[0] = Math.max(left[0], left[1]) + Math.max(right[0], right[1])`
- If we **DO** rob the current node, we mathematically CANNOT rob the children! We are forced to take their "Did Not Rob" values:
  `ans[1] = currentNode.val + left[0] + right[0]`

**3. Base Case:** If the node is `null`, return `[0, 0]`.

```java
class TreeNode {
    int val;
    TreeNode left, right;
}

public class TreeDP {

    public int rob(TreeNode root) {
        // Kick off the Post-Order DFS!
        int[] result = dfs(root);
        // The root returns the final two possibilities. Take the max!
        return Math.max(result[0], result[1]);
    }

    private int[] dfs(TreeNode node) {
        // Base Case: Empty node yields 0 money
        if (node == null) return new int[]{0, 0};

        // 1. Post-Order: Plunge to the children first!
        int[] left = dfs(node.left);
        int[] right = dfs(node.right);

        // 2. The Current Node's DP Logic
        int[] current = new int[2];

        // Option 0: Do NOT rob this node
        // We can safely take the MAX profit from the left and right children
        current[0] = Math.max(left[0], left[1]) + Math.max(right[0], right[1]);

        // Option 1: DO rob this node
        // We are FORCED to take the "Did not rob" (index 0) from the children!
        current[1] = node.val + left[0] + right[0];

        // 3. Pass the two realities up to the parent!
        return current;
    }
}
```

### Pattern 2 — Maximum Path Sum
*Problem:* Find the maximum path sum between ANY two nodes in a Binary Tree. The path does not need to pass through the root!

This introduces a brilliant DP concept: The Global Variable.
Our DFS function calculates the maximum "straight line" path down a single branch (so the parent can use it). But inside the DFS, we also combine the left and right branches to see if the current node forms an incredible "Arch" that beats the global maximum!

```java
public class MaxPathSum {
    int globalMax = Integer.MIN_VALUE;

    public int maxPathSum(TreeNode root) {
        dfs(root);
        return globalMax;
    }

    private int dfs(TreeNode node) {
        if (node == null) return 0;

        // Post-Order: Get max straight-line paths from children.
        // If a child returns a negative number, ignore it! (Math.max(0, ...))
        int leftMax = Math.max(0, dfs(node.left));
        int rightMax = Math.max(0, dfs(node.right));

        // THE ARCH: What if the path goes up the left, through ME, and down the right?
        // This is a closed path. It cannot be passed to the parent!
        int currentArchSum = node.val + leftMax + rightMax;
        
        // Did this arch beat our all-time global record?
        globalMax = Math.max(globalMax, currentArchSum);

        // Return the best "Straight Line" to the parent so they can use us!
        return node.val + Math.max(leftMax, rightMax);
    }
}
```

## REAL_WORLD
**Abstract Syntax Trees (AST)**
When the Java Compiler (Javac) or JavaScript V8 Engine parses your code, it converts it into a Tree of operations (an AST).
For example, `(5 + 3) * 2` becomes a Tree where `*` is the root, `2` is the right child, and `+` is the left child.
To evaluate or optimize this code, the compiler runs DP on the Tree! 
It uses Post-Order traversal. It dives to `(5 + 3)`. The `+` node asks its children for their values. The leaves `5` and `3` return their values. The `+` node calculates `8` and passes `8` UP to the `*` node. The `*` node receives `8` and `2`, calculates `16`, and returns the final answer. All code execution is essentially Post-Order Tree DP!

## INTERVIEW
**Q: You implemented House Robber III. However, instead of returning an `int[]` from your DFS, you created a global HashMap (`Map<TreeNode, Integer>`) to memoize the answers. The interviewer asks: "Returning the array took $O(N)$ time. What is the time complexity of your HashMap approach?"**
A: **It depends on the Map implementation, but usually $O(N \log N)$ or worse.**
If you don't return the `int[]` state up the tree, you are forced to do a Top-Down Recursive DP where you repeatedly query the children from the parent. Because nodes overlap in calculations, you MUST use a HashMap to cache the results.
Checking and inserting into a `HashMap` is theoretically $O(1)$, but calculating HashCodes for Object references has massive overhead. Furthermore, you are doing this for every single node in the tree!
By returning the `int[]` array directly from the DFS, you completely eliminate the need for a global cache! You guarantee that every node is visited exactly once, achieving a mathematically perfect $O(N)$ Time and $O(H)$ Space (where $H$ is the height of the recursion stack). Always prefer passing state arrays UP the tree in Tree DP!
