## WHY
Imagine you are waiting in line at a grocery store checkout. The first person who got in line is the first person who gets served. If you get in line last, you are served last. 

This exact behavior is called **FIFO** (First-In, First-Out). A **Queue** is the fundamental data structure used to model this behavior. In software engineering, Queues are the absolute backbone of Asynchronous Architecture. If your web server receives 10,000 requests in 1 second, it cannot process them all instantly. It places them into a Queue, and background workers pull the oldest requests out one by one, ensuring fairness and preventing the server from crashing.

## THEORY
### FIFO Restriction
Like a Stack, a Queue is an Abstract Data Type. It is backed by an Array or a Linked List, but strictly enforces access at opposite ends:
1. **Enqueue (Offer)**: Add a new element to the *Rear* (Tail) of the queue.
2. **Dequeue (Poll)**: Remove and return the element at the *Front* (Head) of the queue.
3. **Peek**: Look at the Front element without removing it.

### The Array Bottleneck
If you build a Queue using a standard Array, Enqueuing is fast `O(1)` (you just add to the end). But Dequeuing is catastrophic! When you remove the element at index `0` (the front of the line), you must physically shift every other element in the array one slot to the left, which takes `O(N)` time. To fix this, Arrays use a complex "Circular Buffer" algorithm, or developers simply use a Linked List (where adding to Tail and removing from Head are both `O(1)`).

## CODE

### Level 1 — Beginner (Using the Java Queue)
```java
import java.util.Queue;
import java.util.LinkedList;

public class BasicQueue {
    public static void main(String[] args) {
        // In Java, Queue is an INTERFACE. You cannot instantiate it directly.
        // You must instantiate a concrete implementation, like LinkedList!
        Queue<String> line = new LinkedList<>();
        
        // ENQUEUE: Add to the rear of the line
        line.offer("Alice");
        line.offer("Bob");
        line.offer("Charlie");
        
        // DEQUEUE: Serve the person at the front!
        String served = line.poll(); // Removes "Alice"
        System.out.println("Served: " + served);
        
        // PEEK: Who is next?
        System.out.println("Next in line: " + line.peek()); // "Bob"
    }
}
```

### Level 2 — Building Blocks (Linked List Implementation)
```java
class Node {
    int val; Node next;
    Node(int val) { this.val = val; }
}

public class CustomLinkedQueue {
    // We maintain pointers to BOTH ends to achieve O(1) performance
    private Node head; // The Front of the line
    private Node tail; // The Rear of the line
    
    // Time: O(1)
    public void enqueue(int val) {
        Node newNode = new Node(val);
        if (tail == null) {
            // First person in line!
            head = tail = newNode;
            return;
        }
        // Attach to the rear, then move the tail pointer back
        tail.next = newNode;
        tail = newNode;
    }
    
    // Time: O(1)
    public int dequeue() {
        if (head == null) throw new RuntimeException("Queue is Empty!");
        
        int val = head.val;
        head = head.next; // Move head forward, severing the first node
        
        // If we just dequeued the very last element, the queue is now empty.
        // We MUST reset the tail to null as well!
        if (head == null) tail = null; 
        
        return val;
    }
}
```

### Level 3 — Intermediate (Circular Array Implementation)
```java
public class CircularArrayQueue {
    // To solve the O(N) Array shifting problem, we use a Circular Buffer!
    private int[] data;
    private int head = 0;
    private int tail = 0;
    private int size = 0;
    
    public CircularArrayQueue(int capacity) {
        data = new int[capacity];
    }
    
    public void enqueue(int val) {
        if (size == data.length) throw new RuntimeException("Queue Full!");
        
        data[tail] = val;
        // The magical modulo operator wraps the pointer back to 0 
        // if it hits the end of the array!
        tail = (tail + 1) % data.length; 
        size++;
    }
    
    public int dequeue() {
        if (size == 0) throw new RuntimeException("Queue Empty!");
        
        int val = data[head];
        // Wrap the head pointer around just like the tail!
        head = (head + 1) % data.length;
        size--;
        
        return val;
    }
}
```

### Level 4 — Advanced (Breadth-First Search - BFS)
```java
import java.util.Queue;
import java.util.LinkedList;

public class BfsMatrix {
    // The most powerful application of a Queue: Searching a grid Level-by-Level!
    // Example: A zombie outbreak starts at coordinates (2, 2). 
    // It spreads 1 square in all 4 directions every minute.
    
    public static void spreadVirus(int[][] grid, int startRow, int startCol) {
        Queue<int[]> queue = new LinkedList<>();
        
        // Directions: Up, Down, Left, Right
        int[][] directions = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};
        
        // 1. Enqueue the Patient Zero
        queue.offer(new int[]{startRow, startCol});
        grid[startRow][startCol] = 1; // Mark as infected
        
        int minutes = 0;
        
        // 2. Process the queue level-by-level
        while (!queue.isEmpty()) {
            int levelSize = queue.size();
            
            // Process ALL zombies that were infected in the CURRENT minute
            for (int i = 0; i < levelSize; i++) {
                int[] zombie = queue.poll();
                
                // Try to spread to 4 neighbors
                for (int[] dir : directions) {
                    int newRow = zombie[0] + dir[0];
                    int newCol = zombie[1] + dir[1];
                    
                    // If boundary is valid AND it's a healthy human (0)
                    if (newRow >= 0 && newRow < grid.length && 
                        newCol >= 0 && newCol < grid[0].length && 
                        grid[newRow][newCol] == 0) {
                        
                        grid[newRow][newCol] = 1; // Infect!
                        queue.offer(new int[]{newRow, newCol}); // Enqueue for the NEXT minute!
                    }
                }
            }
            minutes++;
        }
        System.out.println("Total minutes to infect entire grid: " + (minutes - 1));
    }
}
```

## REAL_WORLD
In Enterprise Architecture, Queues are the most important decoupling mechanism in existence.
If Amazon's web server receives an order, it does NOT immediately process the credit card, generate the invoice, and email the warehouse. That would take 10 seconds, and the user's browser would freeze.

Instead, the web server creates an "Order Message" and pushes it into a massive Distributed Message Queue (like **Apache Kafka** or **RabbitMQ**). The web server instantly responds "Order Received!" to the user in 50 milliseconds. Completely separately, a fleet of background Java microservices pulls those messages out of the Kafka Queue one by one, processing the credit cards asynchronously. 

If Black Friday hits and Amazon receives 50,000 orders per second, the microservices don't crash. The Queue simply absorbs the massive spike in traffic, holding the messages safely in memory until the microservices have time to dequeue them.

## INTERVIEW
**Q: In Java, why should you use `.offer()` and `.poll()` instead of `.add()` and `.remove()`?**
A: Both sets of methods do the exact same thing, but they handle Edge Cases differently. If a Queue is restricted to 10 elements and is full, calling `.add()` will violently throw an `IllegalStateException` and crash your application. Calling `.offer()` safely returns `false`. Similarly, if the Queue is empty, `.remove()` throws an exception, whereas `.poll()` gracefully returns `null`. In interviews, always use `offer()` and `poll()`.

**Q: Why does a Queue perfectly model Breadth-First Search (BFS)?**
A: Because BFS explores a graph "level by level". You must finish exploring all neighbors at Distance 1 before you are allowed to explore neighbors at Distance 2. A Queue's strict FIFO (First-In, First-Out) nature enforces this mathematically. When you enqueue all neighbors at Distance 1, they are guaranteed to be dequeued BEFORE any of the newly discovered Distance 2 neighbors that are added to the back of the line!

**Q: Why can't we use a Stack for BFS?**
A: A Stack uses LIFO (Last-In, First-Out). If you push all Distance 1 neighbors onto a Stack, and then pop the first one, you will explore its neighbors (Distance 2) and push them onto the Stack. The next time you pop, you will pull a Distance 2 neighbor! You will dive deep into the graph, completely abandoning the other Distance 1 neighbors. This behavior is called Depth-First Search (DFS).
