## WHY
Imagine you are trapped in a physical maze. How do you get out? You pick a path, and you walk straight down it until you hit a dead end. Once you hit a dead end, you trace your steps backward (backtrack) to the last intersection, and try a different path.
This is exactly how **Depth-First Search (DFS)** works. It plunges as deep as mathematically possible into a graph or tree before it ever considers looking at alternative neighbors. 
DFS is incredibly powerful. Because it exhaustively explores paths to their conclusion, it is the foundational algorithm for solving Mazes, Sudoku puzzles, finding Connected Components, and detecting Cycles in graphs!

## THEORY
### The Stack Structure
While BFS uses a Queue (FIFO) to search level-by-level, DFS uses a **Stack (LIFO)**.
1. Push the starting node onto the Stack.
2. Pop a node. Mark it visited.
3. Push all of its *unvisited* neighbors onto the Stack.
4. Repeat.

Because Stacks process the *most recently added* items first, the algorithm physically plunges down a single chain of neighbors before it ever processes the neighbors left behind at the starting node!

## CODE

### Level 1 — Recursive DFS
Because the call stack of a computer program *is* a physical Stack, the easiest and most beautiful way to write DFS is using Recursion!

```java
import java.util.*;

public class GraphDFS {
    HashMap<Integer, List<Integer>> adjList = new HashMap<>();

    public void traverse(int startNode) {
        // The visited set prevents infinite loops!
        HashSet<Integer> visited = new HashSet<>();
        // Launch the recursive plunge!
        dfsRecursive(startNode, visited);
    }

    private void dfsRecursive(int current, HashSet<Integer> visited) {
        // 1. Mark the current node as visited immediately
        visited.add(current);
        System.out.println("Visited Node: " + current);

        // 2. Look at all neighbors
        List<Integer> neighbors = adjList.getOrDefault(current, new ArrayList<>());

        for (int neighbor : neighbors) {
            if (!visited.contains(neighbor)) {
                // 3. PLUNGE! We pause the current loop and dive into the neighbor!
                dfsRecursive(neighbor, visited);
            }
        }
    }
}
```

### Level 2 — Iterative DFS
If a graph is massively deep (e.g., a linked list of 100,000 nodes), recursive DFS will crash with a `StackOverflowError`. 
To fix this, we replace the invisible Call Stack with a physical `Stack` object!

```java
public void dfsIterative(int startNode) {
    Stack<Integer> stack = new Stack<>();
    HashSet<Integer> visited = new HashSet<>();

    stack.push(startNode);

    while (!stack.isEmpty()) {
        // Pop the top of the stack (the deepest node!)
        int current = stack.pop();

        // Have we seen it? If not, process it!
        if (!visited.contains(current)) {
            visited.add(current);
            System.out.println("Visited Node: " + current);

            // Push all neighbors onto the stack!
            for (int neighbor : adjList.getOrDefault(current, new ArrayList<>())) {
                if (!visited.contains(neighbor)) {
                    stack.push(neighbor);
                }
            }
        }
    }
}
```

### Level 3 — Cycle Detection
Imagine a package manager like NPM. Package A depends on B. Package B depends on C. What if Package C accidentally depends on A? You have an infinite loop (a Cycle)!
DFS is the best way to detect cycles in a Directed Graph.
We track three states for a node instead of just two:
1. `UNVISITED`
2. `VISITING` (Currently in the recursive call stack)
3. `VISITED` (Fully completely done processing)

```java
public boolean hasCycle(int numNodes) {
    // 0 = Unvisited, 1 = Visiting, 2 = Visited
    int[] state = new int[numNodes]; 

    for (int i = 0; i < numNodes; i++) {
        if (state[i] == 0) {
            if (dfsDetectCycle(i, state)) return true;
        }
    }
    return false;
}

private boolean dfsDetectCycle(int current, int[] state) {
    // 1. Mark as currently in the stack (VISITING)
    state[current] = 1;

    for (int neighbor : adjList.getOrDefault(current, new ArrayList<>())) {
        if (state[neighbor] == 1) {
            // CRITICAL: We hit a node that is CURRENTLY in our call stack!
            // This mathematically proves a Cycle exists!
            return true;
        }
        
        if (state[neighbor] == 0) {
            // Unvisited. Plunge!
            if (dfsDetectCycle(neighbor, state)) return true;
        }
    }

    // 2. We finished exploring all paths from this node. Mark as fully VISITED.
    state[current] = 2;
    return false;
}
```

## REAL_WORLD
**Finding Connected Components**
Imagine a massive social network. Is everyone on Earth connected by 6 degrees of separation? Or are there completely isolated clusters of users who don't connect to the main network?
These isolated clusters are called "Connected Components".
DFS finds them effortlessly. You write a loop that iterates through all 10 Billion users. If User 1 is unvisited, you launch a DFS. The DFS will paint (mark visited) every single user connected to User 1. Once the DFS finishes, you increment a `componentCount` variable. Then the loop continues to User 2. If User 2 is already painted, you skip them! You continue until you find an unpainted user, proving you found a brand new, isolated network!

## INTERVIEW
**Q: You want to find the Shortest Path out of a maze. You write a DFS algorithm. Does DFS guarantee the shortest path? If not, why?**
A: **No, DFS never guarantees the shortest path!**
DFS blindly plunges down the first available path. Imagine a maze where you can turn Left or turn Right to reach the exit. 
If turning Right takes 3 steps to reach the exit, and turning Left takes 50,000 steps looping around the entire maze to reach the exit...
If your DFS loop happens to check `Left` before `Right`, it will plunge down the 50,000 step path, hit the exit, and return that 50,000 step path as the answer! It never even checked the 3-step path!
If you need the Shortest Path in an unweighted graph or maze, you MUST use BFS, which guarantees the shortest path by searching level-by-level!
