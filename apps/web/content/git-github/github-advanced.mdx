## WHY
Most developers use GitHub purely as a code hosting and PR platform. But GitHub is a massive platform with features that can dramatically improve your productivity, security, and project management. Knowing these advanced features sets you apart as a power user and helps you automate away toil.

## THEORY
### The GitHub Ecosystem
Beyond Git hosting, GitHub offers:
- **GitHub Packages:** Host your own Docker images, npm packages, or Maven artifacts right next to your code.
- **GitHub Projects:** Advanced Kanban boards and spreadsheets to track issues across multiple repositories.
- **Codespaces:** Full cloud-based development environments.
- **Dependabot:** Automated dependency updates.
- **CodeQL (Advanced Security):** Static analysis for vulnerabilities.

## CODE

### GitHub Codespaces
Imagine clicking a button and having a full VS Code instance in your browser, running on a powerful VM, with all dependencies pre-installed.

Configure it for your project:
```json
// .devcontainer/devcontainer.json
{
  "name": "Node.js Environment",
  "image": "mcr.microsoft.com/devcontainers/javascript-node:18",
  "forwardPorts": [3000],
  "postCreateCommand": "npm install",
  "customizations": {
    "vscode": {
      "extensions": ["dbaeumer.vscode-eslint", "esbenp.prettier-vscode"]
    }
  }
}
```
Anyone cloning the repo can instantly start coding without running `npm install` on their local machine!

### Dependabot: Automated Dependency Updates
Keep your dependencies secure and up-to-date automatically.

```yaml
# .github/dependabot.yml
version: 2
updates:
  # Update npm dependencies
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
    open-pull-requests-limit: 10
  
  # Update Docker base images
  - package-ecosystem: "docker"
    directory: "/"
    schedule:
      interval: "weekly"
```
Dependabot will open PRs automatically when a new version of a library is released!

### GitHub CLI Aliases
Create powerful custom commands.

```bash
# Set an alias to delete all local branches that have been merged
gh alias set cleanup '!git branch --merged | grep -v "\*" | xargs -n 1 git branch -d'

# Create an alias to quickly view PRs assigned to you
gh alias set my-prs 'pr list --assignee "@me"'
```

### GitHub Keyboard Shortcuts
While browsing a repository on GitHub:
- Press `?` to show the keyboard shortcuts menu.
- Press `t` to open a file finder (like Cmd+P in VS Code).
- Press `.` (period) to instantly open the current repository in a web-based VS Code editor (`github.dev`).

## REAL_WORLD
**GitHub Packages Registry**
Instead of paying for Docker Hub or an npm Organization account, you can publish packages directly to GitHub.

```yaml
# .github/workflows/publish-docker.yml
name: Publish Docker Image
on:
  push:
    tags: ['v*']

jobs:
  push_to_registry:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Log in to GitHub Container Registry
        run: echo "${{ secrets.GITHUB_TOKEN }}" | docker login ghcr.io -u ${{ github.actor }} --password-stdin
      - name: Build and push
        run: |
          docker build -t ghcr.io/myorg/myapp:${{ github.ref_name }} .
          docker push ghcr.io/myorg/myapp:${{ github.ref_name }}
```

## INTERVIEW
**Q: You want to ensure that no one ever accidentally commits an AWS Secret Key or Database Password to your repository. How does GitHub help you prevent and detect this?**

A: GitHub provides **Secret Scanning**.

1. **Detection (Free for Public, Paid for Private):** GitHub continuously scans repositories for known patterns of secrets (AWS keys, Stripe tokens, Slack tokens). If one is pushed, it alerts the repository admins immediately.
2. **Push Protection (Advanced Security feature):** This is the preventative measure. If enabled, when a developer runs `git push`, GitHub intercepts the push, scans the payload in real-time, and if it finds a secret, it **rejects the push** before it ever enters the repository history.

This is vastly superior to `.gitignore` alone, as it catches mistakes at the exact moment they happen, preventing the secret from ever being exposed in the remote repository's history.
