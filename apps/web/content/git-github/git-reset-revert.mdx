## WHY
You committed something you shouldn't have. Maybe you committed to the wrong branch, included a debugging `console.log`, or made a mistake in the commit message. 
Git gives you two main tools to undo commits: `git reset` (rewrites history) and `git revert` (creates a new undoing commit). Understanding when to use each is critical — using reset on shared branches can corrupt your team's repositories.

## THEORY
### Reset vs Revert
- **`git reset`:** Moves the branch pointer BACKWARDS to an older commit. Rewrites history. Use ONLY on local (not yet pushed) commits.
- **`git revert`:** Creates a NEW commit that is the inverse of a previous commit. Safe for shared branches — adds to history, doesn't rewrite it.

## CODE

### git reset (ONLY for local commits!)
```bash
# You have these commits:
git log --oneline
# d4e5f6c (HEAD → main) WIP: debugging with console.logs
# c3d4e5f Add payment validation logic
# a1b2c3d Add User model

# Remove the last commit, keep changes STAGED
git reset --soft HEAD~1
# HEAD now at: c3d4e5f
# The files from d4e5f6c are still staged!

# Remove the last commit, keep changes UNSTAGED (most common)
git reset HEAD~1
git reset --mixed HEAD~1  # Same thing
# HEAD now at: c3d4e5f
# The files are modified in working directory but not staged

# Remove the last commit AND discard all changes (DANGEROUS — no undo!)
git reset --hard HEAD~1
# HEAD now at: c3d4e5f
# Changes from d4e5f6c are GONE!

# Reset to a specific commit
git reset --hard a1b2c3d  # Reset to the User model commit

# Undo a mistaken reset (the secret command!)
git reflog               # Shows ALL HEAD movements (even resets!)
# d4e5f6c HEAD@{0}: reset: moving to HEAD~1
# c3d4e5f HEAD@{1}: commit: WIP debugging

git reset --hard HEAD@{1}  # Restore to before the reset!
# Git's reflog keeps 90 days of history — you can recover "lost" commits!
```

### git revert (Safe for shared branches!)
```bash
# Scenario: You pushed a bad commit to main that you need to undo
git log --oneline
# d4e5f6c (HEAD, origin/main) Add broken feature  ← Need to undo this!
# c3d4e5f Add payment validation

# Revert creates a NEW commit that undoes d4e5f6c
git revert d4e5f6c
# Opens editor for revert commit message

git log --oneline
# g7h8i9j (HEAD) Revert "Add broken feature"  ← New commit!
# d4e5f6c Add broken feature                   ← Preserved in history
# c3d4e5f Add payment validation

git push origin main    # Safe to push — no history rewrite!

# Revert without opening editor (use -n flag and commit manually)
git revert --no-edit d4e5f6c
```

## REAL_WORLD
**The Decision Tree**
```
Did you already push the commit to a shared remote?
│
├── NO (local only)
│   └── Use git reset (--soft, --mixed, or --hard)
│       └── Clean rewrite, no extra commit in history
│
└── YES (already pushed)
    └── Use git revert
        └── Safe! Creates undo commit. Others' repos remain compatible.
```

**Emergency: Wrong Branch Commit**
```bash
# You committed to main instead of your feature branch!
git log --oneline
# abc123 (HEAD → main) New feature I was working on   ← Should be on feature branch!

# 1. Save the commit hash
# 2. Go to feature branch and cherry-pick it
git checkout feature/my-feature
git cherry-pick abc123

# 3. Remove it from main
git checkout main
git reset --hard HEAD~1   # OK because you haven't pushed yet!
```

## INTERVIEW
**Q: You ran `git reset --hard HEAD~5` to undo 5 commits, but you've realized you need one of those commits back. You didn't push them. Can you recover them?**

A: Yes! Git's reflog is your safety net. Even `--hard` reset doesn't immediately delete commits — they become "orphaned" (no branch points to them) and are cleaned up by garbage collection after 90 days.

```bash
# See all recent HEAD movements
git reflog
# HEAD@{0}: reset: moving to HEAD~5         ← The reset
# HEAD@{1}: commit: Add user authentication  ← Commit 5 you reset
# HEAD@{2}: commit: Fix login bug            ← Commit 4
# HEAD@{3}: commit: Add profile page         ← Commit 3
# HEAD@{4}: commit: WIP auth                 ← Commit 2
# HEAD@{5}: commit: Start auth feature       ← Commit 1

# Option 1: Restore the specific commit you need using cherry-pick
git cherry-pick HEAD@{3}   # Get "Add profile page" commit back

# Option 2: Restore all 5 commits at once
git reset --hard HEAD@{1}  # Jump back to before the reset happened!
# Now you have all 5 commits back.
```
