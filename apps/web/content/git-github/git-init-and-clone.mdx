## WHY
Every Git project starts in one of two ways: initializing a new empty repository (`git init`), or cloning an existing one (`git clone`). These are the entry points to all Git workflows.

## THEORY
### Two Ways to Start
- **`git init`:** Create a brand new repository from scratch. Creates a `.git` folder in your current directory — this folder IS the repository.
- **`git clone`:** Copy an existing repository (local or remote) to your machine. Downloads all commits, branches, and history.

## CODE

### git init
```bash
# Create a new project with Git
mkdir my-new-project
cd my-new-project
git init
# Initialized empty Git repository in /my-new-project/.git/

# What's in the .git folder?
ls -la .git/
# HEAD           ← Points to current branch
# config         ← Repository-local config
# objects/       ← All your commits and file content (compressed)
# refs/          ← Branch and tag pointers

# Add initial files and first commit
echo "# My Project" > README.md
git add README.md
git commit -m "Initial commit"

# Connect to a GitHub repository
git remote add origin git@github.com:username/my-new-project.git
git push -u origin main  # -u sets the upstream tracking
```

### git clone
```bash
# Clone over SSH (preferred — no password needed after key setup)
git clone git@github.com:facebook/react.git

# Clone over HTTPS
git clone https://github.com/facebook/react.git

# Clone into a different directory name
git clone git@github.com:user/project.git my-custom-name

# Shallow clone (only the latest snapshot — no history)
# Much faster for CI/CD!
git clone --depth=1 git@github.com:user/project.git

# Clone a specific branch
git clone -b develop git@github.com:user/project.git

# After cloning:
cd react
git log --oneline -5    # See recent commits
git branch              # * main
git remote -v           # origin git@github.com:facebook/react.git
```

## REAL_WORLD
**When to Use Each**
- `git init`: Starting a brand new project from zero
- `git clone`: Contributing to an existing project, or pulling down your own project on a new machine

For open-source contributions: Fork the repo on GitHub, then clone YOUR fork. Add the original as `upstream` for syncing.

## INTERVIEW
**Q: What is inside the `.git` folder and what would happen if you deleted it?**
A: The `.git` folder is the entire repository — it contains all Git data:
- `HEAD`: A pointer to your current branch
- `objects/`: All commits, file contents (blobs), trees, and tags — compressed and hashed with SHA-1
- `refs/heads/`: All local branch names and their commit pointers
- `refs/tags/`: All tags
- `config`: Repository-specific configuration (remotes, user settings)
- `index`: The staging area (what's staged for the next commit)

If you delete `.git`, your directory becomes a plain folder with no Git history whatsoever. All your commits, all your branches, all your history — completely gone from the local machine. The code files themselves remain, but Git has no memory of their history.

If you had pushed to a remote (GitHub), the history is preserved there. You could re-clone from the remote. But any local-only commits that were never pushed are permanently lost — another reason why pushing frequently is important.
