## WHY
Custom web fonts cause a terrible user experience known as **Layout Shift (CLS)**. The browser renders the page using a fallback font (like Arial), then seconds later, the custom font finishes downloading and the text suddenly changes size and style, pushing the layout around.
The `next/font` module solves this completely. It automatically downloads Google Fonts at build time, hosts them locally, and uses advanced CSS techniques (`size-adjust`) to guarantee zero layout shift.

## THEORY
### How `next/font` Works
1. At build time (`npm run build`), Next.js downloads the font files from Google Fonts.
2. It hosts them locally in your static assets, eliminating external network requests to `fonts.googleapis.com` (better privacy and speed).
3. It generates an optimized fallback font that exactly matches the metrics (width/height) of your custom font.
4. It injects a CSS variable or className into your app.

## CODE

### Using Google Fonts
```tsx
// app/layout.tsx
import { Inter, Roboto_Mono } from 'next/font/google';

// Configure the font (subsets reduce file size drastically)
const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
  variable: '--font-inter', // Creates a CSS variable!
});

const robotoMono = Roboto_Mono({
  subsets: ['latin'],
  display: 'swap',
  variable: '--font-roboto-mono',
});

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    // Apply the CSS variables to the <html> tag
    <html lang="en" className={`${inter.variable} ${robotoMono.variable}`}>
      <body className="font-sans">
        {children}
      </body>
    </html>
  );
}
```

### Tailwind CSS Integration
If you created CSS variables as shown above, you simply configure Tailwind to use them.

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      fontFamily: {
        // Use the CSS variables created by next/font
        sans: ['var(--font-inter)'],
        mono: ['var(--font-roboto-mono)'],
      },
    },
  },
}
```

### Local Fonts
If you bought a custom font (e.g., `.woff2` files) from a foundry, you can optimize it just as easily.

```tsx
// app/layout.tsx
import localFont from 'next/font/local';

// Configure local font
const myCustomFont = localFont({
  src: [
    {
      path: './fonts/CustomFont-Regular.woff2',
      weight: '400',
      style: 'normal',
    },
    {
      path: './fonts/CustomFont-Bold.woff2',
      weight: '700',
      style: 'normal',
    },
  ],
  variable: '--font-custom',
});

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={myCustomFont.variable}>
      <body>{children}</body>
    </html>
  );
}
```

## REAL_WORLD
**The Magic of `size-adjust`**
If you look at the CSS that `next/font` injects into your `<head>`, you'll see something like this:

```css
@font-face {
  font-family: '__Inter_Fallback_e2a3b1';
  src: local("Arial");
  ascent-override: 90.20%;
  descent-override: 22.48%;
  line-gap-override: 0.00%;
  size-adjust: 107.40%;
}
```

Next.js mathematically calculates the exact width and height differences between the Google Font (`Inter`) and the system fallback font (`Arial`). It creates a synthetic, scaled version of Arial that occupies the *exact same pixels* as Inter. When the Inter font finally loads, the letters swap out, but the layout doesn't shift by a single pixel. This guarantees a perfect Core Web Vitals CLS score.

## INTERVIEW
**Q: You use `next/font/google` in your Next.js application. Your company has a strict firewall policy that blocks outgoing requests to Google servers from the production environment. Will the fonts fail to load?**

A: **No, they will load perfectly fine.**

`next/font` downloads the font files from Google servers *at build time* (during `npm run build` on your CI server or local machine). 

When the app is deployed to production, it serves the fonts directly from its own static assets folder. The production server makes zero requests to Google. This is excellent for both privacy (Google doesn't get the IP addresses of your users) and strict enterprise environments.
