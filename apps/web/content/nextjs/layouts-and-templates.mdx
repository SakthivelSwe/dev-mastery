## WHY
Every website has shared UI elements: navigation bars, sidebars, footers. Without a way to define these once and share them across pages, you'd duplicate them in every page component.
Next.js App Router's **Layouts** and **Templates** provide two different mechanisms for shared UI. Understanding the subtle difference between them — and when to use each — is important for building correct applications.

## THEORY
### Layout vs Template
- **Layout (`layout.tsx`):** Persists across navigations. State is preserved. The component stays mounted when navigating between child routes.
- **Template (`template.tsx`):** Creates a NEW instance for every route. State is reset. Each page navigation creates a fresh template.

## CODE

### Layout Hierarchy
```tsx
// app/layout.tsx — Root layout (wraps ALL pages)
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <TopNavigation />      {/* Always visible — never re-renders! */}
        {children}
        <Footer />
      </body>
    </html>
  );
}

// app/dashboard/layout.tsx — Dashboard layout
// Stacks on top of RootLayout! Children see: TopNav > DashboardSidebar > page
export default function DashboardLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="flex">
      <DashboardSidebar />   {/* Visible only on /dashboard/* routes */}
      <main className="flex-1">{children}</main>
    </div>
  );
}

// The final rendered structure for /dashboard/settings:
// <html>
//   <body>
//     <TopNavigation />         ← from app/layout.tsx
//     <div className="flex">    ← from app/dashboard/layout.tsx
//       <DashboardSidebar />
//       <main>
//         <SettingsPage />      ← from app/dashboard/settings/page.tsx
//       </main>
//     </div>
//     <Footer />
//   </body>
// </html>
```

### Template (Resets State on Navigation)
```tsx
// app/dashboard/template.tsx
// Use when you WANT state to reset between page navigations

'use client';
import { useEffect } from 'react';

export default function DashboardTemplate({ children }: { children: React.ReactNode }) {
  // This runs on EVERY navigation to a /dashboard/* page!
  useEffect(() => {
    trackPageView();         // Analytics: track each page visit
    resetScrollPosition();   // Scroll to top on each navigation
  }, []);
  
  return <>{children}</>;
}
```

### Metadata in Layouts
```tsx
// app/dashboard/layout.tsx
import { Metadata } from 'next';

// Set metadata for all /dashboard/* pages
export const metadata: Metadata = {
  title: {
    template: '%s | Dashboard',
    default: 'Dashboard',
  },
};
```

### Nested Layouts Example: Auth-Protected Dashboard
```tsx
// app/(auth)/layout.tsx — Only for auth pages (login, register)
import { auth } from '@/lib/auth';
import { redirect } from 'next/navigation';

export default async function AuthLayout({ children }: { children: React.ReactNode }) {
  const session = await auth();
  
  // If already logged in, redirect away from auth pages
  if (session) redirect('/dashboard');
  
  return (
    <div className="min-h-screen flex items-center justify-center bg-gray-50">
      <div className="max-w-md w-full">
        {children}    {/* Login or Register form */}
      </div>
    </div>
  );
}

// app/(dashboard)/layout.tsx — Only for dashboard pages
export default async function DashboardLayout({ children }: { children: React.ReactNode }) {
  const session = await auth();
  
  // If not logged in, redirect to login
  if (!session) redirect('/login');
  
  return (
    <div className="flex h-screen">
      <Sidebar user={session.user} />
      <div className="flex-1 overflow-auto">
        {children}
      </div>
    </div>
  );
}
```

## REAL_WORLD
**Why Layouts Are Better than Duplicating Navigation**
Without layouts: 20 page components each import and render `<Navigation />` and `<Footer />`.
- Changing the Navigation requires updating 20 files
- Navigation re-renders on every page navigation (animation reset, refetch)

With layouts: One `layout.tsx` defines the shared structure.
- Change Navigation in ONE place
- Navigation stays mounted — no re-render on navigation → smooth UI!

## INTERVIEW
**Q: Your dashboard has a Sidebar that shows the user's notification count. When a user navigates between dashboard pages, you want the notification count to update without re-fetching. Should you use a Layout or Template?**

A: **Layout.** Here's why:

A Layout persists across navigations — the Sidebar component STAYS MOUNTED when navigating from `/dashboard/settings` to `/dashboard/analytics`. No unmount, no remount, no state reset.

The notification count can be stored in a client component's state:
```tsx
// Sidebar.tsx — Client Component
'use client';
import { useState, useEffect } from 'react';

export function Sidebar() {
  const [notificationCount, setNotificationCount] = useState(0);
  
  // Fetches once when Sidebar mounts (stays mounted across navigations!)
  useEffect(() => {
    fetchNotifications().then(setNotificationCount);
    // Set up WebSocket or polling for live updates
    const ws = new WebSocket('/api/notifications');
    ws.onmessage = () => setNotificationCount(c => c + 1);
  }, []);
  
  return <nav><NotificationBell count={notificationCount} /></nav>;
}
```

If you used a Template instead, the Sidebar would unmount and remount on every navigation, resetting the count and triggering a new fetch each time — wasting bandwidth and causing flickering.
