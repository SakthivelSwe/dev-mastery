## WHY
The App Router (introduced in Next.js 13, stable in 14) is a complete reimagining of how routing works in Next.js. It is built on top of React Server Components and introduces powerful new concepts: nested layouts, loading states, error boundaries, and streaming — all defined through file conventions rather than configuration.
The App Router is the future of Next.js. All new projects should use it over the legacy Pages Router.

## THEORY
### File-System Based Routing
In the App Router, your folder structure IS your URL structure:
- `app/page.tsx` → `/`
- `app/about/page.tsx` → `/about`
- `app/blog/[slug]/page.tsx` → `/blog/:slug`
- `app/(marketing)/pricing/page.tsx` → `/pricing` (route group — `(marketing)` not in URL)
- `app/dashboard/@modal/page.tsx` → Parallel route (advanced)

### Special File Conventions
- `page.tsx` → The UI for the route
- `layout.tsx` → Shared UI wrapper (persists between navigations!)
- `loading.tsx` → Auto-shown while data loads (Suspense boundary)
- `error.tsx` → Auto-shown when an error occurs (Error Boundary)
- `not-found.tsx` → Auto-shown for 404s
- `route.ts` → API endpoint (HTTP handlers)

## CODE

### Complete App Router Example
```tsx
// app/layout.tsx — Root layout: wraps EVERY page
import { Sidebar } from '@/components/Sidebar';
import { Header } from '@/components/Header';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <Header />           {/* Always visible on every page */}
        <Sidebar />          {/* Always visible on every page */}
        <main>{children}</main>  {/* Page content changes here */}
      </body>
    </html>
  );
}
```

```tsx
// app/dashboard/layout.tsx — Nested layout: only for /dashboard/* routes
// This layout STACKS on top of the root layout!
export default function DashboardLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="dashboard-container">
      <DashboardNav />  {/* Only visible on dashboard pages! */}
      <div className="content">{children}</div>
    </div>
  );
}
```

### Loading and Error States
```tsx
// app/users/loading.tsx
// Automatically shown while users/page.tsx is loading!
export default function Loading() {
  return (
    <div className="skeleton">
      {[...Array(5)].map((_, i) => (
        <div key={i} className="skeleton-row" />
      ))}
    </div>
  );
}

// app/users/error.tsx
'use client';  // Error boundaries must be Client Components!
export default function Error({
  error,
  reset,
}: {
  error: Error;
  reset: () => void;
}) {
  return (
    <div>
      <h2>Something went wrong!</h2>
      <p>{error.message}</p>
      <button onClick={() => reset()}>Try again</button>
    </div>
  );
}
```

### Dynamic Routes
```tsx
// app/blog/[slug]/page.tsx — Dynamic route
// [slug] matches any URL segment: /blog/my-post, /blog/another-post

export default async function BlogPostPage({
  params,
}: {
  params: { slug: string };
}) {
  const post = await getPostBySlug(params.slug);
  
  if (!post) notFound();  // Shows not-found.tsx!
  
  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}

// Generate static pages at BUILD TIME for all blog posts (SSG!)
export async function generateStaticParams() {
  const posts = await getAllPosts();
  return posts.map(post => ({ slug: post.slug }));
}
```

### Route Groups (Organizing Without Affecting URLs)
```
app/
├── (auth)/
│   ├── login/page.tsx      → /login  (not /auth/login!)
│   └── register/page.tsx   → /register
├── (marketing)/
│   ├── page.tsx            → /  (home)
│   ├── pricing/page.tsx    → /pricing
│   └── about/page.tsx      → /about
└── (dashboard)/
    ├── layout.tsx          → Dashboard layout (auth protected)
    └── dashboard/page.tsx  → /dashboard
```

## REAL_WORLD
**Protecting Routes with Middleware**
```typescript
// middleware.ts (root level — runs before every request!)
import { NextResponse } from 'next/server';
import { auth } from './lib/auth';

export default async function middleware(request: Request) {
  const session = await auth();
  const isProtected = request.url.includes('/dashboard');
  
  if (isProtected && !session) {
    return NextResponse.redirect(new URL('/login', request.url));
  }
  
  return NextResponse.next();
}

export const config = {
  matcher: ['/dashboard/:path*'],  // Only run on dashboard routes
};
```

## INTERVIEW
**Q: In the App Router, what is the difference between a Root Layout and a Page? Why does navigation between pages not cause the entire layout to re-render?**

A: A **Root Layout** (`app/layout.tsx`) is a persistent component that wraps all pages. It renders ONCE when the app first loads and STAYS mounted across navigations. It's perfect for elements that should persist: navigation bars, sidebars, analytics providers, theme providers.

A **Page** (`page.tsx`) is the unique content for a specific route. It unmounts and remounts when navigating between routes.

This works because Next.js uses React's **partial rendering** — when navigating from `/users` to `/posts`, Next.js re-renders only the `page.tsx` component (and any affected nested layouts), while keeping the root layout (with its Header and Sidebar) mounted. This is why navigation feels instant and smooth without a full page reload.

This is fundamentally different from traditional multi-page websites where every navigation fetches a new HTML document from the server, causing a full page flash. Next.js combines SSR's benefits (SEO, fast initial load) with SPA's benefits (instant navigation, no page flashes).
