## WHY
In traditional React (CRA, Vite), routing requires installing React Router, configuring route definitions in JavaScript, and manually wrapping with `<BrowserRouter>`. The routing logic is separate from the page components.
Next.js App Router uses **file-system based routing** — your folder structure is your URL structure. No configuration needed. The convention handles everything: static routes, dynamic routes, nested layouts, route groups, and catch-all routes.

## THEORY
### Routing Conventions
- `app/page.tsx` → `/`
- `app/about/page.tsx` → `/about`
- `app/blog/[slug]/page.tsx` → `/blog/:slug` (dynamic)
- `app/blog/[...slug]/page.tsx` → `/blog/*` (catch-all)
- `app/(marketing)/page.tsx` → `/` (route group — folder not in URL)
- `app/@modal/page.tsx` → Parallel route

### Basic Routes
```
app/
├── page.tsx              → /
├── about/
│   └── page.tsx          → /about
├── blog/
│   ├── page.tsx          → /blog
│   └── [slug]/
│       └── page.tsx      → /blog/any-slug
└── dashboard/
    ├── page.tsx          → /dashboard
    ├── settings/
    │   └── page.tsx      → /dashboard/settings
    └── analytics/
        └── page.tsx      → /dashboard/analytics
```

### Dynamic Routes
```tsx
// app/blog/[slug]/page.tsx
export default function BlogPost({
  params,
}: {
  params: { slug: string }  // Type-safe params!
}) {
  // params.slug = "my-first-post" when URL is /blog/my-first-post
  return <h1>Post: {params.slug}</h1>;
}

// app/shop/[category]/[product]/page.tsx
// URL: /shop/electronics/iphone-15
export default function ProductPage({
  params: { category, product }
}: {
  params: { category: string; product: string }
}) {
  return <div>{category} / {product}</div>;
}
```

### Catch-All Routes
```tsx
// app/docs/[...slug]/page.tsx
// Matches: /docs/intro, /docs/api/authentication, /docs/api/endpoints/users

export default function DocsPage({
  params,
}: {
  params: { slug: string[] }  // Array of path segments!
}) {
  // URL: /docs/api/authentication
  // params.slug = ['api', 'authentication']
  
  const fullPath = params.slug.join('/');  // 'api/authentication'
  
  return <div>Docs: {fullPath}</div>;
}

// Optional catch-all: [[...slug]] also matches the root /docs
```

### Navigation
```tsx
'use client';
import { useRouter, usePathname, useSearchParams } from 'next/navigation';
import Link from 'next/link';

// 1. Link component (preferred for navigation!)
<Link href="/about">About</Link>
<Link href="/blog/my-post">Read Post</Link>
<Link href={{ pathname: '/blog/[slug]', query: { slug: 'my-post' } }}>Post</Link>

// 2. Programmatic navigation
const router = useRouter();
router.push('/dashboard');     // Navigate (adds to history)
router.replace('/login');      // Navigate (replaces history)
router.back();                 // Go back
router.refresh();              // Refresh server components

// 3. Current path
const pathname = usePathname();  // '/blog/my-post'

// 4. Search params
const searchParams = useSearchParams();
const query = searchParams.get('q');  // ?q=nextjs → 'nextjs'
```

### Route Groups
```
app/
├── (auth)/
│   ├── layout.tsx        ← Auth-specific layout (no sidebar, centered form)
│   ├── login/page.tsx    → /login   (not /auth/login!)
│   └── register/page.tsx → /register
└── (dashboard)/
    ├── layout.tsx        ← Dashboard layout (with sidebar)
    └── dashboard/page.tsx → /dashboard
```

## REAL_WORLD
**Active Link Highlighting**
```tsx
'use client';
import Link from 'next/link';
import { usePathname } from 'next/navigation';

function NavLink({ href, children }: { href: string; children: React.ReactNode }) {
  const pathname = usePathname();
  const isActive = pathname === href || pathname.startsWith(`${href}/`);
  
  return (
    <Link
      href={href}
      className={isActive ? 'text-blue-600 font-bold' : 'text-gray-600'}
    >
      {children}
    </Link>
  );
}
```

## INTERVIEW
**Q: In the App Router, what's the difference between `[slug]`, `[...slug]`, and `[[...slug]]` in route segments?**

A:
- **`[slug]`** — Dynamic segment matching exactly ONE path segment. `/blog/my-post` (slug = 'my-post'). Does NOT match `/blog/a/b`.

- **`[...slug]`** — Catch-all segment matching ONE OR MORE path segments. `/docs/api/auth` (slug = ['api', 'auth']). Does NOT match the root `/docs` itself.

- **`[[...slug]]`** — Optional catch-all matching ZERO OR MORE segments. `/docs` (slug = undefined), `/docs/api` (slug = ['api']), `/docs/api/auth` (slug = ['api', 'auth']). Also matches the parent route.

Use case example:
```
/docs → Optional catch-all renders the docs homepage
/docs/getting-started → Same catch-all renders 'getting-started' content
/docs/api/authentication → Same catch-all renders nested docs page
All handled by a SINGLE file: app/docs/[[...slug]]/page.tsx
```
