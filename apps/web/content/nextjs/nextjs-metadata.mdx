## WHY
Search engines index your page's title, description, and Open Graph tags to determine how to rank and display your pages. Social media platforms use Open Graph meta tags for rich link previews. Without proper metadata, your pages look unprofessional in search results and social sharing.
Next.js 13+ App Router has a first-class Metadata API that generates SEO-optimized HTML — including dynamic metadata based on page data.

## THEORY
### Static vs Dynamic Metadata
- **Static metadata:** Defined as an exported constant. Same metadata for all instances of the page.
- **Dynamic metadata:** Generated via `generateMetadata()` function. Can use async data (database, API) to create metadata specific to each record (e.g., each blog post's title).

## CODE

### Static Metadata
```typescript
// app/about/page.tsx
import { Metadata } from 'next';

export const metadata: Metadata = {
  title: 'About Us | MyCompany',
  description: 'Learn about our mission to transform software development education.',
  keywords: ['about', 'company', 'team', 'mission'],
  
  // Open Graph (Facebook, LinkedIn previews)
  openGraph: {
    title: 'About MyCompany',
    description: 'Building the future of developer education',
    url: 'https://mycompany.com/about',
    siteName: 'MyCompany',
    images: [
      {
        url: 'https://mycompany.com/og/about.jpg',
        width: 1200,
        height: 630,
        alt: 'MyCompany Team',
      }
    ],
    locale: 'en_US',
    type: 'website',
  },
  
  // Twitter/X Card
  twitter: {
    card: 'summary_large_image',
    title: 'About MyCompany',
    description: 'Building the future of developer education',
    images: ['https://mycompany.com/og/about.jpg'],
  },
  
  // Canonical URL (important for SEO — prevents duplicate content)
  alternates: {
    canonical: 'https://mycompany.com/about',
  },
};
```

### Root Layout Metadata (Shared Defaults)
```typescript
// app/layout.tsx
export const metadata: Metadata = {
  // Template: page title + site name
  title: {
    template: '%s | DevMastery',    // Page sets the %s part
    default: 'DevMastery',           // Fallback when no title set
  },
  description: 'Master software development with interactive learning paths',
  metadataBase: new URL('https://devmastery.com'),  // Required for relative URLs!
  
  // Favicon and icons
  icons: {
    icon: '/favicon.ico',
    shortcut: '/favicon-16x16.png',
    apple: '/apple-touch-icon.png',
  },
};
```

### Dynamic Metadata (Per-Page Data)
```typescript
// app/blog/[slug]/page.tsx
import { Metadata } from 'next';
import { getPost } from '@/lib/posts';

interface PageProps {
  params: { slug: string };
}

// This function runs on the server and generates metadata for each post!
export async function generateMetadata({ params }: PageProps): Promise<Metadata> {
  const post = await getPost(params.slug);
  
  if (!post) {
    return { title: 'Post Not Found' };
  }
  
  return {
    title: post.title,                    // "How to Master Docker" | DevMastery
    description: post.excerpt,
    openGraph: {
      title: post.title,
      description: post.excerpt,
      type: 'article',
      publishedTime: post.publishedAt.toISOString(),
      authors: [post.author.name],
      images: [{
        url: post.coverImage,
        width: 1200,
        height: 630,
        alt: post.title,
      }],
    },
    alternates: {
      canonical: `https://devmastery.com/blog/${params.slug}`,
    },
  };
}
```

### Robots and Sitemap
```typescript
// app/robots.ts — Generates /robots.txt
import { MetadataRoute } from 'next';

export default function robots(): MetadataRoute.Robots {
  return {
    rules: {
      userAgent: '*',
      allow: '/',
      disallow: ['/admin/', '/api/'],
    },
    sitemap: 'https://devmastery.com/sitemap.xml',
  };
}

// app/sitemap.ts — Generates /sitemap.xml
export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const posts = await getAllPosts();
  
  return [
    { url: 'https://devmastery.com', lastModified: new Date() },
    { url: 'https://devmastery.com/about', lastModified: new Date() },
    ...posts.map(post => ({
      url: `https://devmastery.com/blog/${post.slug}`,
      lastModified: post.updatedAt,
      changeFrequency: 'weekly' as const,
      priority: 0.8,
    })),
  ];
}
```

## INTERVIEW
**Q: Your blog posts have dynamic metadata using `generateMetadata`. But when you share a blog post link on Twitter, it shows generic metadata (not the post-specific title and image). Why?**

A: Several possible causes:

1. **Not cached on Twitter's servers yet:** Twitter caches card previews. Use the [Twitter Card Validator](https://cards-dev.twitter.com/validator) to force a refresh.

2. **`metadataBase` not set:** Relative image URLs won't work without `metadataBase`:
```typescript
// app/layout.tsx
export const metadata: Metadata = {
  metadataBase: new URL('https://myblog.com'),  // Required!
};
```
Without this, `images: ['/og-image.jpg']` becomes an invalid relative URL in the meta tag.

3. **`generateMetadata` returning undefined for some reason:** Check if the slug lookup is actually finding the post and returning real data (not hitting the `if (!post)` fallback).

4. **Twitter is scraping the page before `generateMetadata` finishes:** If the page is rendered dynamically on first request and Twitter's bot is faster than your server's response, it might get an incomplete page. Use SSG or ISR for blog posts so the page is pre-rendered and instantly available.
