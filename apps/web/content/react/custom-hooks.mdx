## WHY
In React, components allow you to extract and reuse *UI logic* (like a beautiful `<Button>` or `<Card>`). But what if you want to extract and reuse *Stateful behavior*?

Imagine you have a `useWindowWidth` feature that attaches a `resize` event listener to the browser window, tracks the width in `useState`, and cleans up the listener in `useEffect`. If you want this exact same behavior in 5 different components, copying and pasting the `useState` and `useEffect` block 5 times violates the DRY (Don't Repeat Yourself) principle.

React allows you to create **Custom Hooks**. A Custom Hook allows you to extract React state and lifecycle logic into a reusable JavaScript function that can be imported and shared across your entire application.

## THEORY
### The Rules of Custom Hooks
A Custom Hook is literally just a standard JavaScript function, with two strict rules:
1. **The name MUST start with `use`** (e.g., `useWindowWidth`, `useFetchUser`). This is not a suggestion; the React ESLint Linter actively looks for functions starting with `use` to enforce Hook rules.
2. **It must call at least one built-in React Hook** (like `useState` or `useEffect`). If it doesn't call a Hook, it's just a normal utility function, not a Custom Hook!

### State Isolation
When Component A and Component B both call `useWindowWidth()`, they **DO NOT share state**. Custom Hooks share *logic*, not *data*. Every time a component calls a Hook, it creates a completely independent, isolated instance of the State for that specific component.

## CODE

### Level 1 — Beginner (Creating a basic Hook)
Let's extract a simple Toggle feature (like opening a modal or expanding an accordion).

```javascript
// 1. Create a file called useToggle.js
import { useState } from 'react';

// 2. The function MUST start with 'use'
export function useToggle(initialValue = false) {
  
  // 3. We use standard React Hooks inside our custom hook!
  const [value, setValue] = useState(initialValue);

  // 4. We build some custom logic
  const toggle = () => setValue(!value);
  const setTrue = () => setValue(true);
  const setFalse = () => setValue(false);

  // 5. We return whatever we want! (Usually an array or an object)
  return [value, toggle, setTrue, setFalse];
}
```

### Level 2 — Building Blocks (Consuming the Hook)
Now we can use our custom `useToggle` hook in any component!

```javascript
import { useToggle } from './useToggle';

export default function App() {
  // We use our custom hook exactly like a built-in React hook!
  const [isModalOpen, toggleModal] = useToggle(false);
  const [isSidebarOpen, toggleSidebar] = useToggle(true);

  return (
    <div>
      <button onClick={toggleSidebar}>
        {isSidebarOpen ? "Close Sidebar" : "Open Sidebar"}
      </button>

      <button onClick={toggleModal}>Open Modal</button>

      {/* Conditional Rendering! */}
      {isModalOpen && <div className="modal">Modal Content!</div>}
    </div>
  );
}
```

### Level 3 — Intermediate (Extracting Side Effects)
The most common use-case for Custom Hooks is extracting complex `useEffect` logic, like fetching data from an API or listening to Browser events.

```javascript
import { useState, useEffect } from 'react';

// A custom hook to track the user's mouse coordinates!
export function useMousePosition() {
  const [position, setPosition] = useState({ x: 0, y: 0 });

  useEffect(() => {
    // The event handler
    const updatePosition = (e) => {
      setPosition({ x: e.clientX, y: e.clientY });
    };

    // Attach the listener on mount
    window.addEventListener('mousemove', updatePosition);

    // Clean it up on unmount!
    return () => window.removeEventListener('mousemove', updatePosition);
  }, []); // Run only once!

  // Return the coordinates
  return position;
}
```

```javascript
import { useMousePosition } from './useMousePosition';

export default function Canvas() {
  // Blazing fast, clean, one-line implementation!
  const { x, y } = useMousePosition();

  return (
    <div>
      Mouse is at: {x}, {y}
    </div>
  );
}
```

### Level 4 — Advanced (Parameterized Hooks - useFetch)
Custom hooks can take parameters and return complex objects representing loading states and errors.

```javascript
import { useState, useEffect } from 'react';

// A robust custom hook for fetching data!
export function useFetch(url) {
  const [data, setData] = useState(null);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    // Reset state when the URL changes
    setIsLoading(true);
    setError(null);

    fetch(url)
      .then(res => {
        if (!res.ok) throw new Error("Network error");
        return res.json();
      })
      .then(json => {
        setData(json);
        setIsLoading(false);
      })
      .catch(err => {
        setError(err.message);
        setIsLoading(false);
      });
  }, [url]); // Automatically re-fetches if the URL parameter changes!

  return { data, isLoading, error };
}
```

## REAL_WORLD
Before Hooks were invented in 2019, extracting stateful logic was a nightmare. Developers had to use terrifying architectural patterns like **Higher-Order Components (HOCs)** or **Render Props**, which resulted in massive "Wrapper Hell" (components wrapped in 6 layers of other components just to get access to data). 

Custom Hooks completely eradicated these patterns overnight. 

Today, the entire React ecosystem is built on Custom Hooks. When you use React Router, you call `useNavigate()`. When you use Redux, you call `useSelector()`. When you use React Hook Form, you call `useForm()`. 

## INTERVIEW
**Q: Can you call a Custom Hook conditionally? (e.g., `if (isAdmin) useAdminData()`)**
A: **NO.** Because Custom Hooks contain built-in React Hooks (like `useState`), they are subject to the exact same Rules of Hooks. They must be called unconditionally at the absolute top level of your component.

**Q: Do two components using the same Custom Hook share the same State?**
A: No. Custom Hooks are a mechanism to reuse *logic*. Every time you call a Custom Hook, the `useState` inside it creates a brand new, isolated piece of memory in the React tree. If you want to share the actual *data* between components, you must use the Context API or Redux.

**Q: Why does the custom hook name have to start with `use`?**
A: If it doesn't start with `use`, the React ESLint plugin cannot identify it as a Hook. It will treat it as a normal JS function and it will NOT warn you if you accidentally put it inside an `if` statement or a `for` loop, leading to catastrophic runtime crashes.

## FEYNMAN CHECK

Explain to a 10-year-old:
> Custom hooks are functions starting with `use` that call other hooks. They extract stateful logic from components — making it reusable, testable in isolation, and shareable across the codebase. The rules of hooks (no conditions, no loops, same call order every render) apply INSIDE custom hooks too. The power: `useFetch` is a custom hook that manages loading state, error state, and data — your component just calls `const { data, loading, error } = useFetch('/api/user')`.
