## WHY
Historically, web development strictly separated technologies: HTML for structure, CSS for styling, and JavaScript for logic. However, this separation was based on *technologies*, not *features*. If you built a "Like Button", the logic and the HTML were stored in entirely different files, making the codebase difficult to maintain.

React introduced **JSX (JavaScript XML)**. JSX allows you to write HTML tags directly inside your JavaScript files. This fundamentally shifted the paradigm: instead of putting JavaScript into your HTML (like Angular or Vue templates do), React puts HTML into your JavaScript. This allows you to harness the full, unbridled power of the JavaScript engine to dynamically generate user interfaces.

## THEORY
### What is JSX?
JSX is **Syntactic Sugar**. Browsers cannot understand JSX. If you send a `<div />` inside a `.js` file to Chrome, it will throw a Syntax Error. 

Before the code reaches the browser, a compiler (like Babel or SWC) scans your files and translates the beautiful JSX tags into ugly, raw JavaScript function calls (`React.createElement()`).

### The Golden Rules of JSX
Because JSX is secretly JavaScript under the hood, it has strict rules that standard HTML doesn't:
1. **You must return a single Root Element**: A function cannot return two separate elements at the same level.
2. **Close every tag**: Even self-closing tags like `<img>` must end with a slash: `<img />`.
3. **CamelCase properties**: Because `class` is a reserved keyword in JS, you must use `className`. Event listeners like `onclick` become `onClick`.

## CODE

### Level 1 — Beginner (The Syntax)
```javascript
// A standard React component returning JSX
export default function UserProfile() {
  const username = "Alice";
  const avatarUrl = "https://example.com/alice.jpg";

  // We use curly braces { } to evaluate ANY JavaScript expression!
  return (
    // 1. We use className instead of class!
    <div className="profile-card">
      
      {/* 2. We inject variables using {} */}
      <h1>{username}'s Profile</h1>
      
      {/* 3. We use {} to bind dynamic data to HTML attributes! */}
      {/* 4. Notice the self-closing tag /> */}
      <img src={avatarUrl} alt="User Avatar" />
      
      <button>Follow</button>
    </div>
  );
}
```

### Level 2 — Building Blocks (The Root Element Rule)
If you try to return two sibling elements, the compiler will instantly crash.

```javascript
// ❌ FATAL ERROR: Adjacent JSX elements must be wrapped!
export function BrokenComponent() {
  return (
    <h1>Title</h1>
    <p>Description</p>
  );
}

// ✅ FIX 1: Wrap in a div (creates a useless DOM node)
export function FixedWithDiv() {
  return (
    <div className="wrapper">
      <h1>Title</h1>
      <p>Description</p>
    </div>
  );
}

// ✅ FIX 2: Use React Fragments <></> (The Enterprise Standard)
// Fragments allow you to group elements without adding a useless <div> to the real DOM!
export function FixedWithFragment() {
  return (
    <>
      <h1>Title</h1>
      <p>Description</p>
    </>
  );
}
```

### Level 3 — Intermediate (Executing JS inside JSX)
Inside the curly braces `{}`, you can execute any valid JavaScript *expression* (something that evaluates to a value). You cannot write statements (like `if`, `for`, or `switch`).

```javascript
export default function MathComponent() {
  const price = 50;
  const taxRate = 0.1;

  return (
    <div>
      {/* You can do math! */}
      <p>Total: ${price + (price * taxRate)}</p>

      {/* You can call functions! */}
      <p>Uppercase Name: {"alice".toUpperCase()}</p>

      {/* ❌ YOU CANNOT DO THIS: */}
      {/* <p> { if (price > 10) return "Expensive" } </p> */}
      
      {/* ✅ YOU MUST DO THIS (Ternary Operator): */}
      <p>{ price > 10 ? "Expensive!" : "Cheap!" }</p>
    </div>
  );
}
```

### Level 4 — Advanced (Under the Hood Compilation)
To truly master React, you must understand what JSX compiles into.

```javascript
// What you write in your IDE:
const myElement = (
  <h1 className="title">
    Hello, World!
  </h1>
);

// What the Babel Compiler turns it into before sending it to the browser:
const myElement = React.createElement(
  'h1',                    // 1. The HTML tag (or Component)
  { className: 'title' },  // 2. The props/attributes object
  'Hello, World!'          // 3. The children (text or other elements)
);
```
Understanding this compilation reveals *why* you can't return two sibling elements: A JavaScript function can only `return` one object at a time!

## REAL_WORLD
Because JSX evaluates pure JavaScript, it is completely immune to **XSS (Cross-Site Scripting) Injection Attacks** by default.

If a malicious hacker sets their database username to `<script>alert('HACKED')</script>`, and you render it using `<h1>{user.name}</h1>`, React will intercept the string. It automatically sanitizes and escapes the HTML, safely printing the literal text `<script>...` to the screen instead of executing the malicious code in the browser. 

*(If you ever actually need to render raw HTML from a string, you must use the terrifyingly named `dangerouslySetInnerHTML` attribute, which forces you to acknowledge you are creating a security vulnerability).*

## INTERVIEW
**Q: Can you use a JavaScript `for` loop inside JSX?**
A: No. A `for` loop is a statement, not an expression. It does not return a value. If you need to loop over an array in JSX, you must use the `.map()` method, which is an expression that returns a new array of JSX elements.

**Q: Why do we use `className` instead of `class`?**
A: JSX compiles down into JavaScript objects that map directly to the Browser's DOM API. In the DOM API, the property to change a class is `element.className`, because the word `class` is a reserved keyword in JavaScript (used for creating Object-Oriented classes). 

**Q: Do I need to `import React from 'react'` to use JSX?**
A: Historically, yes. Because JSX compiled into `React.createElement()`, the `React` object had to be in scope, or the app would crash. However, starting in React 17 (released in 2020), a new JSX Transform was introduced. The compiler now automatically injects a hidden import from `react/jsx-runtime`. Therefore, in modern React, you no longer need to import React just to use JSX!

## FEYNMAN CHECK

Explain to a 10-year-old:
> JSX is NOT HTML inside JavaScript — it is syntactic sugar for `React.createElement` calls. `<div className="box">` compiles to `React.createElement('div', {className:'box'})`. Every JSX attribute is a JavaScript expression. This is why you use `className` (not `class`), `htmlFor` (not `for`), and `onClick` (not `onclick`). JSX is purely a build-time transformation — no JSX exists at runtime.
