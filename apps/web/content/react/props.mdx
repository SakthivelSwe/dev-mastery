## WHY
If you build a `<Button>` component that says "Click Me", that's great. But what if you need one button that says "Login", another that says "Sign Up", and another that says "Submit"? 

You don't want to create three separate components (`<LoginButton>`, `<SignUpButton>`). You want to create ONE reusable `<Button>` component and dynamically pass it the text it should display. 

React solves this with **Props** (short for Properties). Props are how Parent components pass data DOWN to their Child components, allowing you to configure and customize the Child at runtime.

## THEORY
### What are Props?
Props are conceptually identical to arguments passed into a standard JavaScript function. 
When a Parent renders `<Avatar name="Alice" />`, React takes the attribute `name="Alice"`, wraps it into a single JavaScript Object `{ name: "Alice" }`, and passes that object as the *first argument* to the `Avatar` component function.

### The Golden Rule of Props
**Props are strictly Read-Only.** A Child component must NEVER attempt to mutate or change the props it was given. If a Parent passes `age={25}`, the Child cannot do `props.age = 26`. Data in React flows strictly in one direction: Top-Down.

## CODE

### Level 1 — Beginner (Passing and Reading Props)
```javascript
// THE CHILD COMPONENT
// 1. React passes a single 'props' object as the first argument!
export function UserBadge(props) {
  return (
    <div className="badge">
      {/* 2. We read the data out of the object */}
      <h2>{props.name}</h2>
      <p>Role: {props.role}</p>
    </div>
  );
}
```

```javascript
// THE PARENT COMPONENT
import { UserBadge } from './UserBadge';

export default function App() {
  return (
    <div>
      {/* 3. We pass data down using HTML-like attributes! */}
      <UserBadge name="Alice" role="Admin" />
      <UserBadge name="Bob" role="User" />
    </div>
  );
}
```

### Level 2 — Building Blocks (Destructuring Props)
Typing `props.name` and `props.role` over and over gets tedious. The industry standard is to use **JavaScript Object Destructuring** directly in the function signature to instantly unpack the variables!

```javascript
// ✅ Industry Standard: Destructuring
// We instantly unpack 'name' and 'role' from the invisible props object!
export function UserBadge({ name, role }) {
  return (
    <div className="badge">
      <h2>{name}</h2>
      <p>Role: {role}</p>
    </div>
  );
}
```

### Level 3 — Intermediate (Default Props & Passing Data Types)
You aren't limited to passing Strings. You can pass Numbers, Arrays, Objects, or even other Functions! To pass anything other than a literal string, you must use curly braces `{}`.

You can also assign Default values in case the Parent forgets to pass the prop!

```javascript
// 1. We assign a Default Value for 'theme'
export function SettingsCard({ userObject, isPremium, theme = "dark" }) {
  return (
    <div className={`card ${theme}`}>
      <h3>{userObject.username}</h3>
      {isPremium ? <p>💎 Premium Member</p> : <p>Free Tier</p>}
    </div>
  );
}
```

```javascript
// THE PARENT
export default function App() {
  const myUser = { username: "Charlie", age: 30 };

  return (
    <SettingsCard 
      // 2. Passing an Object requires {}
      userObject={myUser} 
      
      // 3. Passing a Boolean requires {}
      isPremium={true} 
      
      // We didn't pass 'theme', so it defaults to "dark"!
    />
  );
}
```

### Level 4 — Advanced (The children Prop)
What if you want to build a generic `<Card>` component, and you want the Parent to decide what goes *inside* the card? 

You can pass JSX into the opening and closing tags of a component. React will automatically grab all of that inner JSX and pass it to the Child via a magical, built-in prop called **`children`**.

```javascript
// THE CHILD (A generic layout wrapper)
export function Card({ title, children }) {
  return (
    <div className="border border-gray-200 shadow-md">
      <h2>{title}</h2>
      
      {/* MAGIC: The Parent's injected JSX is rendered right here! */}
      <div className="card-body">
        {children}
      </div>
      
    </div>
  );
}
```

```javascript
// THE PARENT
export default function Dashboard() {
  return (
    // We pass 'title' as a standard prop...
    <Card title="Sales Metrics">
      
      {/* ...but EVERYTHING inside these tags becomes the 'children' prop! */}
      <img src="chart.png" />
      <button>Download CSV</button>
      
    </Card>
  );
}
```

## REAL_WORLD
If you are using **TypeScript** (which you absolutely should be in Enterprise React), you must strictly define the "Shape" of the Props object using an Interface. 

This gives you massive superpowers: If the Parent forgets to pass a required prop, or passes a String when the Child expects a Number, the IDE will instantly throw a red underline and refuse to compile the code!

```typescript
// ENTERPRISE TYPESCRIPT REACT
interface ButtonProps {
  text: string;
  color?: 'primary' | 'secondary'; // The ? makes it optional!
  isDisabled: boolean;
}

export function Button({ text, color = 'primary', isDisabled }: ButtonProps) {
  return (
    <button disabled={isDisabled} className={`btn-${color}`}>
      {text}
    </button>
  );
}
```

## INTERVIEW
**Q: Can a Child modify its Props?**
A: **Absolutely NOT**. Props are immutable (read-only). If a Child component needs to change the data it was given, it cannot mutate the prop directly. Instead, the Parent must pass down a "Setter Function" as a prop, and the Child must call that function to politely ask the Parent to update the data.

**Q: What happens if you pass a prop but don't define a value? (e.g. `<Input disabled />`)**
A: In JSX, if you pass a prop name with no value, it defaults to the boolean `true`. So `<Input disabled />` is identical to `<Input disabled={true} />`.

**Q: What is Prop Drilling?**
A: It is a notorious React anti-pattern. If the Root component has data, and a Child 5 levels deep needs that data, you have to pass the prop down through 4 intermediate components that don't even care about the data, just to transport it to the bottom. This pollutes the codebase. You solve Prop Drilling by using the Context API or a State Management library like Redux or Zustand.
