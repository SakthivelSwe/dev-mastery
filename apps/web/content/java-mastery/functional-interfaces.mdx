## WHY
We know that Java 8 Lambdas allow you to pass functions around like variables: `names.forEach(name -> print(name))`.
But Java is a strictly, statically typed language. Every variable *must* have a declared Type. If a Lambda is just a function, what is its Type? You can't write `Function f = (x) -> x + 1;` because `Function` wasn't a native keyword.

To solve this without breaking the entire type system, Java introduced the concept of **Functional Interfaces**. A Functional Interface is simply any standard Java `interface` that has exactly ONE abstract method. Because it only has one method, the Java Compiler can definitively look at a Lambda expression and say: "Ah, this lambda perfectly matches the signature of that single method, so the Lambda *is* an instance of this Interface!"

## THEORY
### The `@FunctionalInterface` Annotation
While any interface with a single abstract method works, Java provides the `@FunctionalInterface` annotation. It is completely optional, but highly recommended. It tells the compiler to enforce the rule: if another developer accidentally adds a second abstract method to the interface, the compiler will instantly fail the build, preventing the interface from breaking all the lambdas that rely on it.

### The Big Four (java.util.function)
To prevent developers from creating millions of custom functional interfaces for every little task, the JDK provides 43 built-in functional interfaces in the `java.util.function` package. They all revolve around 4 core paradigms:
1. **`Consumer<T>`**: Takes an input, returns nothing (`void`). Used for printing or modifying external state (e.g., `.forEach()`).
2. **`Supplier<T>`**: Takes no input, returns an object. Used for lazy generation or factories.
3. **`Function<T, R>`**: Takes an input, returns a result of a different type. Used for mapping/transformations (e.g., `.map()`).
4. **`Predicate<T>`**: Takes an input, returns a `boolean`. Used for filtering and matching (e.g., `.filter()`).

## CODE

### Level 1 — Beginner
```java
// Defining our own Functional Interface
@FunctionalInterface
public interface MathOperation {
    // Exactly ONE abstract method
    int operate(int a, int b);
}

public class BasicFI {
    public static void main(String[] args) {
        // The lambda perfectly matches the (int, int) -> int signature!
        MathOperation addition = (a, b) -> a + b;
        MathOperation multiplication = (a, b) -> a * b;
        
        System.out.println(addition.operate(5, 10)); // 15
    }
}
```

### Level 2 — Building Blocks
```java
import java.util.function.Predicate;
import java.util.function.Consumer;

public class BuiltInInterfaces {
    public static void main(String[] args) {
        // PREDICATE: Accepts T, returns boolean
        Predicate<String> isLong = str -> str.length() > 5;
        System.out.println(isLong.test("Hello World")); // true
        
        // CONSUMER: Accepts T, returns void
        Consumer<String> printer = str -> System.out.println("LOG: " + str);
        printer.accept("System Booting...");
    }
}
```

### Level 3 — Intermediate
```java
import java.util.function.Function;
import java.util.function.Supplier;

public class MoreInterfaces {
    public static void main(String[] args) {
        // FUNCTION: Accepts T, returns R (String -> Integer)
        Function<String, Integer> stringLength = str -> str.length();
        System.out.println(stringLength.apply("Java")); // 4
        
        // SUPPLIER: Accepts nothing, returns T
        Supplier<Double> randomGenerator = () -> Math.random();
        System.out.println(randomGenerator.get());
    }
}
```

### Level 4 — Advanced
```java
import java.util.function.Predicate;

public class DefaultMethodComposition {
    public static void main(String[] args) {
        Predicate<String> startsWithA = s -> s.startsWith("A");
        Predicate<String> lengthIs5 = s -> s.length() == 5;
        
        // Advanced: Functional Interfaces can have MULTIPLE Default methods!
        // We can compose them using .and(), .or(), .negate()
        Predicate<String> strictCheck = startsWithA.and(lengthIs5);
        
        System.out.println(strictCheck.test("Apple")); // true
        System.out.println(strictCheck.test("Avocado")); // false
    }
}
```

## REAL_WORLD
The most powerful use of `Supplier<T>` in the real world is **Lazy Logging**.

In high-performance systems, you might have a log statement that does a very heavy toString() calculation:
`logger.debug("Database state: " + massiveDbObject.serialize());`

If the logging level is set to `INFO`, the `debug` log won't actually print. BUT, Java still executes `massiveDbObject.serialize()` anyway to construct the String parameter before passing it to the method! This causes a massive performance hit for absolutely no reason.

Log4j2 and SLF4J solved this using `Supplier`.
`logger.debug(() -> "Database state: " + massiveDbObject.serialize());`

Because we pass a Lambda (`Supplier<String>`) instead of a raw String, the `.serialize()` method is NOT executed! Inside the logger, it checks `if (level == DEBUG)`. If false, it ignores the Lambda. The heavy calculation is completely bypassed!

## INTERVIEW
**Q: Can a Functional Interface have more than one method?**
A: A Functional Interface can have exactly ONE **abstract** method. However, it can have an infinite number of `default` methods and `static` methods. The compiler only cares that there is exactly one undefined method that the Lambda expression will fulfill.

**Q: If you remove the `@FunctionalInterface` annotation, will the lambdas stop working?**
A: No, they will still work perfectly. The annotation is just a compiler directive (like `@Override`). It explicitly tells the compiler to check the interface and throw an error if it finds 0 or 2+ abstract methods, protecting the interface from accidental breakage by other developers.

**Q: What is a `BiFunction`?**
A: Standard Functional Interfaces like `Function<T, R>` and `Predicate<T>` only take exactly one input parameter. If you need a lambda that accepts TWO parameters, you must use the "Bi" variants provided by the JDK, such as `BiFunction<T, U, R>`, `BiConsumer<T, U>`, or `BiPredicate<T, U>`.
