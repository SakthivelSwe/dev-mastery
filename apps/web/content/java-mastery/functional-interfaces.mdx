## WHY

Functional interfaces are the backbone of Java's functional programming capabilities — every lambda expression is an instance of a functional interface. You need to understand them to:

1. **Use lambdas correctly** — Every lambda target type is a functional interface
2. **Use Streams API** — `map()`, `filter()`, `reduce()` all take functional interfaces (`Function`, `Predicate`, `Consumer`)
3. **Design flexible APIs** — Accept lambdas as parameters to make your code configurable
4. **Compose behavior** — `andThen()`, `compose()`, `and()`, `or()`, `negate()` for function composition
5. **Interview** — "What is a functional interface?" followed by "What is the difference between Function and BiFunction?" is very common

## THEORY

### What Is a Functional Interface?

A functional interface is an interface with **exactly one abstract method**. It can have any number of default or static methods.

```java
@FunctionalInterface
public interface Runnable {
    void run(); // exactly ONE abstract method
}

@FunctionalInterface
public interface Comparator<T> {
    int compare(T o1, T o2); // exactly ONE abstract method
    // + many default methods: reversed(), thenComparing(), etc.
    // + static method: naturalOrder(), reverseOrder(), etc.
}
```

The `@FunctionalInterface` annotation is optional but recommended — it causes a compile error if you accidentally add a second abstract method.

### Lambda = Functional Interface Instance

```java
// Anonymous class — verbose
Runnable r1 = new Runnable() {
    @Override public void run() { System.out.println("Hello"); }
};

// Lambda — concise
Runnable r2 = () -> System.out.println("Hello");

// They're the same! A lambda is syntactic sugar for a single-method anonymous class.
// The target type must be a functional interface.
```

### The java.util.function Package — Core Functional Interfaces

| Interface | Abstract Method | Description | Lambda |
|-----------|----------------|-------------|--------|
| `Supplier<T>` | `T get()` | Produces a value, no input | `() -> "hello"` |
| `Consumer<T>` | `void accept(T t)` | Consumes a value, no return | `s -> print(s)` |
| `BiConsumer<T,U>` | `void accept(T t, U u)` | Consumes two values | `(k,v) -> map.put(k,v)` |
| `Function<T,R>` | `R apply(T t)` | Maps T to R | `s -> s.length()` |
| `BiFunction<T,U,R>` | `R apply(T t, U u)` | Maps T,U to R | `(a,b) -> a+b` |
| `UnaryOperator<T>` | `T apply(T t)` | Maps T to T (extends Function) | `s -> s.toUpperCase()` |
| `BinaryOperator<T>` | `T apply(T t1, T t2)` | Maps T,T to T | `(a,b) -> a+b` |
| `Predicate<T>` | `boolean test(T t)` | Tests condition | `s -> s.isEmpty()` |
| `BiPredicate<T,U>` | `boolean test(T t, U u)` | Tests two values | `(a,b) -> a>b` |

**Primitive specializations (avoid boxing):**

| Interface | Avoids boxing for |
|-----------|------------------|
| `IntFunction<R>` | int input |
| `ToIntFunction<T>` | int result |
| `IntUnaryOperator` | int → int |
| `IntBinaryOperator` | int,int → int |
| `IntSupplier` | produces int |
| `IntConsumer` | consumes int |
| `IntPredicate` | tests int |
| (Same for Long, Double) | |

### Function Composition

```java
// Function<T,R> composition
Function<Integer, Integer> doubleIt = x -> x * 2;
Function<Integer, Integer> addTen = x -> x + 10;

// compose: addTen THEN doubleIt (right to left)
Function<Integer, Integer> addTenThenDouble = doubleIt.compose(addTen);
System.out.println(addTenThenDouble.apply(5)); // (5+10)*2 = 30

// andThen: doubleIt THEN addTen (left to right)
Function<Integer, Integer> doubleThenAddTen = doubleIt.andThen(addTen);
System.out.println(doubleThenAddTen.apply(5)); // 5*2+10 = 20
```

### Predicate Composition

```java
Predicate<String> notEmpty = s -> !s.isEmpty();
Predicate<String> startsWithA = s -> s.startsWith("A");
Predicate<String> longEnough = s -> s.length() > 3;

// and(): both conditions
Predicate<String> validName = notEmpty.and(startsWithA).and(longEnough);
System.out.println(validName.test("Alice")); // true
System.out.println(validName.test("Bob"));   // false (doesn't start with A)
System.out.println(validName.test("Al"));    // false (too short)

// or(): either condition
Predicate<String> aOrB = startsWithA.or(s -> s.startsWith("B"));

// negate(): flip condition
Predicate<String> doesNotStartWithA = startsWithA.negate();
```

### Method References

Method references are shorthand for lambdas that call a single method:

| Type | Syntax | Equivalent lambda |
|------|--------|------------------|
| Static method | `ClassName::staticMethod` | `x -> ClassName.staticMethod(x)` |
| Instance method of specific object | `object::instanceMethod` | `x -> object.instanceMethod(x)` |
| Instance method of arbitrary instance | `ClassName::instanceMethod` | `x -> x.instanceMethod()` |
| Constructor | `ClassName::new` | `x -> new ClassName(x)` |

```java
// Static method reference
Function<String, Integer> parseInt = Integer::parseInt;  // x -> Integer.parseInt(x)

// Instance method of specific object
String prefix = "Hello ";
Function<String, String> greet = prefix::concat;  // s -> prefix.concat(s)

// Instance method of arbitrary instance
Function<String, String> toUpper = String::toUpperCase;  // s -> s.toUpperCase()
Predicate<String> isEmpty = String::isEmpty;             // s -> s.isEmpty()

// Constructor reference
Supplier<ArrayList<String>> listFactory = ArrayList::new;  // () -> new ArrayList<>()
Function<String, StringBuilder> sbFactory = StringBuilder::new;  // s -> new StringBuilder(s)
```

## CODE

### Level 1 — Beginner: Using Built-in Functional Interfaces

```java
// FunctionalInterfaceBasics.java
import java.util.function.*;

public class FunctionalInterfaceBasics {
    public static void main(String[] args) {
        // Supplier — produces value with no input
        Supplier<Double> random = Math::random;
        System.out.println(random.get()); // e.g., 0.7342...

        Supplier<String> greeting = () -> "Hello, World!";
        System.out.println(greeting.get());

        // Consumer — consumes value, returns nothing
        Consumer<String> print = System.out::println;
        print.accept("Consumed!");

        Consumer<String> shout = s -> System.out.println(s.toUpperCase());
        shout.accept("hello"); // HELLO

        // andThen — chain consumers
        Consumer<String> printAndShout = print.andThen(shout);
        printAndShout.accept("test"); // test \n TEST

        // Function — transforms T to R
        Function<String, Integer> length = String::length;
        System.out.println(length.apply("hello")); // 5

        Function<Integer, Boolean> isEven = n -> n % 2 == 0;
        System.out.println(isEven.apply(4)); // true

        // Predicate — tests condition
        Predicate<String> isEmail = s -> s.contains("@");
        System.out.println(isEmail.test("user@example.com")); // true
        System.out.println(isEmail.test("not-an-email"));     // false

        // Predicate.not() — negate (Java 11)
        Predicate<String> notEmail = Predicate.not(isEmail);
        System.out.println(notEmail.test("user@example.com")); // false

        // UnaryOperator — T → T
        UnaryOperator<String> trim = String::trim;
        System.out.println(trim.apply("  hello  ")); // "hello"

        // BinaryOperator — T,T → T
        BinaryOperator<Integer> max = Integer::max;
        System.out.println(max.apply(5, 10)); // 10
    }
}
```

### Level 2 — Elementary: Custom Functional Interfaces

```java
// CustomFunctionalInterfaces.java
import java.util.function.*;

public class CustomFunctionalInterfaces {

    // Custom functional interface — throws checked exception
    @FunctionalInterface
    interface CheckedFunction<T, R> {
        R apply(T t) throws Exception;

        // Wrap to regular Function (swallows exception — only for demo)
        static <T, R> Function<T, R> wrap(CheckedFunction<T, R> f) {
            return t -> {
                try { return f.apply(t); }
                catch (Exception e) { throw new RuntimeException(e); }
            };
        }
    }

    // Validator with message
    @FunctionalInterface
    interface Validator<T> {
        ValidationResult validate(T value);

        record ValidationResult(boolean valid, String message) {
            static ValidationResult ok() { return new ValidationResult(true, ""); }
            static ValidationResult fail(String msg) { return new ValidationResult(false, msg); }
        }

        default Validator<T> and(Validator<T> other) {
            return value -> {
                ValidationResult result = this.validate(value);
                if (!result.valid()) return result;
                return other.validate(value);
            };
        }
    }

    public static void main(String[] args) throws Exception {
        // Custom checked function
        CheckedFunction<String, Integer> parseIfPossible = Integer::parseInt;
        Function<String, Integer> safe = CheckedFunction.wrap(parseIfPossible);
        System.out.println(safe.apply("42")); // 42
        // safe.apply("abc") → RuntimeException(NumberFormatException)

        // Validators with composition
        Validator<String> notEmpty = s ->
            s != null && !s.isBlank()
                ? Validator.ValidationResult.ok()
                : Validator.ValidationResult.fail("Cannot be empty");

        Validator<String> minLength = s ->
            s != null && s.length() >= 8
                ? Validator.ValidationResult.ok()
                : Validator.ValidationResult.fail("Min 8 characters");

        Validator<String> hasNumber = s ->
            s != null && s.chars().anyMatch(Character::isDigit)
                ? Validator.ValidationResult.ok()
                : Validator.ValidationResult.fail("Must contain a number");

        Validator<String> passwordValidator = notEmpty.and(minLength).and(hasNumber);

        System.out.println(passwordValidator.validate("password1")); // valid=true
        System.out.println(passwordValidator.validate("short"));     // Min 8 characters
        System.out.println(passwordValidator.validate("longpassword")); // Must contain a number
    }
}
```

### Level 3 — Intermediate: Function Composition and Pipeline

```java
// FunctionPipeline.java
import java.util.function.*;
import java.util.*;
import java.util.stream.*;

public class FunctionPipeline {

    // Data transformation pipeline using Function composition
    record Order(String product, int quantity, double unitPrice, String discount) {}
    record ProcessedOrder(String product, int quantity, double totalPrice, double finalPrice) {}

    static Function<Order, ProcessedOrder> calculateTotal =
        o -> new ProcessedOrder(o.product(), o.quantity(),
            o.quantity() * o.unitPrice(), o.quantity() * o.unitPrice());

    static UnaryOperator<ProcessedOrder> applyDiscount = o -> {
        double discounted = switch (o.totalPrice() > 100 ? "BULK" : "NONE") {
            case "BULK" -> o.totalPrice() * 0.9;
            default -> o.totalPrice();
        };
        return new ProcessedOrder(o.product(), o.quantity(), o.totalPrice(), discounted);
    };

    static Consumer<ProcessedOrder> printReceipt = o ->
        System.out.printf("%-20s qty=%-5d subtotal=$%-8.2f final=$%.2f%n",
            o.product(), o.quantity(), o.totalPrice(), o.finalPrice());

    // Compose the pipeline
    static Function<Order, ProcessedOrder> processOrder =
        calculateTotal.andThen(applyDiscount);

    public static void main(String[] args) {
        List<Order> orders = List.of(
            new Order("Java Book", 1, 45.0, "NONE"),
            new Order("Laptop Stand", 5, 25.0, "BULK"),
            new Order("Keyboard", 2, 80.0, "BULK")
        );

        orders.stream()
            .map(processOrder)
            .forEach(printReceipt);

        // Predicate composition for filtering
        Predicate<Order> isExpensive = o -> o.unitPrice() > 50;
        Predicate<Order> isBulk = o -> o.quantity() >= 3;
        Predicate<Order> isVIPOrder = isExpensive.or(isBulk);

        System.out.println("\nVIP Orders:");
        orders.stream()
            .filter(isVIPOrder)
            .map(o -> o.product())
            .forEach(System.out::println);

        // BiFunction example
        BiFunction<String, Integer, String> repeat = (s, n) -> s.repeat(n);
        System.out.println(repeat.apply("ab", 4)); // abababab

        // Function as parameter — strategy pattern
        List<Integer> nums = List.of(1, 2, 3, 4, 5);
        System.out.println(transform(nums, n -> n * n));   // [1, 4, 9, 16, 25]
        System.out.println(transform(nums, n -> n * 2));   // [2, 4, 6, 8, 10]
    }

    static <T, R> List<R> transform(List<T> items, Function<T, R> mapper) {
        return items.stream().map(mapper).collect(Collectors.toList());
    }
}
```

### Level 4 — Advanced: Partial Application & Currying

```java
// HigherOrderFunctions.java
import java.util.function.*;
import java.util.*;

public class HigherOrderFunctions {

    // Currying — convert multi-arg function to chain of single-arg functions
    static <A, B, C> Function<A, Function<B, C>> curry(BiFunction<A, B, C> f) {
        return a -> b -> f.apply(a, b);
    }

    // Partial application — fix one argument
    static <A, B, C> Function<B, C> partial(BiFunction<A, B, C> f, A a) {
        return b -> f.apply(a, b);
    }

    // Memoize — cache function results
    static <T, R> Function<T, R> memoize(Function<T, R> fn) {
        Map<T, R> cache = new HashMap<>();
        return input -> cache.computeIfAbsent(input, fn);
    }

    // Retry — retry a supplier up to n times
    static <T> Supplier<T> withRetry(Supplier<T> supplier, int maxAttempts) {
        return () -> {
            Exception lastEx = null;
            for (int i = 0; i < maxAttempts; i++) {
                try { return supplier.get(); }
                catch (Exception e) { lastEx = e; }
            }
            throw new RuntimeException("Failed after " + maxAttempts + " attempts", lastEx);
        };
    }

    public static void main(String[] args) {
        // Currying
        BiFunction<Integer, Integer> add = (a, b) -> a + b;
        Function<Integer, Function<Integer, Integer>> curriedAdd = curry(add);
        Function<Integer, Integer> add5 = curriedAdd.apply(5);
        System.out.println(add5.apply(3));  // 8
        System.out.println(add5.apply(10)); // 15

        // Partial application
        BiFunction<String, String, String> format = (template, value) ->
            template.replace("{}", value);
        Function<String, String> formatName = partial(format, "Hello, {}!");
        System.out.println(formatName.apply("Alice")); // Hello, Alice!
        System.out.println(formatName.apply("Bob"));   // Hello, Bob!

        // Memoization — expensive computation cached
        Function<Integer, Long> fibonacci = memoize(new Function<>() {
            final Function<Integer, Long> memo = memoize(this);
            @Override public Long apply(Integer n) {
                if (n <= 1) return (long) n;
                return memo.apply(n-1) + memo.apply(n-2);
            }
        });
        System.out.println(fibonacci.apply(10)); // 55

        // Compose multiple functions into pipeline
        List<Function<String, String>> transformers = List.of(
            String::trim,
            String::toLowerCase,
            s -> s.replace(" ", "_"),
            s -> s.replaceAll("[^a-z0-9_]", "")
        );

        // Fold into single function using reduce
        Function<String, String> slugify = transformers.stream()
            .reduce(Function.identity(), Function::andThen);

        System.out.println(slugify.apply("  Hello World! 2024 ")); // hello_world_2024
    }
}
```

### Level 5 — Expert: Functional Design Patterns

```java
// FunctionalPatterns.java
import java.util.*;
import java.util.function.*;

public class FunctionalPatterns {

    // 1. Result type (monadic error handling)
    sealed interface Result<T> permits Result.Success, Result.Failure {
        record Success<T>(T value) implements Result<T> {}
        record Failure<T>(String error) implements Result<T> {}

        static <T> Result<T> of(Supplier<T> supplier) {
            try { return new Success<>(supplier.get()); }
            catch (Exception e) { return new Failure<>(e.getMessage()); }
        }

        default <R> Result<R> map(Function<T, R> mapper) {
            return switch (this) {
                case Success<T> s -> Result.of(() -> mapper.apply(s.value()));
                case Failure<T> f -> new Failure<>(f.error());
            };
        }

        default <R> Result<R> flatMap(Function<T, Result<R>> mapper) {
            return switch (this) {
                case Success<T> s -> mapper.apply(s.value());
                case Failure<T> f -> new Failure<>(f.error());
            };
        }
    }

    // 2. Command pattern with lambdas
    record Command(String name, Runnable action, Runnable undo) {}

    static class CommandHistory {
        private final Deque<Command> history = new ArrayDeque<>();

        void execute(Command cmd) {
            cmd.action().run();
            history.push(cmd);
        }

        void undo() {
            if (!history.isEmpty()) {
                Command cmd = history.pop();
                System.out.println("Undoing: " + cmd.name());
                cmd.undo().run();
            }
        }
    }

    public static void main(String[] args) {
        // Result type (Railway Oriented Programming)
        Result<Integer> result = Result.of(() -> Integer.parseInt("42"))
            .map(n -> n * 2)
            .map(n -> n + 10);
        System.out.println(result); // Success[value=94]

        Result<Integer> failed = Result.of(() -> Integer.parseInt("abc"))
            .map(n -> n * 2);
        System.out.println(failed); // Failure[error=For input string: "abc"]

        // Command pattern with undo
        List<String> document = new ArrayList<>(List.of("Hello", "World"));
        CommandHistory history = new CommandHistory();

        Command addExclamation = new Command(
            "AddExclamation",
            () -> document.add("!"),
            () -> document.remove(document.size() - 1)
        );
        Command capitalize = new Command(
            "Capitalize",
            () -> document.replaceAll(String::toUpperCase),
            () -> document.replaceAll(String::toLowerCase)
        );

        history.execute(addExclamation);
        System.out.println(document); // [Hello, World, !]
        history.execute(capitalize);
        System.out.println(document); // [HELLO, WORLD, !]
        history.undo();
        System.out.println(document); // [hello, world, !]
        history.undo();
        System.out.println(document); // [hello, world]

        // Function pipeline with error handling
        Function<String, Result<Integer>> safeParseAndDouble =
            s -> Result.of(() -> Integer.parseInt(s))
                       .map(n -> n * 2);

        List.of("10", "abc", "25", "not_a_number", "5")
            .stream()
            .map(safeParseAndDouble)
            .forEach(System.out::println);
    }
}
```

### Stream API with Functional Interfaces

List<Employee> employees = getEmployees();

// Predicate for filtering
Predicate<Employee> isActive = e -> e.getStatus() == Status.ACTIVE;

## INTERVIEW

**Q1: What is a functional interface and what makes it "functional"?**

> **Strong answer:** A functional interface is an interface with exactly one abstract method (SAM — Single Abstract Method). The "functional" part means it represents a single function/behavior. It can have any number of default or static methods — only abstract methods count. The `@FunctionalInterface` annotation is optional but causes a compile-time error if the interface doesn't qualify. Any functional interface can be implemented with a lambda expression — this is what enables Java's lambda syntax. Examples: `Runnable`, `Callable`, `Comparator`, all interfaces in `java.util.function`.

**Q2: What is the difference between Function, Predicate, Consumer, and Supplier?**

> **Strong answer:** `Function<T,R>` — takes T, returns R — for transformation. `Predicate<T>` — takes T, returns boolean — for testing/filtering conditions. `Consumer<T>` — takes T, returns void — for side effects (printing, storing). `Supplier<T>` — takes nothing, returns T — for lazy value production. Memory trick: Supplier supplies, Consumer consumes, Predicate predicates (tests), Function transforms. All four compose: `Function.andThen()`, `Predicate.and()/.or()/.negate()`, `Consumer.andThen()`.

**Q3: What is the difference between `Function.compose()` and `Function.andThen()`?**

> **Strong answer:** Both combine two functions. `f.andThen(g)` = first apply f, then g (left to right): `g(f(x))`. `f.compose(g)` = first apply g, then f (right to left): `f(g(x))`. Mnemonic: `andThen` reads left-to-right like code execution order. `compose` is mathematical function composition notation where `(f∘g)(x) = f(g(x))`. Example: if `double = x*2` and `addTen = x+10`, then `double.andThen(addTen).apply(5) = 20` (5*2+10), but `double.compose(addTen).apply(5) = 30` ((5+10)*2).

**Q4: Why are there primitive specializations like IntFunction, IntPredicate, etc.?**

> **Strong answer:** Java generics don't work with primitives — `Function<int, int>` won't compile. You'd need `Function<Integer, Integer>` which requires autoboxing (wrapping int in Integer object). Autoboxing creates garbage objects and has CPU overhead. For performance-critical code processing millions of numbers, this adds up. Primitive specializations like `IntFunction<R>`, `IntUnaryOperator`, `IntPredicate` work directly with `int` values without boxing. `IntStream.mapToObj()` vs `Stream<Integer>.map()` is the same concept. Always prefer primitive specializations when working with numeric data in performance-sensitive code.

## FEYNMAN CHECK

A functional interface is like a contract that says "you can represent this entire interface with just one piece of behavior." A `Comparator` says "I just need to know how to compare two things — everything else flows from that." A lambda is the behavior you're plugging in. Instead of writing a whole class just to define that one behavior, you write a brief expression. It's like saying "here's how I compare things" in one line instead of writing a five-line class.
