## 1. Architectural Blueprint & Core Concepts

Modern design systems use **OKLCH** (or HSL) color spaces instead of RGB or Hex. OKLCH is perceptually uniform, meaning that changes in lightness map accurately to human perception. This makes programmatically generating accessible color palettes much more reliable.

```
OKLCH Color Model:
  L: Lightness (0% to 100%)
  C: Chroma (Color intensity/saturation)
  H: Hue (Angle: 0 to 360)
```

By defining colors in OKLCH, dark mode transformations can be managed by adjusting the Lightness ($L$) value while keeping Chroma ($C$) and Hue ($H$) constant.

## 2. Figma Workflow & Sync Pipeline

### Figma Setup
1. Define primitive colors inside Figma Variables using Hue-based scales.
2. Set up semantic variables (e.g., `--color-surface-primary`, `--color-text-primary`).
3. Create two variable **Modes**: `Light` and `Dark`.
4. Bind semantic variables to corresponding primitive values for each mode.
5. Export variables to token JSON using the Figma Variables API.

  /* --- Primitive Hues (OKLCH) --- */
  --brand-h: 250;
  --brand-c: 0.25;

  /* --- Semantic Tokens: Light Mode --- */
  --color-bg-primary: oklch(0.98 0.01 var(--brand-h));
  --color-bg-secondary: oklch(0.95 0.02 var(--brand-h));
  --color-text-primary: oklch(0.15 0.02 var(--brand-h));
  --color-text-secondary: oklch(0.45 0.04 var(--brand-h));
  --color-interactive-primary: oklch(0.55 var(--brand-c) var(--brand-h));
  --color-interactive-hover: oklch(0.45 var(--brand-c) var(--brand-h));

[data-theme='dark'] {
  /* --- Semantic Tokens: Dark Mode --- */
  --color-bg-primary: oklch(0.15 0.02 var(--brand-h));
  --color-bg-secondary: oklch(0.20 0.02 var(--brand-h));
  --color-text-primary: oklch(0.95 0.01 var(--brand-h));

}

## 4. Accessibility Compliance (WCAG 2.1 AA)
- **Contrast Check (Success Criterion 1.4.3):** Primary text (`--color-text-primary`) must pass a **4.5:1** contrast ratio against its background (`--color-bg-primary`).
- **Interactive States Contrast (Success Criterion 1.4.11):** Buttons and interactive focus boundaries must maintain a **3:1** contrast ratio against adjacent surfaces.

## 5. React & TypeScript Implementation

A highly functional, accessible Theme Context Provider that injects and manages themes at the application root.

```tsx
import React, { createContext, useContext, useEffect, useState } from 'react';

type Theme = 'light' | 'dark';

interface ThemeContextType {
  theme: Theme;
  toggleTheme: () => void;
}

const ThemeContext = createContext<ThemeContextType | undefined>(undefined);

export const ThemeProvider: React.FC<{ children: React.ReactNode }> = ({ children }) => {
  const [theme, setTheme] = useState<Theme>(() => {
    if (typeof window !== 'undefined') {
      const saved = localStorage.getItem('theme') as Theme;
      if (saved) return saved;
      return window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
    }
    return 'light';
  });

  useEffect(() => {
    const root = window.document.documentElement;
    root.setAttribute('data-theme', theme);
    root.className = theme;
    localStorage.setItem('theme', theme);
  }, [theme]);

  const toggleTheme = () => {
    setTheme((prev) => (prev === 'light' ? 'dark' : 'light'));
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};

export const useTheme = () => {
  const context = useContext(ThemeContext);
  if (!context) throw new Error('useTheme must be used within a ThemeProvider');
  return context;
};

export const ThemeToggler: React.FC = () => {
  const { theme, toggleTheme } = useTheme();

  return (
    <button
      onClick={toggleTheme}
      className="px-4 py-2 rounded-lg font-medium bg-action-primary hover:bg-action-hover text-surface-primary transition-colors focus:outline-none focus:ring-2 focus:ring-action-primary focus:ring-offset-2"
      aria-label={`Switch to ${theme === 'light' ? 'dark' : 'light'} theme`}
    >
      Toggle {theme === 'light' ? 'Dark' : 'Light'} Mode
    </button>
  );
};
```
