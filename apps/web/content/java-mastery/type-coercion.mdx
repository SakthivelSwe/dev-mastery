# Type Coercion and Abstract Equality | Level: 2

Type coercion is the automatic or implicit conversion of values from one data type to another. It is governed by a series of precise algorithms defined in the ECMAScript specification.

---

## 1. The `ToPrimitive` Abstract Operation

When JavaScript needs to convert an object to a primitive value (such as during addition or comparison), it uses the internal `ToPrimitive(input, PreferredType)` operation.

The algorithm works as follows:
1.  If the object has a `[Symbol.toPrimitive](hint)` method, it calls it.
2.  Otherwise, if the `PreferredType` hint is `string`:
    *   Call `input.toString()`. If it returns a primitive, return it.
    *   Call `input.valueOf()`. If it returns a primitive, return it.
    *   Otherwise, throw a `TypeError`.
3.  If the `PreferredType` hint is `number` or default:
    *   Call `input.valueOf()`. If it returns a primitive, return it.
    *   Call `input.toString()`. If it returns a primitive, return it.
    *   Otherwise, throw a `TypeError`.

```javascript
const obj = {
  valueOf() { return 42; },
  toString() { return "hello"; }
};

console.log(obj + 10); // 52 (valueOf is called first for number hint)
console.log(`${obj}`); // "hello" (toString is called first for string hint)
```

## 2. Implicit Coercion: Addition vs. Subtraction

The behavior of standard arithmetic operators is a major source of confusion:

### The Addition Operator (`+`)
The `+` operator performs both numeric addition and string concatenation. If **either** operand is a string (or becomes a string via `ToPrimitive`), it coerces both to strings and concatenates them.

```javascript
console.log(1 + "2");      // "12"
console.log(1 + 2 + "3");  // "33" (1 + 2 executes first -> 3 + "3" -> "33")
console.log("3" + 1 + 2);  // "312" ("3" + 1 -> "31" + 2 -> "312")
console.log([] + {});      // "[object Object]" ([] becomes "", {} becomes "[object Object]")
```
