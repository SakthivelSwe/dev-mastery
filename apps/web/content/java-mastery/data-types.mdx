# JavaScript Data Types — Primitives and References | Level: 1

JavaScript is dynamically typed, meaning variables do not have static types; instead, the values stored in them do. Every value in JavaScript is either a **Primitive** or a **Reference Type (Object)**.

---

## 1. The 7 Primitive Types

Primitives are values that are not objects and have no methods. They are **immutable**—once created, their value cannot be changed. Any operation on a primitive returns a brand new primitive value.

1.  **String:** Represents textual data.
2.  **Number:** Double-precision 64-bit binary format IEEE 754 value. Handles both integers and floats.
3.  **BigInt (ES2020):** Represents integers with arbitrary precision, bypassing the Safe Integer limit of Numbers ($2^{53} - 1$). Created by appending `n` to an integer (e.g., `9007199254740991n`).
4.  **Boolean:** Logical entity containing `true` or `false`.
5.  **Undefined:** Automatically assigned to variables that have been declared but not initialized.
6.  **Null:** Represents the intentional absence of any object value.
7.  **Symbol (ES6):** Unique and immutable primitive value often used as private keys for object properties.

```javascript
let str = "hello";
str.toUpperCase(); // Returns "HELLO", but does not change str
console.log(str); // "hello"
```

---

## 2. The Infamous `typeof null` Bug

One of JavaScript's oldest and most famous design flaws is:

```javascript
console.log(typeof null); // "object"
```

### Why does this happen?
In the first implementation of JavaScript (written by Brendan Eich in 1995), values were stored in 32-bit units, consisting of a **type tag** and the actual value. 

*   The type tag for objects was `000`.
*   `null` was represented as the null pointer (all 32 bits set to 0).
*   Because the pointer was 0, its type tag read as `000` (object). Therefore, `typeof null` returned `"object"`.

A proposal to fix this in ECMAScript was rejected because changing it would break millions of legacy websites that rely on `typeof null === "object"`.

## 3. The Mechanics of `NaN`

`NaN` stands for **Not-a-Number**. Despite its name, its technical type is `"number"`.

```javascript
console.log(typeof NaN); // "number"
```

`NaN` is returned when a mathematical operation fails or results in an undefined numerical value (e.g., `0 / 0`, `Math.sqrt(-1)`, or parsing an invalid string `parseInt("hello")`).

### The Self-Inequality Quirks
`NaN` is the only value in JavaScript that is **not equal to itself**.

```javascript
console.log(NaN === NaN); // false
console.log(NaN == NaN);  // false
```

### Checking for `NaN`
Do not use `==` or `===` to check for `NaN`. Instead, use one of the following:

```javascript
// 1. Global isNaN() - Coerces its argument to a number first (Avoid this!)
console.log(isNaN("hello")); // true (because "hello" coerces to NaN)

// 2. Number.isNaN() - ES6. Checks if value is strictly NaN without coercion (Preferred)
console.log(Number.isNaN("hello")); // false
console.log(Number.isNaN(NaN));     // true

// 3. Object.is() - ES6. Strict equality check that handles NaN correctly
console.log(Object.is(NaN, NaN));   // true
```

## 4. Reference Types: Objects, Arrays, and Functions

Reference types do not store the actual value directly in the variable. Instead, they store a **pointer** (reference) to a location in the system's heap memory where the object is stored.

```javascript
const obj1 = { value: 10 };
const obj2 = obj1; // Copies the pointer, not the object

obj2.value = 20;
console.log(obj1.value); // 20 (both variables point to the same heap object)
```

```
Stack Memory                 Heap Memory
+--------+--------+          +--------------------+
| Name   | Value  |          | Address | Value    |
+--------+--------+          +--------------------+
| obj1   | #001  ─┼─────────►| #001    | {v: 20}  |
| obj2   | #001  ─┼─────────►|                    |
+--------+--------+          +--------------------+
```

## 5. Strict vs. Abstract Equality (`===` vs. `==`)

*   **Abstract Equality (`==`):** Compares two values for equality after performing implicit type coercion if the types do not match.
*   **Strict Equality (`===`):** Compares both the **type** and the **value**. If the types are different, it immediately returns `false`.

```javascript
console.log(5 == "5");   // true (coerced)
console.log(5 === "5");  // false (different types)
```

Always default to `===` to prevent unexpected bugs caused by implicit coercion.

## 6. Falsy Values in JavaScript

When coerced to a boolean (e.g., inside an `if` statement), there are exactly **eight** values that evaluate to `false` (all other values evaluate to `true`):

1.  `false`
2.  `0` (and `-0`, `0n`)
3.  `""` (empty string)
4.  `null`
5.  `undefined`
6.  `NaN`
7.  `document.all` (a legacy browser host object exception)

```javascript
if ("") {
  console.log("This will not execute");
}

if ([]) {
  console.log("This WILL execute (arrays are objects, objects are always truthy)");
}
