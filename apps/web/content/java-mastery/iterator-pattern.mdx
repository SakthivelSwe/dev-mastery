## WHY
If you want to print all the elements in an `ArrayList`, you can use a simple `for` loop with an index counter (`for (int i=0; i < list.size(); i++)`). 
But what if you switch the underlying collection to a `LinkedList`? Using `get(i)` inside a loop on a LinkedList is catastrophic (`O(n^2)` time) because it has to traverse from the head pointer for every single index request.
What if you switch to a `HashSet`? A Set doesn't even have index positions, so `get(i)` won't compile at all!

The Iterator Pattern solves this by completely decoupling the *traversal logic* from the *business logic*. It provides a universal `Iterator` object that knows exactly how to navigate its specific data structure efficiently, allowing you to iterate over Arrays, Linked Lists, Hash Tables, and Trees using the exact same uniform syntax.

## THEORY
### `Iterable` vs `Iterator`
In Java, these are two distinct interfaces:
1. **`java.lang.Iterable`**: Any class that implements this is promising "I can be iterated over". It requires exactly one method: `Iterator<T> iterator()`. Every Collection in Java implements `Iterable`.
2. **`java.util.Iterator`**: The actual pointer object that does the work. It tracks the current position and provides `hasNext()` and `next()`.

### The Enhanced For-Loop
In Java 5, the "Enhanced For-Loop" (`for (String s : list)`) was introduced. This is pure syntax sugar! Behind the scenes, the Java Compiler actually rewrites that code into an explicit `Iterator` while-loop. This means the Enhanced For-Loop inherits all the safety, speed, and polymorphism of the Iterator pattern automatically.

## CODE

### Level 1 — Beginner
```java
import java.util.List;
import java.util.ArrayList;
import java.util.Iterator;

public class BasicIterator {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>(List.of("Alice", "Bob", "Charlie"));
        
        // The old-school, explicit Iterator loop
        Iterator<String> it = names.iterator();
        while (it.hasNext()) {
            String name = it.next();
            System.out.println(name);
        }
    }
}
```

### Level 2 — Building Blocks
```java
import java.util.Set;
import java.util.HashSet;

public class EnhancedForLoop {
    public static void main(String[] args) {
        Set<Integer> numbers = new HashSet<>(Set.of(10, 20, 30));
        
        // This is 100% equivalent to the Iterator while-loop.
        // The compiler generates the Iterator bytecode automatically.
        // It works on ANY class that implements Iterable!
        for (Integer num : numbers) {
            System.out.println(num);
        }
    }
}
```

### Level 3 — Intermediate
```java
import java.util.List;
import java.util.ArrayList;
import java.util.Iterator;

public class SafeRemoval {
    public static void main(String[] args) {
        List<String> tasks = new ArrayList<>(List.of("TODO", "DONE", "TODO"));
        
        // TRAP: Using list.remove() inside a for-loop throws ConcurrentModificationException!
        
        // Correct approach: Use the Iterator's dedicated remove() method
        Iterator<String> it = tasks.iterator();
        while (it.hasNext()) {
            if (it.next().equals("DONE")) {
                it.remove(); // Safely removes the last element returned by next()
            }
        }
        
        System.out.println(tasks); // [TODO, TODO]
    }
}
```

### Level 4 — Advanced
```java
import java.util.Iterator;
import java.util.NoSuchElementException;

// Implementing a Custom Iterable!
class CustomRange implements Iterable<Integer> {
    private final int start, end;
    
    public CustomRange(int start, int end) {
        this.start = start; this.end = end;
    }

    @Override
    public Iterator<Integer> iterator() {
        // Return an anonymous class implementing Iterator
        return new Iterator<>() {
            private int current = start;

            @Override
            public boolean hasNext() {
                return current <= end;
            }

            @Override
            public Integer next() {
                if (!hasNext()) throw new NoSuchElementException();
                return current++;
            }
        };
    }
}

public class Usage {
    public static void main(String[] args) {
        // Because CustomRange implements Iterable, 
        // we can instantly use it in an Enhanced For-Loop!
        for (int num : new CustomRange(1, 5)) {
            System.out.println(num); // Prints 1, 2, 3, 4, 5
        }
    }
}
```

## REAL_WORLD
When building Spring Boot Database Repositories, you might accidentally pull a massive dataset into memory.
If you call `List<User> users = userRepository.findAll()`, Hibernate executes the SQL query, loads all 50,000 users into Heap memory simultaneously, and returns the `ArrayList`. This will crash the server with an `OutOfMemoryError`.

To process massive datasets without crashing, Spring Data JPA provides Stream support, which relies directly on the Iterator pattern. 
Instead of returning a `List`, your repository method returns a `Stream<User>`. Behind the scenes, Hibernate opens an explicit JDBC `ResultSet` Iterator. As you process the stream, the Iterator fetches rows from the PostgreSQL database *one by one*, processes them, and immediately lets the Garbage Collector destroy them. The server memory usage remains flat regardless of whether you process 10 rows or 10 million rows, proving the immense scalability power of the Iterator Pattern.

## INTERVIEW
**Q: Why does modifying a Collection while iterating over it throw a `ConcurrentModificationException`?**
A: All Java Collections maintain an internal `modCount` variable that tracks structural modifications (adds/removes). When you create an `Iterator`, it saves a snapshot of the `modCount`. Every time you call `next()`, the Iterator compares its snapshot to the collection's current `modCount`. If they don't match, it means something bypassed the Iterator and modified the list directly, so it throws the exception in a "Fail-Fast" response to prevent corrupted traversal.

**Q: How do you safely remove elements during iteration?**
A: You must use an explicit `Iterator` and call `iterator.remove()`. The Iterator's remove method safely updates the underlying collection *and* updates its own internal `modCount` snapshot, keeping everything perfectly synchronized and avoiding the exception. (Note: Java 8 introduced `list.removeIf(predicate)` which safely handles this under the hood).

**Q: Can you iterate backward using an Iterator?**
A: A standard `Iterator` only goes forward (`hasNext()`, `next()`). However, `List` implementations provide a `ListIterator` via `list.listIterator()`. The `ListIterator` extends `Iterator` and adds `hasPrevious()` and `previous()` methods, allowing seamless bidirectional traversal.
