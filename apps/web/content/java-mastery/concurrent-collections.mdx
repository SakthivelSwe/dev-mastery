## WHY

Standard collections like `ArrayList`, `HashMap`, `LinkedList` are NOT thread-safe. Using them in multi-threaded code causes data corruption, lost updates, and infinite loops. Concurrent collections solve this:

1. **ConcurrentHashMap** — The go-to thread-safe map (replaces `Hashtable` and `Collections.synchronizedMap`)
2. **CopyOnWriteArrayList** — Thread-safe list where reads are lock-free (great for read-heavy scenarios)
3. **BlockingQueue** — Producer-consumer pattern — the backbone of thread pools
4. **ConcurrentLinkedQueue** — Lock-free queue for high-throughput concurrent access
5. **Interview** — "What concurrent collections do you know?" is in every senior Java interview

## THEORY

### Why Standard Collections Fail in Multi-Threading

```java
// DANGEROUS: HashMap in multi-threaded code
Map<String, Integer> map = new HashMap<>();

// Thread 1 and Thread 2 both doing this:
for (int i = 0; i < 1000; i++) {
    map.put("key" + i, i);
}
// Result: data corruption, random NPE, infinite loop (Java 7), missing entries
```

**Root causes:**
- **Read-write races**: One thread reads while another modifies
- **Compound action races**: `if (!map.containsKey(k)) map.put(k, v)` — not atomic!
- **Iterator invalidation**: `ConcurrentModificationException` when iterating and modifying

### The java.util.concurrent Collections

**Maps:**
| Class | Thread Safety | Notes |
|-------|-------------|-------|
| `HashMap` | ❌ None | Single-threaded only |
| `Hashtable` | ✅ Full lock | Legacy, slow (single lock) |
| `Collections.synchronizedMap` | ✅ Full lock | Wraps HashMap, still slow |
| `ConcurrentHashMap` | ✅ Fine-grained | Fast, recommended |
| `ConcurrentSkipListMap` | ✅ Fine-grained | Sorted, concurrent |

**Lists:**
| Class | Thread Safety | Notes |
|-------|-------------|-------|
| `ArrayList` | ❌ None | Single-threaded |
| `Vector` | ✅ Full lock | Legacy, slow |
| `Collections.synchronizedList` | ✅ Full lock | Wraps ArrayList |
| `CopyOnWriteArrayList` | ✅ Lock-free reads | Write copies full array |
| `ConcurrentLinkedDeque` | ✅ Lock-free | Non-blocking deque |

**Queues:**
| Class | Blocking? | Notes |
|-------|-----------|-------|
| `ArrayBlockingQueue` | ✅ Bounded | Fixed capacity FIFO |
| `LinkedBlockingQueue` | ✅ Optionally bounded | Unbounded or bounded FIFO |
| `PriorityBlockingQueue` | ✅ Unbounded | Priority-ordered |
| `SynchronousQueue` | ✅ Zero capacity | Handoff between threads |
| `ConcurrentLinkedQueue` | ❌ Non-blocking | Lock-free, poll returns null |
| `DelayQueue` | ✅ Unbounded | Elements available after delay |
| `LinkedTransferQueue` | ✅ Unbounded | Can handoff or enqueue |

### ConcurrentHashMap Internals

```
Java 8 ConcurrentHashMap:
- Uses per-bucket locks (synchronized on bin head node)
- Read operations (get): LOCK-FREE — volatile reads only
- Write operations (put): lock only the specific bucket
- computeIfAbsent, merge, compute: atomic compound operations
- Allows 16+ concurrent writers by default

vs synchronized(map) { map.put(k,v) } — locks THE ENTIRE MAP
```

### CopyOnWriteArrayList

```
Write operation (add, remove, set):
1. Lock
2. Copy entire backing array
3. Modify the copy
4. Replace backing array reference
5. Unlock

Read operation (get, iteration):
1. No lock — reads snapshot of current array
2. Readers see consistent view even if writers modify

Best for: read-heavy, rare writes (e.g., listeners, configuration)
Bad for: write-heavy (each write copies entire array = O(n))
```

### BlockingQueue — Producer-Consumer Pattern

```java
BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100); // bounded

// Producer thread
while (true) {
    Task task = generateTask();
    queue.put(task);    // BLOCKS if queue full!
}

// Consumer thread
while (true) {
    Task task = queue.take(); // BLOCKS if queue empty!
    processTask(task);
}
```

## CODE

### Level 1 — Beginner: ConcurrentHashMap Basics

```java
// ConcurrentHashMapBasics.java
import java.util.concurrent.*;
import java.util.*;

public class ConcurrentHashMapBasics {
    public static void main(String[] args) throws InterruptedException {
        ConcurrentHashMap<String, Integer> wordCount = new ConcurrentHashMap<>();

        // Thread-safe operations
        wordCount.put("java", 1);
        wordCount.put("python", 2);
        wordCount.putIfAbsent("java", 99); // won't overwrite existing!
        System.out.println(wordCount.get("java")); // 1

        // Atomic compound operations
        wordCount.computeIfAbsent("go", k -> 0);     // add if missing
        wordCount.computeIfPresent("java", (k, v) -> v + 1); // update if present
        wordCount.compute("python", (k, v) -> v == null ? 1 : v + 1); // update or init
        wordCount.merge("java", 1, Integer::sum);    // add 1, or init to 1 if missing

        System.out.println(wordCount);

        // Multi-threaded increment
        ConcurrentHashMap<String, Integer> freq = new ConcurrentHashMap<>();
        String[] words = "the cat sat on the mat the cat".split(" ");

        List<Thread> threads = new ArrayList<>();
        for (int t = 0; t < 4; t++) {
            final int start = t * 2;
            threads.add(new Thread(() -> {
                for (int i = start; i < Math.min(start + 2, words.length); i++) {
                    freq.merge(words[i], 1, Integer::sum); // atomic!
                }
            }));
        }
        threads.forEach(Thread::start);
        for (Thread t : threads) t.join();

        System.out.println("Word frequencies: " + freq);
        System.out.println("Total words: " + freq.values().stream().mapToInt(Integer::intValue).sum());
    }
}
```

### Level 2 — Elementary: CopyOnWriteArrayList and Thread-Safe Iteration

```java
// CopyOnWriteDemo.java
import java.util.concurrent.*;
import java.util.*;
import java.util.concurrent.atomic.*;

public class CopyOnWriteDemo {

    public static void main(String[] args) throws InterruptedException {
        CopyOnWriteArrayList<String> listeners = new CopyOnWriteArrayList<>();

        // Add listeners
        listeners.add("EventLogger");
        listeners.add("MetricsCollector");
        listeners.add("AuditService");

        // Safe iteration while modifying (no ConcurrentModificationException!)
        Thread modifier = new Thread(() -> {
            try {
                Thread.sleep(10);
                listeners.add("NewListener");     // safe to add while iterating
                listeners.remove("EventLogger");  // safe to remove while iterating
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });

        modifier.start();

        // Iteration sees a SNAPSHOT — won't see mid-iteration changes
        System.out.println("Listeners (snapshot during iteration):");
        for (String listener : listeners) {
            Thread.sleep(5); // simulate work
            System.out.println("  Notifying: " + listener);
        }
        modifier.join();

        System.out.println("\nFinal listeners: " + listeners);

        // CopyOnWriteArraySet
        CopyOnWriteArraySet<String> tags = new CopyOnWriteArraySet<>();
        tags.add("java"); tags.add("spring"); tags.add("java"); // no duplicates!
        System.out.println("Tags: " + tags); // [java, spring]
    }
}
```

### Level 3 — Intermediate: BlockingQueue Producer-Consumer

```java
// ProducerConsumer.java
import java.util.concurrent.*;
import java.util.concurrent.atomic.*;
import java.util.*;

public class ProducerConsumer {

    record Task(int id, String data) {}

    static void runProducerConsumer() throws InterruptedException {
        BlockingQueue<Task> queue = new LinkedBlockingQueue<>(10); // bounded!
        AtomicInteger produced = new AtomicInteger(0);
        AtomicInteger consumed = new AtomicInteger(0);
        AtomicBoolean done = new AtomicBoolean(false);

        // Producer thread
        Thread producer = new Thread(() -> {
            try {
                for (int i = 1; i <= 20; i++) {
                    Task task = new Task(i, "data-" + i);
                    queue.put(task); // blocks if queue is full (10 capacity)
                    produced.incrementAndGet();
                    System.out.printf("Produced Task#%d (queue size: %d)%n", i, queue.size());
                    Thread.sleep(10);
                }
                done.set(true);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });

        // Multiple consumer threads
        List<Thread> consumers = new ArrayList<>();
        for (int c = 0; c < 3; c++) {
            final int consumerId = c + 1;
            consumers.add(new Thread(() -> {
                try {
                    while (!done.get() || !queue.isEmpty()) {
                        Task task = queue.poll(100, TimeUnit.MILLISECONDS);
                        if (task != null) {
                            Thread.sleep(30); // simulate processing time
                            consumed.incrementAndGet();
                            System.out.printf("Consumer-%d processed Task#%d%n",
                                consumerId, task.id());
                        }
                    }
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }));
        }

        producer.start();
        consumers.forEach(Thread::start);
        producer.join();
        consumers.forEach(t -> { try { t.join(); } catch (InterruptedException e) {} });

        System.out.printf("Produced: %d, Consumed: %d%n", produced.get(), consumed.get());
    }

    // Priority-based task queue
    static void priorityQueueDemo() throws InterruptedException {
        PriorityBlockingQueue<Task> priorityQueue = new PriorityBlockingQueue<>(
            10, Comparator.comparingInt(t -> t.id() % 3)); // priority by id mod 3

        for (int i = 10; i >= 1; i--) {
            priorityQueue.offer(new Task(i, "data"));
        }

        System.out.println("\nPriority order:");
        while (!priorityQueue.isEmpty()) {
            System.out.print("Task#" + priorityQueue.take().id() + " ");
        }
        System.out.println();
    }

    public static void main(String[] args) throws InterruptedException {
        runProducerConsumer();
        priorityQueueDemo();
    }
}
```

### Level 4 — Advanced: ConcurrentSkipListMap and Non-Blocking Algorithms

```java
// ConcurrentSkipListDemo.java
import java.util.concurrent.*;
import java.util.*;

public class ConcurrentSkipListDemo {

    public static void main(String[] args) throws InterruptedException {
        // ConcurrentSkipListMap — sorted + concurrent (like TreeMap but thread-safe)
        ConcurrentSkipListMap<Long, String> events = new ConcurrentSkipListMap<>();

        // Multiple threads adding time-stamped events
        List<Thread> threads = new ArrayList<>();
        for (int t = 0; t < 5; t++) {
            final int threadId = t;
            threads.add(new Thread(() -> {
                for (int i = 0; i < 10; i++) {
                    long ts = System.nanoTime();
                    events.put(ts, "Thread-" + threadId + "-Event-" + i);
                }
            }));
        }
        threads.forEach(Thread::start);
        for (Thread t : threads) t.join();

        System.out.println("Total events: " + events.size()); // 50

        // NavigableMap operations — all thread-safe in ConcurrentSkipListMap
        long midPoint = events.firstKey() + (events.lastKey() - events.firstKey()) / 2;
        System.out.println("Events in first half: " + events.headMap(midPoint).size());
        System.out.println("Events in second half: " + events.tailMap(midPoint).size());

        // ConcurrentSkipListSet — sorted + concurrent (like TreeSet but thread-safe)
        ConcurrentSkipListSet<String> words = new ConcurrentSkipListSet<>();
        List<Thread> writers = new ArrayList<>();
        String[] data = "java python go rust java python kotlin swift".split(" ");
        for (String word : data) {
            writers.add(new Thread(() -> words.add(word)));
        }
        writers.forEach(Thread::start);
        for (Thread t : writers) t.join();
        System.out.println("\nSorted unique words: " + words);
        // [go, java, kotlin, python, rust, swift]

        // Lock-free ConcurrentLinkedQueue
        ConcurrentLinkedQueue<String> queue = new ConcurrentLinkedQueue<>();
        queue.offer("first"); queue.offer("second"); queue.offer("third");
        System.out.println("Peek: " + queue.peek());  // first (doesn't remove)
        System.out.println("Poll: " + queue.poll());  // first (removes)
        System.out.println("Size: " + queue.size());  // 2
        System.out.println("Empty? " + queue.isEmpty()); // false
    }
}
```

### Level 5 — Expert: Custom Thread Pool with BlockingQueue

```java
// CustomThreadPool.java
import java.util.concurrent.*;
import java.util.*;
import java.util.concurrent.atomic.*;

/**
 * Simple thread pool built on BlockingQueue.
 * Shows how Executors.newFixedThreadPool() works internally.
 */
public class CustomThreadPool {

    private final BlockingQueue<Runnable> taskQueue;
    private final List<Thread> workers;
    private volatile boolean shutdown = false;
    private final AtomicLong completedTasks = new AtomicLong(0);

    CustomThreadPool(int threads, int queueCapacity) {
        taskQueue = new LinkedBlockingQueue<>(queueCapacity);
        workers = new ArrayList<>(threads);
        for (int i = 0; i < threads; i++) {
            Thread worker = new Thread(() -> {
                while (!shutdown || !taskQueue.isEmpty()) {
                    try {
                        Runnable task = taskQueue.poll(100, TimeUnit.MILLISECONDS);
                        if (task != null) {
                            task.run();
                            completedTasks.incrementAndGet();
                        }
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                        break;
                    }
                }
            }, "CustomPool-Thread-" + i);
            worker.setDaemon(true);
            worker.start();
            workers.add(worker);
        }
    }

    void submit(Runnable task) throws InterruptedException {
        if (shutdown) throw new RejectedExecutionException("Pool is shut down");
        taskQueue.put(task); // blocks if queue full
    }

    boolean submit(Runnable task, long timeout, TimeUnit unit) throws InterruptedException {
        return taskQueue.offer(task, timeout, unit); // returns false if timed out
    }

    void shutdown() throws InterruptedException {
        shutdown = true;
        for (Thread w : workers) w.join(5000);
    }

    long getCompletedTaskCount() { return completedTasks.get(); }
    int getQueueSize() { return taskQueue.size(); }

    public static void main(String[] args) throws Exception {
        CustomThreadPool pool = new CustomThreadPool(4, 50);

        CountDownLatch latch = new CountDownLatch(100);
        for (int i = 0; i < 100; i++) {
            final int taskId = i;
            pool.submit(() -> {
                try {
                    Thread.sleep(10);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                } finally {
                    latch.countDown();
                }
            });
        }

        latch.await();
        System.out.println("Completed: " + pool.getCompletedTaskCount()); // 100
        pool.shutdown();
    }
}
```

## REAL_WORLD

### Rate Limiter with Concurrent Collections

```java
@Service
public class RateLimiter {
    // Token bucket per user — thread-safe
    private final ConcurrentHashMap<String, AtomicInteger> tokens = new ConcurrentHashMap<>();
    private static final int MAX_TOKENS = 100;

    @Scheduled(fixedRate = 60_000) // refill every minute
    public void refillTokens() {
        tokens.forEach((userId, tokenCount) -> tokenCount.set(MAX_TOKENS));
    }

    public boolean tryConsume(String userId) {
        AtomicInteger userTokens = tokens.computeIfAbsent(userId, k -> new AtomicInteger(MAX_TOKENS));
        int remaining;
        do {
            remaining = userTokens.get();
            if (remaining <= 0) return false; // rate limit exceeded
        } while (!userTokens.compareAndSet(remaining, remaining - 1));
        return true; // consumed one token
    }
}
```

## INTERVIEW

**Q1: Why is HashMap not thread-safe and what should you use instead?**

> **Strong answer:** HashMap allows multiple threads to read/write without synchronization. Concurrent puts can cause lost updates, size inconsistency, or infinite loops during resize (Java 7 — linked list cycle). Use `ConcurrentHashMap`: it uses per-bucket fine-grained locking (Java 8+) allowing concurrent reads without locking and concurrent writes to different buckets. Also provides atomic compound operations (`computeIfAbsent`, `merge`) that would require external synchronization with HashMap. `Collections.synchronizedMap(new HashMap<>())` is also thread-safe but slower — it locks the entire map on every operation.

**Q2: When would you use CopyOnWriteArrayList vs ConcurrentLinkedDeque?**

> **Strong answer:** `CopyOnWriteArrayList` is ideal for read-heavy, write-rare scenarios: event listeners, observer lists, configuration lists. Every write copies the entire array — O(n) write cost — so it's bad for frequent writes. Reads are lock-free. Iteration sees a consistent snapshot (no `ConcurrentModificationException`). `ConcurrentLinkedDeque` (or `ConcurrentLinkedQueue`) is lock-free using CAS operations, good for queues with concurrent offer/poll from multiple threads. O(1) amortized both ways. Neither blocks — they return null instead of blocking.

**Q3: What is the difference between BlockingQueue and ConcurrentLinkedQueue?**

> **Strong answer:** Both are thread-safe. Key difference: blocking behavior. `BlockingQueue` (ArrayBlockingQueue, LinkedBlockingQueue, SynchronousQueue) blocks the thread when the queue is full (on `put()`) or empty (on `take()`). This is what makes producer-consumer patterns work without spinning. `ConcurrentLinkedQueue` is non-blocking — `offer()` always returns immediately (true/false) and `poll()` returns null if empty. Use BlockingQueue for producer-consumer patterns where you want backpressure (producers slow down when consumers can't keep up). Use ConcurrentLinkedQueue for lock-free scenarios where you'd rather poll and check null.

## FEYNMAN CHECK

Standard collections are like a notebook that multiple people can't write in simultaneously without corrupting it. Concurrent collections are like a specially designed whiteboard where many people can write in different sections at the same time (ConcurrentHashMap), or where everyone can read freely while a scribe makes a new copy before changing it (CopyOnWriteArrayList), or where there's a well-organized wait system for accessing items (BlockingQueue).
