## 1. Architectural Blueprint & Core Concepts

Design tokens are the atomic decisions of a visual system, stored as structured data (JSON) and transformed into platform-specific deliverables (CSS, Sass, Tailwind, iOS/Android XML). 

An enterprise token system operates on a three-tier taxonomy:
1. **Global Tokens (Tier 1 - Tier 1/Primitive):** Context-agnostic raw values (e.g., `color-blue-500: #3b82f6`). These must never be used directly in components.
2. **Alias/Semantic Tokens (Tier 2):** Context-aware tokens representing intent, meaning, or role (e.g., `color-action-primary: var(--color-blue-500)`).
3. **Component Tokens (Tier 3):** Highly specific tokens scoped to individual components (e.g., `button-primary-bg: var(--color-action-primary)`).

```
[ Global Token: --color-blue-500 (#2563EB) ]
               │
               ▼
[ Semantic Token: --color-action-primary (var(--color-blue-500)) ]
               │
               ▼
[ Component Token: --button-primary-bg (var(--color-action-primary)) ]
```

## 2. Figma Workflow & Sync Pipeline

### Figma Setup & Mappings
In Figma, use **Figma Variables** (or the **Tokens Studio** plugin) to construct your token hierarchy.
1. Define **Collections** for *Primitives* and *Semantic* tokens.
2. Map Semantic variables directly to Primitive variables.
3. Use Figma's native **Modes** (e.g., Light, Dark) on the Semantic collection.

### Token Sync Pipeline
To automate the transformation of design decisions from Figma to code:

```
[ Figma Variables ] ──(Export JSON via API/Tokens Studio)──> [ GitHub Action ] ──> [ Style Dictionary ] ──> [ CSS/JS/Tailwind ]
```

1. **Export:** Export tokens as a W3C-compliant JSON format from Figma.
2. **Transform:** Use **Style Dictionary** to parse the raw JSON.
3. **Distribute:** Generate CSS custom properties and a Tailwind-compatible configuration.

## 3. CSS Custom Properties & Tailwind Configuration

### CSS Custom Properties (`tokens.css`)
```css
:root {
  /* --- Tier 1: Global Primitives --- */
  --blue-500: #2563eb;
  --blue-600: #1d4ed8;
  --gray-100: #f3f4f6;
  --gray-900: #111827;
  --spacing-4: 1rem;
  --font-sans: 'Inter', system-ui, sans-serif;

  /* --- Tier 2: Semantic Tokens (Light Default) --- */
  --color-bg-default: var(--gray-100);
  --color-text-default: var(--gray-900);
  --color-brand-primary: var(--blue-500);
  --color-brand-primary-hover: var(--blue-600);
  --spacing-layout-md: var(--spacing-4);
  --font-family-body: var(--font-sans);
}

@media (prefers-color-scheme: dark) {
  :root {
    --color-bg-default: var(--gray-900);
    --color-text-default: var(--gray-100);
    --color-brand-primary: var(--blue-500);
    --color-brand-primary-hover: var(--blue-600);
  }
}
```

## 4. Accessibility Compliance (WCAG 2.1 AA)
- **Contrast Ratios (Success Criterion 1.4.3):** Text colors mapped via semantic tokens must maintain at least a **4.5:1** contrast ratio against their background for normal text, and **3:1** for large text (18pt/24px or larger).
- **Non-Text Contrast (Success Criterion 1.4.11):** UI boundaries, input focus borders, and active control states must maintain at least a **3:1** contrast ratio against adjacent colors.

## 5. React & TypeScript Implementation

This showcase component renders and validates active design tokens dynamically.

```tsx
import React from 'react';

interface TokenItemProps {
  name: string;
  value: string;
  type: 'color' | 'spacing' | 'font';
}

export const TokenItem: React.FC<TokenItemProps> = ({ name, value, type }) => {
  const isColor = type === 'color';
  return (
    <div 
      className="flex items-center justify-between p-3 rounded-lg border border-gray-200 dark:border-gray-800 bg-bg-default text-text-default shadow-sm"
      role="listitem"
    >
      <div className="flex flex-col">
        <span className="font-mono text-xs font-bold text-gray-500 dark:text-gray-400">{name}</span>
        <span className="text-sm font-semibold">{value}</span>
      </div>
      {isColor && (
        <div 
          className="w-8 h-8 rounded-full border border-gray-300 dark:border-gray-700 shadow-inner" 
          style={{ backgroundColor: value }}
          aria-hidden="true"
        />
      )}
      {type === 'spacing' && (
        <div 
          className="bg-brand-primary h-4 rounded" 
          style={{ width: value }}
          aria-hidden="true"
        />
      )}
    </div>
  );
};

interface TokenShowcaseProps {
  tokens: TokenItemProps[];
}

export const TokenShowcase: React.FC<TokenShowcaseProps> = ({ tokens }) => {
  return (
    <section aria-labelledby="token-showcase-title" className="p-6 space-y-6">
      <h2 id="token-showcase-title" className="text-2xl font-bold text-text-default">
        Active Design Tokens
      </h2>
      <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4" role="list">
        {tokens.map((token) => (
          <TokenItem key={token.name} {...token} />
        ))}
      </div>
    </section>
  );
};
```
