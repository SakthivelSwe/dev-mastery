Title: MongoDB CRUD Operations | Level: 2

### Write Path and Write Concern Mechanics

When a write operation is executed in MongoDB, it goes through several phases to guarantee persistence.

```
[Client App] 
     │
     │  1. Send Write Command
     ▼
[Primary Node] ──(2. Write to WiredTiger Journal/Oplog)
     │
     ├─► [Secondary 1] ──(3. Replay Oplog & Ack) ──┐
     │                                            │  4. Majority Consensus
     └─► [Secondary 2] ──(3. Replay Oplog & Ack) ──┴─► [Client Acked]
```

1.  **In-Memory Buffer:** The write is recorded in the WiredTiger cache.
2.  **Journaling:** The write is recorded in the sequential WiredTiger journal on disk (by default, every 100ms). This guarantees recovery in the event of a hard crash.
3.  **Oplog Generation:** The operation is written to the `oplog.rs` collection, allowing secondary replicas to pull and replay the change.

#### Write Concern Parameters
*   `w: 1`: The primary node acknowledges the write after applying it in memory and writing to its journal (if `j: true`). This is fast but vulnerable to data loss if the primary fails before replicating to secondaries.
*   `w: majority`: The write is acknowledged only after it has been committed to disk/memory of a majority of the replica set nodes. This is the default in MongoDB 5.0+ and prevents rollbacks during elections.
*   `j: true`: Forces the node to write to the journal on disk before acknowledging. Without this, a power loss on the primary could result in lost data even with `w: 1`.
*   `wtimeout`: Specifies a limit in milliseconds to prevent write operations from blocking indefinitely if too few nodes are available.

### Executing CRUD Operations via `mongosh`

Let's work with an `orders` collection.

#### 1. Insert Operations
```javascript
// Insert a single document with explicit write concern
db.orders.insertOne(
  {
    order_id: "ORD-99281",
    customer_id: "CUST-001",
    items: [
      { product_id: "PROD-A", qty: 2, price: NumberDecimal("49.99") },
      { product_id: "PROD-B", qty: 1, price: NumberDecimal("12.50") }
    ],
    status: "PENDING",
    created_at: ISODate()
  },
  { writeConcern: { w: "majority", j: true, wtimeout: 5000 } }
);

// Bulk insert
db.orders.insertMany([
  { order_id: "ORD-99282", customer_id: "CUST-002", items: [], status: "SHIPPED", created_at: ISODate() },
  { order_id: "ORD-99283", customer_id: "CUST-001", items: [], status: "PENDING", created_at: ISODate() }
], { ordered: true }); // If ordered is true, a failure on the second document halts execution
```

#### 2. Read Operations with Projections and Modifiers
```javascript
// Find orders with projection, sorting, skip, and limit
db.orders.find(
  { status: "PENDING" }, // Query Filter
  { order_id: 1, customer_id: 1, _id: 0 } // Projection: 1 to include, 0 to exclude
)
.sort({ created_at: -1 }) // Sort descending
.skip(10)                 // Skip first 10 documents
.limit(5);                // Return maximum 5 documents
```

#### 3. Update Operations
```javascript
// Update a single document using atomic operators
db.orders.updateOne(
  { order_id: "ORD-99281" },
  {
    $set: { status: "PROCESSING", updated_at: ISODate() },
    $push: { items: { product_id: "PROD-C", qty: 1, price: NumberDecimal("9.99") } }
  }
);

// Pull (remove) an item from the array matching a condition
db.orders.updateOne(
  { order_id: "ORD-99281" },
  { $pull: { items: { product_id: "PROD-B" } } }
);

// Update multiple documents
db.orders.updateMany(
  { status: "PENDING", created_at: { $lt: new Date(Date.now() - 86400000) } },
  { $set: { status: "EXPIRED" } }
);
```

#### 4. Replace Operations
Unlike `$set`, which modifies specific fields, `replaceOne` replaces the entire document structure while preserving the `_id`.
```javascript
db.orders.replaceOne(
  { order_id: "ORD-99282" },
  {
    order_id: "ORD-99282",
    customer_id: "CUST-002-UPDATED",
    status: "ARCHIVED",
    replaced: true
  }
);
```

#### 5. Delete Operations
```javascript
// Delete single
db.orders.deleteOne({ order_id: "ORD-99283" });

// Delete many
db.orders.deleteMany({ status: "EXPIRED" });
```

#### 6. BulkWrite Operations
Executes multiple write operations (`insertOne`, `updateOne`, `updateMany`, `deleteOne`, `deleteMany`, `replaceOne`) in a single network roundtrip.
```javascript
db.orders.bulkWrite([
  { insertOne: { document: { order_id: "ORD-99284", status: "PENDING" } } },
  { updateOne: { filter: { order_id: "ORD-99281" }, update: { $set: { status: "COMPLETED" } } } },
  { deleteOne: { filter: { order_id: "ORD-99282" } } }
], { ordered: false }); // unordered allows operations to execute in parallel
```

### Spring Data MongoDB Equivalent

We use `MongoTemplate` to execute complex CRUD operations programmatically.

#### Spring Service Implementation (`OrderRepositoryImpl.java`)
```java
package com.devmastery.mongodb.repository;

import com.devmastery.mongodb.model.Order;
import com.mongodb.WriteConcern;
import com.mongodb.client.result.DeleteResult;
import com.mongodb.client.result.UpdateResult;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.domain.PageRequest;
import org.springframework.data.domain.Sort;
import org.springframework.data.mongodb.core.BulkOperations;
import org.springframework.data.mongodb.core.MongoTemplate;
import org.springframework.data.mongodb.core.query.Criteria;
import org.springframework.data.mongodb.core.query.Query;
import org.springframework.data.mongodb.core.query.Update;
import org.springframework.stereotype.Repository;

import java.math.BigDecimal;
import java.util.List;

@Repository
public class OrderRepositoryImpl {

    private final MongoTemplate mongoTemplate;

    @Autowired
    public OrderRepositoryImpl(MongoTemplate mongoTemplate) {
        this.mongoTemplate = mongoTemplate;
    }

    public Order saveWithWriteConcern(Order order) {
        // Enforce majority write concern programmatically for critical writes
        MongoTemplate templatedWithConcern = new MongoTemplate(
                mongoTemplate.getMongoDatabaseFactory(), 
                mongoTemplate.getConverter()
        );
        templatedWithConcern.setWriteConcern(WriteConcern.MAJORITY.withJournal(true));
        return templatedWithConcern.insert(order);
    }

    public List<Order> findPagedOrders(String status, int page, int size) {
        Query query = new Query();
        query.addCriteria(Criteria.where("status").is(status));
        query.fields().include("orderId").include("customerId").exclude("id");
        query.with(Sort.by(Sort.Direction.DESC, "createdAt"));
        query.with(PageRequest.of(page, size));

        return mongoTemplate.find(query, Order.class);
    }

    public long updateOrderStatusAndAddItem(String orderId, String newStatus, Order.Item item) {
        Query query = new Query(Criteria.where("orderId").is(orderId));
        
        Update update = new Update()
                .set("status", newStatus)
                .push("items", item);

        UpdateResult result = mongoTemplate.updateFirst(query, update, Order.class);
        return result.getModifiedCount();
    }

    public long executeBulkWrite(List<Order> newOrders, String updateStatusId, String deleteId) {
        BulkOperations bulkOps = mongoTemplate.bulkOps(BulkOperations.BulkMode.UNORDERED, Order.class);

        // Queue inserts
        for (Order o : newOrders) {
            bulkOps.insert(o);
        }

        // Queue update
        Query updateQuery = new Query(Criteria.where("orderId").is(updateStatusId));
        Update update = new Update().set("status", "PROCESSING");
        bulkOps.updateOne(updateQuery, update);

        // Queue delete
        Query deleteQuery = new Query(Criteria.where("orderId").is(deleteId));
        bulkOps.remove(deleteQuery);

        // Execute batch in single roundtrip
        return bulkOps.execute().getInsertedCount();
    }
}
```
