## WHY

Inner classes are used everywhere in Java APIs — iterators, event listeners, builder patterns, anonymous callbacks. You need to understand them to:

1. **Read Java APIs** — Iterator, Comparator, ActionListener all use inner/anonymous classes
2. **Write callbacks** — Before lambdas, anonymous inner classes were the only way
3. **Encapsulate helpers** — Static nested classes package related code without exposing it
4. **Understand closures** — Why does an anonymous class need `final` or effectively final variables?
5. **Interview questions** — "Why must outer class variables be effectively final?" is a classic

---

## THEORY

### Four Types of Inner Classes

| Type | Declared Where | Access to Outer | Static? | Use Case |
|------|---------------|-----------------|---------|----------|
| **Static Nested** | Inside class body | Outer static members only | `static` keyword | Helper class, Builder pattern |
| **Inner (Member)** | Inside class body | All outer members (including private) | No `static` | Tight coupling with outer, Iterator |
| **Local** | Inside a method | Local vars (if effectively final) | No | One-off use in method |
| **Anonymous** | Expression | Local vars (if effectively final) | No | Single-use implementation, callbacks |

### 1. Static Nested Class

```java
public class Graph {
    private int vertices;

    // Static nested — doesn't need Graph instance
    // Can only access Graph's static members
    public static class Edge {
        int from, to, weight;

        Edge(int from, int to, int weight) {
            this.from = from;
            this.to = to;
            this.weight = weight;
        }
    }

    public static void main(String[] args) {
        Graph.Edge edge = new Graph.Edge(1, 2, 10); // no Graph instance needed
        // or if inside Graph class:
        Edge edge2 = new Edge(3, 4, 5);
    }
}
```

**When to use:** When the nested class doesn't need access to the outer instance. Prefer this over inner class — it doesn't hold a reference to the outer object (no memory leak risk).

### 2. Inner (Non-Static Member) Class

```java
public class LinkedList<T> {
    private Node<T> head;

    private static class Node<T> {
        T data; Node<T> next;
        Node(T data) { this.data = data; }
    }

    // Inner class — needs access to LinkedList's head
    public class ListIterator implements java.util.Iterator<T> {
        private Node<T> current = head; // accesses outer class field!

        @Override public boolean hasNext() { return current != null; }

        @Override public T next() {
            T data = current.data;
            current = current.next;
            return data;
        }
    }

    public ListIterator iterator() {
        return new ListIterator(); // creates with implicit outer reference
    }
}
// Usage:
LinkedList<Integer> list = new LinkedList<>();
// ... add elements ...
LinkedList<Integer>.ListIterator it = list.iterator(); // requires outer instance
// Or from outside: LinkedList.ListIterator — requires a LinkedList instance
```

**Memory note:** Every inner class instance holds an implicit reference to its enclosing outer class instance. This can cause memory leaks if the inner class outlives the outer class.

### 3. Local Class

```java
public class Validator {
    public boolean validate(String input, String type) {
        // Local class — defined inside method
        class TypeChecker {
            boolean check(String val) {
                return switch (type) { // captures 'type' from enclosing scope (must be effectively final)
                    case "email" -> val.contains("@");
                    case "phone" -> val.matches("\\d{10}");
                    default -> !val.isEmpty();
                };
            }
        }
        return new TypeChecker().check(input);
    }
}
```

**Key rule:** Local classes can access local variables of the enclosing method only if those variables are `final` or **effectively final** (never reassigned after initialization).

### 4. Anonymous Inner Class

```java
// Before Java 8 — anonymous class for single interface implementation
Runnable r = new Runnable() {
    @Override
```

### Why Variables Must Be Effectively Final

```java
void counter() {
    int count = 0;

    // ILLEGAL:
    Runnable r = new Runnable() {
        @Override public void run() {
            count++; // ERROR: count must be effectively final
        }
    };

    count++; // This makes count non-effectively-final
}
```

**Why?** The anonymous class may outlive the method. If `count` was on the stack and the method returned, `count` would be gone. Java captures a copy of the value — so it must be final to ensure consistency (the copy and original always match).

```java
// LEGAL — count is effectively final (never reassigned)
void counter() {
    int count = 10;
    Runnable r = () -> System.out.println("Count: " + count); // OK!
    // count is never changed after initialization
    r.run();
}

// FIX for mutable state in lambdas — use array or AtomicInteger
int[] mutableCount = {0};
Runnable r = () -> mutableCount[0]++; // legal — mutableCount ref is final, array content isn't
```

### Builder Pattern with Static Nested Class

```java
public class Person {
    private final String name;
    private final int age;
    private final String email;
    private final String phone;

    private Person(Builder b) {
        this.name = b.name;
        this.age = b.age;
        this.email = b.email;
        this.phone = b.phone;
    }

    // Static nested Builder
    public static class Builder {
        private final String name; // required
        private int age;           // optional
        private String email;
        private String phone;

        public Builder(String name) { this.name = name; }
        public Builder age(int age) { this.age = age; return this; }
        public Builder email(String email) { this.email = email; return this; }
        public Builder phone(String phone) { this.phone = phone; return this; }
        public Person build() { return new Person(this); }
    }

    @Override public String toString() {
        return name + ", age=" + age + ", email=" + email;
    }
}
// Usage:
Person p = new Person.Builder("Alice")
    .age(30)
    .email("alice@example.com")
    .build();
```

### Level 1 — Beginner: Static vs Non-Static Nested Class

```java
// NestedClassTypes.java
public class NestedClassTypes {

    private int outerField = 10;
    private static int staticField = 20;

    // Static nested — no outer instance needed
    static class StaticNested {
        void show() {
            // outerField — CANNOT access (no outer instance)
            System.out.println("Static field: " + staticField); // OK
        }
    }

    // Inner class — needs outer instance
    class Inner {
        void show() {
            System.out.println("Outer field: " + outerField);   // OK
            System.out.println("Static field: " + staticField); // OK
            System.out.println("Outer instance: " + NestedClassTypes.this); // explicit outer ref
        }
    }

    public static void main(String[] args) {
        // Static nested — no outer instance needed
        StaticNested sn = new StaticNested();
        sn.show();

        // Inner class — needs outer instance
        NestedClassTypes outer = new NestedClassTypes();
        Inner inner = outer.new Inner();
        inner.show();

        // Shorthand inside outer class:
        // Inner inner2 = new Inner(); // equivalent to outer.new Inner()
    }
}
```

### Level 2 — Elementary: Iterator with Inner Class

```java
// CustomStack.java
import java.util.Iterator;
import java.util.NoSuchElementException;

public class CustomStack<T> implements Iterable<T> {
    private Object[] data;
    private int size;

    public CustomStack(int capacity) {
        data = new Object[capacity];
    }

    public void push(T item) {
        if (size == data.length) throw new IllegalStateException("Stack full");
        data[size++] = item;
    }

    @SuppressWarnings("unchecked")
    public T pop() {
        if (size == 0) throw new NoSuchElementException("Stack empty");
        T item = (T) data[--size];
        data[size] = null; // prevent memory leak
        return item;
    }

    public int size() { return size; }

    // Inner class iterator — accesses data and size from outer
    @Override
    public Iterator<T> iterator() {
        return new StackIterator();
    }

    private class StackIterator implements Iterator<T> {
        private int cursor = size; // starts at top

        @Override
        public boolean hasNext() { return cursor > 0; }

        @SuppressWarnings("unchecked")
        @Override
        public T next() {
            if (!hasNext()) throw new NoSuchElementException();
            return (T) data[--cursor]; // iterates top → bottom
        }
    }

    public static void main(String[] args) {
        CustomStack<String> stack = new CustomStack<>(5);
        stack.push("a");
        stack.push("b");
        stack.push("c");

        // Uses inner class iterator
        for (String s : stack) {
            System.out.print(s + " "); // c b a (top to bottom)
        }
        System.out.println();
        System.out.println("Size: " + stack.size()); // 3 (iteration doesn't pop)
    }
}
```

### Level 3 — Intermediate: Anonymous Classes, Comparators, Event Listeners

```java
// AnonymousClassDemo.java
import java.util.*;

public class AnonymousClassDemo {

    // Sort with anonymous Comparator (pre-Java 8 style)
    static void sortWithAnonymous() {
        List<String> names = new ArrayList<>(Arrays.asList("Charlie", "Alice", "Bob", "Dave"));

        // Anonymous class
        Collections.sort(names, new Comparator<String>() {
            @Override
            public int compare(String a, String b) {
                return a.length() != b.length()
                    ? a.length() - b.length() // by length first
                    : a.compareTo(b);         // then alphabetically
            }
        });
        System.out.println(names); // [Bob, Alice, Dave, Charlie]

        // Lambda equivalent (much cleaner)
        names.sort(Comparator.comparingInt(String::length).thenComparing(Comparator.naturalOrder()));
        System.out.println(names);
    }

    // Template method with anonymous class
    abstract static class DataProcessor {
        // Template method
        final void process(int[] data) {
            int[] sorted = sort(data); // hook method
            System.out.println("Result: " + Arrays.toString(sorted));
        }
        abstract int[] sort(int[] data);
    }

    static void templateMethodDemo() {
        // Anonymous class overrides hook method
        DataProcessor ascending = new DataProcessor() {
            @Override
            int[] sort(int[] data) {
                int[] copy = data.clone();
                Arrays.sort(copy);
                return copy;
            }
        };

        DataProcessor descending = new DataProcessor() {
            @Override
            int[] sort(int[] data) {
                Integer[] boxed = new Integer[data.length];
                for (int i = 0; i < data.length; i++) boxed[i] = data[i];
                Arrays.sort(boxed, Collections.reverseOrder());
                int[] result = new int[data.length];
                for (int i = 0; i < data.length; i++) result[i] = boxed[i];
                return result;
            }
        };

        int[] data = {3, 1, 4, 1, 5, 9, 2, 6};
        ascending.process(data);  // Result: [1, 1, 2, 3, 4, 5, 6, 9]
        descending.process(data); // Result: [9, 6, 5, 4, 3, 2, 1, 1]
    }

    public static void main(String[] args) {
        sortWithAnonymous();
        templateMethodDemo();
    }
}
```

### Level 4 — Advanced: Builder Pattern, Effectively Final

```java
// HttpRequest.java — Builder pattern with static nested class
public class HttpRequest {
    private final String method;
    private final String url;
    private final Map<String, String> headers;
    private final String body;
    private final int timeoutMs;

    private HttpRequest(Builder b) {
        this.method = b.method;
        this.url = b.url;
        this.headers = Collections.unmodifiableMap(new HashMap<>(b.headers));
        this.body = b.body;
        this.timeoutMs = b.timeoutMs;
    }

    public String getMethod() { return method; }
    public String getUrl() { return url; }
    public Map<String, String> getHeaders() { return headers; }
    public String getBody() { return body; }
    public int getTimeoutMs() { return timeoutMs; }

    @Override
    public String toString() {
        return method + " " + url + " headers=" + headers.size() + " timeout=" + timeoutMs + "ms";
    }

    // Static nested Builder
    public static class Builder {
        // Required
        private final String method;
        private final String url;
        // Optional with defaults
        private final Map<String, String> headers = new LinkedHashMap<>();
        private String body = null;
        private int timeoutMs = 30_000;

        public Builder(String method, String url) {
            this.method = method.toUpperCase();
            this.url = url;
        }

        public Builder header(String name, String value) {
            headers.put(name, value);
            return this;
        }

        public Builder body(String body) {
            this.body = body;
            return this;
        }

        public Builder timeout(int ms) {
            this.timeoutMs = ms;
            return this;
        }

        public HttpRequest build() {
            if (url == null || url.isBlank()) throw new IllegalStateException("URL is required");
            return new HttpRequest(this);
        }
    }

    // Effectively final demo
    static void effectivelyFinalDemo() {
        String prefix = "Hello"; // effectively final (never reassigned)
        int count = 5;           // effectively final

        // Can use in anonymous/lambda
        Runnable r = () -> {
            for (int i = 0; i < count; i++) {
                System.out.println(prefix + " " + i);
            }
        };
        r.run();

        // prefix = "World"; // uncommenting this → compile error in lambda above
    }

    public static void main(String[] args) {
        HttpRequest req = new HttpRequest.Builder("GET", "https://api.example.com/users")
            .header("Authorization", "Bearer token123")
            .header("Accept", "application/json")
            .timeout(5000)
            .build();

        System.out.println(req);

        HttpRequest post = new HttpRequest.Builder("POST", "https://api.example.com/users")
            .header("Content-Type", "application/json")
            .body("{\"name\":\"Alice\",\"age\":30}")
            .build();

        System.out.println(post);
        effectivelyFinalDemo();
    }
}
```

### Level 5 — Expert: Inner Class Memory Leak & Fix

```java
// InnerClassLeak.java
import java.util.*;

/**
 * Demonstrates memory leak from non-static inner class
 * and how to fix it.
 */
public class InnerClassLeak {

    // LEAKY — HandlerThread holds implicit ref to OuterActivity
    static class OuterActivity {
        private byte[] largeData = new byte[1024 * 1024]; // 1MB

        // PROBLEM: LeakyHandler is a non-static inner class
        // → holds implicit reference to OuterActivity instance
        class LeakyHandler {
            void handleMessage() {
                // uses largeData implicitly
                System.out.println("Data size: " + largeData.length);
            }
        }

        LeakyHandler getHandler() { return new LeakyHandler(); }
    }

    // FIXED — StaticHandler does NOT hold reference to outer class
    static class FixedActivity {
        private byte[] largeData = new byte[1024 * 1024];

        // FIXED: static nested class + WeakReference
        static class StaticHandler {
            private final java.lang.ref.WeakReference<FixedActivity> activityRef;

            StaticHandler(FixedActivity activity) {
                this.activityRef = new java.lang.ref.WeakReference<>(activity);
            }

            void handleMessage() {
                FixedActivity activity = activityRef.get();
                if (activity != null) { // check if activity still exists
                    System.out.println("Data size: " + activity.largeData.length);
                } else {
                    System.out.println("Activity was GC'd");
                }
            }
        }

        StaticHandler getHandler() { return new StaticHandler(this); }
    }

    // Demonstrate the difference
    static void leakDemo() {
        OuterActivity outer = new OuterActivity();
        OuterActivity.LeakyHandler handler = outer.getHandler();
        outer = null; // outer is set to null...
        // BUT handler still holds implicit ref to old outer!
        // OuterActivity cannot be GC'd → 1MB leak
        System.gc();
        handler.handleMessage(); // handler keeps outer alive

        FixedActivity fixed = new FixedActivity();
        FixedActivity.StaticHandler fixedHandler = fixed.getHandler();
        fixed = null; // fixed can now be GC'd — handler only has WeakReference
        System.gc();
        fixedHandler.handleMessage(); // may print "Activity was GC'd"
    }

    // Anonymous class closure — capturing this vs captured variable
    interface Greeting {
        String greet(String name);
    }

    static Greeting createGreeting(String prefix) {
        // 'prefix' is captured — must be effectively final
        return new Greeting() {
            @Override
            public String greet(String name) {
                return prefix + ", " + name + "!";
            }
        };
        // Equivalent lambda: return name -> prefix + ", " + name + "!";
    }

    public static void main(String[] args) {
        leakDemo();

        Greeting hello = createGreeting("Hello");
        Greeting hi = createGreeting("Hi");
        System.out.println(hello.greet("Alice")); // Hello, Alice!
        System.out.println(hi.greet("Bob"));      // Hi, Bob!
    }
}
```

## INTERVIEW

**Q1: What are the four types of inner classes in Java?**

> **Strong answer:** (1) Static nested class — `static` keyword, can only access outer's static members, no implicit outer reference, preferred for helper classes and builders. (2) Inner (member) class — no `static`, holds implicit reference to outer instance, can access all outer members including private. (3) Local class — defined inside a method, can access effectively final local variables. (4) Anonymous class — inline definition of a class without a name, most common for single-use interface/abstract class implementations; replaced by lambdas for functional interfaces in Java 8+.

**Q2: Why must variables captured by an anonymous class be final or effectively final?**

> **Strong answer:** When an anonymous class captures a local variable, it copies the value (because the anonymous class may outlive the method — the local variable would be gone from the stack when the method returns). If you could modify the variable after the copy, you'd have inconsistency — the original and the copy would diverge. To prevent this bug, Java requires the captured variable to never change after initialization (effectively final). With arrays or AtomicInteger, you can work around this because the reference is final (constant) but the content it points to can change.

**Q3: What is the difference between a static nested class and an inner class?**

> **Strong answer:** Static nested class: declared with `static` keyword, doesn't require an outer class instance to instantiate (`new Outer.Nested()`), can only access outer's static members. Inner class: no `static`, requires an outer class instance (`outer.new Inner()`), holds an implicit reference to the outer instance (`OuterClass.this`), can access all outer members including private. Key rule of thumb: if the nested class doesn't need access to outer instance fields, always make it static — prevents memory leaks and is cleaner design.

**Q4: How can non-static inner classes cause memory leaks?**

> **Strong answer:** Every non-static inner class instance holds an invisible reference to its enclosing outer class instance (`OuterClass.this`). If the inner class is passed somewhere with a longer lifecycle (e.g., background thread, event bus, retained in a collection), the outer class instance cannot be garbage collected even after all external references are dropped. Fix: make the nested class static. If it needs to reference the outer class, use a `WeakReference<OuterClass>` — this allows GC to collect the outer class while still being accessible if it's still alive.

## FEYNMAN CHECK

A static nested class is like a guest room in a house — it's part of the house structure but can exist independently. An inner class is like a person who always carries their house with them — you can't have the person without their house following along. Anonymous classes are like temporary contractors — they show up, do one job, and leave without being given a formal name.
