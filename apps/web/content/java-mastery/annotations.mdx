## WHY

Annotations are the foundation of modern Java frameworks — Spring, Hibernate, JUnit, Jackson all rely on them. You need to understand annotations to:

1. **Use frameworks correctly** — `@Autowired`, `@Entity`, `@Test`, `@JsonProperty` are all annotations
2. **Write custom validation** — `@Email`, `@NotNull`, `@Range` — you can build your own
3. **Annotation processing** — Generate code at compile time (Lombok, MapStruct, Dagger)
4. **Meta-programming** — Reflection + annotations = the backbone of every Java framework
5. **Interview** — "How does Spring's @Autowired work?" always leads to annotation + reflection

## THEORY

### What Is an Annotation?

An annotation is **metadata attached to code** — it doesn't change program behavior by itself but can be read by:
- **The compiler** (e.g., `@Override`, `@SuppressWarnings`)
- **Build tools** at compile time (annotation processors: Lombok, MapStruct)
- **Frameworks** at runtime (Spring reads `@Autowired` via reflection)

```java
@Override          // compiler checks this overrides a parent method
@Deprecated        // compiler warns users this is obsolete
@SuppressWarnings  // compiler suppresses specific warnings
@FunctionalInterface // compiler validates exactly one abstract method
```

### Built-in Annotations

**Standard annotations:**

| Annotation | Target | Purpose |
|-----------|--------|---------|
| `@Override` | Method | Ensures the method overrides a parent method |
| `@Deprecated` | Any | Marks as obsolete; generates compiler warning on use |
| `@SuppressWarnings` | Any | Suppresses specified compiler warnings |
| `@FunctionalInterface` | Interface | Validates interface has exactly 1 abstract method |
| `@SafeVarargs` | Method | Suppresses heap pollution warnings for varargs |

**Meta-annotations (annotate other annotations):**

| Meta-annotation | Purpose |
|----------------|---------|
| `@Target` | Where annotation can be applied (TYPE, METHOD, FIELD, etc.) |
| `@Retention` | When annotation is visible (SOURCE, CLASS, RUNTIME) |
| `@Documented` | Include in Javadoc |
| `@Inherited` | Subclasses inherit the annotation |
| `@Repeatable` | Allow same annotation multiple times on one element |

### @Retention Policies

```
SOURCE  → Discarded by compiler. Used for code generation (Lombok).
CLASS   → Retained in .class file but not available at runtime (default).
RUNTIME → Available at runtime via reflection. Used by Spring, JUnit, etc.
```

**Most custom annotations use `RUNTIME`** because they need to be read by frameworks.

### @Target Values

```java
ElementType.TYPE            // class, interface, enum
ElementType.FIELD           // field (including enum constants)
ElementType.METHOD          // method
ElementType.PARAMETER       // method parameter
ElementType.CONSTRUCTOR     // constructor
ElementType.LOCAL_VARIABLE  // local variable
ElementType.ANNOTATION_TYPE // another annotation (meta-annotation)
ElementType.PACKAGE         // package declaration
ElementType.TYPE_PARAMETER  // generic type parameter (Java 8)
ElementType.TYPE_USE        // any use of a type (Java 8)
```

### Defining a Custom Annotation

```java
@Retention(RetentionPolicy.RUNTIME)  // visible at runtime
@Target(ElementType.FIELD)           // can only be on fields
public @interface MinLength {
    int value();                     // annotation element
    String message() default "Too short"; // with default
}
```

**Usage:**
```java
public class User {
    @MinLength(value = 3, message = "Name must be at least 3 chars")
    private String name;

    @MinLength(5)  // shorthand when only 'value' is set
    private String password;
}
```

### Annotation Elements

```java
public @interface Config {
    String name();                    // required String
    int timeout() default 30;         // optional int (default 30)
    String[] tags() default {};       // optional String array
    Class<?> type() default Object.class; // Class type
    RetentionPolicy policy() default RetentionPolicy.RUNTIME; // enum
    Override override() default @Override; // nested annotation
}
```

**Rules:**
- Element types: primitives, String, Class, enum, annotation, arrays of these
- No complex types (List, Map, etc.)
- Default values must be compile-time constants
- `value()` is special — can be used as shorthand: `@Annot("x")` = `@Annot(value="x")`

### Level 1 — Beginner: Built-in Annotations

```java
// BuiltInAnnotations.java
public class BuiltInAnnotations {

    // @Override — compile-time check
    @Override
    public String toString() {
        return "BuiltInAnnotations{}";
    }

    // Trying to override non-existent method → compile error
    // @Override
    // public void notARealMethod() {} // ERROR: does not override

    // @Deprecated — marks obsolete code
    @Deprecated(since = "2.0", forRemoval = true)
    public void oldMethod() {
        System.out.println("This will be removed");
    }

    // @SuppressWarnings — suppress specific warnings
    @SuppressWarnings("unchecked")
    public void uncheckedOperation() {
        java.util.List list = new java.util.ArrayList(); // raw type — unchecked warning suppressed
        list.add("element");
    }

    // @SuppressWarnings with multiple warnings
    @SuppressWarnings({"unchecked", "deprecated"})
    public void multipleWarnings() {
        oldMethod(); // would warn "deprecated" — suppressed
    }

    // @FunctionalInterface
    @FunctionalInterface
    interface Transformer<T, R> {
        R transform(T input);
        // Cannot add another abstract method — @FunctionalInterface enforces this
        // default methods and static methods are fine:
        default Transformer<T, R> andLog() {
            return input -> {
                R result = transform(input);
                System.out.println(input + " → " + result);
                return result;
            };
        }
    }

    public static void main(String[] args) {
        BuiltInAnnotations obj = new BuiltInAnnotations();

        // Using deprecated method — compiler shows warning (unless suppressed)
        @SuppressWarnings("deprecated")
        var unused = obj; // suppress the variable unused warning

        Transformer<String, Integer> lengthOf = String::length;
        Transformer<String, Integer> logging = lengthOf.andLog();
        logging.transform("hello"); // hello → 5
    }
}
```

### Level 2 — Elementary: Custom Annotation + Reflection

```java
// CustomAnnotation.java
import java.lang.annotation.*;
import java.lang.reflect.*;

// 1. Define the annotation
@Retention(RetentionPolicy.RUNTIME)
@Target({ElementType.FIELD, ElementType.METHOD})
public @interface Validate {
    int minLength() default 0;
    int maxLength() default Integer.MAX_VALUE;
    boolean required() default true;
    String message() default "";
}

// 2. Use the annotation
class UserForm {
    @Validate(minLength = 3, maxLength = 50, message = "Name: 3-50 chars")
    private String name;

    @Validate(minLength = 5, message = "Password: min 5 chars")
    private String password;

    @Validate(required = false) // optional field
    private String bio;

    public UserForm(String name, String password, String bio) {
        this.name = name; this.password = password; this.bio = bio;
    }
}

// 3. Process the annotation via reflection
class Validator {
    public static java.util.List<String> validate(Object obj) throws IllegalAccessException {
        java.util.List<String> errors = new java.util.ArrayList<>();
        Class<?> clazz = obj.getClass();

        for (Field field : clazz.getDeclaredFields()) {
            if (!field.isAnnotationPresent(Validate.class)) continue;

            field.setAccessible(true);
            Validate annotation = field.getAnnotation(Validate.class);
            Object value = field.get(obj);

            // Check required
            if (annotation.required() && value == null) {
                errors.add(field.getName() + " is required");
                continue;
            }
            if (value == null) continue;

            // Check string length
            if (value instanceof String s) {
                if (s.length() < annotation.minLength()) {
                    String msg = annotation.message().isEmpty()
                        ? field.getName() + " too short (min " + annotation.minLength() + ")"
                        : annotation.message();
                    errors.add(msg);
                }
                if (s.length() > annotation.maxLength()) {
                    errors.add(field.getName() + " too long (max " + annotation.maxLength() + ")");
                }
            }
        }
        return errors;
    }

    public static void main(String[] args) throws Exception {
        UserForm valid = new UserForm("Alice", "secret123", "Developer");
        System.out.println(validate(valid)); // []

        UserForm invalid = new UserForm("Al", "pwd", null); // name too short, password too short
        System.out.println(validate(invalid)); // [Name: 3-50 chars, Password: min 5 chars]
    }
}
```

### Level 3 — Intermediate: Annotation-Driven Routing (Mini Framework)

```java
// MiniFramework.java
import java.lang.annotation.*;
import java.lang.reflect.*;
import java.util.*;

// HTTP Method annotation
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface GetMapping { String value(); }

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface PostMapping { String value(); }

// Controller marker
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
@interface RestController {}

// Sample controller
@RestController
class UserController {
    @GetMapping("/users")
    public String getUsers() { return "[{\"id\":1,\"name\":\"Alice\"}]"; }

    @GetMapping("/users/{id}")
    public String getUser() { return "{\"id\":1,\"name\":\"Alice\"}"; }

    @PostMapping("/users")
    public String createUser() { return "{\"id\":2,\"name\":\"Bob\"}"; }
}

// Mini router — reads annotations to map paths to handlers
class Router {
    record Route(String method, String path, Method handler, Object controller) {}

    private final List<Route> routes = new ArrayList<>();

    public void register(Object controller) {
        Class<?> clazz = controller.getClass();
        if (!clazz.isAnnotationPresent(RestController.class)) return;

        for (Method method : clazz.getDeclaredMethods()) {
            if (method.isAnnotationPresent(GetMapping.class)) {
                String path = method.getAnnotation(GetMapping.class).value();
                routes.add(new Route("GET", path, method, controller));
            }
            if (method.isAnnotationPresent(PostMapping.class)) {
                String path = method.getAnnotation(PostMapping.class).value();
                routes.add(new Route("POST", path, method, controller));
            }
        }
    }

    public String handle(String httpMethod, String path) throws Exception {
        for (Route route : routes) {
            if (route.method().equals(httpMethod) && pathMatches(route.path(), path)) {
                return (String) route.handler().invoke(route.controller());
            }
        }
        return "404 Not Found";
    }

    private boolean pathMatches(String template, String actual) {
        // Simple matching — treat {id} as wildcard
        String regex = template.replaceAll("\\{[^}]+}", "[^/]+");
        return actual.matches(regex);
    }

    public static void main(String[] args) throws Exception {
        Router router = new Router();
        router.register(new UserController());

        System.out.println(router.handle("GET", "/users"));
        System.out.println(router.handle("GET", "/users/42"));
        System.out.println(router.handle("POST", "/users"));
        System.out.println(router.handle("DELETE", "/users")); // 404
    }
}
```

### Level 4 — Advanced: Repeatable Annotations & Annotation Inheritance

```java
// AdvancedAnnotations.java
import java.lang.annotation.*;
import java.lang.reflect.*;
import java.util.*;

// Repeatable annotation for roles
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@Repeatable(RequiresRoles.class) // must define container annotation
@interface RequiresRole {
    String value();
}

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
@interface RequiresRoles {
    RequiresRole[] value(); // container for repeated annotations
}

// @Inherited — subclasses inherit this annotation
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
@Inherited
@interface Auditable {
    String level() default "INFO";
}

@Auditable(level = "WARN")
class BaseService {
    @RequiresRole("ADMIN")
    @RequiresRole("MODERATOR") // repeated!
    public void sensitiveOperation() {}

    @RequiresRole("USER")
    public void normalOperation() {}
}

class UserService extends BaseService {
    // @Auditable is INHERITED — UserService is also @Auditable(level="WARN")
}

class SecurityInterceptor {
    public static boolean checkAccess(Object obj, String methodName, String userRole) throws Exception {
        Method method = obj.getClass().getMethod(methodName);
        RequiresRole[] roles = method.getAnnotationsByType(RequiresRole.class); // gets all repeated

        if (roles.length == 0) return true; // no restriction
        for (RequiresRole role : roles) {
            if (role.value().equals(userRole)) return true;
        }
        return false;
    }

    public static void main(String[] args) throws Exception {
        BaseService service = new BaseService();

        System.out.println(checkAccess(service, "sensitiveOperation", "ADMIN"));     // true
        System.out.println(checkAccess(service, "sensitiveOperation", "MODERATOR")); // true
        System.out.println(checkAccess(service, "sensitiveOperation", "USER"));      // false
        System.out.println(checkAccess(service, "normalOperation", "USER"));         // true

        // Check inherited annotation
        UserService userService = new UserService();
        Auditable auditable = userService.getClass().getAnnotation(Auditable.class);
        System.out.println("Auditable: " + (auditable != null)); // true (inherited)
        System.out.println("Level: " + auditable.level());       // WARN
    }
}
```

### Level 5 — Expert: Annotation Processor (Compile-Time Code Generation)

```java
// This shows how annotation processors work (like Lombok under the hood)
// A real annotation processor extends javax.annotation.processing.AbstractProcessor

import javax.annotation.processing.*;
import javax.lang.model.*;
import javax.lang.model.element.*;
import java.util.*;

// The annotation to process
@Retention(RetentionPolicy.SOURCE) // only needed at compile time
@Target(ElementType.TYPE)
@interface GenerateBuilder {}

// The processor (runs at compile time, generates source files)
@SupportedAnnotationTypes("GenerateBuilder")
@SupportedSourceVersion(SourceVersion.RELEASE_17)
public class BuilderProcessor extends AbstractProcessor {

    @Override
    public boolean process(Set<? extends TypeElement> annotations, RoundEnvironment roundEnv) {
        for (TypeElement annotation : annotations) {
            for (Element element : roundEnv.getElementsAnnotatedWith(annotation)) {
                if (element.getKind() == ElementKind.CLASS) {
                    generateBuilder((TypeElement) element);
                }
            }
        }
        return true; // annotation claimed
    }

    private void generateBuilder(TypeElement classElement) {
        String className = classElement.getSimpleName().toString();
        String builderName = className + "Builder";
        StringBuilder code = new StringBuilder();
        code.append("public class ").append(builderName).append(" {\n");

        // For each field, generate a setter that returns 'this'
        for (VariableElement field : ElementFilter.fieldsIn(classElement.getEnclosedElements())) {
            String fieldName = field.getSimpleName().toString();
            String fieldType = field.asType().toString();
            code.append("  private ").append(fieldType).append(" ").append(fieldName).append(";\n");
            code.append("  public ").append(builderName).append(" ").append(fieldName)
                .append("(").append(fieldType).append(" ").append(fieldName).append(") {\n");
            code.append("    this.").append(fieldName).append(" = ").append(fieldName).append(";\n");
            code.append("    return this;\n  }\n");
        }
        code.append("  public ").append(className).append(" build() { ... }\n}");

        System.out.println("Generated:\n" + code);
        // In reality: write to processingEnv.getFiler().createSourceFile(builderName)
    }

    // Lombok uses this pattern for @Getter, @Setter, @Builder, @Data
    // MapStruct uses this for @Mapper (generates mapper implementations at compile time)
}
```

## INTERVIEW

**Q1: What are annotations in Java and how are they different from comments?**

> **Strong answer:** Annotations are structured metadata attached to code elements. Unlike comments, annotations are processed by tools: the compiler processes `@Override` to verify method override; build tools like Lombok's annotation processor generates code from `@Data`; frameworks like Spring read `@Autowired` at runtime via reflection to inject dependencies. Annotations have a defined syntax, can carry data (elements), and have a defined retention policy controlling their lifecycle (SOURCE, CLASS, RUNTIME). They're essentially a formalized way to add machine-readable metadata to code.

**Q2: What is the difference between @Retention SOURCE, CLASS, and RUNTIME?**

> **Strong answer:** SOURCE — annotation is discarded by the compiler, never goes into `.class` file. Used for compile-time code generation (Lombok's `@Getter` generates methods during compilation then the annotation is discarded). CLASS — retained in `.class` file but not accessible via reflection at runtime. Default retention. Rarely used directly. RUNTIME — retained and accessible at runtime via `Class.getAnnotation()`, `Method.getAnnotation()` etc. Required for runtime frameworks (Spring, JUnit). Most custom annotations you write will use RUNTIME.

**Q3: How does Spring's @Autowired work under the hood?**

> **Strong answer:** When Spring creates a bean, it uses reflection to scan all fields, constructors, and methods annotated with `@Autowired`. For each `@Autowired` field, Spring calls `field.setAccessible(true)` and then `field.set(beanInstance, dependency)` where the dependency is resolved from the application context. This is why `@Autowired` works even on private fields — reflection bypasses access modifiers. Spring also reads `@Component`, `@Service`, `@Repository`, `@Controller` (all annotations with `@Component` meta-annotation) during classpath scanning to discover which classes should be managed beans.

**Q4: How do you create a custom validation annotation?**

> **Strong answer:** Three steps. (1) Define the annotation with `@Constraint(validatedBy=MyValidator.class)`, `@Target`, and `@Retention(RUNTIME)`. (2) Create a `ConstraintValidator<MyAnnotation, FieldType>` class implementing `initialize()` and `isValid()`. (3) Apply `@Valid` on the object in your controller/service to trigger validation. The framework (Spring/Jakarta) reads the `@Constraint` meta-annotation on your custom annotation, instantiates the validator, and calls `isValid()` during bean validation. You can chain validators and add custom messages with `{key}` interpolation.

## FEYNMAN CHECK

An annotation is like a sticky note on your code. The sticky note doesn't change what the code does — but it tells other people (or tools) something important about it. When you put `@Override` on a method, you're putting a sticky note saying "I believe this overrides a parent method — please verify." When Spring sees `@Autowired`, it reads the sticky note and automatically fills in the dependency for you, like a robot who reads your sticky notes and does what they say.
