# Arrays and Tuples | Level: 2

TypeScript extends JavaScript arrays with static type safety and introduces **Tuples** for fixed-length, position-sensitive arrays.

---

### Typed Arrays

Arrays can be typed using two equivalent syntaxes: `Type[]` (syntactic sugar) or the generic `Array<Type>` syntax.

```typescript
const numbers: number[] = [1, 2, 3];
const strings: Array<string> = ["a", "b", "c"];
```

---

### Readonly Arrays

To prevent mutation of arrays, TypeScript provides `ReadonlyArray<T>` and `readonly T[]`. These types remove all mutating methods like `push`, `pop`, `splice`, or `shift`, and prevent direct index reassignment.

```typescript
const mutable: number[] = [1, 2, 3];
mutable.push(4); // Allowed

const immutable: readonly number[] = [1, 2, 3];
immutable.push(4); 
// error TS2339: Property 'push' does not exist on type 'readonly number[]'.

immutable[0] = 99; 
// error TS2542: Index signature in type 'readonly number[]' only permits reading.
```

---

### Tuples: Fixed-Length, Position-Typed Arrays

A tuple is an array with a fixed number of elements where each element has a specific, known type at a specific position.

```typescript
let response: [string, number];
response = ["Success", 200]; // Valid
response = [200, "Success"]; // Invalid!
// error TS2322: Type 'number' is not assignable to type 'string'.
// error TS2322: Type 'string' is not assignable to type 'number'.
```

#### Optional Tuple Elements
Tuples can have optional elements marked with `?`, which must always be positioned at the end of the tuple definition.

```typescript
type ConnectionConfig = [string, number, boolean?];
const local: ConnectionConfig = ["localhost", 5432];
const remote: ConnectionConfig = ["10.0.0.1", 5432, true];
```

#### Rest Elements in Tuples
Tuples can use spread/rest elements to allow variadic lengths while maintaining certain positional constraints.

```typescript
type StringAndNumbers = [string, ...number[]];
const data1: StringAndNumbers = ["scores", 98, 87, 100];
const data2: StringAndNumbers = ["empty"]; // Also valid
```

#### Labeled Tuples (TypeScript 4.0+)
Labeled tuples provide names to positions. These labels do not affect type compatibility but provide valuable documentation and improve IDE autocomplete.

```typescript
type Point3D = [x: number, y: number, z: number];
const originPoint: Point3D = [0, 0, 0];
```

---

### Real-World Use Cases for Tuples

1.  **Multiple Return Values (React Hook Pattern)**:
    ```typescript
    function useToggle(initialValue: boolean): [boolean, (val: boolean) => void] {
      let state = initialValue;
      const setState = (val: boolean) => { state = val; };
      return [state, setState];
    }
    ```
2.  **CSV/Tabular Row Representation**:
    ```typescript
    type CSVRow = [id: number, email: string, isActive: boolean];
    ```

---

### Compiled JavaScript Output

Tuples and readonly modifiers are purely compile-time concepts. They are compiled down to standard mutable JavaScript arrays.

```typescript
// TypeScript
const readOnlyTuple: readonly [string, number] = ["admin", 1];
```

```javascript
// Compiled JavaScript (Target: ES2022)
const readOnlyTuple = ["admin", 1];
```
