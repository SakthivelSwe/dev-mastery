# Breadth-First Search (BFS)

Breadth-First Search (BFS) is a fundamental graph traversal algorithm designed to explore vertices layer by layer in order of their distance from the source vertex.

## 1. Core Theory & Mathematical Foundations

### The FIFO Queue as a Shortest Path Engine
The mathematical property that makes BFS find the shortest path in an unweighted graph is the **First-In, First-Out (FIFO)** ordering of the queue. 

Let $\delta(s, u)$ be the shortest path distance (number of edges) from source $s$ to vertex $u$. 
If we enqueue vertices in FIFO order, we guarantee that:
$$\text{If } u \text{ is enqueued before } v, \text{ then } \delta(s, u) \le \delta(s, v)$$

During execution, the queue contains at most two adjacent distance levels: some vertices at distance $k$, followed by vertices at distance $k+1$. This prevents us from exploring any vertex at distance $k+2$ before finishing all vertices at distance $k$.

### Visited Set: Breaking Cycles
In an undirected graph or a directed graph with cycles, a node can be reached via multiple paths. Without a tracking mechanism (a `visited` boolean array or hash set), a cycle $A \to B \to A$ will cause the algorithm to infinitely enqueue $A$ and $B$, exhausting memory. 

### BFS Tree vs. Graph
When BFS runs on a graph $G = (V, E)$, it implicitly constructs a **BFS Spanning Tree** $T$. 
* The root of $T$ is the source $s$.
* An edge $(u, v) \in E$ becomes a tree edge in $T$ if $v$ was first discovered via relaxation of $u$.
* Non-tree edges in undirected graphs can only connect vertices whose levels differ by at most 1.

### Level Tracking Techniques
1. **Two-Queue Approach**: Maintain a `currentLevel` queue and a `nextLevel` queue. Swap them when `currentLevel` becomes empty.
2. **Level Counter (Size-based)**: Use a single queue. Before processing a level, capture its size $S = \text{queue.size()}$. Run a loop exactly $S$ times to process all nodes of the current level before incrementing the level counter.

### BFS on Weighted Graphs
BFS **cannot** guarantee shortest paths on weighted graphs. 
* *Counterexample*: Consider a graph with vertices $A, B, C$ and edges $A \to B$ (weight 5), $B \to C$ (weight 1), $A \to C$ (weight 10). 
* BFS discovers $C$ directly from $A$ in 1 step (distance 10) and marks it visited.
* The true shortest path is $A \to B \to C$ with weight 6. Because BFS processes by topological hop-count rather than cost, it yields suboptimal paths on weighted graphs. This necessitates Dijkstra's algorithm.

## 3. Implementation (Java 21)

```java
import java.util.*;

public class BreadthFirstSearch {
    
    public static Map<Integer, Integer> computeShortestPaths(Map<Integer, List<Integer>> adjList, int source) {
        Map<Integer, Integer> distances = new HashMap<>();
        Queue<Integer> queue = new ArrayDeque<>();
        Set<Integer> visited = new HashSet<>();
        
        queue.add(source);
        visited.add(source);
        distances.put(source, 0);
        
        int currentLevel = 0;
        
        while (!queue.isEmpty()) {
            int levelSize = queue.size();
            for (int i = 0; i < levelSize; i++) {
                int curr = queue.poll();
                
                for (int neighbor : adjList.getOrDefault(curr, Collections.emptyList())) {
                    if (!visited.contains(neighbor)) {
                        visited.add(neighbor);
                        distances.put(neighbor, distances.get(curr) + 1);
                        queue.add(neighbor);
                    }
                }
            }
            currentLevel++;
        }
        
        return distances;
    }
}
```

## 4. Complexity Analysis

### Time Complexity: $\mathcal{O}(V + E)$
* **Vertices ($V$)**: Every vertex is enqueued at most once because of the `visited` check. Thus, the outer loop runs exactly $V$ times in a connected graph.
* **Edges ($E$)**: When a vertex is dequeued, we iterate over all its outgoing edges. Summing over all vertices, this loop runs $\sum_{v \in V} \text{deg}(v) = E$ times (for a directed graph) or $2E$ times (for an undirected graph).
* Hence, total time complexity is strictly $\mathcal{O}(V + E)$.

### Space Complexity: $\mathcal{O}(V)$
* **Visited Set**: Stores up to $V$ nodes, requiring $\mathcal{O}(V)$ space.
* **Queue**: In the worst-case scenario, the queue must hold the maximum width of the graph. For a complete binary tree of size $N$, the leaf level contains $\lceil N/2 \rceil$ nodes. Thus, the queue size is $\mathcal{O}(V)$.

## 5. Advanced Interview Practice

### How would you implement BFS on a graph so large that it cannot fit into memory on a single machine?
Use a distributed graph processing framework like Apache Giraph or Pregel. The graph is partitioned across multiple machines (by vertex or edge). BFS is executed using a **Bulk Synchronous Parallel (BSP)** model where machines exchange messages containing active frontier nodes at level $k$, update their local visited states, and synchronize globally before proceeding to level $k+1$.

### Explain Bidirectional BFS. Why does it optimize search space?
Bidirectional BFS runs two simultaneous searches: one forward from the source $s$, and one backward from the target $t$. They meet at a middle node. 
If the branching factor of the graph is $b$ and the shortest path distance is $d$:
* Standard BFS explores $\mathcal{O}(b^d)$ nodes.
* Bidirectional BFS explores $\mathcal{O}(b^{d/2} + b^{d/2}) = \mathcal{O}(b^{d/2})$ nodes, significantly reducing the state space.
