## WHY

`HashMap` and `HashSet` are fast but unordered. `TreeMap` and `TreeSet` give you sorted order with O(log n) operations:

1. **Sorted iteration** — `TreeMap` always iterates in key order; great for leaderboards, indexes
2. **Range queries** — `subMap()`, `headMap()`, `tailMap()` find keys in a range in O(log n) — impossible with HashMap
3. **Ceiling/Floor lookups** — "Find the nearest timestamp before now" — built in
4. **NavigableMap/NavigableSet** — Powerful navigation APIs for time-series, scheduling
5. **Interview** — TreeMap vs HashMap, null keys, compareTo vs equals — classic interview traps

## THEORY

### Red-Black Tree Internals

`TreeMap` (and `TreeSet` which wraps a `TreeMap`) uses a **Red-Black Tree** — a self-balancing BST:

```
Inserting keys: 5, 3, 7, 1, 4, 6, 8

Balanced Red-Black Tree:
        5
       / \
      3   7
     / \ / \
    1  4 6  8

All operations: O(log n) worst case
- No matter what keys you insert, the tree stays balanced
- Max depth = 2 × log₂(n+1)
```

**Why balanced?** A naive BST with sorted inputs degrades to a linked list (O(n)). Red-Black coloring rules guarantee the tree never becomes more than 2× unbalanced:
1. Every node is Red or Black
2. Root is Black
3. No two consecutive Red nodes
4. All paths from root to leaf have same number of Black nodes

### TreeMap vs HashMap vs LinkedHashMap

| | HashMap | LinkedHashMap | TreeMap |
|--|---------|---------------|---------|
| **Order** | None (random) | Insertion or access order | Sorted by key |
| **get/put** | O(1) | O(1) | O(log n) |
| **Range query** | Not supported | Not supported | O(log n) |
| **Memory** | Low | Medium | High (tree nodes) |
| **null key** | 1 null allowed | 1 null allowed | ❌ Never |
| **Use when** | Fast lookup, no order | Need iteration order | Sorted keys, range queries |

### NavigableMap API — TreeMap's Superpower

```java
TreeMap<Integer, String> map = new TreeMap<>();
// keys: 10, 20, 30, 40, 50

map.floorKey(25)      // 20 — largest key ≤ 25
map.ceilingKey(25)    // 30 — smallest key ≥ 25
map.lowerKey(20)      // 10 — largest key < 20
map.higherKey(20)     // 30 — smallest key > 20

map.firstKey()        // 10 — smallest key
map.lastKey()         // 50 — largest key

map.subMap(20, 40)    // {20=..., 30=...} — keys [20, 40)
map.subMap(20, true, 40, true) // {20=..., 30=..., 40=...} — inclusive [20, 40]
map.headMap(30)       // {10=..., 20=...} — keys < 30
map.tailMap(30)       // {30=..., 40=..., 50=...} — keys ≥ 30

map.descendingMap()   // reverse order view
map.pollFirstEntry()  // remove and return smallest entry
map.pollLastEntry()   // remove and return largest entry
```

### TreeSet — The Sorted Set

`TreeSet<E>` implements `NavigableSet<E>` — a sorted set with the same navigation methods:

```java
TreeSet<String> set = new TreeSet<>();
// Elements always sorted: Apple, Banana, Cherry, Zebra

set.floor("Mango")   // "Cherry" — largest ≤ "Mango"
set.ceiling("Mango") // "Zebra" — smallest ≥ "Mango"
set.headSet("Cherry") // [Apple, Banana] — elements before "Cherry"
set.subSet("B", "Z") // elements between "B" and "Z"
```

### Critical Trap: compareTo vs equals

```java
TreeSet/TreeMap use compareTo() for EVERYTHING — equals() is IGNORED!

class Person {
    String name;
    int age;

    @Override
    public int compareTo(Person o) { return Integer.compare(this.age, o.age); }

    @Override
    public boolean equals(Object o) { return name.equals(((Person)o).name); }
}

TreeSet<Person> set = new TreeSet<>();
Person alice1 = new Person("Alice", 30);
Person alice2 = new Person("Alice", 30);
set.add(alice1);
set.add(alice2);
System.out.println(set.size()); // 1! compareTo returns 0 → "same element" to TreeSet

// TRAP: Two different people with same age → TreeSet treats them as duplicates!
Person bob = new Person("Bob", 30); // same age as Alice
set.add(bob);
System.out.println(set.size()); // STILL 1! Because 30.compareTo(30) == 0
```

### Level 1 — Beginner: Basic TreeSet and TreeMap

```java
// BasicTreeCollections.java
import java.util.*;

public class BasicTreeCollections {
    public static void main(String[] args) {
        // TreeSet — always sorted
        TreeSet<String> names = new TreeSet<>();
        names.add("Zebra");
        names.add("Apple");
        names.add("Monkey");
        names.add("Banana");

        System.out.println("Sorted: " + names);           // [Apple, Banana, Monkey, Zebra]
        System.out.println("First: " + names.first());    // Apple
        System.out.println("Last: " + names.last());      // Zebra
        System.out.println("Floor of Lemon: " + names.floor("Lemon")); // Monkey? No: Banana
        System.out.println("Ceiling of Lemon: " + names.ceiling("Lemon")); // Monkey

        // TreeMap — always sorted by key
        TreeMap<Integer, String> rankings = new TreeMap<>();
        rankings.put(3, "Bronze");
        rankings.put(1, "Gold");
        rankings.put(2, "Silver");
        rankings.put(5, "Copper");
        rankings.put(4, "Platinum");

        System.out.println("\nRankings: " + rankings);
        // {1=Gold, 2=Silver, 3=Bronze, 4=Platinum, 5=Copper}

        System.out.println("Top rank: " + rankings.firstEntry());  // 1=Gold
        System.out.println("Bottom rank: " + rankings.lastEntry()); // 5=Copper

        // TreeSet vs HashSet
        HashSet<Integer> hashSet = new HashSet<>(Arrays.asList(5,3,1,4,2));
        TreeSet<Integer> treeSet = new TreeSet<>(Arrays.asList(5,3,1,4,2));
        System.out.println("\nHashSet: " + hashSet); // random order
        System.out.println("TreeSet: " + treeSet);  // [1, 2, 3, 4, 5]
    }
}
```

### Level 2 — Elementary: Custom Comparator and Reverse Order

```java
// CustomComparatorTree.java
import java.util.*;

public class CustomComparatorTree {
    public static void main(String[] args) {
        // Reverse order TreeMap
        TreeMap<String, Double> prices = new TreeMap<>(Comparator.reverseOrder());
        prices.put("Apple", 1.99);
        prices.put("Banana", 0.99);
        prices.put("Zucchini", 2.99);
        prices.put("Cherry", 3.49);

        System.out.println("Reverse order: " + prices);
        // {Zucchini=2.99, Cherry=3.49, Banana=0.99, Apple=1.99}

        // Case-insensitive TreeSet
        TreeSet<String> caseInsensitive = new TreeSet<>(String.CASE_INSENSITIVE_ORDER);
        caseInsensitive.add("Java");
        caseInsensitive.add("java"); // treated as duplicate!
        caseInsensitive.add("SPRING");
        System.out.println("Case-insensitive: " + caseInsensitive); // [Java, SPRING]
        System.out.println("Size: " + caseInsensitive.size()); // 2, not 3

        // TreeMap with custom object keys
        TreeMap<String[], Integer> wordLengths = new TreeMap<>(
            Comparator.comparingInt(arr -> arr.length));
        // keys sorted by array length

        // Sort words by length, then alphabetically
        TreeMap<String, Integer> wordMap = new TreeMap<>(
            Comparator.comparingInt(String::length)
                      .thenComparing(Comparator.naturalOrder())
        );
        for (String word : "the quick brown fox jumped over the lazy dog".split(" ")) {
            wordMap.put(word, word.length());
        }
        System.out.println("\nWords sorted by length then alpha:");
        wordMap.forEach((w, l) -> System.out.printf("  %s (%d)%n", w, l));
    }
}
```

### Level 3 — Intermediate: Range Queries and NavigableMap

```java
// NavigableMapDemo.java
import java.util.*;
import java.time.*;

public class NavigableMapDemo {

    // Time-series: store events by timestamp, query by time range
    record Event(String description) {}

    public static void main(String[] args) {
        TreeMap<Long, Event> events = new TreeMap<>();
        long now = System.currentTimeMillis();

        events.put(now - 3600_000, new Event("1 hour ago: User registered"));
        events.put(now - 1800_000, new Event("30 min ago: Login"));
        events.put(now - 600_000,  new Event("10 min ago: Viewed product"));
        events.put(now - 300_000,  new Event("5 min ago: Added to cart"));
        events.put(now - 60_000,   new Event("1 min ago: Payment started"));
        events.put(now,            new Event("Now: Payment confirmed"));

        // Get all events in last 20 minutes
        long twentyMinAgo = now - 1200_000;
        System.out.println("Events in last 20 minutes:");
        events.tailMap(twentyMinAgo).forEach((ts, e) -> System.out.println("  " + e.description()));

        // Get the most recent event before 2 minutes ago
        long twoMinAgo = now - 120_000;
        Map.Entry<Long, Event> justBefore = events.floorEntry(twoMinAgo);
        System.out.println("\nMost recent event before 2 min ago: " + justBefore.getValue().description());

        // Sliding window — events in the last N minutes (useful for analytics)
        long windowStart = now - 900_000; // 15 minutes
        NavigableMap<Long, Event> window = events.subMap(windowStart, true, now, true);
        System.out.println("\nLast 15 min (" + window.size() + " events):");
        window.forEach((ts, e) -> System.out.println("  " + e.description()));

        // Stock price example — find nearest price tick
        TreeMap<Long, Double> prices = new TreeMap<>();
        prices.put(1000L, 150.00);
        prices.put(1050L, 151.25);
        prices.put(1100L, 149.75);
        prices.put(1150L, 152.00);

        long queryTime = 1075L;
        System.out.println("\nPrice at " + queryTime + ":");
        System.out.println("  Floor (most recent): " + prices.floorEntry(queryTime));
        System.out.println("  Ceiling (next tick): " + prices.ceilingEntry(queryTime));
    }
}
```

### Level 4 — Advanced: TreeSet for Leaderboard and Interval Scheduling

```java
// TreeSetAlgorithms.java
import java.util.*;
import java.util.stream.*;

public class TreeSetAlgorithms {

    // Leaderboard: sorted by score (desc), then name (asc) for ties
    record Player(String name, int score) implements Comparable<Player> {
        @Override public int compareTo(Player o) {
            int cmp = Integer.compare(o.score, this.score); // desc by score
            return cmp != 0 ? cmp : this.name.compareTo(o.name); // asc by name for ties
        }
        @Override public String toString() { return name + ":" + score; }
    }

    static void leaderboardDemo() {
        TreeSet<Player> leaderboard = new TreeSet<>();
        leaderboard.add(new Player("Charlie", 1000));
        leaderboard.add(new Player("Alice", 1500));
        leaderboard.add(new Player("Bob", 1500));   // same score as Alice
        leaderboard.add(new Player("Dave", 800));

        System.out.println("Leaderboard:");
        int rank = 1;
        for (Player p : leaderboard) {
            System.out.printf("  #%d %s%n", rank++, p);
        }
        // #1 Alice:1500 (A before B with same score)
        // #2 Bob:1500
        // #3 Charlie:1000
        // #4 Dave:800

        // Add new player
        leaderboard.add(new Player("Eve", 1200));
        System.out.println("After Eve joins: " + leaderboard);

        // Top 3 players
        System.out.println("Top 3: " +
            leaderboard.stream().limit(3).collect(Collectors.toList()));
    }

    // Interval merging using TreeMap
    static List<int[]> mergeIntervals(int[][] intervals) {
        TreeMap<Integer, Integer> map = new TreeMap<>();
        for (int[] iv : intervals) {
            map.merge(iv[0], iv[1], Math::max);
        }

        List<int[]> result = new ArrayList<>();
        for (Map.Entry<Integer, Integer> e : map.entrySet()) {
            if (result.isEmpty() || result.get(result.size()-1)[1] < e.getKey()) {
                result.add(new int[]{e.getKey(), e.getValue()});
            } else {
                result.get(result.size()-1)[1] = Math.max(result.get(result.size()-1)[1], e.getValue());
            }
        }
        return result;
    }

    public static void main(String[] args) {
        leaderboardDemo();

        System.out.println("\nMerged intervals:");
        int[][] intervals = {{1,3},{2,6},{8,10},{15,18},{7,11}};
        mergeIntervals(intervals).forEach(iv ->
            System.out.printf("  [%d, %d]%n", iv[0], iv[1]));
        // [1, 6], [7, 11], [15, 18]
    }
}
```

### Level 5 — Expert: LRU Cache and Sliding Window Maximum

```java
// TreeMapExpert.java
import java.util.*;

public class TreeMapExpert {

    // Sliding window maximum using TreeMap (frequency map approach)
    static int[] slidingWindowMax(int[] nums, int k) {
        TreeMap<Integer, Integer> window = new TreeMap<>(); // value -> frequency
        int[] result = new int[nums.length - k + 1];

        // Initialize window
        for (int i = 0; i < k; i++) window.merge(nums[i], 1, Integer::sum);
        result[0] = window.lastKey(); // TreeMap gives O(log n) max

        for (int i = k; i < nums.length; i++) {
            // Add new element
            window.merge(nums[i], 1, Integer::sum);
            // Remove leaving element
            int leaving = nums[i - k];
            window.merge(leaving, -1, Integer::sum);
            if (window.get(leaving) == 0) window.remove(leaving);

            result[i - k + 1] = window.lastKey();
        }
        return result;
    }

    // Median of data stream using two TreeMaps (multisets)
    static class MedianFinder {
        TreeMap<Integer, Integer> lower = new TreeMap<>(); // max-heap via last()
        TreeMap<Integer, Integer> upper = new TreeMap<>(); // min-heap via first()
        int lowerSize = 0, upperSize = 0;

        void addNum(int num) {
            // Add to appropriate half
            if (lower.isEmpty() || num <= lower.lastKey()) {
                lower.merge(num, 1, Integer::sum); lowerSize++;
            } else {
                upper.merge(num, 1, Integer::sum); upperSize++;
            }

            // Balance: lower can have 1 more than upper
            while (lowerSize > upperSize + 1) {
                int maxLower = lower.lastKey();
                lower.merge(maxLower, -1, Integer::sum);
                if (lower.get(maxLower) == 0) lower.remove(maxLower);
                lowerSize--;
                upper.merge(maxLower, 1, Integer::sum); upperSize++;
            }
            while (upperSize > lowerSize) {
                int minUpper = upper.firstKey();
                upper.merge(minUpper, -1, Integer::sum);
                if (upper.get(minUpper) == 0) upper.remove(minUpper);
                upperSize--;
                lower.merge(minUpper, 1, Integer::sum); lowerSize++;
            }
        }

        double findMedian() {
            if (lowerSize > upperSize) return lower.lastKey();
            return (lower.lastKey() + upper.firstKey()) / 2.0;
        }
    }

    public static void main(String[] args) {
        // Sliding window max
        int[] nums = {1, 3, -1, -3, 5, 3, 6, 7};
        System.out.println("Sliding window max (k=3): " +
            Arrays.toString(slidingWindowMax(nums, 3)));
        // [3, 3, 5, 5, 6, 7]

        // Median finder
        MedianFinder mf = new MedianFinder();
        int[] stream = {5, 2, 8, 1, 9, 3, 7};
        for (int n : stream) {
            mf.addNum(n);
            System.out.printf("After adding %d: median = %.1f%n", n, mf.findMedian());
        }
    }
}
```

### Event Scheduler

```java
// Schedule events sorted by execution time
TreeMap<Long, List<Runnable>> scheduler = new TreeMap<>();

void schedule(long delayMs, Runnable task) {
    long executeAt = System.currentTimeMillis() + delayMs;
    scheduler.computeIfAbsent(executeAt, k -> new ArrayList<>()).add(task);
}

void tick() {
    long now = System.currentTimeMillis();
    // Get and remove all due tasks
    NavigableMap<Long, List<Runnable>> due = scheduler.headMap(now, true);
    due.values().forEach(tasks -> tasks.forEach(Runnable::run));
    due.clear();
}
```

## INTERVIEW

**Q: What is the time complexity of TreeMap operations?**
> O(log n) for get, put, remove — guaranteed (Red-Black tree always balanced). Compare to HashMap's O(1) average but O(n) worst case (all keys hash to same bucket).

**Q: Can TreeMap have null keys? What about null values?**
> No null keys — TreeMap must call `compareTo()` on keys and `null.compareTo()` throws NPE. But null VALUES are fine — `treeMap.put("key", null)` works. HashMap allows ONE null key.

**Q: Does TreeSet use equals() or compareTo() to detect duplicates?**
> TreeSet uses only `compareTo()` (or Comparator). `equals()` is completely ignored. If `compareTo()` returns 0, the element is considered a duplicate even if `equals()` returns false. This is a common bug: two objects with different identities but same compareTo result "can't both be in a TreeSet."

**Q: When would you use TreeMap over HashMap?**
> Use TreeMap when you need: sorted iteration, range queries (subMap/headMap/tailMap), floor/ceiling lookups (nearest key), or naturally ordered output. Use HashMap when you just need fast key-value lookup with no ordering requirements — it's 10x faster. Use ConcurrentSkipListMap (equivalent of TreeMap) in multi-threaded code.

## FEYNMAN CHECK

A TreeMap is like a phone book that stays alphabetically sorted as you add names. Instead of scrambling entries and using a hash to find them, it maintains a perfectly balanced tree where every lookup walks through at most log₂(n) steps — like binary searching a sorted array. Because it's always sorted, you can instantly find "all names between A and M" or "the name just before Miller."
