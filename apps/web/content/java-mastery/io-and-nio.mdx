## WHY

Every real application reads files, writes logs, sends network data. Java has two I/O systems:

1. **Classic I/O (java.io)** — Stream-based, blocking, still used everywhere for simple file ops
2. **NIO (java.nio)** — Buffer/Channel based, non-blocking I/O, critical for high-performance apps
3. **NIO.2 (java.nio.file)** — Path API, Files utility class — modern file handling since Java 7
4. **Interview** — "What is the difference between InputStream and Channel?" is common
5. **Real-world** — Log parsing, CSV reading, config loading, file uploads all use Java I/O

## THEORY

### java.io — Classic I/O

Two hierarchies:
- **Byte streams**: `InputStream` / `OutputStream` — raw bytes
- **Character streams**: `Reader` / `Writer` — text with character encoding

```
Byte Streams (raw bytes):
InputStream
  ├── FileInputStream
  ├── ByteArrayInputStream
  ├── BufferedInputStream  (wraps any InputStream, adds buffering)
  └── DataInputStream      (reads primitives: int, double, etc.)

OutputStream
  ├── FileOutputStream
  ├── ByteArrayOutputStream
  ├── BufferedOutputStream
  └── DataOutputStream

Character Streams (text):
Reader
  ├── FileReader           (uses system default encoding!)
  ├── InputStreamReader    (wraps InputStream with charset)
  └── BufferedReader       (adds readLine())

Writer
  ├── FileWriter
  ├── OutputStreamWriter
  ├── BufferedWriter
  └── PrintWriter          (adds print(), println(), printf())
```

### Decorator Pattern in I/O

Java I/O uses the Decorator pattern — wrap streams to add functionality:

```java
// Layer 1: Source (raw bytes)
FileInputStream fis = new FileInputStream("data.bin");

// Layer 2: Buffering (reduce system calls)
BufferedInputStream bis = new BufferedInputStream(fis);

// Layer 3: Data reading (read primitives)
DataInputStream dis = new DataInputStream(bis);

int value = dis.readInt();  // reads 4 bytes as int
```

**Key rule:** Always close the outermost stream — it closes all wrapped streams.

### java.nio — New I/O (Java 4+)

Three core concepts:

| Concept | Description |
|---------|-------------|
| **Buffer** | Fixed-size container for data; has position, limit, capacity |
| **Channel** | Bidirectional I/O conduit (vs streams: unidirectional) |
| **Selector** | Monitors multiple channels; enables non-blocking I/O on 1 thread |

```
Stream I/O:  source → [byte-by-byte stream] → destination
Channel I/O: source → [Block of data in Buffer] → destination
```

### java.nio.file — NIO.2 (Java 7+) — Modern File API

```java
Path path = Path.of("/home/user/data.txt");  // immutable path representation

// Files utility class — all common file operations
Files.exists(path)
Files.readString(path)           // read entire file as String
Files.readAllLines(path)         // List<String>
Files.readAllBytes(path)         // byte[]
Files.writeString(path, content) // write String to file
Files.copy(src, dst)             // copy file
Files.move(src, dst)             // move/rename
Files.delete(path)               // delete
Files.createDirectories(path)    // mkdir -p
Files.walk(path)                 // stream all files recursively
Files.lines(path)                // lazy Stream<String> line by line
```

### Path API

```java

p.getFileName()   // report.pdf
```

## CODE

### Level 1 — Beginner: Reading and Writing Files

```java
// FileReadWrite.java
import java.nio.file.*;
import java.io.*;
import java.util.*;

public class FileReadWrite {
    public static void main(String[] args) throws Exception {
        Path file = Path.of("sample.txt");

        // === WRITE ===
        // Method 1: Files.writeString (Java 11+, simplest)
        Files.writeString(file, "Hello, World!\nLine 2\nLine 3");

        // Method 2: Files.write with List<String>
        Files.write(file, List.of("Line 1", "Line 2", "Line 3"),
            StandardOpenOption.APPEND); // append to existing

        // Method 3: BufferedWriter (more control)
        try (BufferedWriter writer = Files.newBufferedWriter(file, StandardOpenOption.APPEND)) {
            writer.write("Appended line");
            writer.newLine();
        }

        // === READ ===
        // Method 1: readString (whole file as String)
        String content = Files.readString(file);
        System.out.println("Content:\n" + content);

        // Method 2: readAllLines (List<String>)
        List<String> lines = Files.readAllLines(file);
        System.out.println("Lines: " + lines.size());

        // Method 3: lines() — lazy Stream (best for large files)
        long wordCount = Files.lines(file)
            .flatMap(line -> Arrays.stream(line.split("\\s+")))
            .filter(word -> !word.isEmpty())
            .count();
        System.out.println("Word count: " + wordCount);

        // Method 4: BufferedReader (classic, most control)
        try (BufferedReader reader = Files.newBufferedReader(file)) {
            String line;
            while ((line = reader.readLine()) != null) {
                System.out.println("> " + line);
            }
        }

        // Cleanup
        Files.delete(file);
    }
}
```

```java
// PathsAndDirectories.java
import java.nio.file.*;
import java.io.*;
import java.util.stream.*;

public class PathsAndDirectories {
    public static void main(String[] args) throws Exception {
        // Path operations
        Path home = Path.of(System.getProperty("user.home"));
        Path docs = home.resolve("Documents");
        Path report = docs.resolve("2024/Q1/report.txt");

        System.out.println("Filename: " + report.getFileName()); // report.txt
        System.out.println("Parent: " + report.getParent());     // .../2024/Q1
        System.out.println("Root: " + report.getRoot());         // / (or C:\ on Windows)

        // Resolve and relativize
        Path base = Path.of("/home/user");
        Path target = Path.of("/home/user/projects/myapp/src/Main.java");
        Path relative = base.relativize(target);
        System.out.println("Relative: " + relative); // projects/myapp/src/Main.java

        // Create directory structure
        Path workDir = Path.of("test_dir/subdir1/subdir2");
        Files.createDirectories(workDir); // like mkdir -p

        // Write some test files
        Files.writeString(workDir.resolve("file1.txt"), "content 1");
        Files.writeString(workDir.resolve("file2.txt"), "content 2");
        Files.writeString(workDir.getParent().resolve("config.json"), "{}");

        // List directory contents
        System.out.println("\nDirect children of test_dir:");
        try (Stream<Path> stream = Files.list(Path.of("test_dir"))) {
            stream.forEach(p -> System.out.println("  " + p));
        }

        // Walk entire tree
        System.out.println("\nAll files in test_dir:");
        try (Stream<Path> stream = Files.walk(Path.of("test_dir"))) {
            stream.filter(Files::isRegularFile)
                  .forEach(p -> System.out.println("  " + p));
        }

        // Find specific files
        System.out.println("\nAll .txt files:");
        try (Stream<Path> stream = Files.walk(Path.of("test_dir"))) {
            stream.filter(p -> p.toString().endsWith(".txt"))
                  .forEach(System.out::println);
        }

        // Cleanup — delete recursively
        try (Stream<Path> stream = Files.walk(Path.of("test_dir"))) {
            stream.sorted(Comparator.reverseOrder()) // delete children before parents
                  .forEach(p -> { try { Files.delete(p); } catch (IOException e) {} });
        }
    }
}
```

### Level 3 — Intermediate: Streams, Buffers, and Large File Processing

```java
// LargeFileProcessing.java
import java.io.*;
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;
import java.util.*;

public class LargeFileProcessing {

    // Process CSV line by line — memory efficient for large files
    static Map<String, Long> countByCategory(Path csvFile) throws IOException {
        Map<String, Long> counts = new LinkedHashMap<>();
        try (BufferedReader reader = Files.newBufferedReader(csvFile)) {
            reader.readLine(); // skip header
            String line;
            while ((line = reader.readLine()) != null) {
                String[] cols = line.split(",", 3);
                if (cols.length >= 2) {
                    counts.merge(cols[1].trim(), 1L, Long::sum);
                }
            }
        }
        return counts;
    }

    // Efficient copy using NIO channels (zero-copy transfer)
    static void copyWithChannels(Path src, Path dst) throws IOException {
        try (FileChannel srcCh = FileChannel.open(src, StandardOpenOption.READ);
             FileChannel dstCh = FileChannel.open(dst, StandardOpenOption.WRITE,
                 StandardOpenOption.CREATE, StandardOpenOption.TRUNCATE_EXISTING)) {
            // transferTo uses OS-level zero-copy (sendfile syscall on Linux)
            srcCh.transferTo(0, srcCh.size(), dstCh);
        }
    }

    // Memory-mapped file — map file to memory for very fast random access
    static int countOccurrences(Path file, byte target) throws IOException {
        try (FileChannel channel = FileChannel.open(file, StandardOpenOption.READ)) {
            MappedByteBuffer buffer = channel.map(
                FileChannel.MapMode.READ_ONLY, 0, channel.size());
            int count = 0;
            while (buffer.hasRemaining()) {
                if (buffer.get() == target) count++;
            }
            return count;
        }
    }

    // ByteBuffer operations
    static void bufferDemo() {
        ByteBuffer buf = ByteBuffer.allocate(10);
        System.out.println("Capacity: " + buf.capacity()); // 10
        System.out.println("Position: " + buf.position()); // 0
        System.out.println("Limit: " + buf.limit());       // 10

        buf.put((byte)'H');
        buf.put((byte)'i');
        buf.put((byte)'!');
        System.out.println("After writing 3 bytes, position: " + buf.position()); // 3

        buf.flip(); // prepare for reading: limit=3, position=0
        System.out.println("After flip - position: " + buf.position() + " limit: " + buf.limit());

        while (buf.hasRemaining()) {
            System.out.print((char) buf.get()); // H i !
        }
        System.out.println();

        buf.clear(); // reset for writing again: position=0, limit=capacity
    }

    public static void main(String[] args) throws Exception {
        bufferDemo();

        // Create test CSV
        Path csv = Path.of("test.csv");
        Files.writeString(csv, "id,category,value\n1,A,100\n2,B,200\n3,A,150\n4,C,300\n5,B,100");
        System.out.println("Category counts: " + countByCategory(csv));

        // NIO channel copy
        Path copy = Path.of("test_copy.csv");
        copyWithChannels(csv, copy);
        System.out.println("Copied. Content matches: " +
            Files.readString(csv).equals(Files.readString(copy)));

        Files.delete(csv);
        Files.delete(copy);
    }
}
```

### Level 4 — Advanced: Custom InputStream/OutputStream

```java
// CustomStreams.java
import java.io.*;
import java.util.*;

public class CustomStreams {

    // Progress-tracking InputStream — wraps any stream and reports progress
    static class ProgressInputStream extends FilterInputStream {
        private final long totalBytes;
        private long readBytes = 0;
        private final java.util.function.Consumer<Integer> progressCallback;

        ProgressInputStream(InputStream in, long totalBytes,
                java.util.function.Consumer<Integer> progressCallback) {
            super(in);
            this.totalBytes = totalBytes;
            this.progressCallback = progressCallback;
        }

        @Override
        public int read() throws IOException {
            int b = super.read();
            if (b != -1) {
                readBytes++;
                int pct = (int)(readBytes * 100 / totalBytes);
                progressCallback.accept(pct);
            }
            return b;
        }

        @Override
        public int read(byte[] b, int off, int len) throws IOException {
            int n = super.read(b, off, len);
            if (n > 0) {
                readBytes += n;
                int pct = (int)(readBytes * 100 / totalBytes);
                progressCallback.accept(pct);
            }
            return n;
        }
    }

    // TeeOutputStream — writes to two streams simultaneously
    static class TeeOutputStream extends OutputStream {
        private final OutputStream first, second;

        TeeOutputStream(OutputStream first, OutputStream second) {
            this.first = first; this.second = second;
        }

        @Override public void write(int b) throws IOException {
            first.write(b); second.write(b);
        }

        @Override public void write(byte[] b, int off, int len) throws IOException {
            first.write(b, off, len); second.write(b, off, len);
        }

        @Override public void flush() throws IOException { first.flush(); second.flush(); }
        @Override public void close() throws IOException { first.close(); second.close(); }
    }

    public static void main(String[] args) throws Exception {
        // Progress tracking
        byte[] data = "This is test content for progress tracking demo".getBytes();
        int[] lastPct = {0};
        try (ProgressInputStream pis = new ProgressInputStream(
                new ByteArrayInputStream(data), data.length,
                pct -> { if (pct != lastPct[0]) { System.out.print(pct + "% "); lastPct[0] = pct; }})) {
            pis.transferTo(OutputStream.nullOutputStream());
        }
        System.out.println("done");

        // Tee output — write to file AND console simultaneously
        ByteArrayOutputStream log = new ByteArrayOutputStream();
        try (TeeOutputStream tee = new TeeOutputStream(System.out, log);
             PrintWriter writer = new PrintWriter(tee)) {
            writer.println("This goes to both stdout and the log buffer");
            writer.printf("Timestamp: %d%n", System.currentTimeMillis());
        }
        System.out.println("Log captured: " + log.toString().length() + " bytes");
    }
}
```

### Level 5 — Expert: NIO Non-Blocking Server with Selector

```java
// NioEchoServer.java — non-blocking I/O with one thread handling many connections
import java.net.*;
import java.nio.*;
import java.nio.channels.*;
import java.util.*;

public class NioEchoServer {

    public static void startServer(int port) throws Exception {
        // Open non-blocking server socket channel
        ServerSocketChannel serverChannel = ServerSocketChannel.open();
        serverChannel.configureBlocking(false);
        serverChannel.bind(new InetSocketAddress(port));
        System.out.println("NIO Echo Server listening on port " + port);

        // Selector monitors multiple channels on ONE thread
        Selector selector = Selector.open();
        serverChannel.register(selector, SelectionKey.OP_ACCEPT);

        ByteBuffer buffer = ByteBuffer.allocate(1024);

        while (true) {
            selector.select(); // blocks until at least one channel is ready
            Iterator<SelectionKey> keys = selector.selectedKeys().iterator();

            while (keys.hasNext()) {
                SelectionKey key = keys.next();
                keys.remove();

                if (key.isAcceptable()) {
                    // New client connection
                    SocketChannel client = serverChannel.accept();
                    client.configureBlocking(false);
                    client.register(selector, SelectionKey.OP_READ);
                    System.out.println("Client connected: " + client.getRemoteAddress());

                } else if (key.isReadable()) {
                    // Client sent data
                    SocketChannel client = (SocketChannel) key.channel();
                    buffer.clear();
                    int bytesRead = client.read(buffer);
                    if (bytesRead == -1) {
                        System.out.println("Client disconnected");
                        client.close();
                        key.cancel();
                    } else {
                        buffer.flip();
                        String received = new String(buffer.array(), 0, buffer.limit());
                        System.out.println("Received: " + received.trim());
                        // Echo back
                        buffer.rewind();
                        client.write(buffer);
                    }
                }
            }
        }
    }

    public static void main(String[] args) throws Exception {
        // One thread handles unlimited clients (vs thread-per-client in classic I/O)
        // This is the pattern used by Netty, Vert.x, Undertow under the hood
        System.out.println("NIO Server demonstrates:");
        System.out.println("1. Non-blocking channels (configureBlocking(false))");
        System.out.println("2. Selector monitors multiple channels in ONE thread");
        System.out.println("3. O(1) channel monitoring regardless of connection count");
        // startServer(8080); // uncomment to run
    }
}
```

## INTERVIEW

**Q1: What is the difference between InputStream and Reader?**

> **Strong answer:** `InputStream` operates on raw bytes — it reads one byte at a time (0-255). `Reader` operates on characters — it handles character encoding (UTF-8, UTF-16) and reads one char at a time (0-65535). Use `InputStream` for binary data (images, PDFs, serialized objects). Use `Reader` for text data. When reading text from a binary stream, wrap it: `new InputStreamReader(inputStream, StandardCharsets.UTF_8)`. Never use `FileReader` directly — it uses the system default encoding which varies by platform.

**Q2: What is NIO and how is it different from classic I/O?**

> **Strong answer:** Classic I/O is stream-based (one byte at a time), unidirectional (InputStream or OutputStream), and always blocking. NIO (Java 4+) uses Buffers and Channels: data moves in blocks (via ByteBuffer), channels are bidirectional, and I/O can be non-blocking with Selectors. NIO.2 (Java 7+) added the `Path` API and `Files` utility class for modern file operations. Classic I/O is simpler for sequential file reading/writing. NIO is better for high-performance servers (one thread handling thousands of connections via Selector), memory-mapped files (faster random access), and zero-copy file transfer (`FileChannel.transferTo()`).

**Q3: What is memory-mapped file I/O?**

> **Strong answer:** Memory-mapped I/O (`FileChannel.map()` → `MappedByteBuffer`) maps a file directly into process address space. The OS handles paging data between file and memory transparently. Benefits: (1) Reading a mapped file is as fast as reading from memory — the OS may have the data already in page cache. (2) Multiple processes can map the same file (shared memory). (3) Random access is O(1) — just access `buffer.get(offset)` without seeking. Used for: large binary file processing, database storage engines, in-process shared memory. Caution: mapped memory stays until GC collects the MappedByteBuffer — no explicit unmap in Java before Java 22 (when `FileChannel.unmap()` was added).

## FEYNMAN CHECK

Classic I/O is like a garden hose — water (bytes) flows one direction, one drop at a time. NIO is like a truck — it carries a big box (Buffer) of bytes both ways (channels are bidirectional). The Selector in NIO is like a dispatcher watching many trucks — when any truck has cargo ready, the dispatcher handles it. One dispatcher (one thread) can manage thousands of trucks efficiently.
