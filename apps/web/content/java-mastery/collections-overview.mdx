## WHY

Most performance and correctness bugs in Java backend systems are data-structure choice bugs in disguise.

Examples:

- using `LinkedList` for random access causes O(n^2) loops
- using `HashMap` where key order matters causes unstable API responses
- using non-thread-safe collections in concurrent code causes corruption/races
- using wrong set type (`HashSet` vs `TreeSet`) wastes CPU or breaks order assumptions

The Java Collections Framework (JCF) gives you many implementations with the same interface. That power is also the trap: same API, very different complexity and memory behavior.

This topic gives you a selection framework so you can choose the right collection the first time.

## THEORY

### Core Hierarchy

```
java.lang.Iterable<E>
        |
java.util.Collection<E>
   |          |          |
  List       Set       Queue/Deque
   |          |            |
ArrayList   HashSet     ArrayDeque
LinkedList  LinkedHashSet PriorityQueue
Vector      TreeSet

java.util.Map<K,V>  (separate from Collection)
   |
HashMap / LinkedHashMap / TreeMap / ConcurrentHashMap
```

### Collection Families at a Glance

| Family | Guarantees | Typical Implementations |
|--------|------------|--------------------------|
| `List` | ordered, duplicates allowed | `ArrayList`, `LinkedList`, `CopyOnWriteArrayList` |
| `Set` | unique elements | `HashSet`, `LinkedHashSet`, `TreeSet`, `EnumSet` |
| `Queue` | FIFO-style access | `ArrayDeque`, `LinkedList`, `PriorityQueue` |
| `Map` | key-value pairs, unique keys | `HashMap`, `LinkedHashMap`, `TreeMap`, `ConcurrentHashMap` |

### Big-O Comparison (Most Used Types)

| Operation | ArrayList | LinkedList | HashMap | TreeMap | HashSet | TreeSet |
|-----------|-----------|------------|---------|---------|---------|---------|
| `get(i)` | O(1) | O(n) | - | - | - | - |
| `add(e)` end | O(1) amortized | O(1) | O(1) avg | O(log n) | O(1) avg | O(log n) |
| `insert(i,e)` | O(n) | O(n) traversal + O(1) link | - | - | - | - |
| `contains(e)` | O(n) | O(n) | O(1) avg by key | O(log n) by key | O(1) avg | O(log n) |
| ordered iteration | insertion order | insertion order | no guarantee | sorted | no guarantee | sorted |

### Choosing List Implementations

| Use Case | Best Choice | Why |
|----------|-------------|-----|
| Random access by index | `ArrayList` | contiguous array, cache-friendly |
| Frequent head/tail deque ops | `ArrayDeque` | better than `LinkedList` for queue/stack |
| Many concurrent reads, few writes | `CopyOnWriteArrayList` | lock-free iteration snapshots |
| Frequent middle insertion with active iterator | `LinkedList` (rare) | O(1) link adjustments once positioned |

Rule: default to `ArrayList`. Reach for `LinkedList` only with a proven benchmark.

### Choosing Map Implementations

| Need | Best Map |
|------|----------|
| Fast lookups, no order | `HashMap` |
| Preserve insertion/access order | `LinkedHashMap` |
| Sorted keys / range queries | `TreeMap` |
| Concurrent updates | `ConcurrentHashMap` |
| Enum keys | `EnumMap` |

### Fail-Fast vs Weakly Consistent Iterators

| Iterator Type | Example | Behavior |
|---------------|---------|----------|
| Fail-fast | `ArrayList`, `HashMap` | detects structural changes and throws `ConcurrentModificationException` |
| Snapshot | `CopyOnWriteArrayList` | iterates over immutable snapshot |
| Weakly consistent | `ConcurrentHashMap` | may see some concurrent updates, never throws CME |

Fail-fast is a bug detector, not a synchronization mechanism.

### `modCount` Mechanism (Why CME Happens)

`AbstractList` maintains `modCount` incremented on structural change.

Iterator captures `expectedModCount` at creation.

Each `next()` checks:

```java
if (modCount != expectedModCount) {
    throw new ConcurrentModificationException();
}
```

Safe removal during iteration is only via `iterator.remove()` (which updates expected state).

### Memory Footprint Notes

- `ArrayList`: reference array + occasional resize
- `LinkedList`: per-node object overhead (`item`, `next`, `prev`) and poor cache locality
- `HashMap`: bucket table + Node objects + load factor expansion
- `TreeMap`: red-black tree nodes (parent/left/right/color pointers)

Memory-heavy structures become GC-heavy structures.

## CODE

### Level 1 - Beginner

```java
import java.util.*;

public class BasicCollectionsDemo {
    public static void main(String[] args) {
        // List: ordered, duplicates allowed
        List<String> list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("Java");
        System.out.println("List: " + list); // [Java, Python, Java]

        // Set: unique elements
        Set<String> set = new HashSet<>();
        set.add("Java");
        set.add("Python");
        set.add("Java");
        System.out.println("Set: " + set); // [Java, Python] (order not guaranteed)

        // Map: key-value
        Map<String, Integer> map = new HashMap<>();
        map.put("Alice", 90);
        map.put("Bob", 85);
        map.put("Alice", 95); // overwrite existing key
        System.out.println("Map: " + map); // {Alice=95, Bob=85}

        // Queue: FIFO
        Queue<String> queue = new ArrayDeque<>();
        queue.offer("first");
        queue.offer("second");
        System.out.println(queue.poll()); // first
        System.out.println(queue.poll()); // second
    }
}
```

### Level 2 - Building Blocks

```java
import java.util.*;

public class OrderingAndImplementations {
    public static void main(String[] args) {
        // HashMap: no iteration order guarantee
        Map<Integer, String> hash = new HashMap<>();
        hash.put(3, "C"); hash.put(1, "A"); hash.put(2, "B");
        System.out.println("HashMap      : " + hash);

        // LinkedHashMap: insertion order
        Map<Integer, String> linked = new LinkedHashMap<>();
        linked.put(3, "C"); linked.put(1, "A"); linked.put(2, "B");
        System.out.println("LinkedHashMap: " + linked); // 3,1,2

        // TreeMap: sorted by key
        Map<Integer, String> tree = new TreeMap<>();
        tree.put(3, "C"); tree.put(1, "A"); tree.put(2, "B");
        System.out.println("TreeMap      : " + tree); // 1,2,3

        // HashSet vs TreeSet
        Set<Integer> hs = new HashSet<>(List.of(5, 2, 9, 1));
        Set<Integer> ts = new TreeSet<>(List.of(5, 2, 9, 1));
        System.out.println("HashSet: " + hs); // arbitrary order
        System.out.println("TreeSet: " + ts); // sorted

        // PriorityQueue: heap order, not sorted iteration order
        PriorityQueue<Integer> pq = new PriorityQueue<>();
        pq.addAll(List.of(5, 1, 4, 2, 3));
        while (!pq.isEmpty()) {
            System.out.print(pq.poll() + " "); // 1 2 3 4 5
        }
        System.out.println();
    }
}
```

### Level 3 - Intermediate

```java
import java.util.*;

public class IteratorSafety {

    // BAD: modifies list directly during for-each
    static void badRemoval(List<String> data) {
        for (String s : data) {
            if (s.startsWith("x")) {
                // data.remove(s); // throws ConcurrentModificationException
            }
        }
    }

    // GOOD: use iterator.remove()
    static void goodRemovalIterator(List<String> data) {
        Iterator<String> it = data.iterator();
        while (it.hasNext()) {
            if (it.next().startsWith("x")) {
                it.remove();
            }
        }
    }

    // BEST modern approach for filtering
    static void goodRemovalPredicate(List<String> data) {
        data.removeIf(s -> s.startsWith("x"));
    }

    public static void main(String[] args) {
        List<String> data = new ArrayList<>(List.of("a", "x1", "b", "x2", "c"));
        goodRemovalIterator(data);
        System.out.println(data); // [a, b, c]

        List<String> data2 = new ArrayList<>(List.of("a", "x1", "b", "x2", "c"));
        goodRemovalPredicate(data2);
        System.out.println(data2); // [a, b, c]

        // Concurrent collection iteration behavior
        Map<Integer, String> chm = new java.util.concurrent.ConcurrentHashMap<>();
        chm.put(1, "A"); chm.put(2, "B");
        for (Map.Entry<Integer, String> e : chm.entrySet()) {
            chm.put(3, "C"); // no CME; weakly consistent iteration
        }
        System.out.println(chm);
    }
}
```

### Level 4 - Advanced

```java
import java.util.*;
import java.util.concurrent.*;

public class ConcurrentCollectionsPatterns {

    // Pattern 1: read-heavy listener registry
    static class ListenerRegistry {
        // Snapshot semantics: safe iteration without locking
        private final CopyOnWriteArrayList<Runnable> listeners = new CopyOnWriteArrayList<>();

        void register(Runnable r) { listeners.addIfAbsent(r); }
        void unregister(Runnable r) { listeners.remove(r); }

        void broadcast() {
            for (Runnable r : listeners) {
                r.run();
            }
        }
    }

    // Pattern 2: scalable counting map
    static class WordCounter {
        private final ConcurrentHashMap<String, LongAdder> counts = new ConcurrentHashMap<>();

        void increment(String word) {
            counts.computeIfAbsent(word, w -> new LongAdder()).increment();
        }

        long get(String word) {
            LongAdder adder = counts.get(word);
            return adder == null ? 0 : adder.sum();
        }

        Map<String, Long> snapshot() {
            Map<String, Long> snap = new TreeMap<>();
            counts.forEach((k, v) -> snap.put(k, v.sum()));
            return snap;
        }
    }

    // Pattern 3: bounded LRU cache with LinkedHashMap
    static class LruCache<K, V> extends LinkedHashMap<K, V> {
        private final int max;
        LruCache(int max) {
            super(max + 1, 1.0f, true);
            this.max = max;
        }
        @Override
        protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
            return size() > max;
        }
    }

    public static void main(String[] args) {
        ListenerRegistry registry = new ListenerRegistry();
        registry.register(() -> System.out.println("listener A"));
        registry.register(() -> System.out.println("listener B"));
        registry.broadcast();

        WordCounter wc = new WordCounter();
        Arrays.stream("a b a c a b".split(" ")).forEach(wc::increment);
        System.out.println(wc.snapshot()); // {a=3, b=2, c=1}

        LruCache<Integer, String> cache = new LruCache<>(3);
        cache.put(1, "A"); cache.put(2, "B"); cache.put(3, "C");
        cache.get(1);
        cache.put(4, "D"); // evicts key 2
        System.out.println(cache.keySet()); // [3, 1, 4]
    }
}
```

### Level 5 - Expert

```java
import java.util.*;
import java.util.concurrent.*;
import java.util.stream.*;

public class CollectionsDiagnosticsAndBenchmark {

    static long timeMs(Runnable r) {
        long s = System.nanoTime();
        r.run();
        return (System.nanoTime() - s) / 1_000_000;
    }

    static void benchmarkIteration() {
        int n = 1_000_000;
        List<Integer> al = new ArrayList<>(n);
        List<Integer> ll = new LinkedList<>();
        for (int i = 0; i < n; i++) {
            al.add(i);
            ll.add(i);
        }

        long alTime = timeMs(() -> {
            long sum = 0;
            for (int v : al) sum += v;
            if (sum == -1) System.out.println(sum);
        });

        long llTime = timeMs(() -> {
            long sum = 0;
            for (int v : ll) sum += v;
            if (sum == -1) System.out.println(sum);
        });

        System.out.println("ArrayList iterate ms = " + alTime);
        System.out.println("LinkedList iterate ms = " + llTime);
    }

    static void benchmarkRandomAccess() {
        int n = 100_000;
        List<Integer> al = IntStream.range(0, n).boxed().collect(Collectors.toCollection(ArrayList::new));
        List<Integer> ll = IntStream.range(0, n).boxed().collect(Collectors.toCollection(LinkedList::new));

        long alTime = timeMs(() -> {
            long sum = 0;
            for (int i = 0; i < n; i++) sum += al.get(i);
            if (sum == -1) System.out.println(sum);
        });

        long llTime = timeMs(() -> {
            long sum = 0;
            for (int i = 0; i < n; i++) sum += ll.get(i); // O(n^2)
            if (sum == -1) System.out.println(sum);
        });

        System.out.println("ArrayList get(i) ms = " + alTime);
        System.out.println("LinkedList get(i) ms = " + llTime);
    }

    static void benchmarkMapContention() throws InterruptedException {
        int threads = 8;
        int ops = 200_000;

        Map<Integer, Integer> syncMap = Collections.synchronizedMap(new HashMap<>());
        ConcurrentHashMap<Integer, Integer> chm = new ConcurrentHashMap<>();

        ExecutorService pool = Executors.newFixedThreadPool(threads);

        long syncTime = timeMs(() -> {
            CountDownLatch latch = new CountDownLatch(threads);
            for (int t = 0; t < threads; t++) {
                pool.submit(() -> {
                    for (int i = 0; i < ops; i++) {
                        syncMap.merge(i % 10_000, 1, Integer::sum);
                    }
                    latch.countDown();
                });
            }
            try { latch.await(); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
        });

        long chmTime = timeMs(() -> {
            CountDownLatch latch = new CountDownLatch(threads);
            for (int t = 0; t < threads; t++) {
                pool.submit(() -> {
                    for (int i = 0; i < ops; i++) {
                        chm.merge(i % 10_000, 1, Integer::sum);
                    }
                    latch.countDown();
                });
            }
            try { latch.await(); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
        });

        pool.shutdown();
        System.out.println("synchronizedMap merge ms = " + syncTime);
        System.out.println("ConcurrentHashMap merge ms = " + chmTime);
    }

    public static void main(String[] args) throws InterruptedException {
        benchmarkIteration();
        benchmarkRandomAccess();
        benchmarkMapContention();
    }
}
```

## REAL_WORLD

### Practical Selection Rules in Backend Systems

1. API response assembly where order matters: use `LinkedHashMap`
2. hot in-memory cache with sorted key ranges: use `TreeMap` + careful synchronization or `ConcurrentSkipListMap`
3. metrics counters under high contention: `ConcurrentHashMap<String, LongAdder>`
4. read-mostly listeners: `CopyOnWriteArrayList`
5. queue/stack operations: `ArrayDeque`, not `LinkedList`

### Example: Event Processor Registry

```java
class EventProcessorRegistry {
    private final Map<String, EventProcessor> byType = new ConcurrentHashMap<>();
    private final List<EventProcessor> ordered = new CopyOnWriteArrayList<>();

    void register(EventProcessor p) {
        byType.put(p.type(), p);
        ordered.add(p);
    }

    EventProcessor get(String type) {
        return byType.get(type);
    }

    void broadcast(Event e) {
        for (EventProcessor p : ordered) {
            p.handle(e);
        }
    }

    interface EventProcessor {
        String type();
        void handle(Event e);
    }
    record Event(String type, String payload) {}
}
```

## INTERVIEW

### Q1: Why is `Map` not part of `Collection` hierarchy?

Weak: Because it stores key-value pairs.

Strong: `Collection<E>` represents a group of elements of one type with operations like `add(E)` and `contains(E)`. `Map<K,V>` is fundamentally different: values are accessed by key and entries are pairs, not single elements. Forcing `Map` into `Collection` would break type and semantic contracts.

---

### Q2: How do fail-fast iterators work internally?

Weak: They throw CME if collection changes.

Strong: Iterators capture `expectedModCount` when created. Structural modifications (`add`, `remove`, resize, clear) increment collection `modCount`. Each iterator operation validates `modCount == expectedModCount`; mismatch throws `ConcurrentModificationException`. This is best-effort bug detection, not thread safety.

---

### Q3: When do you choose `LinkedHashMap` over `HashMap` and `TreeMap`?

Weak: When order matters.

Strong: `LinkedHashMap` keeps insertion (or access) order with near-`HashMap` complexity O(1) avg. Choose it when deterministic iteration order is needed (JSON output, LRU cache, preserving DB result order). Use `TreeMap` only if you need sorted order and range queries (`ceilingKey`, `subMap`), accepting O(log n) operations.

---

### Q4: Why is `CopyOnWriteArrayList` great for reads but bad for writes?

Weak: Because writes copy array.

Strong: Every write operation creates a new backing array and copies all elements (O(n) + allocation). Reads are lock-free and iterate over stable snapshots with no CME. So it excels when read/write ratio is very high (e.g., 10,000:1). It is terrible for frequent writes due to copy overhead and GC churn.

---

### Q5: `ConcurrentHashMap` vs `Collections.synchronizedMap(new HashMap<>())`?

Weak: CHM is faster.

Strong: `synchronizedMap` uses one global monitor lock for all operations, serializing all threads. `ConcurrentHashMap` uses lock-striping/CAS and non-blocking reads, allowing high parallel throughput. Iterators in synchronizedMap must be externally synchronized and are fail-fast; CHM iterators are weakly consistent and do not throw CME.

## FEYNMAN CHECK

Collections are like storage furniture:

- `ArrayList` = drawer with numbered slots (fast by index)
- `LinkedList` = chain of boxes tied by strings (slow random access)
- `HashMap` = locker room with hashed locker numbers (fast lookup, messy order)
- `LinkedHashMap` = locker room + walking order list
- `TreeMap` = alphabetically sorted filing cabinet
- `ConcurrentHashMap` = multi-lane locker room where many people can use different lanes safely

Choosing the wrong furniture means wasting time every time you fetch something.
