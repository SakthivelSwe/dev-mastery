## WHY

Sorting is ubiquitous in programming. Understanding `Comparable` and `Comparator` is essential for:

1. **Collections.sort() and Arrays.sort()** — Only work when objects implement Comparable or you provide a Comparator
2. **TreeMap and TreeSet** — Keep elements sorted; need comparison logic
3. **PriorityQueue** — Orders elements by priority using Comparator
4. **Stream.sorted()** — Sort stream elements
5. **Interview** — "How do you sort a list of custom objects?" is asked constantly

---

## THEORY

### Comparable — Natural Ordering

`Comparable<T>` defines the **natural ordering** of a class:

```java
public class Student implements Comparable<Student> {
    String name;
    double gpa;

    @Override
    public int compareTo(Student other) {
        return Double.compare(this.gpa, other.gpa); // ascending GPA
        // Negative: this < other (this comes first)
        // Zero:     this == other
        // Positive: this > other (other comes first)
    }
}

// Now Student can be sorted naturally
List<Student> students = ...;
Collections.sort(students); // uses compareTo()
Student min = Collections.min(students); // uses compareTo()
TreeSet<Student> sorted = new TreeSet<>(students); // uses compareTo()
```

### Comparator — Custom Ordering

`Comparator<T>` defines an **external** comparison strategy (doesn't modify the class):

```java
// Sort students by name alphabetically
Comparator<Student> byName = (a, b) -> a.name.compareTo(b.name);

// Sort by GPA descending
Comparator<Student> byGpaDesc = (a, b) -> Double.compare(b.gpa, a.gpa);

// Use it
students.sort(byName);
students.sort(byGpaDesc);
```

### compareTo Contract

| Return | Meaning |
|--------|---------|
| `negative` | `this` is less than `other` → `this` comes before `other` |
| `0` | `this` equals `other` → no order preference |
| `positive` | `this` is greater than `other` → `this` comes after `other` |

**The contract must satisfy:**
1. **Antisymmetry**: if `a.compareTo(b) > 0` then `b.compareTo(a) < 0`
2. **Transitivity**: if `a > b && b > c` then `a > c`
3. **Consistency with equals** (recommended): `a.compareTo(b) == 0` ↔ `a.equals(b)`

### Common Mistakes in compareTo

```java
// WRONG — integer overflow with subtraction!
// If a.age = Integer.MIN_VALUE and b.age = 1, a.age - b.age overflows
return a.age - b.age;

// CORRECT — use Integer.compare()
return Integer.compare(a.age, b.age);

// WRONG for floats/doubles — NaN issues
return a.gpa > b.gpa ? 1 : a.gpa < b.gpa ? -1 : 0;

// CORRECT
return Double.compare(a.gpa, b.gpa);

// CORRECT for strings
return a.name.compareTo(b.name);
```

### Comparator Factory Methods (Java 8+)

```java
// Comparator.comparing — extract key and compare
Comparator<Student> byName = Comparator.comparing(s -> s.name);
// or: Comparator.comparing(Student::getName)

// comparingInt, comparingDouble, comparingLong — avoid boxing
Comparator<Student> byAge = Comparator.comparingInt(Student::getAge);
Comparator<Student> byGpa = Comparator.comparingDouble(Student::getGpa);

// reversed() — flip order
Comparator<Student> byGpaDesc = Comparator.comparingDouble(Student::getGpa).reversed();

// thenComparing — secondary sort
Comparator<Student> complex = Comparator
    .comparingDouble(Student::getGpa).reversed()  // primary: GPA desc
    .thenComparing(Student::getName);              // secondary: name asc

// naturalOrder(), reverseOrder() — for already-Comparable types
Comparator<String> natural = Comparator.naturalOrder();
Comparator<String> reversed = Comparator.reverseOrder();

// nullsFirst, nullsLast
Comparator<String> nullsFirst = Comparator.nullsFirst(Comparator.naturalOrder());
```

### Comparable vs Comparator — When to Use Which

| | Comparable | Comparator |
|--|-----------|------------|
| **Defined** | Inside the class | Outside the class |
| **Orderings** | One (natural) | Multiple (strategy) |
| **Modifying class** | Yes (implement interface) | No (external) |
| **Use when** | There's a clear natural order | Need flexible/multiple orderings |
| **Examples** | String, Integer, Date | Sort by any field, multi-level sort |

## CODE

### Level 1 — Beginner: Implementing Comparable

```java
// ComparableDemo.java
import java.util.*;

public class ComparableDemo {

    static class Product implements Comparable<Product> {
        String name;
        double price;
        int stock;

        Product(String name, double price, int stock) {
            this.name = name; this.price = price; this.stock = stock;
        }

        // Natural order: by price ascending
        @Override
        public int compareTo(Product other) {
            return Double.compare(this.price, other.price);
        }

        @Override public String toString() {
            return name + "($" + price + ")";
        }
    }

    public static void main(String[] args) {
        List<Product> products = new ArrayList<>(List.of(
            new Product("Laptop", 999.99, 5),
            new Product("Mouse", 25.00, 100),
            new Product("Keyboard", 75.00, 50),
            new Product("Monitor", 399.00, 20),
            new Product("Headset", 150.00, 30)
        ));

        // Natural ordering (uses compareTo)
        Collections.sort(products);
        System.out.println("By price (natural): " + products);
        // [Mouse($25.0), Keyboard($75.0), Headset($150.0), Monitor($399.0), Laptop($999.99)]

        System.out.println("Min: " + Collections.min(products)); // Mouse($25.0)
        System.out.println("Max: " + Collections.max(products)); // Laptop($999.99)

        // TreeSet uses natural ordering
        TreeSet<Product> sortedSet = new TreeSet<>(products);
        System.out.println("TreeSet: " + sortedSet);
        // Sorted by price, no duplicates

        // binarySearch requires sorted list
        Collections.sort(products);
        int idx = Collections.binarySearch(products, new Product("Headset", 150.00, 0));
        System.out.println("Found at index: " + idx); // 2
    }
}
```

### Level 2 — Elementary: Comparator Chaining

```java
// ComparatorChaining.java
import java.util.*;
import java.util.stream.*;

public class ComparatorChaining {

    record Employee(String name, String dept, double salary, int experience) {}

    public static void main(String[] args) {
        List<Employee> employees = List.of(
            new Employee("Alice", "Engineering", 120_000, 5),
            new Employee("Bob", "Marketing", 80_000, 8),
            new Employee("Charlie", "Engineering", 110_000, 3),
            new Employee("Dave", "Marketing", 95_000, 5),
            new Employee("Eve", "Engineering", 120_000, 7)
        );

        // 1. Sort by department, then salary desc, then name
        Comparator<Employee> complex = Comparator
            .comparing(Employee::dept)
            .thenComparing(Comparator.comparingDouble(Employee::salary).reversed())
            .thenComparing(Employee::name);

        employees.stream()
            .sorted(complex)
            .forEach(e -> System.out.printf("%-15s %-12s $%-10.0f %dy%n",
                e.name(), e.dept(), e.salary(), e.experience()));
        // Engineering sorted: Alice, Eve (same salary, A before E), Charlie
        // Marketing sorted: Dave (95k), Bob (80k)

        System.out.println();

        // 2. Using Comparator.comparing with key extractor
        Comparator<Employee> byExpDesc = Comparator
            .comparingInt(Employee::experience)
            .reversed();

        System.out.println("Most experienced:");
        employees.stream()
            .sorted(byExpDesc)
            .limit(3)
            .forEach(e -> System.out.println("  " + e.name() + " (" + e.experience() + "y)"));

        // 3. Null-safe comparator
        List<String> withNulls = new ArrayList<>(Arrays.asList("banana", null, "apple", null, "cherry"));
        withNulls.sort(Comparator.nullsLast(Comparator.naturalOrder()));
        System.out.println("\nWith nulls last: " + withNulls);
        // [apple, banana, cherry, null, null]

        withNulls.sort(Comparator.nullsFirst(Comparator.naturalOrder()));
        System.out.println("With nulls first: " + withNulls);
        // [null, null, apple, banana, cherry]

        // 4. reverseOrder shorthand
        List<Integer> nums = Arrays.asList(5, 2, 8, 1, 9, 3);
        nums.sort(Comparator.reverseOrder());
        System.out.println("\nReverse sorted: " + nums); // [9, 8, 5, 3, 2, 1]
    }
}
```

### Level 3 — Intermediate: Multi-Level Sort, Stability, PriorityQueue

```java
// SortingPatterns.java
import java.util.*;
import java.util.stream.*;

public class SortingPatterns {

    record Task(String id, int priority, long deadline, String owner) {}

    public static void main(String[] args) {
        List<Task> tasks = new ArrayList<>(List.of(
            new Task("T1", 3, 1000L, "Alice"),
            new Task("T2", 1, 2000L, "Bob"),
            new Task("T3", 2, 1500L, "Alice"),
            new Task("T4", 1, 1000L, "Charlie"),
            new Task("T5", 2, 1500L, "Bob")
        ));

        // 1. Stable sort — Java's sort is stable (equal elements keep original order)
        // Sort by owner first, then by priority
        Comparator<Task> byOwnerThenPriority = Comparator
            .comparing(Task::owner)
            .thenComparingInt(Task::priority);
        tasks.sort(byOwnerThenPriority);
        System.out.println("By owner then priority:");
        tasks.forEach(t -> System.out.printf("  %-10s owner=%-10s priority=%d%n",
            t.id(), t.owner(), t.priority()));

        // 2. PriorityQueue — always returns highest priority first
        // Lower number = higher priority (min-heap behavior)
        PriorityQueue<Task> pq = new PriorityQueue<>(
            Comparator.comparingInt(Task::priority)
                      .thenComparingLong(Task::deadline));

        pq.addAll(tasks);
        System.out.println("\nProcessing by priority:");
        while (!pq.isEmpty()) {
            Task t = pq.poll(); // removes and returns highest priority (lowest number)
            System.out.printf("  Processing %s (priority=%d, deadline=%d)%n",
                t.id(), t.priority(), t.deadline());
        }

        // 3. Custom comparator for version strings "1.2.3"
        List<String> versions = new ArrayList<>(List.of("1.10.0","1.9.0","2.0.0","1.0.1","1.9.5"));
        versions.sort((a, b) -> {
            int[] aParts = Arrays.stream(a.split("\\.")).mapToInt(Integer::parseInt).toArray();
            int[] bParts = Arrays.stream(b.split("\\.")).mapToInt(Integer::parseInt).toArray();
            for (int i = 0; i < Math.min(aParts.length, bParts.length); i++) {
                if (aParts[i] != bParts[i]) return Integer.compare(aParts[i], bParts[i]);
            }
            return Integer.compare(aParts.length, bParts.length);
        });
        System.out.println("\nVersion sort: " + versions);
        // [1.0.1, 1.9.0, 1.9.5, 1.10.0, 2.0.0]
        // Note: "1.10.0" comes AFTER "1.9.5" (numeric, not lexicographic)
    }
}
```

### Level 4 — Advanced: TreeMap with Custom Comparator

```java
// TreeMapComparators.java
import java.util.*;

public class TreeMapComparators {

    public static void main(String[] args) {
        // TreeMap default: natural order (String alphabetical)
        TreeMap<String, Integer> natural = new TreeMap<>();
        natural.put("banana", 2);
        natural.put("apple", 1);
        natural.put("cherry", 3);
        System.out.println("Natural: " + natural);
        // {apple=1, banana=2, cherry=3}

        // TreeMap with reverse order
        TreeMap<String, Integer> reversed = new TreeMap<>(Comparator.reverseOrder());
        reversed.putAll(natural);
        System.out.println("Reversed: " + reversed);
        // {cherry=3, banana=2, apple=1}

        // TreeMap with case-insensitive comparison
        TreeMap<String, Integer> caseInsensitive = new TreeMap<>(String.CASE_INSENSITIVE_ORDER);
        caseInsensitive.put("JAVA", 1);
        caseInsensitive.put("java", 2); // treated as same key!
        System.out.println("Case-insensitive size: " + caseInsensitive.size()); // 1
        System.out.println("Value: " + caseInsensitive.get("Java")); // 2 (overwritten)

        // TreeMap for frequency map sorted by count (not key)
        String text = "the quick brown fox jumps over the lazy dog the fox";
        Map<String, Integer> freq = new HashMap<>();
        for (String w : text.split(" ")) freq.merge(w, 1, Integer::sum);

        // Sort by frequency descending, then alphabetically
        TreeMap<String, Integer> sortedByFreq = new TreeMap<>(
            Comparator.<String, Integer>comparing(freq::get).reversed()
                       .thenComparing(Comparator.naturalOrder()));
        sortedByFreq.putAll(freq);
        System.out.println("\nWord frequencies (by count desc):");
        sortedByFreq.forEach((word, count) ->
            System.out.printf("  %-10s: %d%n", word, count));

        // NavigableMap operations on TreeMap
        TreeMap<Integer, String> scores = new TreeMap<>();
        scores.put(90, "A"); scores.put(80, "B"); scores.put(70, "C");
        scores.put(60, "D"); scores.put(50, "F");

        System.out.println("\nfloorKey(85): " + scores.floorKey(85));   // 80
        System.out.println("ceilingKey(75): " + scores.ceilingKey(75)); // 80
        System.out.println("subMap(60,90): " + scores.subMap(60, true, 90, false)); // {60=D, 70=C, 80=B}
        System.out.println("headMap(80): " + scores.headMap(80));       // {50=F, 60=D, 70=C}
        System.out.println("tailMap(80): " + scores.tailMap(80));       // {80=B, 90=A}
    }
}
```

### Level 5 — Expert: Implementing Comparable with Complex Tie-Breaking

```java
// ComplexComparable.java
import java.util.*;

public class ComplexComparable {

    // Multi-field Comparable with proper equals/hashCode
    static final class SemanticVersion implements Comparable<SemanticVersion> {
        final int major, minor, patch;
        final String preRelease; // null means release; "alpha.1", "beta.2"
        final String build;      // build metadata, ignored for comparison

        SemanticVersion(int major, int minor, int patch, String preRelease, String build) {
            this.major = major; this.minor = minor; this.patch = patch;
            this.preRelease = preRelease; this.build = build;
        }

        static SemanticVersion parse(String version) {
            String[] parts = version.split("-", 2);
            String[] buildParts = parts[0].split("\\+", 2);
            String[] nums = buildParts[0].split("\\.");
            return new SemanticVersion(
                Integer.parseInt(nums[0]),
                Integer.parseInt(nums[1]),
                Integer.parseInt(nums[2]),
                parts.length > 1 ? parts[1] : null,
                buildParts.length > 1 ? buildParts[1] : null
            );
        }

        @Override
        public int compareTo(SemanticVersion other) {
            // Compare major.minor.patch
            int cmp = Integer.compare(major, other.major);
            if (cmp != 0) return cmp;
            cmp = Integer.compare(minor, other.minor);
            if (cmp != 0) return cmp;
            cmp = Integer.compare(patch, other.patch);
            if (cmp != 0) return cmp;

            // Pre-release: release > pre-release
            // "1.0.0" > "1.0.0-alpha"
            if (preRelease == null && other.preRelease == null) return 0;
            if (preRelease == null) return 1;   // this is release, other is pre-release
            if (other.preRelease == null) return -1; // other is release

            // Both have pre-release — compare identifiers
            String[] ids = preRelease.split("\\.");
            String[] otherIds = other.preRelease.split("\\.");
            for (int i = 0; i < Math.min(ids.length, otherIds.length); i++) {
                String id = ids[i], otherId = otherIds[i];
                boolean numA = id.matches("\\d+"), numB = otherId.matches("\\d+");
                if (numA && numB) {
                    cmp = Integer.compare(Integer.parseInt(id), Integer.parseInt(otherId));
                } else if (numA) {
                    cmp = -1; // numeric < alphanumeric
                } else if (numB) {
                    cmp = 1;
                } else {
                    cmp = id.compareTo(otherId); // both alphanumeric
                }
                if (cmp != 0) return cmp;
            }
            return Integer.compare(ids.length, otherIds.length);
        }

        @Override public boolean equals(Object o) {
            if (!(o instanceof SemanticVersion)) return false;
            return compareTo((SemanticVersion) o) == 0;
        }

        @Override public int hashCode() {
            return Objects.hash(major, minor, patch, preRelease);
        }

        @Override public String toString() {
            return major + "." + minor + "." + patch +
                (preRelease != null ? "-" + preRelease : "") +
                (build != null ? "+" + build : "");
        }
    }

    public static void main(String[] args) {
        List<SemanticVersion> versions = new ArrayList<>(List.of(
            SemanticVersion.parse("1.0.0"),
            SemanticVersion.parse("1.0.0-alpha"),
            SemanticVersion.parse("1.0.0-alpha.1"),
            SemanticVersion.parse("1.0.0-beta.2"),
            SemanticVersion.parse("1.0.0-rc.1"),
            SemanticVersion.parse("2.0.0"),
            SemanticVersion.parse("1.9.9")
        ));

        Collections.sort(versions);
        System.out.println("Sorted semantic versions:");
        versions.forEach(v -> System.out.println("  " + v));
        // 1.0.0-alpha
        // 1.0.0-alpha.1
        // 1.0.0-beta.2
        // 1.0.0-rc.1
        // 1.0.0     ← release > pre-release
        // 1.9.9
        // 2.0.0
    }
}
```

## INTERVIEW

**Q1: What is the difference between Comparable and Comparator?**

> **Strong answer:** `Comparable` is implemented by the class itself — it defines the natural ordering. One class, one ordering. `Comparator` is external — you define comparison logic outside the class. Benefits: (1) You can have multiple orderings without modifying the class. (2) You can sort third-party classes you can't modify. Rule of thumb: if there's a clear, obvious "natural" order (Integer by value, String alphabetically), implement `Comparable`. Otherwise, use `Comparator`. You can use both — `TreeSet<String>` uses String's `Comparable`, but you can override with a custom `Comparator` via the constructor.

**Q2: What is wrong with `return a.age - b.age` in compareTo?**

> **Strong answer:** Integer subtraction can overflow! If `a.age = Integer.MIN_VALUE` (-2,147,483,648) and `b.age = 1`, then `a.age - b.age` overflows to a large positive number, making `a` appear greater than `b` — wrong result. Always use `Integer.compare(a.age, b.age)` which correctly returns negative/zero/positive without overflow. Same for `long` — use `Long.compare()`. For doubles, use `Double.compare()` which also handles NaN correctly.

**Q3: How does Comparator.comparing() work?**

> **Strong answer:** `Comparator.comparing(keyExtractor)` creates a Comparator that extracts a sort key from each object and compares by natural ordering of that key. `Comparator.comparing(Student::getGpa)` is equivalent to `(a, b) -> Double.compare(a.getGpa(), b.getGpa())`. You can chain with `.thenComparing()` for multi-level sort, `.reversed()` to flip order, `.thenComparingInt()`, `.thenComparingDouble()` for primitive keys (avoids boxing). The result is a composable, readable Comparator without any boilerplate.

## FEYNMAN CHECK

`Comparable` is like asking someone "how do you naturally compare to others?" — the answer is built into the person. `Comparator` is like a judge at a competition who decides the ranking by specific criteria — the comparison logic is outside the competitors. You need `Comparable` when there's one obvious ordering (students by ID). You need `Comparator` when different contexts require different orderings (sometimes sort by name, sometimes by GPA, sometimes by enrollment date).
