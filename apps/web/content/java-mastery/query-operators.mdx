Title: Query Operators — Comparison, Logical, Array | Level: 2

### Query Optimization and Array Processing Mechanics

MongoDB queries are processed by the Query Optimizer, which evaluates structural features of the query (such as operators, paths, and array nesting) and builds a query plan.

```
       [Incoming Query]
              │
              ▼
    [Query Optimizer] ──(Checks Cache)
              │
      ┌───────┴───────┐
 (Cache Hit)     (Cache Miss)
      │               │
      │               ▼
      │        [Generate Candidate Plans]
      │               │
      │        [Test Plans (IXSCAN vs COLLSCAN)]
      │               │
      │        [Cache Winning Plan]
      ▼               ▼
        [Execute Query Plan]
```

#### Dot Notation and Nested Document Resolution
When querying nested documents (e.g., `"address.city": "Chennai"`), the engine evaluates the path by traversing the BSON hierarchy without parsing the entire document structure, provided there is an index matching the path. If no index exists, a full collection scan (`COLLSCAN`) occurs, parsing every document up to the matched path.

#### Array Query Nuances
Array queries are highly dynamic. When querying an array field without explicit operators (e.g., `tags: "tech"`), MongoDB matches if *any* element in the array is `"tech"`. When using `$elemMatch`, MongoDB evaluates multiple criteria against a *single* element inside the array, preventing cross-element false positives.

### Comprehensive Query Examples using `mongosh`

Let's query a `users` collection.

#### 1. Comparison Operators
```javascript
// Find users where age is between 25 and 40 (inclusive) and status is in ACTIVE or PENDING
db.users.find({
  age: { $gte: 25, $lte: 40 },
  status: { $in: ["ACTIVE", "PENDING"] }
});

// Find users where external ID is not equal to null and not in a specific blacklisted set
db.users.find({
  external_id: { $ne: null, $nin: ["EX-000", "EX-999"] }
});
```

#### 2. Logical Operators
```javascript
// Find users matching ((age >= 18 OR role == 'ADMIN') AND status != 'SUSPENDED')
db.users.find({
  $and: [
    {
      $or: [
        { age: { $gte: 18 } },
        { role: "ADMIN" }
      ]
    },
    { status: { $ne: "SUSPENDED" } }
  ]
});

// Use $nor to find users who do not have status 'ACTIVE' and do not have role 'USER'
db.users.find({
  $nor: [
    { status: "ACTIVE" },
    { role: "USER" }
  ]
});
```

#### 3. Element Operators
```javascript
// Find documents where the 'middle_name' field exists and is of type 'string' (BSON type 2)
db.users.find({
  middle_name: { $exists: true, $type: "string" }
});
```

#### 4. Array Operators
```javascript
// $all: Matches arrays that contain all specified elements, regardless of order
db.users.find({ tags: { $all: ["developer", "speaker"] } });

// $size: Matches arrays with exactly the specified length
db.users.find({ tags: { $size: 3 } });

// $elemMatch: Matches if at least one array element satisfies all conditions
db.users.find({
  addresses: {
    $elemMatch: {
      city: "Chennai",
      type: "WORK"
    }
  }
});
```

#### 5. Dot Notation for Nested Documents
```javascript
// Querying nested object directly
db.users.find({ "metadata.preferences.theme": "dark" });

// Querying element in an array of objects by index
db.users.find({ "addresses.0.zipcode": "600001" });
```

#### 6. Regular Expressions
```javascript
// Case-insensitive regex search anchoring to start of string
db.users.find({ email: { $regex: "^admin@", $options: "i" } });
```

### Spring Data MongoDB Equivalent

We use the `Criteria` API to express these complex logical, element, and array queries.

#### Spring Query Service (`UserQueryRepository.java`)
```java
package com.devmastery.mongodb.repository;

import com.devmastery.mongodb.model.User;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.mongodb.core.MongoTemplate;
import org.springframework.data.mongodb.core.query.Criteria;
import org.springframework.data.mongodb.core.query.Query;
import org.springframework.stereotype.Repository;

import java.util.List;

@Repository
public class UserQueryRepository {

    private final MongoTemplate mongoTemplate;

    @Autowired
    public UserQueryRepository(MongoTemplate mongoTemplate) {
        this.mongoTemplate = mongoTemplate;
    }

    public List<User> findActiveAdultsOrAdmins(int minAge) {
        Query query = new Query();

        // Complex logical construct: (age >= minAge OR role == 'ADMIN') AND status != 'SUSPENDED'
        query.addCriteria(
            new Criteria().andOperator(
                new Criteria().orOperator(
                    Criteria.where("age").gte(minAge),
                    Criteria.where("role").is("ADMIN")
                ),
                Criteria.where("status").ne("SUSPENDED")
            )
        );

        return mongoTemplate.find(query, User.class);
    }
