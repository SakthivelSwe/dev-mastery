## WHY

At its core, programming is manipulating data. You take inputs, evaluate them through logic, calculate new values, and produce outputs. Without operators, a programming language would be completely static. 

However, mathematical manipulation in computers is not as simple as high-school algebra. If you add two extremely large positive numbers together in a computer, the result might suddenly become a massive negative number! If you check whether a complex file stream is ready before checking if the file actually exists, your program will crash with a Null Pointer Exception. 

Operators in Java define the precise rules of computation, logical evaluation, and byte-level bit manipulation. Understanding these operators deeply—specifically concepts like short-circuit evaluation, integer promotion, and operator precedence—is the difference between code that works purely by accident and code that is provably correct under all edge cases.

## THEORY

An **Expression** in Java is any combination of variables, literals, method calls, and operators that evaluates to a single value.

### Arithmetic and Integer Promotion
Java supports standard arithmetic (`+`, `-`, `*`, `/`, `%`). 
**The Trap**: When performing arithmetic on smaller types (`byte`, `short`, `char`), Java automatically applies **Integer Promotion**. It converts them to `int` before performing the calculation.
```java
byte b1 = 10;
byte b2 = 20;
// byte sum = b1 + b2; // COMPILER ERROR! b1+b2 evaluates to an 'int'
byte sum = (byte) (b1 + b2); // Requires explicit cast back to byte
```

### Logical and Short-Circuit Operators
Java has bitwise logical operators (`&`, `|`) and short-circuit logical operators (`&&`, `||`).
- `A && B` (Short-Circuit AND): If `A` is `false`, Java completely skips evaluating `B` because the result is guaranteed to be `false`. This is extremely useful for null-checks.
- `A & B` (Bitwise AND): Evaluates BOTH `A` and `B`, regardless of the value of `A`.

### Bitwise and Shift Operators
These operators manipulate numbers directly at the binary level.
- `>>` (Signed Right Shift): Shifts bits to the right. It fills the new leftmost bits with the *sign bit* (preserves whether the number is negative or positive). Effectively divides by 2.
- `>>>` (Unsigned Right Shift): Shifts bits to the right, but always fills the new leftmost bits with `0`, regardless of the sign. Unique to Java because Java doesn't have unsigned numeric types.
- `<<` (Left Shift): Shifts bits to the left, filling the right with `0`. Effectively multiplies by 2.

### Compound Assignment Hidden Casts
Operators like `+=`, `-=`, `*=` are not just syntax sugar. They include a **hidden cast**.
```java
short s = 10;
s = s + 5; // ERROR: s + 5 is an int
s += 5;    // SUCCESS: compiles to s = (short)(s + 5)
```

## CODE

### Level 1 — Beginner
```java
public class BasicOperators {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;
        
        // Integer division truncates the decimal!
        System.out.println(a / b); // Prints 3, NOT 3.33
        
        // Modulo gets the remainder
        System.out.println(a % b); // Prints 1
        
        // Post-increment (evaluates, THEN increments)
        int x = 5;
        System.out.println(x++); // Prints 5
        System.out.println(x);   // Prints 6
    }
}
```

### Level 2 — Building Blocks
```java
public class ShortCircuit {
    public static void main(String[] args) {
        String text = null;
        
        // The JVM evaluates left-to-right.
        // text != null is FALSE. 
        // Because it's &&, the JVM short-circuits and skips text.length() > 0.
        // This prevents a NullPointerException!
        if (text != null && text.length() > 0) {
            System.out.println("Valid text");
        }
    }
}
```

### Level 3 — Intermediate
```java
public class BitMasking {
    public static void main(String[] args) {
        // Flags stored as bits in a single byte:
        // 00000001 = READ (1)
        // 00000010 = WRITE (2)
        // 00000100 = EXECUTE (4)
        
        byte READ = 1;
        byte WRITE = 2;
        
        // Combine permissions using Bitwise OR
        byte myPermissions = (byte) (READ | WRITE); // 00000011 (3)
        
        // Check permissions using Bitwise AND
        boolean canWrite = (myPermissions & WRITE) == WRITE;
        System.out.println("Can write? " + canWrite); // true
    }
}
```

### Level 4 — Advanced
```java
public class OverflowTrap {
    public static void main(String[] args) {
        int max = Integer.MAX_VALUE; // 2,147,483,647 (011111...1)
        
        // Adding 1 flips the sign bit!
        int overflow = max + 1;
        
        System.out.println(overflow); // Prints -2147483648
        
        // To safely calculate without silent overflow, use Math.addExact
        try {
            Math.addExact(max, 1);
        } catch (ArithmeticException e) {
            System.out.println("Safely caught the overflow!");
        }
    }
}
```

## REAL_WORLD

In heavily optimized Java libraries like `java.util.concurrent.ConcurrentHashMap` or hashing algorithms, you will rarely see division (`/`) or modulo (`%`) operators used in hot paths. 

Why? Because CPU instructions for division are notoriously slow compared to bitwise operators. 

If you look at the source code of `HashMap` in the JDK, you will see how it calculates the array bucket index for a given key. Instead of doing `hash % array.length`, it does this:
```java
// REAL JDK HashMap Source Code (simplified)
int index = (n - 1) & hash;
```
Because `HashMap` heavily enforces that the array size `n` is always a power of 2, `(n - 1)` creates a bitmask (e.g., `16 - 1 = 15`, which is `00001111` in binary). Doing a bitwise `&` with the hash is mathematically identical to `hash % n`, but executes drastically faster at the CPU level.

Another common real-world issue is the **Ternary Operator NPE trap**.
```java
Integer a = null;
int b = 2;
// If the condition is true, Java attempts to unbox 'a' to match 'b's primitive type!
// This throws a NullPointerException silently on the line below.
int result = (true) ? a : b; 
```
When using the ternary operator `? :`, if the second and third operands are a mix of primitives and wrappers, Java will automatically attempt to unbox the wrapper to match the primitive. If the wrapper is null, it crashes.

## INTERVIEW

**Q: What is the difference between `>>` and `>>>` in Java?**
A: Both shift bits to the right. The signed right shift `>>` preserves the sign bit (the leftmost bit), meaning negative numbers stay negative. The unsigned right shift `>>>` ignores the sign and always fills the new leftmost bits with `0`, making the result positive. Java added `>>>` because unlike C++, Java does not have unsigned integer data types.

**Q: Explain Short-Circuit evaluation and why it is critical.**
A: Short-circuit operators (`&&`, `||`) stop evaluating expressions as soon as the outcome is determined. In `A && B`, if `A` is false, `B` is never executed. This is critical for preventing runtime exceptions (like checking if an object is null before calling a method on it) and avoiding expensive, unnecessary method calls.

**Q: Will the following code compile? `byte b = 5; b = b + 2;`**
A: No, it will cause a compiler error. Due to Integer Promotion, the expression `b + 2` is automatically promoted to an `int` during calculation. You cannot assign an `int` back to a `byte` variable without an explicit cast `b = (byte)(b + 2)`. However, `b += 2` would compile because compound assignment operators include a hidden cast.

**Q: What happens if an integer overflows in Java? Does it throw an exception?**
A: No, standard arithmetic operators gracefully overflow and underflow silently. Adding 1 to `Integer.MAX_VALUE` will wrap around to `Integer.MIN_VALUE` due to Two's Complement binary representation. If you want an exception to be thrown on overflow, you must use the `Math.addExact()` utility methods introduced in Java 8.
