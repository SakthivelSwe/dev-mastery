## WHY
Before Java 5, collections like `ArrayList` only held `Object` references. If you had a list of Strings and another developer added an `Integer` to it, the compiler said nothing. Your application crashed at runtime with `ClassCastException` — often in production, far from the original bug.

Generics shift these errors from **Runtime** (crash in production) to **Compile-Time** (build fails). By declaring `ArrayList<String>`, you contractually tell the compiler: "Only Strings go in here." Adding an `Integer` becomes a build failure, not a 3am pager alert.

Why this matters beyond collections:
1. **Type-safe APIs**: `Optional<User>` tells callers exactly what they'll receive
2. **Code reuse without duplication**: One `Repository<T, ID>` works for ALL entity types
3. **Expressive algorithms**: Sort `List<T extends Comparable<T>>` without knowing T's actual type
4. **Null safety**: `Optional<T>` explicitly communicates that a value may be absent

## THEORY

### How Generics Work: Type Parameters
A type parameter is a placeholder for a real type, written in angle brackets `<T>`:
```java
class Box<T> {        // T is the type parameter (could be anything)
    private T item;   // 'item' will be whatever type the caller specifies
}

Box<String>  sBox = new Box<>();  // T = String
Box<Integer> iBox = new Box<>();  // T = Integer
Box<User>    uBox = new Box<>();  // T = User
```

Common type parameter naming conventions:
| Letter | Meaning | Example |
|--------|---------|---------|
| `T` | Type | `Box<T>` |
| `E` | Element (collections) | `List<E>` |
| `K`, `V` | Key, Value | `Map<K, V>` |
| `N` | Number | `<N extends Number>` |
| `R` | Return type | `Function<T, R>` |

### Type Erasure — The Critical JVM Detail
This is the most important thing to understand about Java generics. At **compile time**, the compiler enforces all `<T>` rules. But during compilation to bytecode, the compiler **erases** all generic type information:

```java
// Source code (what you write):
List<String> names = new ArrayList<>();
names.add("Alice");
String s = names.get(0);

// Bytecode equivalent (what the JVM actually runs):
List names = new ArrayList();       // Raw List — no type info!
names.add("Alice");                  // No type check
String s = (String) names.get(0);   // Compiler inserts cast automatically
```

**Why Type Erasure?** Backward compatibility with Java 1.4. Generic class files must run on JVMs that knew nothing about generics.

**Consequences:**
- `list instanceof List<String>` → compilation error (type info erased at runtime)
- `new T[10]` → compilation error (JVM doesn't know what `T` is at runtime)
- `T.class` → compilation error

### Bounded Type Parameters
Restrict what types `T` can be:

| Syntax | Meaning | Can Call |
|--------|---------|---------|
| `<T>` | Any type | Only Object methods |
| `<T extends Number>` | Number or its subtypes | All Number methods |
| `<T extends Comparable<T>>` | Types that can compare themselves | `compareTo()` |
| `<T extends A & B>` | T must be both A AND B | Methods of both |

### Wildcards and PECS
Wildcards (`?`) add flexibility when dealing with generic collections you don't create:

| Wildcard | Meaning | Can Read? | Can Write? | Use for |
|----------|---------|-----------|------------|---------|
| `<?>` | Unknown type | As Object | ❌ No | Read-only pass-through |
| `<? extends T>` | T or any subtype | As T | ❌ No | **Producer** (you read FROM it) |
| `<? super T>` | T or any supertype | As Object | ✅ Yes | **Consumer** (you write TO it) |

**PECS Rule**: **P**roducer **E**xtends, **C**onsumer **S**uper

## CODE

### Level 1 — Beginner: Generic Collections
```java
import java.util.*;

public class GenericCollections {
    public static void main(String[] args) {
        // ---- Type-safe List ----
        List<String> names = new ArrayList<>(); // Diamond operator infers <String>
        names.add("Alice");
        names.add("Bob");
        // names.add(100);     // ❌ COMPILE ERROR — type safety!

        String first = names.get(0);  // No cast needed!
        System.out.println(first.toUpperCase()); // "ALICE"

        // ---- Type-safe Map ----
        Map<String, Integer> scores = new HashMap<>();
        scores.put("Alice", 95);
        scores.put("Bob", 87);
        // scores.put(42, "oops"); // ❌ COMPILE ERROR

        // No cast: we know it's an Integer
        int alice = scores.get("Alice");
        System.out.println("Alice's score: " + alice);

        // ---- Iterating generics ----
        for (Map.Entry<String, Integer> entry : scores.entrySet()) {
            System.out.println(entry.getKey() + " → " + entry.getValue());
        }

        // ---- Set deduplication with generics ----
        Set<Integer> uniqueNumbers = new LinkedHashSet<>();
        uniqueNumbers.addAll(List.of(3, 1, 4, 1, 5, 9, 2, 6, 5, 3));
        System.out.println(uniqueNumbers); // [3, 1, 4, 5, 9, 2, 6] — duplicates removed
    }
}
```

### Level 2 — Building Blocks: Writing Your Own Generic Classes
```java
// Generic Pair — holds any two related values of any types
public class Pair<A, B> {
    private final A first;
    private final B second;

    public Pair(A first, B second) {
        this.first = first;
        this.second = second;
    }

    public A getFirst()  { return first; }
    public B getSecond() { return second; }

    // Static factory method — cleaner API
    public static <A, B> Pair<A, B> of(A a, B b) {
        return new Pair<>(a, b);
    }

    @Override
    public String toString() {
        return "(" + first + ", " + second + ")";
    }
}

// Generic Stack implementation
public class GenericStack<T> {
    private final java.util.Deque<T> storage = new java.util.ArrayDeque<>();

    public void push(T item) { storage.push(item); }

    public T pop() {
        if (isEmpty()) throw new java.util.EmptyStackException();
        return storage.pop();
    }

    public T peek() {
        if (isEmpty()) throw new java.util.EmptyStackException();
        return storage.peek();
    }

    public boolean isEmpty() { return storage.isEmpty(); }
    public int size() { return storage.size(); }

    public static void main(String[] args) {
        Pair<String, Integer> nameScore = Pair.of("Alice", 95);
        System.out.println(nameScore); // (Alice, 95)

        GenericStack<String> stack = new GenericStack<>();
        stack.push("first");
        stack.push("second");
        stack.push("third");

        while (!stack.isEmpty()) {
            System.out.println(stack.pop()); // third, second, first
        }
    }
}
```

### Level 3 — Intermediate: Bounded Type Parameters and Generic Methods
```java
import java.util.*;

public class BoundedGenerics {

    // T must implement Comparable — we can call compareTo()
    public static <T extends Comparable<T>> T findMax(List<T> list) {
        if (list.isEmpty()) throw new NoSuchElementException("Empty list");
        T max = list.get(0);
        for (T item : list) {
            if (item.compareTo(max) > 0) {
                max = item;
            }
        }
        return max;
    }

    // Multiple bounds: T must extend Shape AND implement Printable
    public static <T extends Shape & Printable> void renderAndPrint(T item) {
        item.draw();    // Shape method
        item.print();   // Printable method
    }

    // Bounded wildcards — PECS in action
    // PRODUCER: reads from 'source', use extends
    public static <T> void copy(List<? extends T> source, List<? super T> dest) {
        for (T item : source) {
            dest.add(item);  // Polymorphic add — T is compatible with super T
        }
    }

    // Statistics method: accepts List of any Number subtype
    public static <N extends Number> double average(List<N> numbers) {
        return numbers.stream()
                      .mapToDouble(Number::doubleValue)
                      .average()
                      .orElse(0.0);
    }
    
    public static void main(String[] args) {
        System.out.println(findMax(List.of(3, 7, 1, 9, 4)));    // 9
        System.out.println(findMax(List.of("banana", "apple", "cherry"))); // cherry

        System.out.println(average(List.of(10, 20, 30)));        // 20.0
        System.out.println(average(List.of(1.5, 2.5, 3.0)));    // 2.333...

        List<Integer> ints  = new ArrayList<>(List.of(1, 2, 3));
        List<Number>  nums  = new ArrayList<>();
        copy(ints, nums);   // Integer extends Number — safe!
        System.out.println(nums); // [1, 2, 3]
    }
}
```

### Level 4 — Advanced: PECS, Wildcards Deep Dive, and Type Tokens
```java
import java.util.*;
import java.util.function.*;

public class AdvancedGenerics {

    // PECS Classic Example — Collections.sort pattern
    public static <T extends Comparable<? super T>> void sort(List<T> list) {
        // T must be comparable to itself OR its supertypes
        // This allows Integer list to be sorted even though Integer implements Comparable<Integer>
        // and we might have List<? extends Integer>
        for (int i = 0; i < list.size() - 1; i++) {
            for (int j = 0; j < list.size() - 1 - i; j++) {
                if (list.get(j).compareTo(list.get(j + 1)) > 0) {
                    T temp = list.get(j);
                    list.set(j, list.get(j + 1));
                    list.set(j + 1, temp);
                }
            }
        }
    }

    // Generic method with Function composition
    public static <T, R, V> Function<T, V> compose(
            Function<T, R> first, Function<R, V> second) {
        return t -> second.apply(first.apply(t));
    }

    // Type Token pattern — using Class<T> to carry type info despite erasure
    public static <T> Optional<T> cast(Object obj, Class<T> type) {
        if (type.isInstance(obj)) {
            return Optional.of(type.cast(obj));
        }
        return Optional.empty();
    }

    // Generic event system
    static class EventBus {
        private final Map<Class<?>, List<Consumer<?>>> handlers = new HashMap<>();

        @SuppressWarnings("unchecked")
        public <T> void subscribe(Class<T> eventType, Consumer<T> handler) {
            handlers.computeIfAbsent(eventType, k -> new ArrayList<>()).add(handler);
        }

        @SuppressWarnings("unchecked")
        public <T> void publish(T event) {
            List<Consumer<?>> eventHandlers = handlers.getOrDefault(event.getClass(), List.of());
            for (Consumer<?> handler : eventHandlers) {
                ((Consumer<T>) handler).accept(event);
            }
        }
    }

    public static void main(String[] args) {
        // Compose functions: String → int (length) → String (format)
        Function<String, String> pipeline = compose(
            String::length,
            len -> "Length: " + len
        );
        System.out.println(pipeline.apply("Hello, Generics!")); // "Length: 16"

        // Type token pattern
        Object obj = "Hello";
        Optional<String> str = cast(obj, String.class);
        Optional<Integer> num = cast(obj, Integer.class);
        System.out.println(str.isPresent()); // true
        System.out.println(num.isPresent()); // false

        // Event bus with generics
        EventBus bus = new EventBus();
        bus.subscribe(String.class, s -> System.out.println("String event: " + s));
        bus.subscribe(Integer.class, n -> System.out.println("Integer event: " + n));

        bus.publish("Hello World");    // "String event: Hello World"
        bus.publish(42);               // "Integer event: 42"
    }
}
```

### Level 5 — Expert: Generic API Design for Production
```java
import java.util.*;
import java.util.function.*;

// Production-grade Result type — eliminates null returns and exceptions for business logic
public sealed interface Result<T>
    permits Result.Ok, Result.Err {

    record Ok<T>(T value) implements Result<T> {}
    record Err<T>(String code, String message, Throwable cause) implements Result<T> {
        Err(String code, String message) { this(code, message, null); }
    }

    static <T> Result<T> ok(T value) { return new Ok<>(value); }
    static <T> Result<T> err(String code, String message) { return new Err<>(code, message); }
    static <T> Result<T> err(String code, String message, Throwable cause) { return new Err<>(code, message, cause); }

    default boolean isOk()  { return this instanceof Ok; }
    default boolean isErr() { return this instanceof Err; }

    default T getOrThrow() {
        return switch (this) {
            case Ok<T> ok   -> ok.value();
            case Err<T> err -> throw new RuntimeException("[" + err.code() + "] " + err.message(), err.cause());
        };
    }

    default T getOrElse(T fallback) {
        return switch (this) {
            case Ok<T> ok   -> ok.value();
            case Err<T> err -> fallback;
        };
    }

    default <R> Result<R> map(Function<T, R> mapper) {
        return switch (this) {
            case Ok<T> ok   -> Result.ok(mapper.apply(ok.value()));
            case Err<T> err -> Result.err(err.code(), err.message(), err.cause());
        };
    }

    default <R> Result<R> flatMap(Function<T, Result<R>> mapper) {
        return switch (this) {
            case Ok<T> ok   -> mapper.apply(ok.value());
            case Err<T> err -> Result.err(err.code(), err.message(), err.cause());
        };
    }
}

// Usage: type-safe, no exceptions for business logic
class UserRegistrationService {
    Result<Long> register(String email, String password) {
        if (email == null || !email.contains("@")) {
            return Result.err("INVALID_EMAIL", "Invalid email format");
        }
        if (password.length() < 8) {
            return Result.err("WEAK_PASSWORD", "Password must be at least 8 characters");
        }
        // Simulate DB save
        long id = System.currentTimeMillis();
        System.out.println("Registered user: " + email + " with ID=" + id);
        return Result.ok(id);
    }

    public static void main(String[] args) {
        UserRegistrationService service = new UserRegistrationService();

        Result<Long> r1 = service.register("invalid", "pass");
        Result<Long> r2 = service.register("alice@example.com", "weak");
        Result<Long> r3 = service.register("alice@example.com", "SecurePass123");

        System.out.println(r1); // Err[INVALID_EMAIL]
        System.out.println(r2); // Err[WEAK_PASSWORD]
        System.out.println(r3.getOrElse(-1L)); // The ID

        // Chaining with map
        String message = r3.map(id -> "User created with ID: " + id)
                           .getOrElse("Registration failed");
        System.out.println(message);
    }
}
```

## REAL_WORLD

### Spring Data JPA — Generic Repository Pattern
Spring Data JPA's entire design is based on generics:

```java
// Spring's built-in repository interface (simplified)
public interface CrudRepository<T, ID> {
    <S extends T> S save(S entity);
    Optional<T> findById(ID id);
    Iterable<T> findAll();
    long count();
    void deleteById(ID id);
}

// Your specific repositories extend it with concrete types
public interface UserRepository extends JpaRepository<User, Long> {
    // Spring generates the full implementation at runtime using reflection
    // to read the type parameters <User, Long>

    Optional<User> findByEmail(String email);
    List<User> findByRole(String role);

    @Query("SELECT u FROM User u WHERE u.createdAt > :since")
    List<User> findRecentUsers(@Param("since") LocalDateTime since);
}

public interface OrderRepository extends JpaRepository<Order, UUID> {
    List<Order> findByUserAndStatusIn(User user, List<String> statuses);
}
```
Spring reads the `<User, Long>` type parameters at startup using reflection (technically reading the **GenericSignature** bytecode attribute that IS preserved despite erasure) and generates an appropriate SQL-backed implementation automatically.

### Java Generics Best Practices

```java
// ❌ AVOID: Raw types — lose all type safety
List list = new ArrayList();
list.add("hello");
list.add(100); // No error — dangerous!

// ✅ USE: Parameterized types always
List<String> list = new ArrayList<>();

// ❌ AVOID: Casting from Object when a generic works
Object result = process();
String str = (String) result; // ClassCastException risk

// ✅ USE: Generic return types
<T> T process(Class<T> type) { ... }
String str = process(String.class); // Type-safe

// ❌ AVOID: Unbounded wildcard when a concrete type is known
void print(List<?> items) { // Can't add anything!
    for (Object item : items) { ... }
}

// ✅ USE: Bounded wildcard or concrete type
<T extends Displayable> void print(List<T> items) {
    for (T item : items) { item.display(); } // Can call display()!
}
```

## INTERVIEW

**Q: What is Type Erasure?**
A: During compilation, the Java compiler removes all generic type parameters from bytecode. `List<String>` becomes a raw `List` holding `Object` references. The compiler inserts automatic casts and enforces type safety only at compile time. This was done for backward compatibility with Java 1.4. Consequence: you cannot use generic types in `instanceof` checks or `new` expressions at runtime.

**Q: Explain the PECS rule.**
A: PECS = **Producer Extends, Consumer Super**. If a method reads from a generic collection (the collection "produces" data for you), use `<? extends T>` — you can read objects as type T but cannot add to it. If a method writes to a generic collection (the collection "consumes" data from you), use `<? super T>` — you can add type T objects but can only read them as Object.

**Q: Can you create `T[] array = new T[10];`?**
A: No — compilation error. Arrays require their exact element type at runtime for `ArrayStoreException` checking. Due to Type Erasure, `T` is replaced by `Object` (or its bound), so the JVM can't create a properly typed array. Work around: use `(T[]) new Object[n]` with an `@SuppressWarnings`, or prefer `ArrayList<T>`.

**Q: What is the difference between `<T extends Number>` and `<? extends Number>`?**
A: `<T extends Number>` defines a named type parameter — you can use `T` by name throughout the method/class to maintain the relationship between parameters and return types. `<? extends Number>` is an anonymous wildcard — used when you only need to say "some subtype of Number" but don't need to reference it by name again.

**Q: How does Spring Data JPA read generic type parameters despite Type Erasure?**
A: The Java compiler actually DOES preserve generic type information in a special bytecode attribute called `GenericSignature` (in `.class` files). This is NOT the same as what `instanceof` and `new` operations can see at runtime. Spring uses `ResolvableType` which reads this bytecode attribute via Reflection to determine that `UserRepository extends JpaRepository<User, Long>` uses `User` as `T` and `Long` as `ID`.

## FEYNMAN CHECK

Explain to a 10-year-old:
> Generics are like **labeled boxes**. Without generics, you have an unlabeled box — anyone can put anything in it (string, number, apple), and when you reach in to get something, you have no idea what you'll pull out. With `Box<Apple>`, the label says "Apple Box Only." The store clerk (compiler) checks every item going in. If you try to put an orange in, the clerk stops you right there at the store — instead of letting you get home and find a rotten orange in your fruit bowl.
