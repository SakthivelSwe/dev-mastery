## WHY

Records (Java 16) and sealed classes (Java 17) transform how Java models data:

1. **Records** — Eliminate boilerplate: no more writing equals, hashCode, toString, getters for data classes
2. **Sealed classes** — Restrict class hierarchies for exhaustive pattern matching
3. **Pattern matching** — Records + sealed classes + switch expressions = powerful algebraic data types
4. **Immutability by default** — Records are immutable, encouraging safer code
5. **Interview** — "What are records in Java?" is now in every Java 17+ interview

## THEORY

### Records — Immutable Data Carriers

A record is a special class that automatically generates: constructor, getters, equals, hashCode, toString.

```java
// Traditional data class — 50+ lines with Lombok or manual code
public class Point {
    private final int x;
    private final int y;
    public Point(int x, int y) { this.x = x; this.y = y; }
    public int x() { return x; }
    public int y() { return y; }
    @Override public boolean equals(Object o) { ... }
    @Override public int hashCode() { return Objects.hash(x, y); }
    @Override public String toString() { return "Point[x=" + x + ", y=" + y + "]"; }
}

// Record — 1 line!
public record Point(int x, int y) {}
```

**What the compiler generates for `record Point(int x, int y)`:**

| Generated | What |
|-----------|------|
| `Point(int x, int y)` | Canonical constructor |
| `int x()` | Accessor (not `getX()` — just `x()`) |
| `int y()` | Accessor |
| `equals(Object)` | Structural equality (x and y match) |
| `hashCode()` | Based on all components |
| `toString()` | `Point[x=3, y=4]` |

### Record Characteristics

```java
public record Person(String name, int age) {
    // 1. Compact constructor — validate/normalize without repeating assignments
    Person {  // called "compact canonical constructor"
        if (age < 0) throw new IllegalArgumentException("Age cannot be negative");
        name = name.trim(); // can modify parameters before assignment
    }

    // 2. Custom constructor (must call canonical constructor)
    Person(String name) {
        this(name, 0); // delegate to canonical
    }

    // 3. Additional methods
    boolean isAdult() { return age >= 18; }

    // 4. Static methods and fields
    static final Person UNKNOWN = new Person("Unknown", -1);

    // 5. Can implement interfaces
    // implements Comparable<Person> { @Override int compareTo(Person o) {...} }
}
```

**What records CANNOT have:**
- Instance fields (only the declared components)
- `extends` another class (records implicitly extend `Record`)
- `native` methods

### Sealed Classes — Restricting Hierarchies

A `sealed` class/interface restricts which classes can extend/implement it:

```java
// Sealed class — only Shape can be extended by Circle, Rectangle, Triangle
public sealed class Shape permits Circle, Rectangle, Triangle {
    abstract double area();
}

public final class Circle extends Shape {        // final — no further extension
    double radius;
    Circle(double r) { radius = r; }
    @Override double area() { return Math.PI * radius * radius; }
}

public final class Rectangle extends Shape {
    double width, height;
    Rectangle(double w, double h) { width = w; height = h; }
    @Override double area() { return width * height; }
}

public non-sealed class Triangle extends Shape { // non-sealed — can be extended further
    double base, height;
    Triangle(double b, double h) { base = b; height = h; }
    @Override double area() { return 0.5 * base * height; }
}
```

**Three modifiers for permitted subclasses:**
- `final` — no further subclasses
- `sealed` — further restricted subclasses
- `non-sealed` — unrestricted (anyone can extend)

### Sealed Classes + Pattern Matching = Exhaustive Switch

```java
// Exhaustive switch — compiler knows ALL possible subtypes!
double calcArea(Shape shape) {
    return switch (shape) {
        case Circle c -> Math.PI * c.radius() * c.radius();
        case Rectangle r -> r.width() * r.height();
        case Triangle t -> 0.5 * t.base() * t.height();
        // No default needed — compiler verifies all cases covered
    };
}

// If you add a new Shape subclass and forget to add a case → compile error!
```

### Records as Domain Model (Algebraic Data Types)

```java
// Model HTTP response as sealed hierarchy
sealed interface HttpResponse<T> permits Ok, NotFound, Error {
    record Ok<T>(T body, int statusCode) implements HttpResponse<T> {}
    record NotFound(String path) implements HttpResponse<Void> {}

        case HttpResponse.Ok<User> ok -> displayUser(ok.body());
```

## CODE

### Level 1 — Beginner: Basic Records

```java
// RecordBasics.java
public class RecordBasics {

    record Point(int x, int y) {}
    record Person(String name, int age, String email) {}
    record RGB(int r, int g, int b) {
        // Compact constructor with validation
        RGB {
            if (r < 0 || r > 255 || g < 0 || g > 255 || b < 0 || b > 255)
                throw new IllegalArgumentException("RGB values must be 0-255");
        }
        String toHex() { return String.format("#%02X%02X%02X", r, g, b); }
    }

    public static void main(String[] args) {
        Point p1 = new Point(3, 4);
        Point p2 = new Point(3, 4);
        Point p3 = new Point(5, 6);

        // Auto-generated accessors (not getX()!)
        System.out.println(p1.x()); // 3
        System.out.println(p1.y()); // 4

        // Auto-generated equals (structural)
        System.out.println(p1.equals(p2)); // true
        System.out.println(p1.equals(p3)); // false
        System.out.println(p1 == p2);      // false (different objects)

        // Auto-generated hashCode
        System.out.println(p1.hashCode() == p2.hashCode()); // true

        // Auto-generated toString
        System.out.println(p1); // Point[x=3, y=4]
        System.out.println(new Person("Alice", 30, "alice@example.com"));
        // Person[name=Alice, age=30, email=alice@example.com]

        // Records in collections
        java.util.Set<Point> set = new java.util.HashSet<>();
        set.add(p1);
        set.add(p2); // NOT added — equal to p1
        System.out.println(set.size()); // 1

        // RGB with custom method
        RGB red = new RGB(255, 0, 0);
        System.out.println(red.toHex()); // #FF0000
        // new RGB(300, 0, 0); // throws IllegalArgumentException
    }
}
```

### Level 2 — Elementary: Records with Custom Constructors, Implements

```java
// RecordAdvanced.java
import java.util.*;

public class RecordAdvanced {

    // Record implementing Comparable
    record Version(int major, int minor, int patch) implements Comparable<Version> {
        // Compact constructor
        Version {
            if (major < 0 || minor < 0 || patch < 0)
                throw new IllegalArgumentException("Version components cannot be negative");
        }

        @Override
        public int compareTo(Version other) {
            if (major != other.major) return Integer.compare(major, other.major);
            if (minor != other.minor) return Integer.compare(minor, other.minor);
            return Integer.compare(patch, other.patch);
        }

        @Override public String toString() { return major + "." + minor + "." + patch; }

        static Version parse(String version) {
            String[] parts = version.split("\\.");
            return new Version(Integer.parseInt(parts[0]),
                Integer.parseInt(parts[1]), Integer.parseInt(parts[2]));
        }
    }

    // Nested records for complex data
    record Address(String street, String city, String country) {}
    record Employee(String name, double salary, Address address) {}

    public static void main(String[] args) {
        // Version comparison
        List<Version> versions = new ArrayList<>(List.of(
            Version.parse("2.1.0"),
            Version.parse("1.9.5"),
            Version.parse("2.0.1"),
            Version.parse("1.9.9")
        ));
        Collections.sort(versions);
        System.out.println(versions); // [1.9.5, 1.9.9, 2.0.1, 2.1.0]

        // Records as keys in Map
        Map<Version, String> changelog = new HashMap<>();
        changelog.put(Version.parse("2.1.0"), "Added records support");
        changelog.put(Version.parse("2.0.0"), "Major refactor");
        System.out.println(changelog.get(new Version(2, 1, 0))); // Works! equals() is correct

        // Nested records
        Employee emp = new Employee("Alice", 100_000,
            new Address("123 Main St", "San Francisco", "USA"));
        System.out.println(emp.name() + " in " + emp.address().city());
    }
}
```

### Level 3 — Intermediate: Sealed Classes + Pattern Matching

```java
// SealedClasses.java
public class SealedClasses {

    // Sealed interface for expression tree (AST)
    sealed interface Expr permits Num, Add, Multiply, Negate {
        record Num(double value) implements Expr {}
        record Add(Expr left, Expr right) implements Expr {}
        record Multiply(Expr left, Expr right) implements Expr {}
        record Negate(Expr expr) implements Expr {}
    }

    // Exhaustive evaluation — no default needed!
    static double evaluate(Expr expr) {
        return switch (expr) {
            case Expr.Num n -> n.value();
            case Expr.Add a -> evaluate(a.left()) + evaluate(a.right());
            case Expr.Multiply m -> evaluate(m.left()) * evaluate(m.right());
            case Expr.Negate n -> -evaluate(n.expr());
        };
    }

    // Pretty print
    static String prettyPrint(Expr expr) {
        return switch (expr) {
            case Expr.Num n -> String.valueOf(n.value());
            case Expr.Add a -> "(" + prettyPrint(a.left()) + " + " + prettyPrint(a.right()) + ")";
            case Expr.Multiply m -> "(" + prettyPrint(m.left()) + " * " + prettyPrint(m.right()) + ")";
            case Expr.Negate n -> "-" + prettyPrint(n.expr());
        };
    }

    public static void main(String[] args) {
        // Build expression tree for: (3 + 4) * -(2)
        Expr expr = new Expr.Multiply(
            new Expr.Add(new Expr.Num(3), new Expr.Num(4)),
            new Expr.Negate(new Expr.Num(2))
        );

        System.out.println(prettyPrint(expr)); // ((3.0 + 4.0) * -2.0)
        System.out.println(evaluate(expr));    // -14.0

        // Pattern matching with guards (Java 21)
        Object obj = 42;
        String result = switch (obj) {
            case Integer i when i > 100 -> "large number: " + i;
            case Integer i when i > 0   -> "positive: " + i;
            case Integer i              -> "non-positive: " + i;
            case String s               -> "string: " + s;
            default                     -> "other: " + obj;
        };
        System.out.println(result); // positive: 42
    }
}
```

### Level 4 — Advanced: Domain Modeling with Sealed + Records

```java
// DomainModeling.java
import java.util.*;

public class DomainModeling {

    // Payment processing domain
    sealed interface PaymentMethod permits PaymentMethod.CreditCard,
            PaymentMethod.BankTransfer, PaymentMethod.Crypto {

        record CreditCard(String number, String expiry, String cvv) implements PaymentMethod {
            CreditCard {
                number = number.replaceAll("[^0-9]", "");
                if (number.length() != 16) throw new IllegalArgumentException("Invalid card number");
            }
            // Don't expose CVV
            @Override public String toString() {
                return "CreditCard[****" + number.substring(12) + "]";
            }
        }

        record BankTransfer(String accountNumber, String routingNumber, String bankName) implements PaymentMethod {}

        record Crypto(String currency, String walletAddress) implements PaymentMethod {
            Crypto { if (!Set.of("BTC","ETH","USDT").contains(currency))
                throw new IllegalArgumentException("Unsupported currency: " + currency); }
        }
    }

    sealed interface PaymentResult permits PaymentResult.Success, PaymentResult.Failure {
        record Success(String transactionId, double amount) implements PaymentResult {}
        record Failure(String errorCode, String message) implements PaymentResult {}
    }

    static PaymentResult processPayment(PaymentMethod method, double amount) {
        return switch (method) {
            case PaymentMethod.CreditCard cc -> {
                System.out.println("Processing credit card: " + cc);
                yield new PaymentResult.Success("TXN-" + System.nanoTime(), amount);
            }
            case PaymentMethod.BankTransfer bt -> {
                System.out.println("Processing bank transfer to " + bt.bankName());
                yield new PaymentResult.Success("WIRE-" + System.nanoTime(), amount);
            }
            case PaymentMethod.Crypto crypto -> {
                System.out.println("Processing " + crypto.currency() + " transfer");
                yield new PaymentResult.Success("CRYPTO-" + System.nanoTime(), amount);
            }
        };
    }

    public static void main(String[] args) {
        var methods = List.of(
            new PaymentMethod.CreditCard("4532015112830366", "12/26", "123"),
            new PaymentMethod.BankTransfer("12345678", "021000021", "Chase"),
            new PaymentMethod.Crypto("ETH", "0x1234...abcd")
        );

        for (PaymentMethod method : methods) {
            PaymentResult result = processPayment(method, 99.99);
            switch (result) {
                case PaymentResult.Success s ->
                    System.out.println("✓ Transaction: " + s.transactionId());
                case PaymentResult.Failure f ->
                    System.out.println("✗ Error: " + f.message());
            }
        }
    }
}
```

### Level 5 — Expert: Records with Generic Types & Interop

```java
// RecordsExpert.java
import java.util.*;
import java.util.function.*;

public class RecordsExpert {

    // Generic record
    record Pair<A, B>(A first, B second) {
        // Map over first element
        <C> Pair<C, B> mapFirst(Function<A, C> f) {
            return new Pair<>(f.apply(first), second);
        }

        // Swap
        Pair<B, A> swap() { return new Pair<>(second, first); }
    }

    // Result type using sealed + records
    sealed interface Result<T> permits Result.Ok, Result.Err {
        record Ok<T>(T value) implements Result<T> {}
        record Err<T>(String error) implements Result<T> {}

        static <T> Result<T> of(java.util.concurrent.Callable<T> action) {
            try { return new Ok<>(action.call()); }
            catch (Exception e) { return new Err<>(e.getMessage()); }
        }

        default <R> Result<R> map(Function<T, R> f) {
            return switch (this) {
                case Ok<T> ok -> Result.of(() -> f.apply(ok.value()));
                case Err<T> err -> new Err<>(err.error());
            };
        }

        default T getOrElse(T defaultValue) {
            return switch (this) {
                case Ok<T> ok -> ok.value();
                case Err<T> err -> defaultValue;
            };
        }
    }

    // Record as value object in domain
    record Money(long cents, String currency) implements Comparable<Money> {
        Money {
            Objects.requireNonNull(currency);
            if (!currency.equals(currency.toUpperCase()) || currency.length() != 3)
                throw new IllegalArgumentException("Currency must be 3-letter ISO code");
        }

        Money add(Money other) {
            if (!currency.equals(other.currency)) throw new IllegalArgumentException("Currency mismatch");
            return new Money(cents + other.cents, currency);
        }

        Money multiply(double factor) {
            return new Money((long)(cents * factor), currency);
        }

        @Override public int compareTo(Money o) {
            if (!currency.equals(o.currency)) throw new IllegalArgumentException("Cannot compare different currencies");
            return Long.compare(cents, o.cents);
        }

        @Override public String toString() {
            return String.format("%s %.2f", currency, cents / 100.0);
        }
    }

    public static void main(String[] args) {
        // Generic Pair
        Pair<String, Integer> p = new Pair<>("hello", 5);
        Pair<Integer, Integer> mapped = p.mapFirst(String::length);
        System.out.println(mapped); // Pair[first=5, second=5]
        System.out.println(p.swap()); // Pair[first=5, second=hello]

        // Result monad
        Result<Integer> r1 = Result.of(() -> Integer.parseInt("42"))
            .map(n -> n * 2);
        System.out.println(r1); // Ok[value=84]

        Result<Integer> r2 = Result.of(() -> Integer.parseInt("not-a-number"));
        System.out.println(r2.getOrElse(0)); // 0

        // Money value object
        Money price = new Money(1999, "USD"); // $19.99
        Money tax = price.multiply(0.08);     // 8% tax
        Money total = price.add(tax);
        System.out.println(price + " + " + tax + " = " + total);
        // USD 19.99 + USD 1.60 = USD 21.59
    }
}
```

## INTERVIEW

**Q1: What are Java records and what do they generate automatically?**

> **Strong answer:** Records (Java 16) are immutable data carriers. Declaring `record Point(int x, int y)` generates: a canonical constructor, accessor methods `x()` and `y()` (not `getX()`), `equals()` based on all components, `hashCode()` based on all components, and `toString()` returning `Point[x=3, y=4]`. Records cannot have instance fields beyond the declared components, cannot extend other classes (implicitly extend `java.lang.Record`), and are implicitly `final`. They can implement interfaces, have additional methods, and add validation in compact constructors.

**Q2: What are sealed classes and what problem do they solve?**

> **Strong answer:** Sealed classes (Java 17) restrict which classes can extend them using `permits`. Problem they solve: in a normal hierarchy, anyone can create new subclasses — you can't write exhaustive switch expressions because someone might add a new subclass. With sealed classes, the compiler knows ALL possible subtypes, enabling exhaustive pattern matching switch expressions without a `default` case. If you add a new permitted subclass and forget to handle it in a switch, the compiler errors. This is particularly powerful for domain modeling, expression trees, result types (Ok/Error), and algebraic data types.

**Q3: How is a record different from a regular class with Lombok @Value?**

> **Strong answer:** Lombok `@Value` generates similar code at compile time via annotation processing. Records are a JVM-level feature — the JVM natively understands records (there's a `java.lang.Record` base class and bytecode metadata). Key differences: (1) Record accessor methods are named after the component (`x()`), not `getX()`. (2) Records can't extend other classes; Lombok classes can. (3) Records support compact constructors for validation. (4) Records work natively with pattern matching in switch expressions. (5) No Lombok dependency needed — records are pure Java. For most DTO use cases, records are preferred over Lombok in modern Java.

## FEYNMAN CHECK

A record is like a named tuple with superpowers. Instead of writing a class with constructors, getters, and equals/hashCode, you declare what data the record holds and Java generates all the plumbing. A sealed class is like an enum for class hierarchies — it says "these and only these classes can be my subtypes." Together they let you model choices in your domain (Success or Failure, Circle or Rectangle or Triangle) and the compiler verifies you've handled all cases.
