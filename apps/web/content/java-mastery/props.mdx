# Props — Passing Data, TypeScript Types, Children

## Props are Read-Only (Immutable Snapshots)

Props represent the configuration of a component passed down from its parent. In React, **props are strictly read-only**.

React enforces this immutability. In Development and Strict Mode, React freezes the props object. Attempting to mutate a prop directly will throw a runtime error.

```typescript
interface UserWidgetProps {
  user: { name: string };
}

export const UserWidget: React.FC<UserWidgetProps> = ({ user }) => {
  // CRITICAL VIOLATION:
  // user.name = "Bob"; // Throws error in strict mode!
  
  // CORRECT: Treat as read-only, create a new reference if modification is needed
  const uppercaseName = user.name.toUpperCase();
  
  return <div>{uppercaseName}</div>;
};
```

## The Prop Drilling Problem

**Prop Drilling** occurs when data must be passed through multiple layers of intermediate components that do not actually consume the data themselves, solely to reach a deeply nested target component.

```
[App] (holds user state)
  └── [Header]
        └── [Navigation]
              └── [UserAvatar] (needs user state)
```

While acceptable for 2-3 levels, deep prop drilling makes components rigid, tightly coupled, and difficult to refactor. Solutions include:
1. **Component Composition** (passing the rendered leaf component down as a prop).
2. **Context API** (for global/semi-global state).
3. **State Management Libraries** (Redux, TanStack Query).

## Spreading Props: Power and Peril

The object spread operator (`{...props}`) allows you to pass all keys of an object as individual props to an element.

```typescript
import React from 'react';

interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant: 'primary' | 'secondary';
}

export const Button: React.FC<ButtonProps> = ({ variant, ...rest }) => {
  const className = `btn btn-${variant}`;
  
  return (
    // rest contains all standard HTML button props (onClick, disabled, type, etc.)
    <button className={className} {...rest} />
  );
};
```

### The Security and Performance Risk of Spreading:
If you spread an unvetted object (e.g., raw API responses) directly onto a DOM element, you risk:
1. Passing invalid HTML attributes, triggering console warnings in the browser.
2. Accidentally exposing sensitive data to the DOM as plain-text HTML attributes.
3. Triggering unnecessary re-renders if the spread object changes its reference on every render.

## Default Props: Modern TypeScript Approach

In class components, default props were defined via static `defaultProps` property. In modern functional components, standard **ES6 default parameters** are the standard, clean, and type-safe approach.

```typescript
interface AlertProps {
  message: string;
  type?: 'info' | 'warning' | 'error'; // Optional prop
}

// Default values assigned directly in the destructuring assignment
export const Alert: React.FC<AlertProps> = ({ 
  message, 
  type = 'info' // Default parameter
}) => {
  return <div className={`alert alert-${type}`}>{message}</div>;
};
```

## Typing Children: React Node vs. React Element

TypeScript provides several types for handling components and child nodes:

| Type | Description | Includes |
| :--- | :--- | :--- |
| `React.ReactNode` | Most permissive type. Represents anything that can be rendered by React. | `ReactElement`, `string`, `number`, `iterable`, `null`, `undefined` |
| `React.ReactElement` | Represents an actual virtual DOM object node. | Only JSX elements (e.g. `<div />` or `<MyComponent />`) |
| `React.FC<T>` | Function Component generic interface. | Explicitly types the function signature and returned nodes. |

```typescript
import React from 'react';

interface LayoutProps {
  // Use ReactNode for children to allow strings, arrays, or elements
  children: React.ReactNode;
  
  // Use ReactElement to enforce that only a SINGLE, valid React JSX element is passed
  header: React.ReactElement;
}

export const Layout: React.FC<LayoutProps> = ({ children, header }) => {
  return (
    <div className="layout">
      <header className="site-header">{header}</header>
      <main className="site-content">{children}</main>
    </div>
  );
};
```

## Component as a Prop Pattern (Render Props)

Instead of passing raw data, you can pass a function that returns a component. This is a powerful pattern for separating logic from UI presentation.

```typescript
import React, { useState } from 'react';

interface MouseTrackerProps {
  // A function prop that takes coordinates and returns a React Node
  render: (coords: { x: number; y: number }) => React.ReactNode;
}

export const MouseTracker: React.FC<MouseTrackerProps> = ({ render }) => {
  const [coords, setCoords] = useState({ x: 0, y: 0 });

  const handleMouseMove = (event: React.MouseEvent) => {
    setCoords({
      x: event.clientX,
      y: event.clientY,
    });
  };

  return (
    <div style={{ height: '200px', border: '1px solid black' }} onMouseMove={handleMouseMove}>
      {render(coords)}
    </div>
  );
};

// Usage:
export const App: React.FC = () => (
  <MouseTracker 
    render={({ x, y }) => (
      <h1>The mouse pointer is at ({x}, {y})</h1>
    )}
  />
);
```
