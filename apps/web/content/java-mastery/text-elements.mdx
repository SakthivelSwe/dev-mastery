# Text Elements — Headings, Paragraphs, Inline Elements

### 1. Deep-Dive Technical Explanation

#### The Document Outline Algorithm
The HTML5 specification originally proposed an outline algorithm where sectioning structures (like `<section>` and `<article>`) would dynamically determine the heading level of nested `<h1>` tags. However, **no browser or screen reader ever implemented this algorithm**. 

To this day, you must explicitly use heading levels `<h1>` through `<h6>` to build an accessible document outline. Skipping heading levels (e.g., jumping from `<h1>` to `<h3>`) breaks the hierarchical tree model used by assistive technologies to index the page.

#### Semantic Distinctions of Inline Formatting Elements
Inline text elements are not merely visual styles; they carry deep semantic weight in the Accessibility Tree:

- **`<strong>` vs `<b>`**: `<strong>` represents strong importance, seriousness, or urgency, and is announced with altered emphasis by some screen readers. `<b>` is historically "bold" but in HTML5 represents a stylistic offset without conveying extra importance (such as key terms in an abstract).
- **`<em>` vs `<i>`**: `<em>` represents stress emphasis (changing the meaning of the sentence if spoken aloud). `<i>` represents idiomatic text, technical terms, or thoughts, offset from the normal prose without extra emphasis.
- **`<mark>`**: Represents highlighted text for reference or notation purposes, often used to show search term matches.
- **`<small>`**: Represents side-comments or small print, such as copyright disclaimers or legal terms.
- **`<time>`**: Machine-readable date/time representation. The `datetime` attribute provides standard ISO 8601 formatting for parsers, while the inner text displays human-readable output.

```
Source HTML: <p>Review by <time datetime="2026-03-31">March 31</time></p>
DOM Node: HTMLTimeElement
Accessibility Tree: "Review by March 31" (Semantic date metadata attached to the node)
```

### 2. Complete, Valid HTML5 Example

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Semantic Typography and Text Elements</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>

  <main class="content-container">
    <article class="prose-article">
      <header>
        <h1>Advanced Web Accessibility: Core Principles</h1>
        <p class="byline">Published on <time datetime="2026-03-31">March 31, 2026</time> by DevMastery</p>
      </header>

      <section>
        <h2>The Importance of Logical Document Outlines</h2>
        <p>
          Constructing a logical outline is <strong>absolutely critical</strong> for screen reader users. 
          When headings are structured hierarchically, users can navigate the document swiftly. 
          Using structural elements like <code>&lt;h1&gt;</code> through <code>&lt;h6&gt;</code> 
          creates a predictable roadmap.
        </p>
      </section>

      <section>
        <h2>Inline Semantics and Visual Distinctions</h2>
        <p>
          We must distinguish between purely typographic changes and semantic shifts. For instance, 
          taxonomic designations like the scientific name of the bald eagle, <i>Haliaeetus leucocephalus</i>, 
          require the <code>&lt;i&gt;</code> element. In contrast, words requiring direct voice 
          stress should utilize <em>emphasis</em>.
        </p>
        <p>
          When displaying search results, matching queries must be wrapped in the 
          <mark class="search-highlight">highlighted term</mark> element.
        </p>
      </section>

      <footer>
        <p><small>The information contained within this document is subject to change without notice.</small></p>
      </footer>
    </article>
  </main>

</body>
</html>
```

  margin: 0;
  font-family: Georgia, Cambria, "Times New Roman", Times, serif;

  max-width: 700px;
  margin: 0 auto;
