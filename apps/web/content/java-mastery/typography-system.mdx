## 1. Architectural Blueprint & Core Concepts

Fluid typography dynamically scales text sizes relative to the viewport dimensions using CSS mathematical operations like `clamp()`. This prevents layout shifts (CLS) and eliminates the need for aggressive media-query overrides.

### Fluid Typography Formula
To calculate a fluid typography value:
$$\text{Fluid Size} = \text{clamp}(\text{Min Size}, \text{Preferred Value}, \text{Max Size})$$

The preferred value uses viewport units (`vw`) combined with a relative unit (`rem`) offset to preserve user-initiated browser zoom settings.

## 2. Figma Workflow & Sync Pipeline

### Figma Setup
1. Define a base typography hierarchy using Figma's Text Styles.
2. Use variables mapped to **min-width** (e.g., `320px`) and **max-width** (e.g., `1440px`) breakpoints.
3. Design layouts using a responsive scale (e.g., Minor Third $1.200$ for mobile, Perfect Fourth $1.333$ for desktop).
4. Utilize plugins like **Fluid Type Scale Generator** to map static Figma text frames to dynamic CSS variables.

## 3. CSS Custom Properties & Tailwind Configuration

### CSS Custom Properties (`typography.css`)
```css
:root {
  /* Scale Constraints */
  --bp-min: 320;
  --bp-max: 1440;
  
  /* Fluid Typographic Scale */
  /* Formula: clamp(minPx, minRem + (maxPx - minPx) * ((100vw - minBp) / (maxBp - minBp)), maxPx) */
  --font-size-sm: clamp(0.875rem, 0.8rem + 0.375vw, 1rem);
  --font-size-base: clamp(1rem, 0.95rem + 0.25vw, 1.125rem);
  --font-size-lg: clamp(1.25rem, 1.1rem + 0.75vw, 1.5rem);
  --font-size-xl: clamp(1.5rem, 1.3rem + 1vw, 2.25rem);
  --font-size-2xl: clamp(2rem, 1.6rem + 2vw, 3.5rem);

  /* Line Heights */
  --line-height-tight: 1.2;
  --line-height-snug: 1.4;
  --line-height-normal: 1.6;
}
```

## 4. Accessibility Compliance (WCAG 2.1 AA)
- **Visual Presentation (Success Criterion 1.4.8):** Line spacing (leading) must be at least **1.5** times the font size, and paragraph spacing must be at least **2** times the line spacing.
- **Resize Text (Success Criterion 1.4.4):** Avoid absolute units (`px`) in fluid text calculations. Always use relative units (`rem`, `em`) inside `clamp()` to ensure text scales up correctly when users resize browser settings up to 200%.

## 5. React & TypeScript Implementation

A highly flexible, polymorphic `<Text>` component supporting multiple typographic levels and semantic HTML tags.

```tsx
import React from 'react';

type TextVariant = 'sm' | 'base' | 'lg' | 'xl' | '2xl';
type TextWeight = 'normal' | 'medium' | 'bold';

interface TextProps<C extends React.ElementType> {
  as?: C;
  variant?: TextVariant;
  weight?: TextWeight;
  children: React.ReactNode;
  className?: string;
}

export const Text = <C extends React.ElementType = 'p'>({
  as,
  variant = 'base',
  weight = 'normal',
  children,
  className = '',
  ...props
}: TextProps<C> & Omit<React.ComponentPropsWithoutRef<C>, keyof TextProps<C>>) => {
  const Component = as || 'p';

  const variantClasses: Record<TextVariant, string> = {
    sm: 'text-fluid-sm leading-fluid-snug',
    base: 'text-fluid-base leading-fluid-normal',
    lg: 'text-fluid-lg leading-fluid-snug',
    xl: 'text-fluid-xl leading-fluid-tight font-semibold tracking-tight',
    '2xl': 'text-fluid-2xl leading-fluid-tight font-bold tracking-tighter',
  };

  const weightClasses: Record<TextWeight, string> = {
    normal: 'font-normal',
    medium: 'font-medium',
    bold: 'font-bold',
  };

  return (
    <Component
      className={`${variantClasses[variant]} ${weightClasses[weight]} ${className}`}
      {...props}
    >
      {children}
    </Component>
  );
};
```
