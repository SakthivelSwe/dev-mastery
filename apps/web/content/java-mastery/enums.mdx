## WHY

Enums are the right way to model a fixed set of constants. They prevent entire classes of bugs:

1. **Type safety** — `if (status == 4)` is error-prone; `if (status == Status.ACTIVE)` is self-documenting and compile-checked
2. **Behavior in constants** — Java enums can have fields, methods, and implement interfaces — far more powerful than other languages
3. **Switch expressions** — Java 14+ exhaustive switch with enums prevents missing cases
4. **Singleton pattern** — `enum Singleton` is the only truly safe singleton in Java (handles serialization, reflection attacks)
5. **State machines** — Enum + behavior is the canonical Java state machine pattern

## THEORY

### Enum Basics

An enum is a special class where instances are known at compile time:

```java
public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}
```

Under the hood, the compiler generates roughly:
```java
public final class Day extends Enum<Day> {
    public static final Day MONDAY = new Day("MONDAY", 0);
    public static final Day TUESDAY = new Day("TUESDAY", 1);
    // ...
    private static final Day[] $VALUES = {MONDAY, TUESDAY, ...};

    public static Day[] values() { return $VALUES.clone(); }
    public static Day valueOf(String name) { ... }
}
```

### Enum Built-in Methods

| Method | Returns | Example |
|--------|---------|---------|
| `name()` | String | `Day.MONDAY.name()` → `"MONDAY"` |
| `ordinal()` | int (0-based) | `Day.MONDAY.ordinal()` → `0` |
| `toString()` | String | same as `name()` by default |
| `values()` | E[] (static) | `Day.values()` → all enum constants |
| `valueOf(String)` | E (static) | `Day.valueOf("MONDAY")` → `Day.MONDAY` |
| `compareTo(E)` | int | by ordinal |
| `equals(Object)` | boolean | `==` and `.equals()` are the same for enums |

### Enums with Fields and Methods

```java
public enum Planet {
    MERCURY(3.303e+23, 2.4397e6),
    VENUS  (4.869e+24, 6.0518e6),
    EARTH  (5.976e+24, 6.37814e6),
    MARS   (6.421e+23, 3.3972e6);

    private final double mass;    // in kilograms
    private final double radius;  // in meters
    static final double G = 6.67300E-11;

    Planet(double mass, double radius) {
        this.mass = mass;
        this.radius = radius;
    }

    double surfaceGravity() {
        return G * mass / (radius * radius);
    }

    double surfaceWeight(double otherMass) {
        return otherMass * surfaceGravity();
    }
}
// Usage:
double earthWeight = 75.0;
double mass = earthWeight / Planet.EARTH.surfaceGravity();
for (Planet p : Planet.values()) {
    System.out.printf("Weight on %s: %.2f%n", p, p.surfaceWeight(mass));
}
```

### Abstract Methods in Enum (Per-Constant Behavior)

```java
public enum Operation {
    PLUS("+") {
        @Override public double apply(double x, double y) { return x + y; }
    },
    MINUS("-") {
        @Override public double apply(double x, double y) { return x - y; }
    },
    TIMES("*") {
        @Override public double apply(double x, double y) { return x * y; }
    },
        DIVIDE("/") {
        @Override public double apply(double x, double y) { return x / y; }
    };

    private final String symbol;

    Operation(String symbol) { this.symbol = symbol; }

    public abstract double apply(double x, double y);

    @Override public String toString() { return symbol; }
}
// Usage:
for (Operation op : Operation.values()) {
    System.out.printf("%.1f %s %.1f = %.1f%n", 2.0, op, 4.0, op.apply(2.0, 4.0));
}
// 2.0 + 4.0 = 6.0
// 2.0 - 4.0 = -2.0
// 2.0 * 4.0 = 8.0
// 2.0 / 4.0 = 0.5
```

### Enum in Switch (Java 14+ exhaustive switch expression)

```java
enum Status { PENDING, PROCESSING, COMPLETED, FAILED }

// Old style (switch statement — not exhaustive)
switch (status) {
    case PENDING: return "Waiting..."; break;
    case COMPLETED: return "Done!"; break;
    // Missing PROCESSING and FAILED → compiler won't warn!
}

// New style (switch expression — exhaustive, compile-time check)
String message = switch (status) {
    case PENDING -> "Waiting...";
    case PROCESSING -> "In progress...";
    case COMPLETED -> "Done!";
    case FAILED -> "Error occurred";
    // If you add a new Status value and forget to add a case → compile error!
};
```

### EnumSet and EnumMap

| | Description | Performance |
|--|-------------|-------------|
| `EnumSet` | Set of enum constants | Backed by bit vector — O(1), extremely fast |
| `EnumMap` | Map with enum keys | Array-backed — faster than HashMap |

```java
// EnumSet — much faster than HashSet<Day>
EnumSet<Day> weekdays = EnumSet.range(Day.MONDAY, Day.FRIDAY);
EnumSet<Day> weekend = EnumSet.of(Day.SATURDAY, Day.SUNDAY);
EnumSet<Day> allDays = EnumSet.allOf(Day.class);

// EnumMap — faster than HashMap<Status, Handler>
EnumMap<Status, String> handlers = new EnumMap<>(Status.class);
handlers.put(Status.PENDING, "queueHandler");
handlers.put(Status.FAILED, "errorHandler");
```

## CODE

### Level 1 — Beginner: Basic Enum Usage

```java
// EnumBasics.java
public class EnumBasics {
    enum Season { SPRING, SUMMER, AUTUMN, WINTER }

    public static void main(String[] args) {
        Season s = Season.SUMMER;

        // name() — always returns declaration name
        System.out.println(s.name());     // SUMMER
        System.out.println(s.toString()); // SUMMER
        System.out.println(s.ordinal());  // 1

        // values() — iterate all constants
        for (Season season : Season.values()) {
            System.out.println(season.ordinal() + ": " + season);
        }

        // valueOf() — parse from String
        Season spring = Season.valueOf("SPRING");
        System.out.println(spring == Season.SPRING); // true! (enum == is safe)

        // Switch
        String activity = switch (s) {
            case SPRING -> "Gardening";
            case SUMMER -> "Swimming";
            case AUTUMN -> "Hiking";
            case WINTER -> "Skiing";
        };
        System.out.println(activity); // Swimming

        // Enum in if
        if (s == Season.SUMMER || s == Season.SPRING) {
            System.out.println("Warm season");
        }
    }
}
```

### Level 2 — Elementary: Enums with Fields

```java
// HttpStatus.java
public enum HttpStatus {
    OK(200, "OK"),
    CREATED(201, "Created"),
    BAD_REQUEST(400, "Bad Request"),
    UNAUTHORIZED(401, "Unauthorized"),
    FORBIDDEN(403, "Forbidden"),
    NOT_FOUND(404, "Not Found"),
    INTERNAL_SERVER_ERROR(500, "Internal Server Error"),
    SERVICE_UNAVAILABLE(503, "Service Unavailable");

    private final int code;
    private final String description;

    HttpStatus(int code, String description) {
        this.code = code;
        this.description = description;
    }

    public int getCode() { return code; }
    public String getDescription() { return description; }

    // Reverse lookup — find enum by code
    public static HttpStatus fromCode(int code) {
        for (HttpStatus status : values()) {
            if (status.code == code) return status;
        }
        throw new IllegalArgumentException("Unknown HTTP status code: " + code);
    }

    public boolean isSuccess() { return code >= 200 && code < 300; }
    public boolean isClientError() { return code >= 400 && code < 500; }
    public boolean isServerError() { return code >= 500 && code < 600; }

    @Override
    public String toString() { return code + " " + description; }

    public static void main(String[] args) {
        HttpStatus status = HttpStatus.NOT_FOUND;
        System.out.println(status);              // 404 Not Found
        System.out.println(status.getCode());    // 404
        System.out.println(status.isClientError()); // true

        HttpStatus found = HttpStatus.fromCode(200);
        System.out.println(found);              // 200 OK
        System.out.println(found.isSuccess());  // true
    }
}
```

### Level 3 — Intermediate: State Machine with Enum

```java
// OrderStateMachine.java
public enum OrderState {
    PENDING {
        @Override public OrderState next() { return PROCESSING; }
        @Override public OrderState cancel() { return CANCELLED; }
        @Override public boolean canShip() { return false; }
    },
    PROCESSING {
        @Override public OrderState next() { return SHIPPED; }
        @Override public OrderState cancel() { return CANCELLED; }
        @Override public boolean canShip() { return false; }
    },
    SHIPPED {
        @Override public OrderState next() { return DELIVERED; }
        @Override public OrderState cancel() {
            throw new IllegalStateException("Cannot cancel shipped order");
        }
        @Override public boolean canShip() { return true; }
    },
    DELIVERED {
        @Override public OrderState next() {
            throw new IllegalStateException("Order already delivered");
        }
        @Override public OrderState cancel() {
            throw new IllegalStateException("Cannot cancel delivered order");
        }
        @Override public boolean canShip() { return false; }
    },
    CANCELLED {
        @Override public OrderState next() {
            throw new IllegalStateException("Cannot advance cancelled order");
        }
        @Override public OrderState cancel() { return this; }
        @Override public boolean canShip() { return false; }
    };

    public abstract OrderState next();
    public abstract OrderState cancel();
    public abstract boolean canShip();

    public static void main(String[] args) {
        OrderState state = OrderState.PENDING;
        System.out.println("State: " + state);          // PENDING

        state = state.next();
        System.out.println("State: " + state);          // PROCESSING

        state = state.next();
        System.out.println("State: " + state);          // SHIPPED
        System.out.println("Can ship: " + state.canShip()); // true

        state = state.next();
        System.out.println("State: " + state);          // DELIVERED

        try {
            state = state.cancel();
        } catch (IllegalStateException e) {
            System.out.println("Error: " + e.getMessage()); // Cannot cancel delivered order
        }
    }
}
```

### Level 4 — Advanced: EnumSet, EnumMap, Strategy Pattern

```java
// EnumPatterns.java
import java.util.*;

public class EnumPatterns {

    enum Permission { READ, WRITE, EXECUTE, DELETE, ADMIN }
    enum UserRole { GUEST, USER, MODERATOR, ADMIN }

    // Permission system using EnumSet — bit-vector backed, very fast
    static final Map<UserRole, EnumSet<Permission>> ROLE_PERMISSIONS = new EnumMap<>(UserRole.class);

    static {
        ROLE_PERMISSIONS.put(UserRole.GUEST, EnumSet.of(Permission.READ));
        ROLE_PERMISSIONS.put(UserRole.USER, EnumSet.of(Permission.READ, Permission.WRITE));
        ROLE_PERMISSIONS.put(UserRole.MODERATOR,
            EnumSet.of(Permission.READ, Permission.WRITE, Permission.DELETE));
        ROLE_PERMISSIONS.put(UserRole.ADMIN, EnumSet.allOf(Permission.class));
    }

    static boolean hasPermission(UserRole role, Permission perm) {
        return ROLE_PERMISSIONS.get(role).contains(perm);
    }

    // Strategy pattern with enum
    enum SortStrategy {
        BUBBLE {
            @Override public int[] sort(int[] arr) {
                arr = arr.clone();
                for (int i = 0; i < arr.length - 1; i++)
                    for (int j = 0; j < arr.length - 1 - i; j++)
                        if (arr[j] > arr[j+1]) { int t = arr[j]; arr[j]=arr[j+1]; arr[j+1]=t; }
                return arr;
            }
        },
        SELECTION {
            @Override public int[] sort(int[] arr) {
                arr = arr.clone();
                for (int i = 0; i < arr.length; i++) {
                    int min = i;
                    for (int j = i+1; j < arr.length; j++) if (arr[j] < arr[min]) min = j;
                    int t = arr[i]; arr[i] = arr[min]; arr[min] = t;
                }
                return arr;
            }
        };
        public abstract int[] sort(int[] arr);
    }

    public static void main(String[] args) {
        // Permission checks
        System.out.println(hasPermission(UserRole.USER, Permission.READ));    // true
        System.out.println(hasPermission(UserRole.GUEST, Permission.WRITE));  // false
        System.out.println(hasPermission(UserRole.ADMIN, Permission.DELETE)); // true

        // EnumSet operations (set algebra)
        EnumSet<Permission> userPerms = ROLE_PERMISSIONS.get(UserRole.USER);
        EnumSet<Permission> modPerms = ROLE_PERMISSIONS.get(UserRole.MODERATOR);
        EnumSet<Permission> extraForMod = EnumSet.copyOf(modPerms);
        extraForMod.removeAll(userPerms);
        System.out.println("Extra mod perms: " + extraForMod); // [DELETE]

        // Strategy with enum
        int[] data = {5, 2, 8, 1, 9};
        System.out.println(Arrays.toString(SortStrategy.BUBBLE.sort(data)));
        System.out.println(Arrays.toString(SortStrategy.SELECTION.sort(data)));
    }
}
```

### Level 5 — Expert: Enum Singleton, Serialization Safety, Reflection Safety

```java
// EnumSingleton.java
import java.io.*;

/**
 * Demonstrates why enum is the best singleton implementation.
 * Effective Java Item 3: Prefer enum singleton.
 */
public enum EnumSingleton {
    INSTANCE;

    private int counter = 0;

    public void increment() { counter++; }
    public int getCounter() { return counter; }

    // Enum singletons are automatically:
    // 1. Thread-safe (JVM class loading is thread-safe)
    // 2. Serialization-safe (JVM handles readResolve automatically)
    // 3. Reflection-safe (cannot create enum instance via reflection)

    public static void main(String[] args) throws Exception {
        EnumSingleton s1 = EnumSingleton.INSTANCE;
        s1.increment(); s1.increment();
        System.out.println(s1.getCounter()); // 2

        // Prove it's the same instance
        EnumSingleton s2 = EnumSingleton.INSTANCE;
        System.out.println(s1 == s2); // true

        // Reflection attack — fails for enum (unlike regular singleton)
        try {
            java.lang.reflect.Constructor<EnumSingleton> c =
                EnumSingleton.class.getDeclaredConstructor(String.class, int.class);
            c.setAccessible(true);
            EnumSingleton hack = c.newInstance("HACKED", 99);
        } catch (Exception e) {
            System.out.println("Reflection blocked: " + e.getMessage());
            // Cannot reflectively create enum objects
        }

        // Serialization safety test
        byte[] serialized;
        try (ByteArrayOutputStream bos = new ByteArrayOutputStream();
             ObjectOutputStream oos = new ObjectOutputStream(bos)) {
            oos.writeObject(s1);
            serialized = bos.toByteArray();
        }
        EnumSingleton deserialized;
        try (ObjectInputStream ois = new ObjectInputStream(new ByteArrayInputStream(serialized))) {
            deserialized = (EnumSingleton) ois.readObject();
        }
        System.out.println("Same after deserialization: " + (s1 == deserialized)); // true!
    }
}
```

## INTERVIEW

**Q1: How are enums different from using integer constants?**

> **Strong answer:** Integer constants (like `static final int STATUS_ACTIVE = 1`) have no type safety — you can pass any int where a status is expected. Enums are their own type — the compiler rejects passing a wrong type. Enums also provide: `name()` and `toString()` for readable output, `values()` to iterate all constants, ability to add fields/methods, natural use in switch expressions with exhaustiveness checking. Also, adding a new constant to an enum with exhaustive switch causes compile errors at all switch sites — forcing you to handle it. With ints, you'd miss it silently.

**Q2: Can enums have constructors and methods in Java?**

> **Strong answer:** Yes — Java enums are full classes. They can have constructors (always private), fields, instance methods, abstract methods (each constant provides its own implementation), static methods, and can implement interfaces. The constructor is called once per constant at class loading time. This makes Java enums far more powerful than enums in C/C++ or even C# — you can attach behavior to each constant, creating type-safe strategy patterns or state machines.

**Q3: Why is enum the best way to implement the Singleton pattern?**

> **Strong answer:** Three guarantees regular singleton implementations can't easily provide: (1) Thread-safety — JVM class loading is thread-safe; enum is initialized exactly once. (2) Serialization-safety — regular singletons need special `readResolve()` method; enum handles it automatically (deserialization returns the same instance). (3) Reflection-safety — `Constructor.newInstance()` on an enum throws `IllegalArgumentException`. With a regular class, a determined attacker could call the private constructor via reflection. Enum blocks all three attack vectors.

**Q4: What is EnumSet and when would you use it over HashSet?**

> **Strong answer:** `EnumSet` is a specialized `Set` implementation for enum types, backed by a bit vector (single `long` for enums with ≤64 constants). Operations like `add`, `contains`, iteration are O(1) and extremely fast — just bit operations. Memory: one `long` vs HashSet's 48 bytes per entry. Use EnumSet whenever you need a set of enum values. Common pattern: permission systems, feature flags, day-of-week sets. `EnumSet.of(...)`, `EnumSet.range(...)`, `EnumSet.allOf(...)`, `EnumSet.noneOf(...)` are all factory methods.

## FEYNMAN CHECK

An enum in Java is like a closed club with a fixed membership list. The members (constants) are known at compile time — you can't add new ones at runtime. Unlike a list of numbers, each member has a real identity, can carry data (like a name badge with information), and can have behaviors (like knowing how to introduce themselves). The compiler ensures you never accidentally let a non-member into the club.
