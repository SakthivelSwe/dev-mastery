# Topological Sort

Topological Sort is a linear ordering of vertices in a Directed Acyclic Graph (DAG) such that for every directed edge $u \to v$, vertex $u$ comes before $v$ in the ordering.

## 1. Core Theory & Mathematical Foundations

### The DAG Constraint
A topological sort is **only** defined on Directed Acyclic Graphs (DAGs). 
* *Proof*: Suppose a graph has a cycle $v_1 \to v_2 \to \dots \to v_k \to v_1$. In any topological ordering, $v_1$ must appear before $v_2$, which must appear before $v_k$, which must appear before $v_1$. This is a contradiction ($v_1 < v_1$), making ordering impossible.

### Kahn’s Algorithm (BFS-based)
Kahn’s algorithm operates on the concept of **in-degrees** (the number of incoming edges to a node).
1. Compute the in-degree of every vertex.
2. Initialize a queue with all vertices having an in-degree of `0` (nodes with no dependencies).
3. While the queue is not empty:
   * Dequeue a vertex $u$ and append it to the topological order.
   * For each outgoing edge $u \to v$, decrement the in-degree of $v$.
   * If $v$'s in-degree becomes `0`, enqueue $v$.

### DFS Approach (Reverse Post-Order)
Using DFS, we traverse the graph and push vertices onto a stack (or list) only when they are finalized (i.e., when the recursive call for that node returns). The final topological order is the reverse of this post-order sequence.

### Cycle Detection
* **In Kahn's Algorithm**: If the final topological order contains fewer than $V$ vertices, a cycle exists. This is because any node involved in a cycle (or downstream of one) will never have its in-degree reduced to `0`, preventing it from entering the queue.
* **In DFS**: If we encounter a node currently in the recursion stack (a "Gray" node in tri-color marking), a cycle is detected.

## 2. Step-by-Step Concrete Trace (Kahn's Algorithm)

Consider the graph:
```text
0 -> 1, 0 -> 2
1 -> 3
2 -> 3
```

### Initial State
* `In-degrees`: `[0: 0, 1: 1, 2: 1, 3: 2]`
* `Queue`: `[0]`
* `Order`: `[]`

### Iterations

| Step | Dequeued Node | Neighbors Affected | In-degree Updates | Queue State | Order State |
|:---|:---|:---|:---|:---|:---|
| 1 | `0` | `1`, `2` | `1` becomes 0, `2` becomes 0 | `[1, 2]` | `[0]` |
| 2 | `1` | `3` | `3` becomes 1 | `[2]` | `[0, 1]` |
| 3 | `2` | `3` | `3` becomes 0 | `[3]` | `[0, 1, 2]` |
| 4 | `3` | None | None | `[]` | `[0, 1, 2, 3]` |
