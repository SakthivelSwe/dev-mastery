# Basic Types — string, number, boolean, any, unknown | Level: 2

TypeScript maps directly to JavaScript's runtime primitive types while adding explicit safety constructs. Understanding the boundaries between these types is critical for writing deterministic programs.

---

### Primitive Types in TypeScript

JavaScript has seven primitive types. TypeScript provides corresponding type annotations for them:

```typescript
const username: string = "Jane Doe";
const age: number = 42; // Handles integers, floats, NaN, and Infinity (IEEE 754)
const isActive: boolean = true;
const bigIntVal: bigint = 9007199254740991n; // Requires target ES2020+
const uniqueKey: symbol = Symbol("key");
```

---

### Special Types: `any` vs. `unknown`

TypeScript provides two top-level types (types that can represent any value), but they have vastly different safety implications.

#### `any` (The Escape Hatch)
The `any` type completely opts out of type checking. It allows you to assign any value to it, assign it to any type, and access any property on it without compile-time errors. **Avoid using `any` in production code.**

```typescript
let looseValue: any = { greet: () => "Hello" };
looseValue.nonExistentMethod(); // Compiles fine, crashes at runtime!
let myNumber: number = looseValue; // Allowed, potentially breaking 'myNumber' type safety
```

#### `unknown` (The Safe Alternative)
The `unknown` type represents any value, but it is **type-safe**. You cannot interact with an `unknown` value (call methods, read properties, pass it to typed functions) until you perform some form of **type narrowing** to prove its type.

```typescript
let secureValue: unknown = "Hello World";

// Compile-time error! TS doesn't know if it has a 'length' property.
const len1 = secureValue.length; 
// error TS18046: 'secureValue' is of type 'unknown'.

// Correct approach: Narrow the type first
if (typeof secureValue === "string") {
  const len2 = secureValue.length; // Safe and allowed!
}
```

### Control Flow Types: `void` vs. `never`

#### `void`
`void` represents the absence of a value, typically used as the return type of functions that do not return a value. In JavaScript, a function without a return statement implicitly returns `undefined`. In TypeScript, `void` is not identical to `undefined`, though a `void`-returning function can return `undefined` safely.

```typescript
function logMessage(msg: string): void {
  console.log(msg);
}
```

#### `never`
`never` represents the type of values that **never occur**. It is used for functions that never return (e.g., they throw an exception or enter an infinite loop) or to represent impossible states in type narrowing.

```typescript
function throwError(msg: string): never {
  throw new Error(msg);
}

function infiniteLoop(): never {
  while (true) {}
}
```

### Type Annotations vs. Type Inference

You do not need to annotate every variable. TypeScript is highly capable of inferring types based on the assigned value (contextual typing).

```typescript
// Explicit Annotation
let explicitUser: string = "Alice";

// Type Inference (Preferred)
let inferredUser = "Alice"; // TypeScript automatically infers type 'string'
```

*Rule of thumb*: Let TypeScript infer types for local variables. Use explicit annotations for function parameters, return types, and public API boundaries to enforce contracts.

### `null` and `undefined` with `strictNullChecks`

In non-strict mode, `null` and `undefined` can be assigned to any type (e.g., you could assign `null` to a `number` variable). With `"strict": true` (which enables `strictNullChecks`), they are treated as distinct, independent types.

```typescript
let count: number = 10;
count = null; 
// error TS2322: Type 'null' is not assignable to type 'number'.
```

#### Structural and Conceptual Difference
*   `undefined`: Represents a variable that has been declared but not yet assigned a value, or a missing property.
*   `null`: Represents an intentional absence of any object value.

```typescript
type UserResponse = {
  id: string;
  middleName: string | null; // Intentional absence
  phoneNumber?: string;     // Optional (implicitly can be undefined)
};
```

### Code Compilation and Type Erasure

```typescript
// TypeScript
const compute = (val: unknown): number => {
  if (typeof val === "number") {
    return val * 2;
  }
  return 0;
};
```

```javascript
// Compiled JavaScript (Target: ES2022)
const compute = (val) => {
    if (typeof val === "number") {
        return val * 2;
    }
    return 0;
};
```
