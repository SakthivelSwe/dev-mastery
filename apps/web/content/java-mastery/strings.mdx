## WHY

Strings are the most used type in Java — every application handles them. You MUST know:

1. **Why String is immutable** — thread safety, String pool, security (passwords, file paths)
2. **String Pool trap** — `==` vs `.equals()` causes production bugs
3. **StringBuilder vs String concatenation** — `+` in a loop = O(n²) time and memory waste
4. **String methods** — 40+ methods; most interviews test at least 5
5. **Real-world** — HTTP parsing, JSON, user input validation all use strings constantly

## THEORY

### String Immutability

`String` is **immutable** — once created, its value cannot change. Here's why:

```java
String s = "hello";
s.toUpperCase();       // does NOT change s
System.out.println(s); // still "hello"

s = s.toUpperCase();   // creates NEW String object "HELLO"
System.out.println(s); // "HELLO"
```

**Why immutable?**
- **Security**: File paths, class names, network connections passed as strings can't be tampered with
- **Thread safety**: Multiple threads can share the same String without synchronization
- **String Pool**: Pooling only works if strings can't change (if they could, changing one pooled string would corrupt all references to it)
- **HashCode caching**: `String.hashCode()` is cached on first call — safe because value never changes

### String Pool (String Intern Pool)

The JVM maintains a pool of String literals in the Heap (Metaspace area in older JVMs):

```
String s1 = "Java";       // Creates "Java" in pool
String s2 = "Java";       // REUSES existing pool entry
String s3 = new String("Java"); // Creates new object on heap (outside pool)
String s4 = s3.intern();  // Returns pool reference

s1 == s2  → true  (same pool object)
s1 == s3  → false (different heap objects)
s1 == s4  → true  (intern() returned pool reference)
s1.equals(s3) → true (same content)
```

**Memory diagram:**
```
String Pool (Heap):
┌──────────┐
│  "Java"  │ ← s1, s2, s4 all point here
└──────────┘

Heap (regular):
┌──────────┐
│  "Java"  │ ← s3 points here (separate object)
└──────────┘
```

### String vs StringBuilder vs StringBuffer

| | String | StringBuilder | StringBuffer |
|--|--------|---------------|--------------|
| **Mutability** | Immutable | Mutable | Mutable |
| **Thread safe** | Yes (immutable) | No | Yes (synchronized) |
| **Performance** | Slowest for concat | Fastest | Slower than SB |
| **Use when** | Fixed/few concatenations | Single thread, many changes | Multi-thread string building |

**Performance example:**
```java
// BAD — O(n²) — creates new String each iteration
String result = "";
for (int i = 0; i < 10000; i++) {
    result += i;  // creates 10000 intermediate String objects!
}

// GOOD — O(n) — mutates the same StringBuilder
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 10000; i++) {
    sb.append(i);
}
String result = sb.toString();
```

### String Comparison — The Most Common Bug

```java
// WRONG — compares object references
if (userInput == "admin") { ... }   // Almost always false!

// RIGHT — compares content
if ("admin".equals(userInput)) { ... }  // "admin" first to avoid NPE

// WRONG — case sensitive
if (userInput.equals("Admin")) { ... }

// RIGHT — case insensitive
if ("admin".equalsIgnoreCase(userInput)) { ... }
```

```java
// StringBasics.java
public class StringBasics {
    public static void main(String[] args) {
        String name = "DevMastery";

        // Length
        System.out.println("Length: " + name.length()); // 10

        // Access characters
        System.out.println("First char: " + name.charAt(0)); // D
        System.out.println("Last char: " + name.charAt(name.length() - 1)); // y

        // Case conversion
        System.out.println("Upper: " + name.toUpperCase()); // DEVMASTERY

        // Substring
        System.out.println("Sub: " + name.substring(0, 3)); // Dev

        // Contains, starts, ends
        System.out.println("Contains 'ast': " + name.contains("ast")); // true
        System.out.println("Starts 'Dev': " + name.startsWith("Dev")); // true
        System.out.println("Ends 'ery': " + name.endsWith("ery"));     // true

        // IndexOf
        System.out.println("Index of 'M': " + name.indexOf('M')); // 3

        // Replace
        System.out.println(name.replace("Dev", "Java")); // JavaMastery

        // Concat
        String greeting = "Hello, " + name + "!";
        System.out.println(greeting); // Hello, DevMastery!

        // Trim
        String padded = "   hello   ";
        System.out.println("|" + padded.trim() + "|"); // |hello|
    }
}
```

### Level 2 — Elementary: String Comparison & Pool

```java
// StringComparison.java
public class StringComparison {
    public static void main(String[] args) {
        // String pool demonstration
        String s1 = "Java";
        String s2 = "Java";
        String s3 = new String("Java");
        String s4 = s3.intern();

        System.out.println("s1 == s2: " + (s1 == s2));      // true  (same pool object)
        System.out.println("s1 == s3: " + (s1 == s3));      // false (different object)
        System.out.println("s1 == s4: " + (s1 == s4));      // true  (intern returns pool)
        System.out.println("s1.equals(s3): " + s1.equals(s3)); // true (same content)

        // The bug: using == for string comparison
        String input = new String("admin"); // simulates user input
        if (input == "admin") {
            System.out.println("This won't print!"); // WRONG
        }
        if ("admin".equals(input)) {
            System.out.println("Correct way!"); // RIGHT
        }
        
        // Null-safe comparison
        String nullStr = null;
        // nullStr.equals("test")  → NullPointerException!
        System.out.println("admin".equals(nullStr));      // false — safe
        System.out.println(java.util.Objects.equals(nullStr, "admin")); // false — also safe

        // compareTo
        System.out.println("apple".compareTo("banana")); // negative (a < b)
        System.out.println("banana".compareTo("apple")); // positive (b > a)
        System.out.println("apple".compareTo("apple"));  // 0 (equal)

        // Case-insensitive compare
        System.out.println("Hello".equalsIgnoreCase("hello")); // true
        System.out.println("Hello".compareToIgnoreCase("hello")); // 0
    }
}
```

### Level 3 — Intermediate: StringBuilder & String Manipulation Algorithms

```java
// StringAlgorithms.java
public class StringAlgorithms {

    // 1. Reverse a string using StringBuilder
    public static String reverse(String s) {
        return new StringBuilder(s).reverse().toString();
    }

    // 2. Check palindrome
    public static boolean isPalindrome(String s) {
        String cleaned = s.toLowerCase().replaceAll("[^a-z0-9]", "");
        int left = 0, right = cleaned.length() - 1;
        while (left < right) {
            if (cleaned.charAt(left) != cleaned.charAt(right)) return false;
            left++;
            right--;
        }
        return true;
    }

    // 3. Count occurrences of a character
    public static int countChar(String s, char c) {
        int count = 0;
        for (char ch : s.toCharArray()) {
            if (ch == c) count++;
        }
        return count;
    }

    // 4. First non-repeating character
    public static char firstNonRepeating(String s) {
        java.util.LinkedHashMap<Character, Integer> freq = new java.util.LinkedHashMap<>();
        for (char c : s.toCharArray()) {
            freq.merge(c, 1, Integer::sum);
        }
        for (java.util.Map.Entry<Character, Integer> e : freq.entrySet()) {
            if (e.getValue() == 1) return e.getKey();
        }
        return '\0';
    }

    // 5. Anagram check
    public static boolean isAnagram(String s, String t) {
        if (s.length() != t.length()) return false;
        int[] count = new int[26];
        for (char c : s.toCharArray()) count[c - 'a']++;
        for (char c : t.toCharArray()) {
            count[c - 'a']--;
            if (count[c - 'a'] < 0) return false;
        }
        return true;
    }

    public static void main(String[] args) {
        System.out.println(reverse("hello"));                      // olleh
        System.out.println(isPalindrome("A man a plan a canal Panama")); // true
        System.out.println(countChar("hello world", 'l'));         // 3
        System.out.println(firstNonRepeating("leetcode"));         // l
        System.out.println(isAnagram("listen", "silent"));         // true

        // StringBuilder performance demo
        long start = System.currentTimeMillis();
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < 100_000; i++) sb.append("x");
        System.out.println("StringBuilder: " + (System.currentTimeMillis() - start) + "ms");
    }
}
```

```java
import java.util.*;

public class StringAdvanced {

        String csv = "a,b,,c,";

        System.out.println(Arrays.toString(parts)); // [a, b, , c, ]
        System.out.println(parts.length); // 5

        // 4. Regex — validate email
        String emailRegex = "^[a-zA-Z0-9+_.-]+@[a-zA-Z0-9.-]+$";
        System.out.println("test@email.com".matches(emailRegex)); // true
        System.out.println("invalid-email".matches(emailRegex));  // false

        // 5. Regex — extract all numbers from string
        String text = "Order 123 has 45 items worth $678.90";
        Matcher m = Pattern.compile("\\d+\\.?\\d*").matcher(text);
        while (m.find()) {
            System.out.print(m.group() + " "); // 123 45 678 90
        }
        System.out.println();

        // 6. Java 11+ methods
        String multiline = "  hello  \n  world  \n  java  ";
        multiline.lines()
                 .map(String::strip)
                 .filter(s -> !s.isBlank())
                 .forEach(System.out::println);

        // 7. String.chars() — stream of characters
        "hello".chars()
               .filter(c -> "aeiou".indexOf(c) >= 0)
               .mapToObj(c -> String.valueOf((char) c))
               .forEach(System.out::print); // eo
        System.out.println();

        // 8. repeat (Java 11)
        System.out.println("=-".repeat(20)); // separator line

        // 9. Formatted (Java 15)
        String result = "%-10s: %d".formatted("Score", 100);
        System.out.println(result); // Score     : 100
    }
}
```

### Level 5 — Expert: String Interning, Memory Optimization, Custom Tokenizer

```java
// StringExpert.java
import java.util.*;
import java.util.concurrent.*;
import java.util.stream.*;

public class StringExpert {

    /**
     * Custom CSV tokenizer — handles quoted fields, escaped commas.
     * Production-grade, handles: "field1","field with, comma","field3"
     */
    public static List<String> parseCsv(String line) {
        List<String> fields = new ArrayList<>();
        boolean inQuotes = false;
        StringBuilder current = new StringBuilder();

        for (int i = 0; i < line.length(); i++) {
            char c = line.charAt(i);
            if (c == '"') {
                if (inQuotes && i + 1 < line.length() && line.charAt(i + 1) == '"') {
                    current.append('"'); // escaped quote
                    i++;
                } else {
                    inQuotes = !inQuotes;
                }
            } else if (c == ',' && !inQuotes) {
                fields.add(current.toString().trim());
                current.setLength(0); // reset without new allocation
            } else {
                current.append(c);
            }
        }
        fields.add(current.toString().trim());
        return fields;
    }

    /**
     * String interning benchmark — shows memory savings.
     * In large-scale systems (e.g., reading millions of records),
     * intern() can reduce memory by 10x for repeated strings.
     */
    public static void internBenchmark() {
        int count = 1_000_000;

        // Without intern — 1M separate String objects
        long before = Runtime.getRuntime().totalMemory() - Runtime.getRuntime().freeMemory();
        String[] withoutIntern = new String[count];
        for (int i = 0; i < count; i++) {
            withoutIntern[i] = new String("status_active"); // each is a new object
        }
        long after = Runtime.getRuntime().totalMemory() - Runtime.getRuntime().freeMemory();
        System.out.printf("Without intern: %,d bytes%n", after - before);

        // With intern — all reference same pool object
        before = Runtime.getRuntime().totalMemory() - Runtime.getRuntime().freeMemory();
        String[] withIntern = new String[count];
        for (int i = 0; i < count; i++) {
            withIntern[i] = "status_active".intern(); // all share one object
        }
        after = Runtime.getRuntime().totalMemory() - Runtime.getRuntime().freeMemory();
        System.out.printf("With intern:    %,d bytes%n", after - before);
    }

    /**
     * Pattern precompilation — always compile patterns outside loops.
     * Pattern.compile() is expensive; reuse the compiled Pattern object.
     */
    private static final Pattern IP_PATTERN =
        Pattern.compile("^((25[0-5]|2[0-4]\\d|[01]?\\d\\d?)\\.){3}(25[0-5]|2[0-4]\\d|[01]?\\d\\d?)$");

    public static boolean isValidIp(String ip) {
        return IP_PATTERN.matcher(ip).matches();
    }

    public static void main(String[] args) {
        // CSV parsing
        String csv = "\"Alice\",\"New York, NY\",\"30\",\"Engineer\"";
        System.out.println(parseCsv(csv)); // [Alice, New York, NY, 30, Engineer]

        // IP validation
        System.out.println(isValidIp("192.168.1.1"));  // true
        System.out.println(isValidIp("999.1.1.1"));    // false
        System.out.println(isValidIp("256.0.0.1"));    // false

        // String deduplication (Java 8u20+ G1GC feature)
        // -XX:+UseStringDeduplication — JVM automatically deduplicates heap strings

        // String compression (Java 9+ Compact Strings)
        // Latin-1 strings stored as byte[] (1 byte/char) instead of char[] (2 bytes/char)
        // Automatic — saves ~30% memory for typical applications
        System.out.println(isValidIp("10.0.0.1"));    // true
    }
}
```

### SQL Query Builder (StringBuilder pattern)

```java
public String buildQuery(String table, List<String> conditions, int limit) {
    StringBuilder sql = new StringBuilder("SELECT * FROM ");
    sql.append(table);
    if (!conditions.isEmpty()) {
        sql.append(" WHERE ");
        sql.append(String.join(" AND ", conditions));
    }
    if (limit > 0) {
        sql.append(" LIMIT ").append(limit);
    }
    return sql.toString();
}
// Result: "SELECT * FROM users WHERE active = 1 AND age > 18 LIMIT 50"
```

## INTERVIEW

**Q1: Why is String immutable in Java?**

> **Strong answer:** Three reasons. (1) Security — strings are used as keys for network connections, file paths, class names. If mutable, an attacker could change them after a security check. (2) Thread safety — immutable objects are inherently thread-safe without synchronization. (3) String Pool — the JVM reuses String literals in a pool. If strings were mutable, changing one pooled string would corrupt all references to it. Also, `hashCode()` is cached because the value never changes.

**Q2: What is the difference between `==` and `.equals()` for Strings?**

> **Strong answer:** `==` compares object references (memory addresses). `.equals()` compares the content (character by character). String literals are stored in the String Pool, so two literals with the same value share the same object — `==` returns true. But `new String("hello")` creates a new heap object outside the pool — `==` returns false even if content is identical. Always use `.equals()` for content comparison. Use `"literal".equals(variable)` (Yoda condition) to avoid NullPointerException.

**Q3: When would you use StringBuilder over String?**

> **Strong answer:** When doing repeated concatenation in a loop. String concatenation with `+` creates a new String object on every iteration — O(n²) time and memory. StringBuilder mutates the same internal buffer — O(n). Rule: use String for simple fixed concatenation (the compiler optimizes it anyway); use StringBuilder for loops, dynamic building, or when you need methods like `insert()`, `delete()`, `reverse()`. Use StringBuffer only when multiple threads are accessing the same builder (rare in practice — prefer thread-local StringBuilder).

**Q4: What is String.intern() and when should you use it?**

> **Strong answer:** `intern()` returns the canonical (pooled) instance of a string. If the pool already contains a string equal to this one, it returns that. Otherwise it adds it to the pool and returns it. Use case: when reading millions of records where certain strings repeat (e.g., country codes, status values), interning saves memory by ensuring all equal strings share the same object. Caution: the pool is in Metaspace — internning too many unique strings can cause `OutOfMemoryError: Metaspace`. Java's G1GC has `-XX:+UseStringDeduplication` as an automatic alternative.

**Q5: How does String concatenation work internally in Java?**

> **Strong answer:** The Java compiler optimizes string concatenation. In Java 8 and below, `a + b + c` is compiled to `new StringBuilder().append(a).append(b).append(c).toString()`. In Java 9+, the compiler uses `invokedynamic` with `StringConcatFactory`, which generates optimized concatenation strategies at runtime based on the types involved — avoiding StringBuilder entirely for some cases and using more efficient strategies. This is why concatenation in loops is still bad (each iteration = new concatenation chain), but a single expression like `"Hello " + name + "!"` is fine.

## FEYNMAN CHECK

A String in Java is like a printed book — once printed, you can't change it. If you want to "add a chapter," you need to print a whole new book with the chapter added. That's why Java keeps a library (String Pool) of common books — if two people want the same book, they get the same copy rather than two identical copies. StringBuilder is like writing on a whiteboard — you can freely erase and add text before printing your final book.
