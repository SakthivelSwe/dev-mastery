## WHY
When building large systems, you often find overlapping behaviors. A `Dog`, `Cat`, and `Bird` all need an `eat()` method. Copy-pasting the `eat()` logic into 50 different animal classes is a maintenance disaster. If the feeding logic changes, you have to find and update 50 files. 

Inheritance solves this by establishing an "IS-A" relationship. By creating a parent `Animal` class, the child classes automatically inherit all non-private state and behavior. This promotes massive code reuse and creates hierarchical classifications (e.g., an `Admin` IS-A `User`).

## THEORY
### vtable (Virtual Method Table)
At the JVM level, how does a child class override a parent's method? The JVM uses a **Virtual Method Table (vtable)**. Every class has a vtable in the Metaspace. It's an array of pointers to the actual method bytecode. 
When a child inherits from a parent, it copies the parent's vtable. If the child `Overrides` a method, the JVM overwrites the pointer in the child's vtable to point to the new child bytecode. When the method is invoked at runtime, the JVM looks up the pointer in the exact object's vtable, ensuring the correct overridden method is called (Dynamic Dispatch).

### Constructor Chaining and `super()`
When you instantiate a child object (`new Dog()`), the JVM MUST construct the parent part of the object first. Therefore, the first line of every constructor is implicitly `super()`, which calls the parent's constructor. This chains all the way up to `java.lang.Object`.

### The Diamond Problem
Java strictly forbids multiple inheritance for classes (a class cannot extend two classes). If `A` extends `B` and `C`, and both have a `start()` method, the compiler wouldn't know which `start()` to inherit. This is the Diamond Problem, and Java avoids it entirely by enforcing single class inheritance.

## CODE

### Level 1 — Beginner
```java
class Animal {
    public void eat() { System.out.println("Eating..."); }
}

// Dog IS-A Animal
class Dog extends Animal {
    public void bark() { System.out.println("Woof!"); }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.eat(); // Inherited from Animal
        d.bark();
    }
}
```

### Level 2 — Building Blocks
```java
class User {
    protected String role;
    
    public User(String role) {
        this.role = role;
    }
}

class Admin extends User {
    public Admin() {
        // MUST call parent constructor first!
        super("ADMIN"); 
    }
}
```

### Level 3 — Intermediate
```java
class Parent {
    // Cannot be overridden by children
    public final void strictMethod() { }
}

// Cannot be inherited from at all
final class ImmutableClass { }
```

### Level 4 — Advanced
```java
// Method Hiding (Static methods)
class Base {
    public static void print() { System.out.println("Base"); }
}
class Child extends Base {
    // This does NOT override, it HIDES the parent method
    public static void print() { System.out.println("Child"); }
}
```

## REAL_WORLD
In standard Enterprise Java, deep inheritance trees are actually considered an **Anti-Pattern**. 
"Composition over Inheritance" is the modern standard. Deep hierarchies (e.g., `Admin -> Moderator -> User -> BaseEntity -> Object`) become extremely rigid and brittle. If you change a field in `BaseEntity`, it ripples through the entire system and breaks child classes unexpectedly (the Fragile Base Class problem).

Spring Boot avoids inheritance for business logic. Instead of a `UserService` extending a `DatabaseService`, the `UserService` uses **Composition** by `@Autowired`-injecting the `UserRepository` into it. 

## INTERVIEW
**Q: Why doesn't Java support multiple inheritance?**
A: To avoid the Diamond Problem. If Class C extended Class A and Class B, and both A and B defined a method with the exact same signature but different implementations, the JVM would not know which method C should inherit. 

**Q: What is the purpose of the `super` keyword?**
A: `super` is used to access members of the direct parent class. `super()` invokes the parent's constructor (and must be the first statement in the child constructor), while `super.methodName()` allows a child to call the parent's version of an overridden method.
