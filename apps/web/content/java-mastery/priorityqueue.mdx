## WHY

A regular Queue processes items in insertion order (FIFO). PriorityQueue processes in priority order — the most important item always comes out first:

1. **Task scheduling** — Process high-priority tasks first, regardless of when they arrived
2. **Dijkstra's algorithm** — Always expand the shortest-distance node next
3. **Top-K elements** — Efficient O(n log k) solution for finding top-K in a stream
4. **Merge K sorted lists** — Merge in O(n log k) instead of O(nk)
5. **Interview** — PriorityQueue with custom Comparator is asked in 50%+ of coding interviews

## THEORY

### Binary Heap — The Internal Structure

A PriorityQueue is backed by a **Binary Min-Heap** (default) or Max-Heap (with `Comparator.reverseOrder()`):

```
Binary Min-Heap — Parent is ALWAYS ≤ both children:

          1         ← always the minimum (index 0)
         / \
        3   2
       / \ / \
      7  5 4  8

Array representation: [1, 3, 2, 7, 5, 4, 8]
                       [0][1][2][3][4][5][6]

Children of index i: left = 2i+1, right = 2i+2
Parent of index i: (i-1)/2
```

### Time Complexity

| Operation | Complexity | Why |
|-----------|-----------|-----|
| `peek()` | **O(1)** | Minimum is always at index 0 |
| `add(e)` | **O(log n)** | Sift up: swap with parent until heap property restored |
| `poll()` | **O(log n)** | Remove root, put last element at root, sift down |
| `remove(e)` | **O(n)** | Must find element first (no indexing) |
| `contains(e)` | **O(n)** | Linear scan |
| `size()` | **O(1)** | Stored field |

**Key insight:** `peek()` is O(1) — always use `peek()` to look at priority element without removing.

### Min-Heap vs Max-Heap

```java
// Default: MIN-HEAP (smallest element has highest priority)
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.add(5); minHeap.add(1); minHeap.add(3);
System.out.println(minHeap.poll()); // 1 (minimum)

// MAX-HEAP (largest element has highest priority)
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
maxHeap.add(5); maxHeap.add(1); maxHeap.add(3);
System.out.println(maxHeap.poll()); // 5 (maximum)

// Custom priority
PriorityQueue<Task> taskQueue = new PriorityQueue<>(
    Comparator.comparingInt(Task::getPriority) // lower number = higher priority
);
```

### PriorityQueue API

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();

// Add elements
pq.add(5);        // throws exception if capacity exceeded (rare)
pq.offer(3);      // returns false if capacity exceeded (prefer this)

// Inspect top
pq.peek();        // returns min without removing, null if empty
pq.element();     // returns min without removing, throws if empty

// Remove top
pq.poll();        // removes and returns min, null if empty
pq.remove();      // removes and returns min, throws if empty

// Other
pq.size()
pq.isEmpty()
pq.contains(x)   // O(n)
pq.remove(x)      // O(n) remove specific element
pq.toArray()      // array snapshot (NOT sorted!)
```

**Warning:** Iterating a PriorityQueue does NOT iterate in priority order:

```java
PriorityQueue<Integer> pq = new PriorityQueue<>(List.of(3,1,4,1,5));
System.out.println(pq); // [1, 1, 4, 3, 5] — heap array order, not sorted!

// To get in sorted order, use poll():
while (!pq.isEmpty()) System.out.print(pq.poll() + " "); // 1 1 3 4 5
```

## CODE

### Level 1 — Beginner: Basic PriorityQueue Operations

```java
// PriorityQueueBasics.java
import java.util.*;

public class PriorityQueueBasics {
    public static void main(String[] args) {
        // Min-heap (default)
        PriorityQueue<Integer> pq = new PriorityQueue<>();
        pq.offer(5);
        pq.offer(1);
        pq.offer(8);
        pq.offer(3);
        pq.offer(7);

        System.out.println("Peek (min): " + pq.peek()); // 1
        System.out.println("Size: " + pq.size());       // 5
        System.out.println("Internal array (not sorted): " + pq);

        // Poll extracts in sorted (ascending) order
        System.out.print("Poll order: ");
        while (!pq.isEmpty()) {
            System.out.print(pq.poll() + " "); // 1 3 5 7 8
        }
        System.out.println();

        // Max-heap
        PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
        maxHeap.addAll(List.of(5, 1, 8, 3, 7));
        System.out.print("\nMax-heap poll order: ");
        while (!maxHeap.isEmpty()) {
            System.out.print(maxHeap.poll() + " "); // 8 7 5 3 1
        }
        System.out.println();

        // String priority queue
        PriorityQueue<String> stringPQ = new PriorityQueue<>();
        stringPQ.addAll(List.of("banana", "apple", "cherry", "date"));
        System.out.print("\nString PQ (alphabetical): ");
        while (!stringPQ.isEmpty()) System.out.print(stringPQ.poll() + " ");
        // apple banana cherry date
        System.out.println();
    }
}
```

### Level 2 — Elementary: Custom Priority with Comparator

```java
// CustomPriority.java
import java.util.*;

public class CustomPriority {

    record Task(String name, int priority, long arrivalTime) {}
    record Patient(String name, String condition, int severity) {} // 1=critical, 5=minor

    public static void main(String[] args) {
        // Task queue: lower priority number = processed first (like OS scheduling)
        PriorityQueue<Task> taskQueue = new PriorityQueue<>(
            Comparator.comparingInt(Task::priority)
                      .thenComparingLong(Task::arrivalTime) // tie-break by arrival
        );

        taskQueue.offer(new Task("Database Backup", 3, 1000L));
        taskQueue.offer(new Task("Critical Error Fix", 1, 2000L));
        taskQueue.offer(new Task("Send Report Email", 4, 1500L));
        taskQueue.offer(new Task("User Authentication", 1, 1000L)); // same priority as Critical

        System.out.println("Processing tasks by priority:");
        while (!taskQueue.isEmpty()) {
            Task t = taskQueue.poll();
            System.out.printf("  [P%d] %s%n", t.priority(), t.name());
        }
        // [P1] User Authentication (arrived first at t=1000)
        // [P1] Critical Error Fix (same priority, arrived later)
        // [P3] Database Backup
        // [P4] Send Report Email

        // ER triage: severity 1 = most critical
        PriorityQueue<Patient> er = new PriorityQueue<>(
            Comparator.comparingInt(Patient::severity));

        er.offer(new Patient("Alice", "Broken arm", 3));
        er.offer(new Patient("Bob", "Heart attack", 1));
        er.offer(new Patient("Charlie", "Mild cough", 5));
        er.offer(new Patient("Dave", "Appendicitis", 2));

        System.out.println("\nER triage order:");
        while (!er.isEmpty()) {
            Patient p = er.poll();
            System.out.printf("  [Severity %d] %s — %s%n", p.severity(), p.name(), p.condition());
        }
        // Severity 1: Bob (heart attack)
        // Severity 2: Dave (appendicitis)
        // Severity 3: Alice (broken arm)
        // Severity 5: Charlie (cough)
    }
}
```

### Level 3 — Intermediate: Top-K and Merge K Sorted Lists

```java
// PriorityQueueAlgorithms.java
import java.util.*;
import java.util.stream.*;

public class PriorityQueueAlgorithms {

    // Top-K most frequent elements — O(n log k)
    public static int[] topKFrequent(int[] nums, int k) {
        Map<Integer, Integer> freq = new HashMap<>();
        for (int n : nums) freq.merge(n, 1, Integer::sum);

        // Min-heap of size k — maintains top k frequent
        PriorityQueue<Map.Entry<Integer, Integer>> minHeap =
            new PriorityQueue<>(Comparator.comparingInt(Map.Entry::getValue));

        for (Map.Entry<Integer, Integer> entry : freq.entrySet()) {
            minHeap.offer(entry);
            if (minHeap.size() > k) minHeap.poll(); // evict least frequent
        }

        return minHeap.stream().mapToInt(Map.Entry::getKey).toArray();
    }

    // Find K-th largest element — O(n log k)
    public static int findKthLargest(int[] nums, int k) {
        PriorityQueue<Integer> minHeap = new PriorityQueue<>(); // size k min-heap
        for (int n : nums) {
            minHeap.offer(n);
            if (minHeap.size() > k) minHeap.poll();
        }
        return minHeap.peek(); // top of size-k min-heap = k-th largest
    }

    // Merge K sorted arrays — O(n log k) where n = total elements
    public static int[] mergeKSorted(int[][] arrays) {
        // Entry: (value, arrayIndex, elementIndex)
        PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));

        for (int i = 0; i < arrays.length; i++) {
            if (arrays[i].length > 0) pq.offer(new int[]{arrays[i][0], i, 0});
        }

        List<Integer> result = new ArrayList<>();
        while (!pq.isEmpty()) {
            int[] curr = pq.poll();
            result.add(curr[0]);
            int arrIdx = curr[1], elemIdx = curr[2];
            if (elemIdx + 1 < arrays[arrIdx].length) {
                pq.offer(new int[]{arrays[arrIdx][elemIdx+1], arrIdx, elemIdx+1});
            }
        }

        return result.stream().mapToInt(Integer::intValue).toArray();
    }

    public static void main(String[] args) {
        System.out.println("Top-2 frequent: " +
            Arrays.toString(topKFrequent(new int[]{1,1,1,2,2,3}, 2))); // [1, 2]

        System.out.println("3rd largest in [3,2,1,5,6,4]: " +
            findKthLargest(new int[]{3,2,1,5,6,4}, 3)); // 4

        int[][] arrays = {{1,4,7},{2,5,8},{3,6,9}};
        System.out.println("Merged: " + Arrays.toString(mergeKSorted(arrays)));
        // [1, 2, 3, 4, 5, 6, 7, 8, 9]
    }
}
```

### Level 4 — Advanced: Dijkstra's Algorithm with PriorityQueue

```java
// DijkstraAlgorithm.java
import java.util.*;

public class DijkstraAlgorithm {

    // Weighted graph as adjacency list
    // edges[u] = list of (v, weight) pairs
    static int[] dijkstra(List<List<int[]>> graph, int src) {
        int n = graph.size();
        int[] dist = new int[n];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[src] = 0;

        // Min-heap: (distance, node)
        PriorityQueue<int[]> pq = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
        pq.offer(new int[]{0, src});

        while (!pq.isEmpty()) {
            int[] curr = pq.poll();
            int d = curr[0], u = curr[1];

            if (d > dist[u]) continue; // stale entry — skip

            for (int[] edge : graph.get(u)) {
                int v = edge[0], w = edge[1];
                int newDist = dist[u] + w;
                if (newDist < dist[v]) {
                    dist[v] = newDist;
                    pq.offer(new int[]{newDist, v});
                }
            }
        }
        return dist;
    }

    // Schedule meeting rooms — count minimum rooms needed
    static int minMeetingRooms(int[][] intervals) {
        Arrays.sort(intervals, Comparator.comparingInt(a -> a[0]));
        PriorityQueue<Integer> endTimes = new PriorityQueue<>(); // min-heap of end times

        for (int[] meeting : intervals) {
            if (!endTimes.isEmpty() && endTimes.peek() <= meeting[0]) {
                endTimes.poll(); // reuse this room (it's free)
            }
            endTimes.offer(meeting[1]); // allocate room ending at meeting[1]
        }

        return endTimes.size(); // active rooms = minimum needed
    }

    public static void main(String[] args) {
        // Graph: 5 nodes
        int n = 5;
        List<List<int[]>> graph = new ArrayList<>();
        for (int i = 0; i < n; i++) graph.add(new ArrayList<>());
        // edges: (from, to, weight)
        int[][] edges = {{0,1,4},{0,2,1},{2,1,2},{1,3,1},{2,3,5},{3,4,3}};
        for (int[] e : edges) {
            graph.get(e[0]).add(new int[]{e[1], e[2]});
        }

        int[] dist = dijkstra(graph, 0);
        System.out.println("Shortest distances from node 0:");
        for (int i = 0; i < n; i++) System.out.println("  To " + i + ": " + dist[i]);
        // To 0: 0, To 1: 3 (0→2→1), To 2: 1, To 3: 4, To 4: 7

        // Meeting rooms
        int[][] meetings = {{0,30},{5,10},{15,20}};
        System.out.println("\nMin meeting rooms needed: " + minMeetingRooms(meetings)); // 2
    }
}
```

### Level 5 — Expert: A* Search and Huffman Coding

```java
// HuffmanCoding.java
import java.util.*;

public class HuffmanCoding {

    static class Node implements Comparable<Node> {
        char ch;
        int freq;
        Node left, right;

        Node(char ch, int freq) { this.ch = ch; this.freq = freq; }
        Node(Node l, Node r) { left = l; right = r; freq = l.freq + r.freq; }

        boolean isLeaf() { return left == null && right == null; }

        @Override public int compareTo(Node o) { return Integer.compare(freq, o.freq); }
    }

    // Build Huffman tree using PriorityQueue
    static Node buildHuffmanTree(String text) {
        Map<Character, Integer> freq = new HashMap<>();
        for (char c : text.toCharArray()) freq.merge(c, 1, Integer::sum);

        PriorityQueue<Node> pq = new PriorityQueue<>();
        freq.forEach((c, f) -> pq.offer(new Node(c, f)));

        while (pq.size() > 1) {
            Node left = pq.poll();   // lowest frequency
            Node right = pq.poll();  // second lowest
            pq.offer(new Node(left, right)); // merge
        }

        return pq.poll(); // root of Huffman tree
    }

    // Generate codes
    static Map<Character, String> generateCodes(Node root) {
        Map<Character, String> codes = new HashMap<>();
        generateCodesHelper(root, "", codes);
        return codes;
    }

    static void generateCodesHelper(Node node, String code, Map<Character, String> codes) {
        if (node == null) return;
        if (node.isLeaf()) { codes.put(node.ch, code.isEmpty() ? "0" : code); return; }
        generateCodesHelper(node.left, code + "0", codes);
        generateCodesHelper(node.right, code + "1", codes);
    }

    public static void main(String[] args) {
        String text = "abracadabra";
        Node root = buildHuffmanTree(text);
        Map<Character, String> codes = generateCodes(root);

        System.out.println("Huffman codes for '" + text + "':");
        codes.entrySet().stream()
            .sorted(Map.Entry.comparingByKey())
            .forEach(e -> System.out.printf("  '%c': %s%n", e.getKey(), e.getValue()));

        // Encode
        StringBuilder encoded = new StringBuilder();
        for (char c : text.toCharArray()) encoded.append(codes.get(c));
        System.out.println("Encoded (" + encoded.length() + " bits): " + encoded);
        System.out.println("Original: " + text.length() * 8 + " bits (ASCII)");
        System.out.printf("Compression: %.1f%%%n",
            100.0 * (1 - encoded.length() / (text.length() * 8.0)));
    }
}
```

### Top-N Popular Products

```java
// Efficiently find top 10 products by views without sorting 1M products
public List<Product> getTop10ByViews(List<Product> products) {
    // Size-10 min-heap: always keeps the 10 highest-view products
    PriorityQueue<Product> top10 = new PriorityQueue<>(
        Comparator.comparingLong(Product::getViews)); // min-heap

    for (Product p : products) {
        top10.offer(p);
        if (top10.size() > 10) top10.poll(); // remove smallest-views product
    }

    // Convert to sorted list (highest first)
    List<Product> result = new ArrayList<>(top10);
    result.sort(Comparator.comparingLong(Product::getViews).reversed());
    return result;
}
// O(n log 10) = O(n) vs O(n log n) for full sort
```

## INTERVIEW

**Q: What data structure backs a Java PriorityQueue?**
> A binary min-heap, stored as an `Object[]` array. The minimum element is always at index 0 (O(1) peek). The children of element at index `i` are at `2i+1` and `2i+2`. Insert (sift up) and remove (sift down) are O(log n).

**Q: Why does iterating a PriorityQueue not yield elements in sorted order?**
> PriorityQueue stores a heap — a partial ordering where every parent ≤ children, but siblings have no defined order. The array [1, 3, 2, 7, 5, 4] is a valid min-heap. Iteration traverses the backing array in index order — not priority order. To iterate in order, repeatedly call `poll()`, which is O(n log n).

**Q: How do you find the K-th largest element efficiently?**
> Use a min-heap of size K. Iterate all elements: add each to the heap; if size exceeds K, remove the minimum. After processing all elements, the heap contains the K largest elements and the root (minimum of heap) is the K-th largest. Time: O(n log k), much better than O(n log n) full sort when k << n.

**Q: How does Dijkstra's algorithm use PriorityQueue?**
> Dijkstra's uses a min-heap ordered by distance from the source. The key insight: always expand the unvisited node with the minimum known distance. PriorityQueue `poll()` gives that node in O(log n). When a shorter path to a neighbor is found, add the neighbor with new distance to the queue. The greedy property — always expand the closest unvisited node — guarantees shortest paths in non-negative-weight graphs.

## FEYNMAN CHECK

A PriorityQueue is like an emergency room triage system. Patients don't wait in arrival order — the most critical patient is always treated next. Internally, it's like a tournament bracket (heap) where the winner (most critical/lowest priority number) always bubbles to the top. Adding a new patient takes a few rounds of comparisons to place them correctly in the bracket (O(log n)). Calling the next patient is instant — they're always at the front (O(1) peek).
