# CSS Selectors — Combinators, Pseudo-classes, :has() | Level: 1

CSS selectors determine how the browser matches elements in the DOM tree. Modern CSS selectors have evolved beyond simple matching into a declarative querying engine, complete with relational logic (`:has()`) and zero-specificity styling hooks (`:where()`).

---

## 1. Combinators: Navigating the DOM Tree

Combinators define the relationship between two or more simple selectors:

*   **Descendant Selector (space)**: Matches all matching elements nested anywhere inside the ancestor.
    ```css
    article p { color: oklch(0.3 0.01 240); }
    ```
*   **Child Selector (`>`)**: Matches only direct children of the parent element.
    ```css
    article > p { font-weight: 600; }
    ```
*   **Adjacent Sibling Selector (`+`)**: Matches the immediately following sibling element.
    ```css
    h1 + p { margin-block-start: 0.5rem; }
    ```
*   **General Sibling Selector (`~`)**: Matches any following sibling elements, even if not immediately adjacent.
    ```css
    h1 ~ p { color: oklch(0.4 0.02 240); }
    ```

---

## 2. Attribute Selectors

Attribute selectors match elements based on the presence, value, or substring patterns of their HTML attributes:

```css
/* 1. Exact Match */
[type="password"] { border-color: oklch(0.6 0.2 20); }

/* 2. Starts With (Prefix Match) */
[href^="https://"] { padding-inline-end: 1rem; }

/* 3. Ends With (Suffix Match) */
[href$=".pdf"]::after { content: " 📄"; }

/* 4. Contains Substring */
[class*="card-"] { border-radius: 8px; }
```
