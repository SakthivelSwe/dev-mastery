## WHY
A Stack is a LIFO (Last-In, First-Out) data structure. Think of it like a stack of plates in a cafeteria; the last plate you put on top of the pile is the very first plate you take off. 

In algorithmic interviews, Stacks are the absolute perfect tool for **Parsing and Matching**. Anytime you encounter a problem involving parentheses, nested structures, reversing order, or "going back one step" (like a browser's Back button), a Stack is almost certainly the optimal solution.

## THEORY
### Push and Pop
In most modern programming languages (like JavaScript and Python), you don't need to import a special Stack class. A standard Array is used as a Stack by simply restricting yourself to two methods:
- `push(item)`: Adds an item to the end (top) of the array. ($O(1)$ time).
- `pop()`: Removes and returns the item from the end (top) of the array. ($O(1)$ time).

## CODE

### Level 1 — Beginner (Valid Parentheses)
This is the quintessential Stack problem. It is asked constantly.
*Problem: Given a string containing just the characters '(', ')', '{', '}', '[' and ']', determine if the input string is valid (every open bracket is closed by the same type of bracket).*

```javascript
function isValid(s) {
    const stack = [];
    const map = {
        ')': '(',
        '}': '{',
        ']': '['
    };
    
    for (let char of s) {
        // If it's a closing bracket...
        if (map[char]) {
            // Pop the top element from the stack (or a dummy value if empty)
            const topElement = stack.length === 0 ? '#' : stack.pop();
            
            // If the popped element doesn't match the required opening bracket, fail!
            if (topElement !== map[char]) {
                return false;
            }
        } else {
            // If it's an opening bracket, push it onto the stack!
            stack.push(char);
        }
    }
    
    // If the stack is completely empty at the end, all brackets were matched perfectly!
    return stack.length === 0;
}
```

### Level 2 — Building Blocks (Min Stack)
*Problem: Design a stack that supports push, pop, top, and retrieving the minimum element in constant $O(1)$ time.*
*Trick:* Standard stacks can pop in $O(1)$, but finding the minimum usually requires scanning the whole stack $O(N)$. To achieve $O(1)$, you use **Two Stacks**! One for the data, and one that strictly tracks the minimum value seen so far.

```javascript
class MinStack {
    constructor() {
        this.stack = [];
        this.minStack = []; // Stores the minimum value at each corresponding level
    }

    push(val) {
        this.stack.push(val);
        
        // If minStack is empty, or the new val is smaller/equal to the current min, push it!
        if (this.minStack.length === 0 || val <= this.minStack[this.minStack.length - 1]) {
            this.minStack.push(val);
        }
    }

    pop() {
        const poppedVal = this.stack.pop();
        
        // If the value we just popped was the current minimum, pop it from minStack too!
        if (poppedVal === this.minStack[this.minStack.length - 1]) {
            this.minStack.pop();
        }
    }

    top() {
        return this.stack[this.stack.length - 1];
    }

    getMin() {
        return this.minStack[this.minStack.length - 1];
    }
}
```

### Level 3 — Intermediate (Evaluate Reverse Polish Notation)
*Problem: Evaluate the value of an arithmetic expression in Reverse Polish Notation. Valid operators are +, -, *, and /.*
*(Example: `["2","1","+","3","*"]` evaluates to `((2 + 1) * 3) = 9`)*

```javascript
function evalRPN(tokens) {
    const stack = [];
    
    for (let token of tokens) {
        if (token === '+' || token === '-' || token === '*' || token === '/') {
            // Pop the LAST TWO numbers (Order matters for division/subtraction!)
            const num2 = stack.pop(); 
            const num1 = stack.pop();
            
            let result;
            if (token === '+') result = num1 + num2;
            else if (token === '-') result = num1 - num2;
            else if (token === '*') result = num1 * num2;
            else if (token === '/') result = Math.trunc(num1 / num2); // Round towards zero
            
            stack.push(result);
        } else {
            // It's a number, convert to integer and push!
            stack.push(parseInt(token));
        }
    }
    
    // The final answer is the only item left in the stack
    return stack.pop();
}
```

### Level 4 — Advanced (Daily Temperatures)
*Problem: Given an array of daily temperatures, return an array such that answer[i] is the number of days you have to wait after the $i^{th}$ day to get a warmer temperature.*
*Trick:* This uses a highly specialized **Monotonic Decreasing Stack**. The stack strictly holds temperatures that are waiting for a warmer day.

```javascript
function dailyTemperatures(temperatures) {
    const n = temperatures.length;
    const answer = new Array(n).fill(0);
    const stack = []; // Stores INDICES of temperatures
    
    for (let i = 0; i < n; i++) {
        const currentTemp = temperatures[i];
        
        // While stack is not empty AND current day is WARMER than the day at the top of the stack
        while (stack.length > 0 && currentTemp > temperatures[stack[stack.length - 1]]) {
            // We found a warmer day for the index at the top of the stack!
            const prevIndex = stack.pop();
            // Calculate the difference in days
            answer[prevIndex] = i - prevIndex; 
        }
        
        // Push the current index onto the stack to wait for a warmer day
        stack.push(i);
    }
    
    return answer;
}
```

## REAL_WORLD
**The Call Stack and Recursion**
When you write a recursive function (a function that calls itself), how does the computer remember where to return to when the deepest function finally finishes? 
The Operating System uses a Stack! It pushes the memory address and local variables of the current function onto the **Call Stack** before jumping into the new function. When the new function finishes, it `pops` the Call Stack to resume exactly where it left off. If you write infinite recursion, you push too many items and crash the memory—hence the famous error: **Stack Overflow**.

## INTERVIEW
**Q: I implemented a Stack in JavaScript using an Array, but I used `shift()` and `unshift()` instead of `push()` and `pop()`. The interviewer failed me. Why?**
A: **Time Complexity!** `push()` and `pop()` operate on the very end of the array. The computer simply adds or removes memory at the end in $O(1)$ constant time. 
`unshift()` and `shift()` operate on the *beginning* (Index 0) of the array. If an array has 100,000 items, and you `unshift()` a new item into Index 0, the computer must manually shift all 100,000 existing items one slot to the right in memory to make room. This turns your Stack operations into $O(N)$ linear time, completely destroying the performance benefits of using a Stack in the first place.
