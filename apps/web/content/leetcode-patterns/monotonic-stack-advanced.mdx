## WHY
You are given a histogram (a bar chart) and asked to find the largest rectangular area that can fit inside the bars. Or you are given an array of numbers and asked to find the "Next Greater Element" for every single number. 

A standard $O(N^2)$ nested loop is easy to write but fails large test cases. The **Monotonic Stack** is a hyper-specialized variant of the Stack pattern. By strictly enforcing that the elements inside the stack are ALWAYS sorted (either strictly increasing or strictly decreasing), we can solve these notoriously difficult "Next/Previous Greater/Smaller" problems in $O(N)$ linear time!

## THEORY
### Maintaining Monotonicity
"Monotonic" is a math term meaning "always goes in one direction."
- **Monotonic Decreasing Stack:** The elements from bottom to top are getting smaller. (E.g., `[10, 8, 5, 2]`). If a `7` tries to enter, the `2` and `5` must be violently kicked out (popped) to maintain the decreasing order!
- **Monotonic Increasing Stack:** The elements from bottom to top are getting larger. (E.g., `[2, 5, 8, 10]`).

Whenever an element is popped from a monotonic stack, a massive realization occurs: *The element that caused the pop is the exact "Next Greater/Smaller Element" for the element being popped!*

## CODE

### Level 1 — Beginner (Next Greater Element)
*Problem: Given an array of integers, find the Next Greater Element for every element. If it doesn't exist, output -1.*
*(Example: `[2, 1, 2, 4, 3]` -> `[4, 2, 4, -1, -1]`)*

```javascript
function nextGreaterElements(nums) {
    const n = nums.length;
    const result = new Array(n).fill(-1);
    
    // The stack will store INDICES, not the actual values!
    // We use a Monotonic Decreasing Stack.
    const stack = []; 
    
    for (let i = 0; i < n; i++) {
        const currentNum = nums[i];
        
        // While stack is not empty AND the current number is GREATER than 
        // the number sitting at the top of the stack...
        while (stack.length > 0 && currentNum > nums[stack[stack.length - 1]]) {
            // We just found the "Next Greater Element" for the item at the top of the stack!
            const indexToResolve = stack.pop();
            result[indexToResolve] = currentNum;
        }
        
        // Push the current index onto the stack to wait for its own Next Greater Element
        stack.push(i);
    }
    
    return result;
}
```

### Level 2 — Building Blocks (Daily Temperatures)
*Problem: Given an array of temperatures, return an array answering how many days you have to wait for a warmer temperature.*

This is literally the exact same code as Level 1, but instead of returning the *value* of the Next Greater Element, we return the *distance* (difference in indices).

```javascript
function dailyTemperatures(temperatures) {
    const n = temperatures.length;
    const answer = new Array(n).fill(0);
    const stack = []; // Monotonic Decreasing Stack
    
    for (let i = 0; i < n; i++) {
        // Did we find a warmer day?
        while (stack.length > 0 && temperatures[i] > temperatures[stack[stack.length - 1]]) {
            const prevIndex = stack.pop();
            // The wait time is the current day minus the day we were waiting!
            answer[prevIndex] = i - prevIndex;
        }
        stack.push(i);
    }
    
    return answer;
}
```

### Level 3 — Intermediate (Largest Rectangle in Histogram)
*Problem: Given an array of integers representing the bar heights of a histogram (where width of each bar is 1), return the area of the largest rectangle in the histogram.*

*Trick:* A rectangle is bottlenecked by its shortest bar. If we know the "Next Smaller Element" on the right, and the "Previous Smaller Element" on the left, we can mathematically calculate exactly how wide a rectangle can stretch before it hits a wall! We use a **Monotonic Increasing Stack** to find these boundaries!

```javascript
function largestRectangleArea(heights) {
    let maxArea = 0;
    const stack = []; // Monotonic Increasing Stack of indices
    
    for (let i = 0; i <= heights.length; i++) {
        // We use 0 as a dummy height at the very end to force the stack to completely empty!
        const currentHeight = i === heights.length ? 0 : heights[i];
        
        // If the current bar is SMALLER than the bar at the top of the stack,
        // it means the bar at the top of the stack can no longer expand to the right!
        while (stack.length > 0 && currentHeight < heights[stack[stack.length - 1]]) {
            
            // The height of the rectangle we are finalizing
            const h = heights[stack.pop()];
            
            // If the stack is empty, it means this bar was the absolute shortest so far,
            // so it can stretch all the way back to index 0!
            // Otherwise, it stretches back to whatever is sitting directly below it in the stack.
            const width = stack.length === 0 ? i : i - stack[stack.length - 1] - 1;
            
            maxArea = Math.max(maxArea, h * width);
        }
        
        stack.push(i);
    }
    
    return maxArea;
}
```

### Level 4 — Advanced (Trapping Rain Water)
*Problem: Given `n` non-negative integers representing an elevation map where the width of each bar is 1, compute how much water it can trap after raining.*
*(Note: There is a Two-Pointer solution, but the Monotonic Stack solution is incredibly elegant for understanding valleys).*

*Trick:* We use a **Monotonic Decreasing Stack**. Whenever we find a bar that is TALLER than the top of the stack, it means we just found the right wall of a valley! The element below it in the stack is the left wall! We calculate the water trapped in that horizontal valley layer, and pop it.

```javascript
function trap(height) {
    let totalWater = 0;
    const stack = []; // Stores indices
    
    for (let i = 0; i < height.length; i++) {
        // We found a Right Wall that is taller than the bottom of the valley!
        while (stack.length > 0 && height[i] > height[stack[stack.length - 1]]) {
            
            // The bottom of the valley
            const valleyBottomIndex = stack.pop();
            
            // If there is no Left Wall, the water just spills out the left side! Break!
            if (stack.length === 0) break;
            
            const leftWallIndex = stack[stack.length - 1];
            
            // The distance between the Left Wall and Right Wall
            const width = i - leftWallIndex - 1;
            
            // Water fills up to the SHORTEST of the two walls, minus the dirt at the bottom
            const boundedHeight = Math.min(height[leftWallIndex], height[i]) - height[valleyBottomIndex];
            
            totalWater += (width * boundedHeight);
        }
        
        stack.push(i);
    }
    
    return totalWater;
}
```

## REAL_WORLD
**Stock Market Span Analysis**
In financial software, traders often want to calculate the "Stock Span"—the number of consecutive days prior to today where the stock price was less than or equal to today's price. 
Instead of looking backwards through months of historical data every single day ($O(N^2)$), trading algorithms use a Monotonic Decreasing Stack. By instantly popping days that are cheaper than today, the algorithm can calculate the Stock Span in real-time streaming $O(1)$ amortized time per day, maintaining extreme latency requirements.

## INTERVIEW
**Q: I completely understand the standard Stack pattern (Valid Parentheses), but I struggle to identify when to upgrade to a Monotonic Stack during an interview.**
A: **Look for the keywords: "Next/Previous", "Greater/Smaller", "Maximum/Minimum span".** 
If the problem requires finding the nearest element to the left or right that breaks a certain threshold (e.g., the next warmer day, the next shorter bar, the next cheapest item), a brute force nested loop will always take $O(N^2)$. The Monotonic Stack is practically the only data structure in computer science that solves these "Nearest Greater/Smaller" range queries in linear $O(N)$ time.
