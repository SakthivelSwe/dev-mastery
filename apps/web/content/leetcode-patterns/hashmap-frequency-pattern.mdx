## WHY
Imagine you are given an array of 100,000 numbers and asked to find the most frequent element. A naive approach would be a nested loop: for every number, scan the entire array again to count how many times it appears. This results in $O(N^2)$ time complexity, which is disastrously slow and will instantly fail any coding interview.

The **Hashmap Frequency Pattern** eliminates the inner loop. By utilizing a Hash Map (or Object/Dictionary) to store the frequency of elements as we traverse the array exactly once, we drop the time complexity to a blistering $O(N)$.

## THEORY
### The Time-Space Tradeoff
A Hash Map provides $O(1)$ average time complexity for both insertions and lookups. 
This pattern is the ultimate example of the **Time-Space Tradeoff** in computer science: we gladly sacrifice $O(N)$ auxiliary memory (to store the Hash Map) in order to achieve $O(N)$ time complexity. Memory is cheap; execution time is expensive.

## CODE

### Level 1 — Beginner (Counting Frequencies)
The core foundation of this pattern is building the frequency map.
*Problem: Given an array of integers, return a map of how many times each integer appears.*

```javascript
function buildFrequencyMap(arr) {
    const map = {};
    
    for (let i = 0; i < arr.length; i++) {
        const num = arr[i];
        
        // If the number is already in the map, increment its count
        if (map[num] !== undefined) {
            map[num]++;
        } else {
            // Otherwise, initialize its count to 1
            map[num] = 1;
        }
    }
    
    return map;
}

console.log(buildFrequencyMap([1, 2, 2, 3, 1, 1])); 
// Output: { '1': 3, '2': 2, '3': 1 }
```
*(Pro-tip: In JavaScript, you can write this much cleaner: `map[num] = (map[num] || 0) + 1;`)*

### Level 2 — Building Blocks (Two Sum)
The most famous LeetCode problem of all time (Problem #1) is perfectly solved using a Hash Map.
*Problem: Given an array of numbers and a `target`, return the indices of the two numbers that add up to the target.*

```javascript
function twoSum(nums, target) {
    // Map stores { number: index }
    const map = new Map();
    
    for (let i = 0; i < nums.length; i++) {
        const currentNum = nums[i];
        // What number do we mathematically NEED to hit the target?
        const complement = target - currentNum;
        
        // Did we see the complement earlier in the array?
        if (map.has(complement)) {
            return [map.get(complement), i];
        }
        
        // If not, record the current number and its index for future reference
        map.set(currentNum, i);
    }
    
    return [];
}
```

### Level 3 — Intermediate (Valid Anagram)
*Problem: Given two strings `s` and `t`, return true if `t` is an anagram of `s` (contains the exact same characters in any order).*

```javascript
function isAnagram(s, t) {
    if (s.length !== t.length) return false;
    
    const countMap = {};
    
    // Step 1: Build the frequency map for the first string
    for (let char of s) {
        countMap[char] = (countMap[char] || 0) + 1;
    }
    
    // Step 2: "Spend" the characters using the second string
    for (let char of t) {
        // If the character doesn't exist, or we ran out of them, it's not an anagram!
        if (!countMap[char]) {
            return false;
        }
        countMap[char]--;
    }
    
    return true;
}
```

### Level 4 — Advanced (Subarray Sum Equals K)
This problem combines the Prefix Sum pattern with the Hashmap pattern!
*Problem: Given an array of integers and an integer `k`, return the total number of continuous subarrays whose sum equals `k`.*

```javascript
function subarraySum(nums, k) {
    let count = 0;
    let currentSum = 0;
    
    // Map stores { prefixSum: frequency }
    // Initialize with { 0: 1 } to handle subarrays that start at index 0
    const prefixMap = new Map();
    prefixMap.set(0, 1);
    
    for (let i = 0; i < nums.length; i++) {
        currentSum += nums[i];
        
        // If (currentSum - k) exists in the map, it means there is a previous 
        // prefix sum that we can chop off to leave exactly 'k' remaining!
        const difference = currentSum - k;
        if (prefixMap.has(difference)) {
            count += prefixMap.get(difference);
        }
        
        // Add the current sum to the map
        prefixMap.set(currentSum, (prefixMap.get(currentSum) || 0) + 1);
    }
    
    return count;
}
```

## REAL_WORLD
**Detecting Fraudulent Activity**
Imagine you work for a bank processing millions of credit card transactions per hour. You need to alert the security team if the exact same credit card number is used more than 5 times within a one-minute window. 

Sorting the transactions by credit card number would take $O(N \log N)$ time, introducing massive latency to the real-time pipeline. By feeding the transactions into an in-memory Hash Map (or a Redis cache) and incrementing the frequency counter in $O(1)$ time, the system can instantly trigger an alert the millisecond a card hits a frequency of 5.

## INTERVIEW
**Q: When should I use a JavaScript `Map` vs a standard JavaScript Object `{}` for frequency counting?**
A: You should almost always use a `Map` in an interview for three reasons:
1. **Key Types:** Standard Objects convert all keys to strings (e.g., the number `5` becomes `"5"`). A `Map` preserves the exact data type.
2. **Iteration:** A `Map` is strictly ordered by insertion time and is easily iterable using `for...of`.
3. **Prototype Pollution:** Standard Objects inherit built-in methods (like `.toString`). If an array contains the string `"toString"`, doing `map["toString"]++` on a standard Object will crash your algorithm! A `Map` is a pure data structure immune to this.
