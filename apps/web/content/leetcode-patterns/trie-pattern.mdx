## WHY
If you have a dictionary of 100,000 words, and you want to know if any word starts with the prefix "app" (e.g., apple, app, application), how do you do it? 
A Hash Map is useless here; it can only do exact string matching. A linear search over an array takes $O(N)$ time, comparing every single string, which is far too slow for real-time search autocomplete.

The **Trie** (pronounced "Try") is a specialized tree data structure designed specifically for blazing-fast prefix matching and string validation. In a Trie, looking up a word (or prefix) takes $O(L)$ time, where $L$ is the length of the word, completely regardless of how many millions of words are stored in the tree!

## THEORY
### Nodes and Edges
Unlike a Binary Tree (which has 2 children), a Trie Node can have up to 26 children (one for each letter of the English alphabet). 
Each node itself doesn't actually store the string "apple". It just stores a single character, and points to the next character. 
The word is formed by the *path* you take down the tree. 

Crucially, every node must have a boolean flag (`isEndOfWord`) to indicate if the current path forms a complete, valid dictionary word, or if it is just a prefix for a longer word.

## CODE

### Level 1 — Beginner (Building the TrieNode)
The foundation of the Trie is the Node class. In JavaScript, we usually use a Hash Map (or Object) to store the children, making it easy to support any character (not just a-z).

```javascript
class TrieNode {
    constructor() {
        // Map characters to their corresponding TrieNode
        this.children = {}; 
        
        // Is this node the absolute end of a valid inserted word?
        this.isEndOfWord = false; 
    }
}
```

### Level 2 — Building Blocks (Insert and Search)
*Problem: Implement a Trie class with `insert`, `search`, and `startsWith` methods.*

```javascript
class Trie {
    constructor() {
        this.root = new TrieNode();
    }

    // O(L) time where L is the length of the word
    insert(word) {
        let currentNode = this.root;
        
        for (let char of word) {
            // If the letter doesn't exist in the current node's children, create it!
            if (!currentNode.children[char]) {
                currentNode.children[char] = new TrieNode();
            }
            // Traverse down to the next node
            currentNode = currentNode.children[char];
        }
        
        // We finished inserting all letters. Mark the final node as a valid word!
        currentNode.isEndOfWord = true;
    }

    // O(L) time. Returns TRUE only if the exact word exists.
    search(word) {
        let currentNode = this.root;
        
        for (let char of word) {
            // If at any point the letter is missing, the word doesn't exist!
            if (!currentNode.children[char]) {
                return false;
            }
            currentNode = currentNode.children[char];
        }
        
        // We found the path! But is it a full word, or just a prefix?
        return currentNode.isEndOfWord;
    }

    // O(L) time. Returns TRUE if ANY word in the Trie starts with this prefix.
    startsWith(prefix) {
        let currentNode = this.root;
        
        for (let char of prefix) {
            if (!currentNode.children[char]) {
                return false;
            }
            currentNode = currentNode.children[char];
        }
        
        // We successfully traced the prefix. We don't care if it's a full word!
        return true; 
    }
}
```

### Level 3 — Intermediate (Word Dictionary with Dots)
*Problem: Design a data structure that supports adding words and searching. The search word can contain the '.' dot character to represent ANY one letter.*
*(Example: `search("b.d")` matches "bad", "bed", "bid").*

*Trick:* Because the dot can be anything, we must abandon simple iteration and use **DFS (Recursion)** to explore all possible children when we hit a dot!

```javascript
class WordDictionary {
    constructor() {
        this.root = new TrieNode();
    }

    addWord(word) {
        // Standard Trie Insert (Same as Level 2)
        let node = this.root;
        for (let char of word) {
            if (!node.children[char]) node.children[char] = new TrieNode();
            node = node.children[char];
        }
        node.isEndOfWord = true;
    }

    search(word) {
        // Kick off the recursive DFS helper function
        return this.dfsSearch(word, 0, this.root);
    }
    
    dfsSearch(word, index, node) {
        // Base case: We reached the end of the word. Is it valid?
        if (index === word.length) {
            return node.isEndOfWord;
        }
        
        const char = word[index];
        
        if (char === '.') {
            // It's a wildcard! We must try ALL available children at this level.
            for (let childKey in node.children) {
                // If ANY of the recursive paths return true, we found a match!
                if (this.dfsSearch(word, index + 1, node.children[childKey])) {
                    return true;
                }
            }
            // None of the paths worked
            return false;
        } else {
            // It's a normal character. Does it exist?
            if (!node.children[char]) {
                return false;
            }
            // Just traverse down normally
            return this.dfsSearch(word, index + 1, node.children[char]);
        }
    }
}
```

### Level 4 — Advanced (Word Search II - Matrix + Trie)
*Problem: Given an `m x n` board of characters and a list of strings `words`, return all words on the board. You can move horizontally or vertically.*

*Trick:* Searching a matrix for 100,000 different words using pure DFS will instantly Time Limit Exceed. You MUST build a Trie of the dictionary words, and then run a DFS over the Matrix, constantly checking the Trie to see if the current path is valid. If it's not a valid prefix in the Trie, you instantly terminate the DFS branch (Pruning!).

*(Implementation is too long for this snippet, but the core architecture is:*
1. *Insert all `words` into a `Trie`.*
2. *Loop over every `(row, col)` in the matrix.*
3. *Run `DFS(row, col, trieNode)`.*
4. *If `trieNode.children[matrixChar]` exists, continue DFS. Else, abort early!)*

## REAL_WORLD
**Routers and IP Forwarding**
When an internet packet hits a massive Cisco router, the router needs to look at the destination IP address (e.g., `192.168.1.50`) and match it against a massive routing table containing hundreds of thousands of network prefixes to figure out which cable to send it down. 
Routers use a highly optimized, binary version of a Trie called a **Radix Tree** (or Patricia Trie) to perform Longest Prefix Matching in hardware-accelerated milliseconds.

## INTERVIEW
**Q: I built a Trie for an autocomplete feature. It works perfectly, but it consumes a massive amount of RAM compared to just storing the strings in a Hash Map. Why?**
A: **The Node Overhead.** In a Hash Map, the string "apple" is stored contiguously in memory as 5 bytes. 
In a naive Trie, the word "apple" requires 5 distinct Node objects. In JavaScript or Java, every single object has substantial memory overhead (pointers to the class prototype, memory headers, garbage collection metadata). A single Node might consume 32-64 bytes! If your Trie has millions of nodes with few overlapping prefixes, the object overhead absolutely dwarfs the actual string data. In production, developers optimize this using Arrays instead of Objects for the children, or by compressing linear branches into single nodes (Radix Tries).
