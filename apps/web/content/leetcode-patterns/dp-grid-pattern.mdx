## WHY
You are playing a video game. Your character spawns in the top-left corner of a grid map. You need to reach the treasure chest in the bottom-right corner, but you can only move **Right** or **Down**. The grid is littered with obstacles and different terrain costs. 
How many distinct paths can you take? What is the minimum cost to reach the end?

The **DP 2D Grid Pattern** is one of the most visually intuitive forms of Dynamic Programming. Because you can only move Right or Down, you know mathematically that any cell `(row, col)` can ONLY be reached from exactly two places: the cell immediately ABOVE it, or the cell immediately to the LEFT of it.

## THEORY
### The State Transition
In a grid problem, the state is simply your coordinates `(i, j)`.
If you want to know the "Minimum Cost" to reach cell `(i, j)`, the formula is:
`Current Cell Cost + Math.min( Cost to reach the cell ABOVE, Cost to reach the cell LEFT )`

By initializing a 2D matrix (Tabulation) and iterating through the rows and columns sequentially from top to bottom, left to right, we guarantee that the "Above" and "Left" cells have already been fully calculated before we ever need them!

## CODE

### Level 1 — Beginner (Unique Paths)
*Problem: A robot is located at the top-left corner of an `m x n` grid. The robot can only move either down or right at any point in time. The robot is trying to reach the bottom-right corner. How many possible unique paths are there?*

```javascript
function uniquePaths(m, n) {
    // Create an M x N matrix initialized with 1s
    const dp = Array.from({ length: m }, () => Array(n).fill(1));
    
    // Why did we initialize with 1s?
    // Because if you are anywhere in the first row, you can only move RIGHT. 
    // There is exactly 1 way to get there.
    // If you are anywhere in the first column, you can only move DOWN. (1 way).
    
    // Start at row 1, col 1 (skip the first row and col)
    for (let row = 1; row < m; row++) {
        for (let col = 1; col < n; col++) {
            
            // The number of paths to get HERE is the sum of the paths 
            // from the cell ABOVE and the cell LEFT!
            dp[row][col] = dp[row - 1][col] + dp[row][col - 1];
            
        }
    }
    
    // The bottom-right corner contains the total sum of all possible paths!
    return dp[m - 1][n - 1];
}
```

### Level 2 — Building Blocks (Unique Paths II - Obstacles)
*Problem: Now consider if some obstacles are added to the grid. An obstacle and space are marked as `1` and `0` respectively in the grid. A path cannot pass through an obstacle.*

```javascript
function uniquePathsWithObstacles(obstacleGrid) {
    const m = obstacleGrid.length;
    const n = obstacleGrid[0].length;
    
    // If the starting cell itself is an obstacle, it's impossible to move anywhere!
    if (obstacleGrid[0][0] === 1) return 0;
    
    const dp = Array.from({ length: m }, () => Array(n).fill(0));
    
    // Start position has 1 path
    dp[0][0] = 1;
    
    // Pre-fill the first column (If we hit an obstacle, all cells BELOW it become unreachable 0s)
    for (let i = 1; i < m; i++) {
        dp[i][0] = (obstacleGrid[i][0] === 0 && dp[i - 1][0] === 1) ? 1 : 0;
    }
    
    // Pre-fill the first row (If we hit an obstacle, all cells RIGHT of it become unreachable 0s)
    for (let j = 1; j < n; j++) {
        dp[0][j] = (obstacleGrid[0][j] === 0 && dp[0][j - 1] === 1) ? 1 : 0;
    }
    
    // Standard iteration
    for (let i = 1; i < m; i++) {
        for (let j = 1; j < n; j++) {
            if (obstacleGrid[i][j] === 0) {
                dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
            } else {
                // It's an obstacle! 0 paths can flow through here.
                dp[i][j] = 0; 
            }
        }
    }
    
    return dp[m - 1][n - 1];
}
```

### Level 3 — Intermediate (Minimum Path Sum)
*Problem: Given an `m x n` grid filled with non-negative numbers, find a path from top left to bottom right, which minimizes the sum of all numbers along its path.*

```javascript
function minPathSum(grid) {
    const m = grid.length;
    const n = grid[0].length;
    
    // We can do this IN-PLACE by mutating the original grid to save O(m*n) space!
    
    // Pre-fill first row (Can only come from the left)
    for (let j = 1; j < n; j++) {
        grid[0][j] += grid[0][j - 1];
    }
    
    // Pre-fill first column (Can only come from above)
    for (let i = 1; i < m; i++) {
        grid[i][0] += grid[i - 1][0];
    }
    
    for (let i = 1; i < m; i++) {
        for (let j = 1; j < n; j++) {
            
            // To minimize the sum, look ABOVE and LEFT, pick the SMALLEST one, 
            // and add it to the current cell!
            grid[i][j] += Math.min(grid[i - 1][j], grid[i][j - 1]);
            
        }
    }
    
    return grid[m - 1][n - 1];
}
```

### Level 4 — Advanced (Maximal Square)
*Problem: Given an `m x n` binary matrix filled with `0`'s and `1`'s, find the largest square containing only `1`'s and return its area.*

*Trick:* The state transition changes! A square depends on THREE cells, not two. To form a $2 \times 2$ square, the cell you are on must be a `1`, AND the cell ABOVE, LEFT, and DIAGONAL-UP-LEFT must all be valid squares themselves!

```javascript
function maximalSquare(matrix) {
    if (matrix.length === 0) return 0;
    const m = matrix.length;
    const n = matrix[0].length;
    
    // dp[i][j] represents the SIDE LENGTH of the maximum square 
    // whose bottom-right corner is positioned at (i, j)
    const dp = Array.from({ length: m + 1 }, () => Array(n + 1).fill(0));
    
    let maxSideLength = 0;
    
    // 1-indexed loop to avoid out-of-bounds checks!
    for (let i = 1; i <= m; i++) {
        for (let j = 1; j <= n; j++) {
            
            if (matrix[i - 1][j - 1] === '1') {
                // If it's a 1, it forms a square dictated by the WEAKEST link
                // among its Left, Top, and Diagonal neighbors!
                dp[i][j] = Math.min(
                    dp[i][j - 1],       // Left
                    dp[i - 1][j],       // Top
                    dp[i - 1][j - 1]    // Diagonal Top-Left
                ) + 1;
                
                maxSideLength = Math.max(maxSideLength, dp[i][j]);
            }
            
        }
    }
    
    // The question asks for AREA, so we square the max side length
    return maxSideLength * maxSideLength;
}
```

## REAL_WORLD
**Image Resizing (Seam Carving)**
If you take a landscape photograph and try to compress its width using standard CSS or Photoshop resizing, the people in the photo get squished and look distorted. 
Content-Aware Scaling (Seam Carving) prevents this! The algorithm treats the image as a massive Grid of pixels. It assigns an "Energy Cost" to each pixel (high cost for people/faces, low cost for empty sky/grass). It then runs a **DP Grid Minimum Path Sum** algorithm from the top of the image to the bottom, finding the vertical path (a "Seam") with the lowest total energy cost. It deletes that exact seam of pixels, making the image 1 pixel thinner without touching the people at all!

## INTERVIEW
**Q: My DP matrix solutions always use $O(M \times N)$ space. An interviewer asked me to optimize the space complexity of `Unique Paths` to $O(N)$. How is that mathematically possible when the grid has $M$ rows?**
A: Because of the State Transition rule! To calculate row `i`, you ONLY look at data from row `i - 1` (the row immediately above you). You completely ignore row `i - 2` and `i - 3`. 
Therefore, you don't need a massive $M \times N$ matrix. You only need to keep **two single arrays** in memory: the `previousRow` and the `currentRow`. You calculate the `currentRow`, and then when the loop repeats, the `currentRow` becomes the new `previousRow`. This immediately drops the space complexity from $O(M \times N)$ down to $O(N)$!
