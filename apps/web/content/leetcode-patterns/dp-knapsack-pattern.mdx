## WHY
Imagine you are a thief breaking into a house. You have a backpack (knapsack) that can hold a maximum of 10 pounds. You see 5 items, each with a different weight and a different dollar value. You cannot take fractions of an item (you either take the whole TV, or you leave it). How do you maximize your profit without breaking the backpack?

The **0/1 Knapsack Problem** is the absolute grandfather of all Dynamic Programming (DP) algorithms. It teaches you how to optimize decisions under strict constraints. If you master this pattern, you can solve almost any "pick or don't pick" problem in computer science.

## THEORY
### The Core DP Equation
At every single item, you have exactly two choices:
1. **Take it:** The profit increases, but the backpack's remaining capacity decreases.
2. **Leave it:** The profit stays the same, and the backpack's capacity stays the same.

The optimal solution for the *current* item is simply:
`Math.max( Profit if I take it, Profit if I leave it )`. 
By using a 2D Array to cache these calculations (Memoization/Tabulation), we drop the time complexity from an impossible $O(2^N)$ to a highly efficient $O(N \times W)$ (where $W$ is the backpack capacity).

## CODE

### Level 1 — Beginner (Recursive Brute Force)
*Warning: This solution will instantly Time Limit Exceed on LeetCode ($O(2^N)$), but it is the mandatory first step to understanding DP!*

```javascript
// weights = [1, 2, 3], values = [10, 15, 40], capacity = 6
function knapsackRecursive(weights, values, capacity, currentIndex) {
    // Base Case: If we ran out of items, or the backpack is full, profit is 0!
    if (capacity <= 0 || currentIndex >= weights.length) {
        return 0;
    }
    
    // Choice 1: Leave the current item
    let profitIfLeave = knapsackRecursive(weights, values, capacity, currentIndex + 1);
    
    // Choice 2: Take the current item (ONLY if it physically fits in the bag!)
    let profitIfTake = 0;
    if (weights[currentIndex] <= capacity) {
        profitIfTake = values[currentIndex] + knapsackRecursive(
            weights, 
            values, 
            capacity - weights[currentIndex], 
            currentIndex + 1
        );
    }
    
    // Return the maximum of the two choices!
    return Math.max(profitIfLeave, profitIfTake);
}
```

### Level 2 — Building Blocks (Top-Down Memoization)
To fix the $O(2^N)$ time complexity, we cache the results of our recursive calls. If we ever reach the exact same `(currentIndex, capacity)` state again, we just return the saved answer instantly!

```javascript
function knapsackMemoized(weights, values, maxCapacity) {
    // Create a 2D cache array filled with -1
    // Rows = number of items, Cols = capacity + 1
    const memo = Array.from({ length: weights.length }, () => 
                 Array(maxCapacity + 1).fill(-1));
                 
    function solve(capacity, currentIndex) {
        if (capacity <= 0 || currentIndex >= weights.length) return 0;
        
        // CHECK THE CACHE!
        if (memo[currentIndex][capacity] !== -1) {
            return memo[currentIndex][capacity];
        }
        
        let profitIfLeave = solve(capacity, currentIndex + 1);
        
        let profitIfTake = 0;
        if (weights[currentIndex] <= capacity) {
            profitIfTake = values[currentIndex] + solve(
                capacity - weights[currentIndex], 
                currentIndex + 1
            );
        }
        
        // SAVE TO THE CACHE before returning!
        memo[currentIndex][capacity] = Math.max(profitIfLeave, profitIfTake);
        return memo[currentIndex][capacity];
    }
    
    return solve(maxCapacity, 0);
}
```

### Level 3 — Intermediate (Bottom-Up Tabulation)
Instead of recursion, we build a 2D matrix from the ground up using a nested loop. This avoids the risk of Maximum Call Stack Size Exceeded errors.

```javascript
function knapsackTabulation(weights, values, capacity) {
    const n = weights.length;
    // dp[i][c] represents the max profit using the first 'i' items with capacity 'c'
    const dp = Array.from({ length: n + 1 }, () => Array(capacity + 1).fill(0));
    
    for (let i = 1; i <= n; i++) {
        for (let c = 1; c <= capacity; c++) {
            
            // Note: i-1 because the dp array has an extra 0th row for base cases
            const currentWeight = weights[i - 1];
            const currentValue = values[i - 1];
            
            // If we LEAVE the item, the profit is whatever we had in the row above
            let profitIfLeave = dp[i - 1][c];
            
            // If we TAKE the item...
            let profitIfTake = 0;
            if (currentWeight <= c) {
                // Current Value + the best profit we could get with the REMAINING capacity
                profitIfTake = currentValue + dp[i - 1][c - currentWeight];
            }
            
            dp[i][c] = Math.max(profitIfLeave, profitIfTake);
        }
    }
    
    return dp[n][capacity];
}
```

### Level 4 — Advanced (Partition Equal Subset Sum)
*Problem: Given an integer array `nums`, return true if you can partition the array into two subsets such that the sum of the elements in both subsets is equal.*

*Trick:* This is secretly the 0/1 Knapsack problem! If the total sum of the array is 22, we just need to find if there is a subset (a "knapsack") that can hold exactly 11 (which is 22 / 2).

```javascript
function canPartition(nums) {
    const totalSum = nums.reduce((a, b) => a + b, 0);
    
    // If the total sum is odd, it is mathematically impossible to split in half!
    if (totalSum % 2 !== 0) return false;
    
    const targetCapacity = totalSum / 2;
    
    // We can optimize the Tabulation matrix down to a single 1D array!
    // dp[c] will be true if we can create a subset that sums to EXACTLY capacity 'c'
    const dp = new Array(targetCapacity + 1).fill(false);
    
    // Base case: We can always make a sum of 0 by choosing no elements
    dp[0] = true;
    
    for (let num of nums) {
        // We MUST iterate backwards! If we iterate forwards, we might use the 
        // exact same number multiple times in the same loop (Unbounded Knapsack).
        for (let c = targetCapacity; c >= num; c--) {
            // We can make capacity 'c' if we could ALREADY make capacity (c - num)
            dp[c] = dp[c] || dp[c - num];
        }
    }
    
    return dp[targetCapacity];
}
```

## INTERVIEW
**Q: In the Level 4 `canPartition` solution, why did we optimize the 2D array down to a 1D array?**
A: **Space Complexity Optimization.** In the 2D Tabulation method, to calculate row `i`, you only ever look at values from row `i - 1` (the row immediately above it). You never look at `i - 2` or `i - 3`. Therefore, keeping the entire massive matrix in memory is a complete waste of RAM. By using a single 1D array and updating it in place (iterating backwards to prevent overwriting values we still need), we drop the space complexity from $O(N \times W)$ to $O(W)$, which is highly impressive to an interviewer.
