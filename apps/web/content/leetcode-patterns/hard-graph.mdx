## WHY
Basic BFS and DFS are sufficient for 90% of graph problems involving unweighted edges or simple connectivity. But what if the edges have different costs (e.g., Highway A takes 5 minutes, Highway B takes 50 minutes)? What if you need to find the optimal flow of data through a pipeline network? What if the graph is so massive you have to calculate shortest paths from *every* node to *every other* node?

In Hard Graph problems, we leave basic traversal behind and step into the realm of famous named algorithms: **Dijkstra's, Bellman-Ford, and Floyd-Warshall**. 

## THEORY
### Weighted Graphs and Priority Queues
In a standard BFS, a Queue ensures we process nodes strictly by how many *hops* away they are. 
In a Weighted Graph, a 1-hop path might cost 100, while a 3-hop path might cost 5. To ensure we always process the mathematically *cheapest* path first, we replace the standard Queue with a **Min-Priority Queue (Heap)**. This forces the algorithm to constantly re-evaluate and prioritize exploration based on cumulative cost, not hops.

## CODE

### Level 1 — Beginner (Dijkstra's Algorithm)
*Problem: You are given a directed, weighted graph. Find the shortest path (minimum total cost) from a starting node `S` to all other nodes. (Assumes NO negative weights).*

```javascript
// Assuming we have a standard MinPriorityQueue class in JavaScript
function dijkstra(n, edges, startNode) {
    // 1. Build the Adjacency List: { node: [[neighbor, weight], ...] }
    const graph = Array.from({ length: n }, () => []);
    for (let [u, v, weight] of edges) {
        graph[u].push([v, weight]);
    }
    
    // 2. Track the minimum distance to every node. Initialize to Infinity.
    const distances = new Array(n).fill(Infinity);
    distances[startNode] = 0;
    
    // 3. Min-Heap stores: { cost, node }
    const pq = new MinPriorityQueue({ priority: (item) => item.cost });
    pq.enqueue({ cost: 0, node: startNode });
    
    while (!pq.isEmpty()) {
        const { cost: currentCost, node: currentNode } = pq.dequeue().element;
        
        // Optimization: If we already found a cheaper way to this node earlier, 
        // ignore this outdated, more expensive path!
        if (currentCost > distances[currentNode]) continue;
        
        // Explore all neighbors
        for (let [neighbor, edgeWeight] of graph[currentNode]) {
            const newTotalCost = currentCost + edgeWeight;
            
            // Did we just discover a CHEAPER path to this neighbor?
            if (newTotalCost < distances[neighbor]) {
                distances[neighbor] = newTotalCost;     // Update the record!
                pq.enqueue({ cost: newTotalCost, node: neighbor }); // Queue it for exploration!
            }
        }
    }
    
    return distances;
}
```

### Level 2 — Building Blocks (Network Delay Time)
*Problem: You are given a network of `n` nodes. You send a signal from a certain node `k`. Return the minimum time it takes for all `n` nodes to receive the signal. If it is impossible, return -1.*

*Trick:* This is literally just Dijkstra's Algorithm! We find the shortest path to *every* node. The total time taken for the entire network to receive the signal is simply the maximum of those shortest paths!

```javascript
function networkDelayTime(times, n, k) {
    // Graph is 1-indexed!
    const graph = Array.from({ length: n + 1 }, () => []);
    for (let [u, v, w] of times) {
        graph[u].push([v, w]);
    }
    
    const distances = new Array(n + 1).fill(Infinity);
    distances[k] = 0;
    
    const pq = new MinPriorityQueue({ priority: (item) => item.time });
    pq.enqueue({ time: 0, node: k });
    
    while (!pq.isEmpty()) {
        const { time: currentTime, node: currentNode } = pq.dequeue().element;
        
        if (currentTime > distances[currentNode]) continue;
        
        for (let [neighbor, travelTime] of graph[currentNode]) {
            const newTime = currentTime + travelTime;
            if (newTime < distances[neighbor]) {
                distances[neighbor] = newTime;
                pq.enqueue({ time: newTime, node: neighbor });
            }
        }
    }
    
    // Find the maximum time it took to reach any node
    let maxDelay = 0;
    for (let i = 1; i <= n; i++) {
        if (distances[i] === Infinity) return -1; // A node was completely unreachable!
        maxDelay = Math.max(maxDelay, distances[i]);
    }
    
    return maxDelay;
}
```

### Level 3 — Intermediate (Bellman-Ford Algorithm)
*Problem: Find the shortest path from node A to node B. WARNING: The graph contains NEGATIVE weight edges (e.g., traveling down this road actually gives you +10 gallons of gas back).*

Dijkstra's Algorithm mathematically collapses if there are negative weights. It uses a greedy approach and assumes costs always increase. **Bellman-Ford** solves this by relaxing EVERY single edge in the graph `V - 1` times!

```javascript
function bellmanFord(n, edges, startNode) {
    const distances = new Array(n).fill(Infinity);
    distances[startNode] = 0;
    
    // Relax all edges (V - 1) times
    for (let i = 0; i < n - 1; i++) {
        // Did we make any updates in this iteration?
        let updated = false; 
        
        for (let [u, v, weight] of edges) {
            // If u is reachable, and the path through u is cheaper...
            if (distances[u] !== Infinity && distances[u] + weight < distances[v]) {
                distances[v] = distances[u] + weight;
                updated = true;
            }
        }
        
        // Optimization: If we ran a full loop and didn't update anything, we are done early!
        if (!updated) break; 
    }
    
    // Cycle Detection! 
    // If we run it ONE MORE TIME, and distances STILL decrease, 
    // it means there is a Negative Weight Cycle (an infinite loop of infinite profit)!
    for (let [u, v, weight] of edges) {
        if (distances[u] !== Infinity && distances[u] + weight < distances[v]) {
            throw new Error("Graph contains a negative-weight cycle!");
        }
    }
    
    return distances;
}
```

### Level 4 — Advanced (Floyd-Warshall - All Pairs Shortest Path)
*Problem: Given a weighted graph, find the shortest path from EVERY node to EVERY OTHER node!*
Running Dijkstra's algorithm `V` times takes $O(V \times E \log V)$. 
**Floyd-Warshall** does it elegantly in $O(V^3)$ using a 3D Dynamic Programming approach!

```javascript
function floydWarshall(n, edges) {
    // dp[i][j] represents the shortest distance from node i to node j
    const dp = Array.from({ length: n }, () => new Array(n).fill(Infinity));
    
    // Distance from a node to itself is always 0
    for (let i = 0; i < n; i++) dp[i][i] = 0;
    
    // Initialize the matrix with the known direct edges
    for (let [u, v, weight] of edges) {
        dp[u][v] = weight;
    }
    
    // The Core DP Transition:
    // For every intermediate node 'k', can we find a cheaper path from 'i' to 'j' 
    // by traveling THROUGH 'k' instead of going directly?
    for (let k = 0; k < n; k++) {
        for (let i = 0; i < n; i++) {
            for (let j = 0; j < n; j++) {
                
                if (dp[i][k] !== Infinity && dp[k][j] !== Infinity) {
                    dp[i][j] = Math.min(dp[i][j], dp[i][k] + dp[k][j]);
                }
                
            }
        }
    }
    
    return dp;
}
```

## REAL_WORLD
**Google Maps & GPS Navigation**
When you ask Google Maps for driving directions, it does not run standard Dijkstra's. The road network of North America has over 20 million intersections (nodes). Standard Dijkstra's would expand radially in all directions (even backwards!) taking minutes to calculate.
Modern GPS uses a hyper-optimized variation of Dijkstra called **A* (A-Star) Search**. It injects a heuristic (like the straight-line distance to the destination) into the Priority Queue math, actively pulling the search algorithm *towards* the destination, heavily pruning useless backward roads and dropping calculation times to milliseconds.

## INTERVIEW
**Q: I used Dijkstra's Algorithm on a graph with negative edges. Sometimes it gave the right answer, but sometimes it failed. Why?**
A: Dijkstra's makes a strict, greedy mathematical assumption: *The first time I pop a node off the Min-Heap, I have found the absolute shortest possible path to that node.* It assumes that any subsequent paths discovered later will logically cost *more* because all edge weights are positive. 
If negative edges exist, a path discovered later might actually be cheaper (because adding an edge subtracts from the total cost). Dijkstra's will blindly ignore this cheaper path because it already permanently locked in the initial answer. You must use Bellman-Ford for negative weights.
