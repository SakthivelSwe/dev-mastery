## WHY
If you want to find the shortest driving route from New York to Los Angeles, or if you want to know how many "degrees of separation" (mutual friends) exist between you and Kevin Bacon on Facebook, you need an algorithm that searches radially outward in expanding concentric circles.

**Breadth-First Search (BFS)** is the algorithm for the job. While DFS dives deep into a single dead-end, BFS explores every possible move 1 step away, then every move 2 steps away, etc. It is mathematically guaranteed to find the absolute shortest path in an unweighted graph.

## THEORY
### The Queue and The Visited Set
Unlike a Binary Tree (where paths only go down), Graphs can have cycles (Node A points to Node B, which points back to Node A). If you do not track where you have been, your algorithm will become trapped in an infinite loop. 

Graph BFS requires two things:
1. **A Queue (FIFO):** To ensure we process nodes in strict order of their distance from the start.
2. **A Visited Set:** A Hash Set to track nodes we have already seen, preventing cycles.

```javascript
    0: [1, 2],

    2: [0, 3],
    3: [1, 2, 4],
    4: [3]
};

function bfsPrint(graph, startNode) {
    const queue = [startNode];
    const visited = new Set();
    
    // Mark the starting node as visited instantly!
    visited.add(startNode);
    
    while (queue.length > 0) {
        const current = queue.shift(); // Dequeue
        console.log(current);          // Process the node
        
        // Explore all neighbors of the current node
        for (let neighbor of graph[current]) {
            // ONLY enqueue if we haven't seen it before
            if (!visited.has(neighbor)) {
                visited.add(neighbor);
                queue.push(neighbor);
            }
        }
    }
}
```

### Level 2 — Building Blocks (Shortest Path in Unweighted Graph)
*Problem: Find the shortest path length (number of edges) from a `start` node to an `end` node.*

*Trick:* To track the distance, we enqueue an Array/Tuple containing both the Node AND its distance from the start!

```javascript
function shortestPath(graph, start, end) {
    if (start === end) return 0;
    
    // Queue stores: [node, distance_from_start]
    const queue = [[start, 0]];
    const visited = new Set([start]);
    
    while (queue.length > 0) {
        const [current, distance] = queue.shift();
        
        for (let neighbor of graph[current]) {
            // Did we find the target?
            if (neighbor === end) {
                return distance + 1;
            }
            
            if (!visited.has(neighbor)) {
                visited.add(neighbor);
                queue.push([neighbor, distance + 1]);
            }
        }
    }
    
    // Target is unreachable!
    return -1; 
}
```

### Level 3 — Intermediate (Rotting Oranges - Multi-Source BFS)
*Problem: You are given an `m x n` grid. `0` is an empty cell, `1` is a fresh orange, `2` is a rotten orange. Every minute, any fresh orange adjacent to a rotten orange becomes rotten. Return the minimum number of minutes until no cell has a fresh orange.*

*Trick:* This is a **Multi-Source BFS**. Instead of starting from a single node, we put ALL rotten oranges into the Queue at Minute 0! They will all radiate outward simultaneously.

```javascript
function orangesRotting(grid) {
    const rows = grid.length;
    const cols = grid[0].length;
    const queue = [];
    let freshCount = 0;
    
    // Step 1: Initialize the Queue with ALL rotten oranges, and count the fresh ones
    for (let r = 0; r < rows; r++) {
        for (let c = 0; c < cols; c++) {
            if (grid[r][c] === 2) {
                queue.push([r, c]);
            } else if (grid[r][c] === 1) {
                freshCount++;
            }
        }
    }
    
    // If there are no fresh oranges to begin with, it takes 0 minutes!
    if (freshCount === 0) return 0;
    
    let minutes = 0;
    const directions = [[1, 0], [-1, 0], [0, 1], [0, -1]];
    
    // Step 2: Multi-Source BFS (Level-Order)
    while (queue.length > 0 && freshCount > 0) {
        const levelSize = queue.length;
        
        for (let i = 0; i < levelSize; i++) {
            const [r, c] = queue.shift();
            
            for (let [dr, dc] of directions) {
                const newR = r + dr;
                const newC = c + dc;
                
                // If it's a valid coordinate and it's a FRESH orange...
                if (newR >= 0 && newR < rows && newC >= 0 && newC < cols && grid[newR][newC] === 1) {
                    // Rot it!
                    grid[newR][newC] = 2;
                    freshCount--;
                    // Push it to the queue so IT can rot its neighbors next minute!
                    queue.push([newR, newC]);
                }
            }
        }
        // One full minute has passed (we processed one entire layer of the queue)
        minutes++;
    }
    
    return freshCount === 0 ? minutes : -1;
}
```

### Level 4 — Advanced (Word Ladder)
*Problem: A transformation sequence from word `beginWord` to word `endWord` changes exactly one letter at a time, and every intermediate word must exist in the `wordList`. Return the number of words in the shortest transformation sequence.*
*(Example: "hit" -> "hot" -> "dot" -> "dog" -> "cog")*

*Trick:* This is secretly a Graph! Every word is a Node. An Edge exists between two words if they differ by exactly one letter. Because we want the shortest sequence, it is a BFS problem!

```javascript
function ladderLength(beginWord, endWord, wordList) {
    const wordSet = new Set(wordList);
    if (!wordSet.has(endWord)) return 0;
    
    // Queue stores: [currentWord, sequenceLength]
    const queue = [[beginWord, 1]];
    
    while (queue.length > 0) {
        const [word, length] = queue.shift();
        
        if (word === endWord) return length;
        
        // Try changing EVERY single letter in the word to every letter 'a'-'z'
        for (let i = 0; i < word.length; i++) {
            for (let c = 97; c <= 122; c++) { // ASCII 'a' to 'z'
                const newChar = String.fromCharCode(c);
                const newWord = word.substring(0, i) + newChar + word.substring(i + 1);
                
                // If this mutated word exists in our dictionary...
                if (wordSet.has(newWord)) {
                    // Enqueue it!
                    queue.push([newWord, length + 1]);
                    // Mark as visited by immediately removing it from the dictionary Set!
                    wordSet.delete(newWord); 
                }
            }
        }
    }
    
    return 0;
}
```

## REAL_WORLD
**Web Crawlers & Search Engines**
When Google indexes the internet, how does Googlebot traverse from website to website? It uses Graph BFS! It starts at a popular website (like Wikipedia or CNN), extracts all the `<a>` links on the homepage, and pushes them into a massive distributed Queue. It then visits all those pages (Level 1), extracts their links (Level 2), and so on. BFS ensures Google maps out the most highly connected and important parts of the internet first before diving into deep, isolated corners of the web.

## INTERVIEW
**Q: I wrote a BFS algorithm to find the shortest path, but it randomly failed some test cases where the edges had different weights (e.g., Road A takes 5 minutes, Road B takes 20 minutes). Why didn't BFS work?**
A: Standard BFS mathematically assumes that **every single edge in the graph has a weight of exactly 1**. It guarantees the shortest path *by number of edges* (e.g., 3 hops is better than 5 hops). 
If the edges have different weights/costs, a 1-hop path that costs 100 is worse than a 3-hop path that costs 5. 
To find the shortest path in a **Weighted Graph**, standard BFS fails completely. You must upgrade to **Dijkstra's Algorithm**, which uses a Min-Heap (Priority Queue) instead of a standard Queue to constantly explore the cheapest available path.
