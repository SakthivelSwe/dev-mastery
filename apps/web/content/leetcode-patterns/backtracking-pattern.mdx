## WHY
Imagine you are trying to solve a Sudoku puzzle, or find a path through a maze. You pick a number, and you go down that path. If you hit a dead end, you **backtrack**, erase that number, and try the next possible option.

**Backtracking** is a systematic algorithmic technique for generating all possible configurations of a search space. It is basically a specialized form of DFS (Depth-First Search) used to solve Constraint Satisfaction Problems (Combinations, Permutations, Subsets).

## THEORY
### The Core Architecture
Every Backtracking algorithm follows the exact same three-step blueprint:
1. **Make a Choice:** Add an element to your current temporary path/subset.
2. **Explore:** Recursively call the backtracking function to explore further down this path.
3. **Undo the Choice (Backtrack):** Remove the element you just added so that the loop can move on to the next choice in a clean state.

## CODE

### Level 1 — Beginner (Subsets / Power Set)
*Problem: Given an integer array `nums` of unique elements, return all possible subsets (the power set).*
*(Example: `[1,2]` returns `[[], [1], [2], [1,2]`)*

```javascript
function subsets(nums) {
    const result = [];
    
    // path: The current subset we are building
    // index: The current element in 'nums' we are deciding to include or exclude
    function backtrack(path, index) {
        // 1. Record the current valid subset
        // We MUST create a copy of the array [...path] because arrays are passed by reference!
        result.push([...path]);
        
        // 2. Iterate through all remaining choices
        for (let i = index; i < nums.length; i++) {
            
            // ACTION: Make a choice (Include the number)
            path.push(nums[i]);
            
            // EXPLORE: Move to the next index
            backtrack(path, i + 1);
            
            // UNDO (BACKTRACK): Remove the number to try the next one in the loop!
            path.pop();
        }
    }
    
    // Kick off the recursion with an empty subset starting at index 0
    backtrack([], 0);
    return result;
}
```

### Level 2 — Building Blocks (Permutations)
*Problem: Given an array `nums` of distinct integers, return all the possible permutations.*
*(Example: `[1,2,3]` returns 6 different orderings).*

*Trick:* Unlike Subsets (where order doesn't matter and we just move forward `i + 1`), in Permutations, order matters! We must consider ALL numbers from index 0 every time, but we cannot reuse a number that is already in the current path.

```javascript
function permute(nums) {
    const result = [];
    
    function backtrack(path) {
        // Base Case: If the path is the same length as nums, we have a full permutation!
        if (path.length === nums.length) {
            result.push([...path]);
            return;
        }
        
        for (let i = 0; i < nums.length; i++) {
            // CONSTRAINT: We cannot reuse numbers that are already in the path!
            if (path.includes(nums[i])) {
                continue; 
            }
            
            // ACTION
            path.push(nums[i]);
            
            // EXPLORE
            backtrack(path);
            
            // UNDO (BACKTRACK)
            path.pop();
        }
    }
    
    backtrack([]);
    return result;
}
```
*(Note: `path.includes()` is $O(N)$ time. In production, you would use a boolean `visited` array or a Hash Set to track used numbers in $O(1)$ time).*

### Level 3 — Intermediate (Combination Sum)
*Problem: Given an array of distinct integers `candidates` and a target integer `target`, return a list of all unique combinations where the chosen numbers sum to target. You may reuse the same number an unlimited number of times!*

```javascript
function combinationSum(candidates, target) {
    const result = [];
    
    // currentSum tracks the sum to avoid having to run a .reduce() on the path array
    function backtrack(path, currentSum, index) {
        // Base Case 1: We hit the target exactly!
        if (currentSum === target) {
            result.push([...path]);
            return;
        }
        // Base Case 2: We overshot the target. This branch is a dead end! (Pruning)
        if (currentSum > target) {
            return;
        }
        
        for (let i = index; i < candidates.length; i++) {
            // ACTION
            path.push(candidates[i]);
            
            // EXPLORE
            // CRITICAL DIFFERENCE: We pass 'i' instead of 'i + 1' because 
            // the problem allows us to reuse the EXACT same number again!
            backtrack(path, currentSum + candidates[i], i);
            
            // UNDO
            path.pop();
        }
    }
    
    backtrack([], 0, 0);
    return result;
}
```

### Level 4 — Advanced (N-Queens)
*Problem: The n-queens puzzle is the problem of placing $n$ queens on an $n \times n$ chessboard such that no two queens attack each other. Return all distinct solutions.*

*Trick:* We place one queen per row. To optimize the constraint checking, we use Sets to track which columns and diagonals are currently under attack!

```javascript
function solveNQueens(n) {
    const result = [];
    
    // Board state: tracks which column the queen was placed in for each row
    const board = []; 
    
    // Sets to track active attack lines in O(1) time
    const cols = new Set();
    const posDiag = new Set(); // (row + col) is constant for positive diagonals
    const negDiag = new Set(); // (row - col) is constant for negative diagonals
    
    function backtrack(row) {
        // Base Case: We successfully placed a queen in every row!
        if (row === n) {
            result.push(formatBoard(board, n));
            return;
        }
        
        // Try every column in the current row
        for (let col = 0; col < n; col++) {
            // CONSTRAINT CHECK: Is this square under attack?
            if (cols.has(col) || posDiag.has(row + col) || negDiag.has(row - col)) {
                continue;
            }
            
            // ACTION
            cols.add(col);
            posDiag.add(row + col);
            negDiag.add(row - col);
            board.push(col);
            
            // EXPLORE (Move down to the next row)
            backtrack(row + 1);
            
            // UNDO (BACKTRACK)
            cols.delete(col);
            posDiag.delete(row + col);
            negDiag.delete(row - col);
            board.pop();
        }
    }
    
    backtrack(0);
    return result;
}

// Helper to convert the array of column indices into the classic ".Q.." string format
function formatBoard(board, n) {
    return board.map(col => '.'.repeat(col) + 'Q' + '.'.repeat(n - col - 1));
}
```

## REAL_WORLD
**Regular Expression Engines**
When you write a complex Regex pattern with greedy quantifiers like `.*(a|b)`, the engine must try to match the string. If the engine matches the `.*` to the entire string, but then realizes it needs to match an `a` at the end, it hits a failure state. The Regex engine literally executes a **Backtracking algorithm** in C++, stepping backwards one character at a time, undoing its previous matches, and trying alternative branching paths until the regex succeeds or completely fails. 
(This is also why poorly written regexes can cause ReDoS attacks by triggering exponential $O(2^N)$ backtracking recursion).

## INTERVIEW
**Q: I wrote a Backtracking solution to return Subsets. When I logged it to the console inside the base case, the arrays looked perfect (`[1]`, `[1,2]`). But the final result returned by my function was just an array of empty arrays: `[[], [], []]`. Why?**
A: **Pass-By-Reference.** In JavaScript, arrays are objects. When you write `result.push(path)`, you are NOT pushing the values. You are pushing a memory pointer to the exact same `path` array. 
Because your backtracking algorithm is designed to eventually `pop()` every single item out of the `path` array until it is empty at the end, the final result will just be a bunch of pointers pointing to the now-empty array!
You MUST create a shallow copy of the array the exact moment you push it to the result: `result.push([...path])` or `result.push(path.slice())`.
