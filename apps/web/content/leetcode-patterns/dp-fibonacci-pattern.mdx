## WHY
The Fibonacci sequence ($0, 1, 1, 2, 3, 5, 8...$) is the "Hello World" of Dynamic Programming. 
Every algorithm student is taught to solve it recursively: `fib(n) = fib(n-1) + fib(n-2)`. 

While this mathematically works, it is computationally disastrous. To calculate `fib(5)`, the computer recalculates `fib(2)` three separate times. For `fib(50)`, it performs trillions of redundant calculations, completely freezing the browser. By applying the **Fibonacci DP Pattern**, we learn how to eliminate redundant work and solve sequential stepping problems in $O(N)$ time.

## THEORY
### Overlapping Subproblems
Dynamic Programming requires two things:
1. **Optimal Substructure:** The main problem can be solved by combining the solutions to smaller subproblems.
2. **Overlapping Subproblems:** The exact same subproblems are asked multiple times. 

If a problem asks you to calculate the $N^{th}$ step based on the $(N-1)$ and $(N-2)$ steps, it belongs to the Fibonacci family.

## CODE

### Level 1 — Beginner (Climbing Stairs)
*Problem: You are climbing a staircase. It takes `n` steps to reach the top. Each time you can either climb 1 or 2 steps. In how many distinct ways can you climb to the top?*

```javascript
function climbStairs(n) {
    // Edge cases
    if (n === 1) return 1;
    if (n === 2) return 2;
    
    // dp[i] represents the number of ways to reach step 'i'
    const dp = new Array(n + 1).fill(0);
    
    // Base cases (You can hardcode the first few steps)
    dp[1] = 1;
    dp[2] = 2;
    
    for (let i = 3; i <= n; i++) {
        // The core Fibonacci logic:
        // To get to step 3, you either stepped from step 2, or you jumped from step 1!
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    
    return dp[n];
}
```

### Level 2 — Building Blocks (Space Optimization)
If you look closely at Level 1, to calculate `dp[10]`, we ONLY need `dp[9]` and `dp[8]`. We do not care about `dp[1]` or `dp[2]` anymore. 
Creating a massive array of size $N$ consumes $O(N)$ memory. We can optimize this to $O(1)$ constant space by just using two variables!

```javascript
function climbStairsOptimized(n) {
    if (n === 1) return 1;
    if (n === 2) return 2;
    
    let twoStepsBehind = 1; // dp[1]
    let oneStepBehind = 2;  // dp[2]
    
    for (let i = 3; i <= n; i++) {
        let current = oneStepBehind + twoStepsBehind;
        
        // Shift the pointers forward for the next loop iteration!
        twoStepsBehind = oneStepBehind;
        oneStepBehind = current;
    }
    
    return oneStepBehind;
}
```

### Level 3 — Intermediate (House Robber)
*Problem: You are a professional robber. Each house has a certain amount of money. The only constraint is that you cannot rob two adjacent houses. Find the maximum amount of money you can rob tonight.*
*(Example: `[1, 2, 3, 1]` -> Rob house 1 and 3 -> `1 + 3 = 4`)*

This is Fibonacci with a twist. The decision isn't just adding the previous two steps; it requires picking the Maximum value!

```javascript
function rob(nums) {
    if (nums.length === 0) return 0;
    if (nums.length === 1) return nums[0];
    
    // dp[i] represents the max money we can have when standing at house 'i'
    const dp = new Array(nums.length).fill(0);
    
    // Base Cases
    dp[0] = nums[0];
    dp[1] = Math.max(nums[0], nums[1]);
    
    for (let i = 2; i < nums.length; i++) {
        // The Decision:
        // Option 1: Rob the current house (must add money from 2 houses ago)
        // Option 2: Skip the current house (keep the money from the house right next door)
        dp[i] = Math.max(
            nums[i] + dp[i - 2], 
            dp[i - 1]
        );
    }
    
    return dp[nums.length - 1];
}
```

### Level 4 — Advanced (Decode Ways)
*Problem: A message containing letters from A-Z can be encoded into numbers (`A -> 1`, `B -> 2`, ... `Z -> 26`). Given a string of digits, return the number of ways to decode it.*
*(Example: `"226"` can be `"BZ" (2, 26)`, `"VF" (22, 6)`, or `"BBF" (2, 2, 6)`. Answer: 3)*

```javascript
function numDecodings(s) {
    // If the string starts with a '0', it's instantly invalid (no letter maps to 0)
    if (s == null || s.length === 0 || s[0] === '0') return 0;
    
    const n = s.length;
    // dp[i] represents the number of ways to decode a string of length 'i'
    const dp = new Array(n + 1).fill(0);
    
    // Base cases
    dp[0] = 1; // There is 1 way to decode an empty string
    dp[1] = 1; // We already checked that s[0] isn't '0'
    
    for (let i = 2; i <= n; i++) {
        // Single digit decoding (e.g. "6" -> "F")
        // Get the character at the current position (i-1)
        const singleDigit = parseInt(s.substring(i - 1, i));
        if (singleDigit >= 1 && singleDigit <= 9) {
            // We can form a valid letter! Inherit the branches from 1 step back.
            dp[i] += dp[i - 1];
        }
        
        // Double digit decoding (e.g. "26" -> "Z")
        const doubleDigit = parseInt(s.substring(i - 2, i));
        if (doubleDigit >= 10 && doubleDigit <= 26) {
            // We can form a valid letter! Inherit the branches from 2 steps back.
            dp[i] += dp[i - 2];
        }
    }
    
    return dp[n];
}
```

## REAL_WORLD
**Algorithmic Trading & State Machines**
In quantitative finance, algorithms decide whether to BUY, SELL, or HOLD a stock on any given day based on complex historical states. A variation of this DP pattern (State Machine DP, like the famous "Best Time to Buy and Sell Stock with Cooldown" problem) is exactly how basic algorithmic trading systems map out the maximum possible profit trajectory over a massive time series of price data.

## INTERVIEW
**Q: If I use recursion with Memoization (Top-Down) instead of a `for` loop (Bottom-Up Tabulation), is it worse?**
A: **Technically, yes, but in an interview, no.** 
Top-Down Recursion + Memoization has the exact same Big-O Time and Space complexity ($O(N)$) as Tabulation. However, recursion requires the Operating System to add a new frame to the Call Stack for every single function call. For $N = 100,000$, this will trigger a "Maximum Call Stack Size Exceeded" error (Stack Overflow) in languages like JavaScript and Python. Tabulation (the `for` loop) operates entirely on the heap, avoiding call stack limits entirely. 
That being said, if recursion makes more sense to your brain during a stressful interview, *always* write the Memoized recursive solution first. It proves you understand the DP logic. You can optimize it to Tabulation if you have time remaining.
