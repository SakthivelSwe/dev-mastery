## WHY
Trees are hierarchical data structures. Unlike Arrays or Linked Lists (where you simply start at index 0 and walk to the end), there are multiple ways to "walk" through a Tree. 
If an interviewer asks you to build a file-system explorer, calculate the maximum depth of an org chart, or invert a binary tree, you absolutely must know the two foundational algorithms for exploring nodes: **Depth-First Search (DFS)** and **Breadth-First Search (BFS)**.

## THEORY
### DFS vs BFS
- **DFS (Depth-First Search):** Dives as deep as physically possible down a single branch before backtracking. It is almost always implemented recursively. It is extremely memory-efficient for deep, narrow trees.
- **BFS (Breadth-First Search):** Scans level-by-level horizontally. It visits all immediate children before moving to the grandchildren. It is always implemented iteratively using a **Queue**. It is perfect for finding the "Shortest Path" to a target.

## CODE

### Level 1 — Beginner (Pre, In, and Post Order DFS)
DFS in a Binary Tree comes in three flavors, determined entirely by *when* you process the current node.

```javascript
// A standard Binary Tree Node class
class TreeNode {
    constructor(val = 0, left = null, right = null) {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}

// 1. PRE-ORDER (Node, Left, Right)
// Good for duplicating a tree.
function preOrder(node) {
    if (!node) return;
    console.log(node.val); // Process the node FIRST!
    preOrder(node.left);
    preOrder(node.right);
}

// 2. IN-ORDER (Left, Node, Right)
// MAGIC TRICK: In a Binary Search Tree (BST), In-Order traversal guarantees 
// the nodes will be visited in perfectly sorted numerical order!
function inOrder(node) {
    if (!node) return;
    inOrder(node.left);
    console.log(node.val); // Process the node IN THE MIDDLE!
    inOrder(node.right);
}

// 3. POST-ORDER (Left, Right, Node)
// Good for deleting a tree (you must delete the children before the parent).
function postOrder(node) {
    if (!node) return;
    postOrder(node.left);
    postOrder(node.right);
    console.log(node.val); // Process the node LAST!
}
```

### Level 2 — Building Blocks (Level-Order BFS)
To traverse a tree level-by-level, you cannot use recursion. You must use a `while` loop and a `Queue` (FIFO).
*Problem: Return an array of arrays, where each inner array represents a horizontal level of the tree.*

```javascript
function levelOrder(root) {
    if (!root) return [];
    
    const result = [];
    const queue = [root]; // Initialize queue with the root
    
    while (queue.length > 0) {
        // How many nodes are on the current level?
        const levelSize = queue.length;
        const currentLevelNodes = [];
        
        // Loop EXACTLY levelSize times to process this specific level
        for (let i = 0; i < levelSize; i++) {
            // Dequeue the front node
            const currentNode = queue.shift(); 
            currentLevelNodes.push(currentNode.val);
            
            // If the node has children, enqueue them for the NEXT level!
            if (currentNode.left) queue.push(currentNode.left);
            if (currentNode.right) queue.push(currentNode.right);
        }
        
        // Add the finished level to the final result
        result.push(currentLevelNodes);
    }
    
    return result;
}
```

### Level 3 — Intermediate (Maximum Depth of Binary Tree)
*Problem: Given the root of a binary tree, return its maximum depth (the number of nodes along the longest path from root to a leaf).*

This is a classic "Bottom-Up" DFS problem. You ask the left and right children for their depth, take the maximum, and add 1 for yourself!

```javascript
function maxDepth(root) {
    // Base Case: If the node is null, its depth is 0.
    if (root === null) {
        return 0;
    }
    
    // Recursive Step: Ask the children for their depths!
    const leftDepth = maxDepth(root.left);
    const rightDepth = maxDepth(root.right);
    
    // The total depth is the max of the children, plus 1 for the current node.
    return Math.max(leftDepth, rightDepth) + 1;
}
```

### Level 4 — Advanced (Lowest Common Ancestor)
*Problem: Given a binary tree, find the lowest common ancestor (LCA) of two given nodes `p` and `q`.*
*(The LCA is the lowest node in the tree that has both `p` and `q` as descendants).*

```javascript
function lowestCommonAncestor(root, p, q) {
    // Base Case 1: Reached the bottom of a branch without finding p or q
    if (root === null) return null;
    
    // Base Case 2: We actually found p or q! Return this node UP the chain!
    if (root === p || root === q) return root;
    
    // Search the left and right subtrees
    const leftSearchResult = lowestCommonAncestor(root.left, p, q);
    const rightSearchResult = lowestCommonAncestor(root.right, p, q);
    
    // The Core Logic:
    // If BOTH the left and right search returned a node, it means 'p' is on one side
    // and 'q' is on the other. Therefore, the CURRENT node is the lowest common ancestor!
    if (leftSearchResult !== null && rightSearchResult !== null) {
        return root;
    }
    
    // Otherwise, pass whatever we found (p, q, or null) back up the chain
    return leftSearchResult !== null ? leftSearchResult : rightSearchResult;
}
```

## REAL_WORLD
**React's Virtual DOM Reconciliation**
When React compares the old Virtual DOM tree with the new Virtual DOM tree to figure out what changed, how does it traverse the trees?
React uses a highly optimized **Breadth-First Search (Heuristic $O(N)$)**. 
Why? Because in UI development, elements are much more likely to move sideways (e.g., swapping two sibling <li> elements) than they are to be moved vertically deep down into another branch. By comparing the trees level-by-level horizontally, React can quickly identify sibling changes and batch DOM updates with incredible efficiency.

## INTERVIEW
**Q: I implemented BFS in JavaScript using `queue.shift()`. For a tree with 1,000 nodes, it passed. For a tree with 100,000 nodes, it failed with a Time Limit Exceeded error. Why?**
A: **The JavaScript Array Shift Problem.** In Python or Java, a `Deque` (Double-Ended Queue) allows you to pop from the front in $O(1)$ time. In JavaScript, standard Arrays do not support $O(1)$ front removal. When you run `queue.shift()`, JS removes the first item, and then is forced to mathematically re-index all 99,999 remaining items in memory ($O(N)$ time). Doing this inside a loop turns a perfectly good $O(N)$ BFS algorithm into a devastatingly slow $O(N^2)$ algorithm. 
In a strict interview environment, you either need to implement a custom Linked-List Queue class, or use an index pointer (`let head = 0`) instead of `shift()` to simulate dequeuing without modifying the array.
