## WHY
If Breadth-First Search (BFS) is like a puddle of water slowly expanding outwards, **Depth-First Search (DFS)** is like a laser beam. 
DFS plunges deep into a graph, following a single path until it hits a dead-end, and only then does it backtrack. While BFS is perfect for finding the *shortest* path, DFS is the undisputed king of exploring **Connectivity**. If you need to map out connected islands, detect cycles in a dependency graph, or find all possible paths from A to B, DFS is mathematically superior and requires much less code (via recursion).

## THEORY
### Recursive Graph Traversal
The architecture of Graph DFS is identical to Tree DFS, with one critical addition: the `visited` set. Trees only flow downwards; graphs can loop backwards. If you recursively visit a neighbor without checking if you've already been there, your call stack will overflow instantly.

### Level 1 — Beginner (Number of Islands)
*Problem: Given an `m x n` 2D binary grid which represents a map of '1's (land) and '0's (water), return the number of islands. An island is surrounded by water and is formed by connecting adjacent lands horizontally or vertically.*

*Trick:* We scan the grid. Every time we find a piece of land, we increment our Island Counter, and then unleash a destructive DFS laser beam that infects all connected land and turns it to water ('0') so we never count it again!

```javascript
function numIslands(grid) {
    if (!grid || grid.length === 0) return 0;
    
    let islandCount = 0;
    
    for (let r = 0; r < grid.length; r++) {
        for (let c = 0; c < grid[0].length; c++) {
            // We found an unvisited piece of land!
            if (grid[r][c] === '1') {
                islandCount++;
                // Unleash the DFS to sink the entire island!
                sinkIslandDFS(grid, r, c);
            }
        }
    }
    
    return islandCount;
}

function sinkIslandDFS(grid, r, c) {
    // Base Case / Boundary Check: 
    // Are we out of bounds, or is this water? Return!
    if (r < 0 || r >= grid.length || c < 0 || c >= grid[0].length || grid[r][c] === '0') {
        return;
    }
    
    // Turn the land to water (This acts as our "Visited Set" in-place!)
    grid[r][c] = '0';
    
    // Recursively explore all 4 directions!
    sinkIslandDFS(grid, r + 1, c); // Down
    sinkIslandDFS(grid, r - 1, c); // Up
    sinkIslandDFS(grid, r, c + 1); // Right
    sinkIslandDFS(grid, r, c - 1); // Left
}
```

### Level 2 — Building Blocks (Max Area of Island)
*Problem: Find the maximum area (number of connected '1's) of an island in the given 2D array.*
*Trick:* We modify the DFS function to *return a mathematical value*! The area of a cell is `1 + Area of neighbors`.

```javascript
function maxAreaOfIsland(grid) {
    let maxArea = 0;
    
    for (let r = 0; r < grid.length; r++) {
        for (let c = 0; c < grid[0].length; c++) {
            if (grid[r][c] === '1') {
                const currentArea = getAreaDFS(grid, r, c);
                maxArea = Math.max(maxArea, currentArea);
            }
        }
    }
    
    return maxArea;
}

function getAreaDFS(grid, r, c) {
    if (r < 0 || r >= grid.length || c < 0 || c >= grid[0].length || grid[r][c] === '0') {
        return 0; // Water has an area of 0!
    }
    
    // Sink it so we don't count it twice!
    grid[r][c] = '0';
    
    // My area is 1, PLUS whatever the recursive calls find in all 4 directions!
    return 1 + 
           getAreaDFS(grid, r + 1, c) + 
           getAreaDFS(grid, r - 1, c) + 
           getAreaDFS(grid, r, c + 1) + 
           getAreaDFS(grid, r, c - 1);
}
```

### Level 3 — Intermediate (Course Schedule - Cycle Detection)
*Problem: There are a total of `numCourses` courses you have to take. Some courses have prerequisites (e.g., `[0, 1]` means to take course 0 you have to first take course 1). Can you finish all courses?*

*Trick:* This is a **Directed Graph Cycle Detection** problem. If A requires B, and B requires A, you have a paradox (a cycle), and you can never graduate. 
We use a 3-State Visited Array:
`0 = Unvisited`, `1 = Visiting (Currently in my recursion stack)`, `2 = Fully Processed`.
If during our DFS we encounter a node marked as `1`, we found a back-edge (a Cycle)!

```javascript
function canFinish(numCourses, prerequisites) {
    // 1. Build the Adjacency List (Graph)
    const graph = Array.from({ length: numCourses }, () => []);
    for (let [course, pre] of prerequisites) {
        graph[pre].push(course);
    }
    
    // 2. Initialize the State Array (0: Unvisited)
    const state = new Array(numCourses).fill(0);
    
    function hasCycleDFS(node) {
        // If we hit a node that is CURRENTLY in our recursion path... CYCLE!
        if (state[node] === 1) return true;
        // If we hit a node we already fully verified earlier... it's safe!
        if (state[node] === 2) return false;
        
        // Mark as "Visiting"
        state[node] = 1;
        
        // Explore all neighbors
        for (let neighbor of graph[node]) {
            if (hasCycleDFS(neighbor)) return true;
        }
        
        // We finished exploring all paths from this node safely!
        // Mark as "Fully Processed"
        state[node] = 2;
        return false;
    }
    
    // 3. We must run DFS on EVERY node, because the graph might be disconnected!
    for (let i = 0; i < numCourses; i++) {
        if (state[i] === 0) {
            if (hasCycleDFS(i)) return false; // Cycle detected! Cannot graduate.
        }
    }
    
    return true;
}
```

### Level 4 — Advanced (Word Search)
*Problem: Given an `m x n` grid of characters and a string `word`, return true if the word exists in the grid. The word can be constructed from letters of sequentially adjacent cells (horizontal or vertical). The same cell may not be used more than once.*

*Trick:* This requires **DFS with Backtracking**. We mark a cell as visited as we dive deep, but if that path fails, we must *un-visit* it so that a different path can reuse it!

```javascript
function exist(board, word) {
    const rows = board.length;
    const cols = board[0].length;
    
    function dfs(r, c, index) {
        // Base Case 1: If the index matches the word length, we found the whole word!
        if (index === word.length) return true;
        
        // Base Case 2: Boundary check AND letter match check
        if (r < 0 || r >= rows || c < 0 || c >= cols || board[r][c] !== word[index]) {
            return false;
        }
        
        // ACTION: Mark the cell as visited temporarily (using a dummy character)
        const tempChar = board[r][c];
        board[r][c] = '#';
        
        // EXPLORE: Branch out in all 4 directions looking for the NEXT letter (index + 1)
        const found = dfs(r + 1, c, index + 1) || 
                      dfs(r - 1, c, index + 1) || 
                      dfs(r, c + 1, index + 1) || 
                      dfs(r, c - 1, index + 1);
                      
        // BACKTRACK (UNDO): Restore the cell to its original letter for other paths to use!
        board[r][c] = tempChar;
        
        return found;
    }
    
    for (let r = 0; r < rows; r++) {
        for (let c = 0; c < cols; c++) {
            // Kick off the DFS anytime we find the FIRST letter of the word!
            if (board[r][c] === word[0]) {
                if (dfs(r, c, 0)) return true;
            }
        }
    }
    
    return false;
}
```

## REAL_WORLD
**NPM and Package Managers**
When you run `npm install react`, how does the computer know what order to download and install the dependencies? React requires `loose-envify`, which requires `js-tokens`. 
NPM builds a Directed Acyclic Graph (DAG) of all dependencies. It runs a variation of DFS called **Topological Sort**. By diving to the absolute deepest node (the package that has zero dependencies of its own) and installing it first, it mathematically guarantees that by the time it works its way back up to React, all required sub-dependencies are already installed! 
(If a cycle is detected using the algorithm in Level 3, NPM crashes with a "Circular Dependency" error).

## INTERVIEW
**Q: I wrote the `numIslands` DFS solution recursively. The interviewer asked me what the Space Complexity was. I said $O(1)$ because I modified the grid in-place and didn't create a `Visited` Array. They said I was wrong. Why?**
A: **The Call Stack is not free.** Even though you didn't create a manual array or hash map, recursion relies on the Operating System's Call Stack. In the absolute worst-case scenario (where the entire $M \times N$ grid is a single massive island snaking around like a maze), the DFS function will recursively call itself $M \times N$ times before hitting a base case. Therefore, the implicit Space Complexity is actually $O(M \times N)$ due to the call stack depth! To achieve true $O(1)$ space, you would have to rewrite it iteratively using a manual Queue/Stack, though this is rarely expected.
