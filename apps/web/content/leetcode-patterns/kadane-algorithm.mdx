## WHY
In 1984, Jay Kadane formulated a brilliant algorithm that solves the "Maximum Subarray Problem" in blazing fast $O(N)$ time. 
Given an array of numbers (including negatives), find the contiguous subarray that has the largest mathematical sum. A naive approach (checking every possible subarray) requires $O(N^2)$ time. **Kadane's Algorithm** proves that you only need to iterate through the array exactly once, making a simple, greedy mathematical decision at every single step.

It is a masterpiece of Dynamic Programming and a highly frequent question in technical interviews.

## THEORY
### The Core Philosophy
As you iterate through the array, at each index, you ask yourself one simple question:
**"Am I better off adding the current number to the existing running total, OR should I throw away the past entirely and start a brand new subarray starting with just this number?"**

If the running total ever drops below zero, it is mathematically poisoning your future sums. You should instantly reset the running total to 0.

## CODE

### Level 1 — Beginner (Maximum Subarray)
*Problem: Given an integer array `nums`, find the contiguous subarray (containing at least one number) which has the largest sum and return its sum.*

```javascript
function maxSubArray(nums) {
    // Initialize both to the first element to handle arrays with all negative numbers!
    let max_so_far = nums[0];
    let current_max = nums[0];
    
    for (let i = 1; i < nums.length; i++) {
        // The core Kadane decision:
        // Do I attach to the previous sum, or do I start fresh?
        current_max = Math.max(nums[i], current_max + nums[i]);
        
        // Did we just beat the all-time high score?
        max_so_far = Math.max(max_so_far, current_max);
    }
    
    return max_so_far;
}

console.log(maxSubArray([-2, 1, -3, 4, -1, 2, 1, -5, 4]));
// Output: 6 (The subarray is [4, -1, 2, 1])
```

### Level 2 — Building Blocks (Returning the Subarray Elements)
What if the interviewer asks you to return the actual subarray itself, not just the sum? We need to track the indices!

```javascript
function findMaxSubArrayElements(nums) {
    let max_so_far = nums[0];
    let current_max = nums[0];
    
    let start_idx = 0;
    let end_idx = 0;
    let temp_start_idx = 0; // Tracks the start of the CURRENT running total
    
    for (let i = 1; i < nums.length; i++) {
        // If starting fresh is better than adding to the current sum...
        if (nums[i] > current_max + nums[i]) {
            current_max = nums[i];
            temp_start_idx = i; // Reset the starting pointer!
        } else {
            current_max += nums[i];
        }
        
        // If we beat the high score, officially record the indices!
        if (current_max > max_so_far) {
            max_so_far = current_max;
            start_idx = temp_start_idx;
            end_idx = i;
        }
    }
    
    // Return the sliced array using the recorded indices
    return nums.slice(start_idx, end_idx + 1);
}
```

### Level 3 — Intermediate (Maximum Product Subarray)
*Problem: Find the contiguous subarray that has the largest PRODUCT.*
*(Warning: `[-2, 3, -4]` has a max sum of `3`, but a max product of `24` because two negatives multiply to make a massive positive! Kadane's algorithm must be modified).*

Because a negative number instantly flips the sign, we must track BOTH the Maximum Product *and* the Minimum Product (because the minimum negative could become a massive positive on the very next loop!).

```javascript
function maxProduct(nums) {
    if (nums.length === 0) return 0;
    
    let max_so_far = nums[0];
    let current_max = nums[0];
    let current_min = nums[0];
    
    for (let i = 1; i < nums.length; i++) {
        const num = nums[i];
        
        // If the current number is negative, it will FLIP the max and min!
        // So we swap them!
        if (num < 0) {
            let temp = current_max;
            current_max = current_min;
            current_min = temp;
        }
        
        // The standard Kadane decision
        current_max = Math.max(num, current_max * num);
        current_min = Math.min(num, current_min * num);
        
        max_so_far = Math.max(max_so_far, current_max);
    }
    
    return max_so_far;
}
```

### Level 4 — Advanced (Maximum Circular Subarray)
*Problem: The array is circular (the last element connects to the first). Find the maximum subarray sum.*
*Trick:* The max subarray is either a normal subarray (found by standard Kadane) OR it wraps around the edges. If it wraps around the edges, it means the *middle* of the array is the Minimum Subarray! 

```javascript
function maxSubarraySumCircular(nums) {
    let total_sum = 0;
    
    let max_so_far = nums[0];
    let current_max = 0;
    
    let min_so_far = nums[0];
    let current_min = 0;
    
    for (let i = 0; i < nums.length; i++) {
        const num = nums[i];
        total_sum += num;
        
        // Standard Kadane for MAX
        current_max = Math.max(num, current_max + num);
        max_so_far = Math.max(max_so_far, current_max);
        
        // Standard Kadane for MIN
        current_min = Math.min(num, current_min + num);
        min_so_far = Math.min(min_so_far, current_min);
    }
    
    // Edge case: If all numbers are negative, total_sum == min_so_far!
    // Wrapping around would yield 0, which is incorrect. Just return max_so_far.
    if (max_so_far < 0) {
        return max_so_far;
    }
    
    // The answer is either the normal MAX, or the Total Sum minus the normal MIN
    return Math.max(max_so_far, total_sum - min_so_far);
}
```

## REAL_WORLD
**Computer Vision (Finding the brightest spot)**
Kadane's Algorithm is heavily used in 2D image processing to find the brightest rectangular section of an image (e.g., detecting the sun or a lightbulb in a photograph). By converting the image pixels to a 2D matrix of light intensities, and adapting Kadane's algorithm to 2D (often combined with the Prefix Sum matrix pattern), computer vision systems can identify the brightest sub-matrix in $O(N^3)$ time instead of an impossible $O(N^6)$ brute force time.

## INTERVIEW
**Q: I initialized `current_max = 0` and `max_so_far = 0` before the loop. The algorithm worked for most test cases but failed when the array contained ONLY negative numbers (e.g. `[-5, -2, -9]`). It returned 0 instead of -2. Why?**
A: Because an empty subarray has a sum of 0, if you initialize your variables to 0, you are implicitly telling the algorithm: "If everything is negative, just pick nothing (0)." If the problem explicitly states that a subarray MUST contain at least one element, you must initialize your max variables to the **first element in the array** (`nums[0]`) or `-Infinity`. This forces the algorithm to pick the "least bad" negative number rather than defaulting to 0.
