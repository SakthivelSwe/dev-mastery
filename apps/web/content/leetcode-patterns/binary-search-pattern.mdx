## WHY
If you are asked to find a specific word in a 1,000-page dictionary, you don't start at page 1 and read every single word until you find it ($O(N)$ time). You open the dictionary to the exact middle. If the word you are looking for comes alphabetically *after* the page you are on, you instantly tear the first half of the dictionary out and throw it in the trash. You just eliminated 500 pages of work in a single step!

This is **Binary Search**. It is the most powerful search algorithm in computer science, turning an $O(N)$ search space into an incredibly fast $O(\log N)$ search space.

## THEORY
### The Core Concept
Binary search relies on three pointers: `left`, `right`, and `mid`.
It requires the search space (the array) to be **sorted**.

1. Calculate `mid = left + (right - left) / 2`.
2. Check if the element at `mid` is the target.
3. If the target is greater than the element at `mid`, discard the left half by moving `left = mid + 1`.
4. If the target is smaller than the element at `mid`, discard the right half by moving `right = mid - 1`.
5. Repeat until the target is found, or `left > right` (meaning the target does not exist).

## CODE

### Level 1 — Beginner (Standard Binary Search)
**Problem:** Given an array of integers `nums` which is sorted in ascending order, and an integer `target`, write a function to search `target` in `nums`. If `target` exists, then return its index. Otherwise, return `-1`.

```java
// Time Complexity: O(log N) | Space Complexity: O(1)
public int search(int[] nums, int target) {
    int left = 0;
    int right = nums.length - 1;
    
    // We use <= because the target might be exactly at the left/right pointer
    while (left <= right) {
        
        // CAUTION: (left + right) / 2 can cause integer overflow!
        // Always use this formula for mid:
        int mid = left + (right - left) / 2;
        
        if (nums[mid] == target) {
            return mid; // Target found!
        } else if (nums[mid] < target) {
            // Target is strictly greater, discard the left half
            left = mid + 1;
        } else {
            // Target is strictly smaller, discard the right half
            right = mid - 1;
        }
    }
    
    return -1; // Target not found
}
```

### Level 2 — Building Blocks (First or Last Occurrence)
What if the array has duplicates, and you need to find the *first* occurrence of the target?
`[1, 2, 2, 2, 2, 2, 3]` (Target: 2)

If a standard binary search lands on the 4th index, it returns immediately. But the *first* occurrence is actually at the 1st index!

```java
public int findFirstOccurrence(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    int firstOccurrence = -1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;
        
        if (nums[mid] == target) {
            // We found the target, but is it the FIRST one?
            // Record the index, but KEEP SEARCHING the left half just in case!
            firstOccurrence = mid;
            right = mid - 1;
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    
    return firstOccurrence;
}
```

### Level 3 — Intermediate (Search in a Rotated Sorted Array)
**Problem:** A sorted array was rotated at an unknown pivot (e.g., `[0,1,2,4,5,6,7]` might become `[4,5,6,7,0,1,2]`). Find the target.

Even though it's rotated, it's still *mostly* sorted. You can still use Binary Search by determining which half of the array is perfectly sorted, and checking if the target falls within that sorted boundary.

```java
public int searchRotated(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) return mid;
        
        // 1. Is the Left half perfectly sorted?
        if (nums[left] <= nums[mid]) {
            // Does the target fall within this perfectly sorted left half?
            if (target >= nums[left] && target < nums[mid]) {
                right = mid - 1; // It does! Search left.
            } else {
                left = mid + 1;  // It doesn't. Search right.
            }
        } 
        // 2. Otherwise, the Right half MUST be perfectly sorted.
        else {
            // Does the target fall within this perfectly sorted right half?
            if (target > nums[mid] && target <= nums[right]) {
                left = mid + 1;  // It does! Search right.
            } else {
                right = mid - 1; // It doesn't. Search left.
            }
        }
    }
    return -1;
}
```

### Level 4 — Advanced (Binary Search on the Answer)
This is the ultimate evolution of Binary Search. Sometimes, you aren't searching through an array at all. You are searching for an optimal mathematical answer.

**Problem (Koko Eating Bananas):** Koko wants to eat all her bananas within `H` hours. What is the *minimum* eating speed (`K` bananas/hour) she can choose?

You know the speed `K` must be between `1` and the `max(piles)`.
Instead of checking every single speed starting from 1 (which would be $O(N)$), you run a Binary Search on the *possible speeds*!

```java
public int minEatingSpeed(int[] piles, int h) {
    // The search space is the minimum and maximum possible eating speed!
    int left = 1;
    int right = getMaxSpeed(piles);
    int optimalSpeed = right;
    
    while (left <= right) {
        int midSpeed = left + (right - left) / 2;
        
        // Helper function simulating if this speed works
        if (canFinishInTime(piles, midSpeed, h)) {
            optimalSpeed = midSpeed; 
            // It worked! But can we eat even slower? (Search the left half)
            right = midSpeed - 1;
        } else {
            // Too slow. We must eat faster. (Search the right half)
            left = midSpeed + 1;
        }
    }
    return optimalSpeed;
}
```

## REAL_WORLD
**Database Indexing**
Relational databases (like PostgreSQL and MySQL) use B-Trees for their indexes. A B-Tree is essentially a massive, highly-optimized generalization of a Binary Search Tree. When you run `SELECT * FROM users WHERE id = 12345`, the database executes a binary search through the B-Tree index, finding the exact user out of a billion rows in 3 or 4 disk reads!

## INTERVIEW
**Q: Why do we write `mid = left + (right - left) / 2` instead of `mid = (left + right) / 2`?**
A: **Integer Overflow.** In Java and C++, a 32-bit integer has a maximum value of roughly 2.14 billion. If you have a massive array and `left` is 1 billion, and `right` is 2 billion, `left + right` equals 3 billion. This exceeds the maximum integer capacity, causing an overflow (resulting in a negative number!), which will throw an `ArrayIndexOutOfBoundsException`. Subtracting `left` from `right` guarantees the calculation never exceeds the bounds of a standard integer.
