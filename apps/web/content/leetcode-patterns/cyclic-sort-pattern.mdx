## WHY

When inputs are integers in a known range 1..n, generic O(n log n) sorting wastes the structure. Before cyclic sort, engineers reached for HashSet dedup or full sorts to find missing/duplicate numbers — extra O(n) memory and slower. Cyclic sort exploits the value→index mapping: number v belongs at index v−1, so one pass of swaps places everything home. The failure mode it prevents: O(n) extra space blowing memory limits on 10M-element arrays, and subtle off-by-one bugs when scanning for the first missing positive. Senior engineers recognise "1..n range" as the trigger for O(n) time, O(1) space.

## THEORY

Cyclic sort repeatedly swaps the element at index i to its correct slot v−1 until i holds the right value; each swap places ≥1 number permanently, so total swaps ≤ n.

```mermaid
flowchart TD
    A[i=0] --> B{a[i] in range and a[i]!=a[a[i]-1]?}
    B -->|yes| C[swap a[i], a[a[i]-1]]
    C --> B
    B -->|no| D[i++]
    D --> E{i<n?}
    E -->|yes| B
    E -->|no| F[scan for first i where a[i]!=i+1]
```

1. Map value v → index v−1.
2. Swap until home; missing index reveals answer.

| Approach | Time | Space |
|----------|------|-------|
| HashSet | O(n) | O(n) |
| Sort | O(n log n) | O(1) |
| Cyclic | O(n) | O(1) |

Misconception: cyclic sort needs distinct values — duplicates are fine, swap stops when target slot already correct.

## CODE

### Level 1 — Beginner: Place numbers home
```java
for(int i=0;i<n;){int j=a[i]-1;if(a[i]>0&&a[i]<=n&&a[i]!=a[j]){int t=a[i];a[i]=a[j];a[j]=t;}else i++;}
```

### Level 2 — Intermediate: Missing number
```java
int i=0;while(i<n){int j=a[i];if(j<n&&a[i]!=a[j]){int t=a[i];a[i]=a[j];a[j]=t;}else i++;}
for(i=0;i<n;i++)if(a[i]!=i)return i;return n;
```

### Level 3 — Advanced: First missing positive
```java
for(int i=0;i<n;){int j=a[i]-1;if(a[i]>0&&a[i]<=n&&a[i]!=a[j]){int t=a[i];a[i]=a[j];a[j]=t;}else i++;}
for(int i=0;i<n;i++)if(a[i]!=i+1)return i+1;return n+1;
```

### Level 4 — Expert: Find all duplicates O(1) space
```java
List<Integer> dups=new ArrayList<>();
for(int i=0;i<n;){int j=a[i]-1;if(a[i]!=a[j]){int t=a[i];a[i]=a[j];a[j]=t;}else i++;}
for(int i=0;i<n;i++)if(a[i]!=i+1)dups.add(a[i]);
return dups;
```

## INTERVIEW

**Q1 (Junior):** When is cyclic sort applicable? A: When values are in 1..n; map value to index for O(n) O(1).
**Q2 (Junior):** Why ≤ n swaps? A: Each swap fixes one element permanently.
**Q3 (Mid):** First missing positive approach? A: Place 1..n home, scan first mismatch.
**Q4 (Mid):** Duplicates handling? A: Stop when target slot already correct.
**Q5 (Senior):** vs HashSet? A: Same time, O(1) space — fits memory-limited 10M arrays.
