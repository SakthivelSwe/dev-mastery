## WHY
A website without links is just a static document. The entire concept of the "World Wide Web" was born from the ability to hyperlink distinct documents together across the globe. By mastering the Anchor tag (`<a>`) and the Image tag (`<img>`), you transform a boring text file into an interactive, visually engaging web page.

## THEORY
### Attributes
Both Links and Images heavily rely on **HTML Attributes**. An attribute is a piece of extra information injected directly into the opening tag of an element. It configures the element's behavior. Attributes always follow the `name="value"` format.

## CODE

### Level 1 — Beginner (The Anchor Tag)
The `<a>` (Anchor) tag creates a hyperlink. It requires the `href` (Hypertext Reference) attribute, which tells the browser exactly where to go when the user clicks the text.

```html
<!-- An Absolute URL (Links to an entirely different website) -->
<a href="https://www.google.com">Search on Google</a>

<!-- A Relative URL (Links to another page on your OWN website) -->
<a href="/about.html">About Us</a>
<a href="/contact">Contact Page</a>
```

### Level 2 — Building Blocks (Advanced Links)
Links don't just go to other web pages. They can trigger actions!

```html
<!-- Opens the user's default email client! -->
<a href="mailto:support@devmastery.com">Email Support</a>

<!-- Opens the user's phone dialer! -->
<a href="tel:+18005551234">Call Us</a>

<!-- Forces the link to open in a BRAND NEW TAB -->
<a href="https://github.com" target="_blank">View my GitHub</a>
```
*(Security Note: When using `target="_blank"`, you should historically include `rel="noopener noreferrer"` to prevent a rare security vulnerability where the new tab could hijack the original tab. Modern browsers now do this automatically, but it is still considered a best practice).*

### Level 3 — Intermediate (The Image Tag)
The `<img>` tag is unique. It is a **Void Element** (also known as a Self-Closing tag). Because an image doesn't wrap around text, it doesn't need a closing `</img>` tag!

The `<img>` tag absolutely requires two attributes:
1. `src` (Source): Where is the image file located?
2. `alt` (Alternate Text): A text description of the image.

```html
<!-- An image loaded from an external server -->
<img src="https://example.com/cute-dog.jpg" alt="A fluffy golden retriever playing in the grass">

<!-- An image loaded from your own project folder -->
<img src="/images/logo.png" alt="DevMastery Company Logo">
```
**Why is `alt` so important?** 
If the image link breaks, the browser displays the `alt` text instead. More importantly, screen readers read the `alt` text out loud to visually impaired users. If you omit the `alt` tag, your website fails accessibility audits.

### Level 4 — Advanced (Images inside Links)
What if you want to make an image clickable? You simply wrap the `<img>` tag entirely inside the `<a>` tag!

```html
<a href="https://twitter.com/devmastery" target="_blank">
  <!-- The user clicks the logo, and it opens Twitter! -->
  <img src="/icons/twitter.svg" alt="Follow us on Twitter">
</a>
```

## REAL_WORLD
**Jump Links (Anchor Hashes)**
Have you ever clicked a link and the page instantly scrolled all the way down to a specific section? This is called a Jump Link.
You do this by assigning an `id` attribute to the destination element, and then using a `#` in the `href`.

```html
<!-- 1. The Navigation Menu -->
<a href="#pricing">Go to Pricing</a>

<!-- ... 5000 lines of content ... -->

<!-- 2. The Destination (The browser scrolls exactly to here!) -->
<h2 id="pricing">Our Pricing Plans</h2>
```

## INTERVIEW
**Q: I have a decorative background image (a cool swirl pattern) that I want to put on my website. Should I use an `<img>` tag?**
A: **No.** The `<img>` tag has semantic meaning. It tells search engines and screen readers, "This image is an important piece of the content." If an image is purely decorative (it provides zero informational value to a blind user), you should NOT put it in your HTML. You should use a `<div class="hero">` and use the CSS `background-image` property to apply the swirl pattern. Keep HTML strictly for meaningful content!
