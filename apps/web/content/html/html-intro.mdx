## WHY
Every website on the internet — from Google to Netflix to your bank's portal — starts with HTML. HTML (HyperText Markup Language) is the foundational skeleton of all web pages. It is not a programming language; it is a **markup language** that uses tags to define the structure and meaning of content.

Without HTML, a browser sees raw, unstyled text. HTML tells the browser: "This text is a Heading. This text is a Paragraph. This is a list of items. This is a clickable link." The browser then renders each element with its default visual style based on its semantic meaning.

## THEORY
### What is the DOM?
When a browser loads an HTML file, it parses the tags and builds a tree data structure in memory called the **DOM (Document Object Model)**. Every HTML tag becomes a "Node" in this tree. JavaScript, CSS, and the browser itself interact with the DOM — not directly with the HTML file.

### Block vs Inline Elements
HTML elements are broadly categorized by their default layout behavior:
- **Block elements** (`<div>`, `<p>`, `<h1>`, `<ul>`): Take up the full width of their container, always starting on a new line.
- **Inline elements** (`<span>`, `<a>`, `<strong>`, `<img>`): Only take up as much width as their content, sitting inline within a line of text.

## CODE

### Level 1 — Beginner (The Basic Document Structure)
Every HTML page must follow this exact boilerplate structure.

```html
<!DOCTYPE html>
<!-- Tells the browser this is modern HTML5 (not old HTML4) -->

<html lang="en">
<!-- The root element. 'lang' helps screen readers and search engines -->

<head>
  <!-- The HEAD contains metadata — information ABOUT the page, not displayed on screen -->
  <meta charset="UTF-8">
  <!-- Declares the character encoding (UTF-8 supports ALL languages) -->
  
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <!-- CRITICAL for mobile! Tells mobile browsers not to zoom out -->
  
  <meta name="description" content="Learn HTML fundamentals on Dev Mastery.">
  <!-- The description shown under your link in Google search results -->
  
  <title>HTML Fundamentals | Dev Mastery</title>
  <!-- Displayed in the browser tab and used as the title in search results -->
  
  <link rel="stylesheet" href="styles.css">
  <!-- Link an external CSS file -->
</head>

<body>
  <!-- The BODY contains everything visible to the user -->
  <h1>Hello, World!</h1>
  <p>This is a paragraph of text.</p>
</body>

</html>
```

### Level 2 — Building Blocks (Core Text Elements)
```html
<!-- Headings: h1 to h6 (h1 is the most important — use only ONE per page!) -->
<h1>Main Page Heading</h1>
<h2>Section Title</h2>
<h3>Sub-section</h3>

<!-- Paragraphs: The default block of text -->
<p>This is a paragraph. It creates a block of text with spacing above and below.</p>

<!-- Inline text formatting -->
<p>
  You can make text <strong>bold (important)</strong> or <em>italic (emphasized)</em>.
  You can also <u>underline</u> or show <code>inline code</code>.
  Or add a line break<br>right here.
</p>

<!-- Links: The a (anchor) tag -->
<a href="https://www.example.com" target="_blank" rel="noopener noreferrer">
  Visit Example.com (opens in new tab)
</a>

<!-- Images: Self-closing tag. 'alt' is REQUIRED for accessibility! -->
<img src="/images/hero.jpg" alt="A developer coding at their desk" width="800" height="600">
```

### Level 3 — Intermediate (Lists, Tables, and Forms)
```html
<!-- Unordered List (bullets) -->
<ul>
  <li>JavaScript</li>
  <li>React</li>
  <li>TypeScript</li>
</ul>

<!-- Ordered List (numbers) -->
<ol>
  <li>Install Node.js</li>
  <li>Run npm install</li>
  <li>Start the server</li>
</ol>

<!-- A Basic HTML Form -->
<form action="/api/login" method="POST">
  <label for="email">Email Address:</label>
  <!-- 'for' must match the input's 'id' for accessibility -->
  <input type="email" id="email" name="email" required placeholder="alice@example.com">

  <label for="password">Password:</label>
  <input type="password" id="password" name="password" required minlength="8">
  
  <button type="submit">Login</button>
</form>
```

### Level 4 — Advanced (Semantic HTML5 Elements)
Semantic elements give meaning to your layout, improving SEO and accessibility. Always prefer them over generic `<div>` tags!

```html
<header>
  <!-- Site-wide header: Logo, main navigation -->
  <nav>
    <a href="/">Home</a>
    <a href="/about">About</a>
    <a href="/blog">Blog</a>
  </nav>
</header>

<main>
  <!-- The primary unique content of the page (only ONE per page!) -->
  <article>
    <!-- A standalone, self-contained piece of content (a blog post, a news article) -->
    <h1>Understanding Flexbox</h1>
    
    <section>
      <!-- A thematic grouping of content within the article -->
      <h2>The Flex Container</h2>
      <p>...</p>
    </section>
    
    <aside>
      <!-- Content tangentially related to the main content (a sidebar, an ad) -->
      <p>Related Articles: ...</p>
    </aside>
  </article>
</main>

<footer>
  <!-- Site-wide footer: Copyright, links -->
  <p>&copy; 2025 Dev Mastery. All rights reserved.</p>
</footer>
```

## REAL_WORLD
**HTML IS the first step in Search Engine Optimization (SEO)**.

Google's crawler reads your HTML to understand what your page is about. Critical SEO rules:
1. **One `<h1>` per page** — Google uses it to understand the page's primary topic.
2. **Descriptive `<title>` tag** — Appears as the blue link in Google results.
3. **`<meta name="description">` tag** — Appears as the grey snippet under the blue link.
4. **Semantic HTML** — Using `<article>`, `<nav>`, `<header>` helps Google understand the page's structure vs content.
5. **`alt` text on all images** — The only way Google can "see" your images.

## INTERVIEW
**Q: What is the difference between `<b>` and `<strong>`, and between `<i>` and `<em>`?**
A: `<b>` and `<i>` are purely **presentational** — they make text visually bold or italic, with no semantic meaning. `<strong>` means the text has high **importance**. `<em>` means the text is **emphasized**. Screen readers for blind users will read `<strong>` content with vocal stress, while `<b>` is ignored. Always prefer semantic tags!

**Q: What does `<!DOCTYPE html>` do?**
A: It is not a tag, but a declaration. It tells the browser to render the page in "standards mode" using the modern HTML5 specification. Without it, the browser enters "quirks mode" and tries to be backwards-compatible with Netscape Navigator from 1997, causing unpredictable CSS and layout behavior.

**Q: What is the `data-*` attribute?**
A: Custom attributes prefixed with `data-` allow you to embed arbitrary data in HTML elements without affecting how the browser renders them. They are accessible via JavaScript: `element.dataset.userId`. This is how you can attach metadata to DOM elements without polluting the class attribute with non-styling information.
