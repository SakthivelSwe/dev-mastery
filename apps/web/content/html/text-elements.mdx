## WHY
When you visit a website, the browser receives a raw string of text from the server. Without HTML, that text would render as one massive, unreadable paragraph. HTML Text Elements provide semantic structure, telling the browser exactly which text is a headline, which is a paragraph, and which should be emphasized. Structuring text properly is the absolute foundation of web development.

## THEORY
### Semantics vs Styling
A common beginner mistake is using an `<h1>` tag just because they want the text to look big and bold. **HTML is for Semantics (meaning); CSS is for Styling (appearance).** 

An `<h1>` tells search engines (like Google) and screen readers (for visually impaired users) that this is the most important topic on the page. If you just want big text, use a `<p>` tag and style it with CSS!

## CODE

### Level 1 — Beginner (Headings & Paragraphs)
There are six levels of headings, `<h1>` through `<h6>`.
- You should almost always have exactly ONE `<h1>` per page.
- Headings must be nested logically (don't jump from `<h2>` straight to `<h4>`).

```html
<!-- The main title of the page -->
<h1>Welcome to DevMastery</h1>

<!-- A subsection -->
<h2>Our Courses</h2>

<!-- A standard paragraph of text -->
<p>
  Learn the skills you need to become a senior developer.
  We offer interactive coding challenges and deep architectural dives.
</p>
```

### Level 2 — Building Blocks (Inline Formatting)
Sometimes you need to emphasize a specific word *inside* a paragraph.
Historically, developers used `<b>` (bold) and `<i>` (italic). Today, you should use `<strong>` and `<em>`. 
- `<b>` just makes text bold.
- `<strong>` makes text bold AND tells a screen reader to emphasize the word audibly!

```html
<p>
  Warning: You must <strong>never</strong> share your password.
</p>

<p>
  I am <em>so</em> excited to learn HTML!
</p>

<!-- The <br> tag forces a line break without starting a new paragraph -->
<p>
  123 Main Street<br>
  New York, NY 10001
</p>

<!-- The <hr> tag draws a thematic break (horizontal line) -->
<hr>
```

### Level 3 — Intermediate (Quotes and Code)
If you are building a blog or documentation site, you need specialized tags for quoting external sources and displaying source code.

```html
<!-- Blockquote is for long, multi-line quotes -->
<blockquote cite="https://en.wikipedia.org/wiki/Tim_Berners-Lee">
  The Web does not just connect machines, it connects people.
</blockquote>

<!-- The <q> tag is for short, inline quotes (it automatically adds quotation marks!) -->
<p>As my teacher always said, <q>Practice makes perfect.</q></p>

<!-- The <code> tag formats text with a monospace font -->
<p>To start the server, run the <code>npm start</code> command.</p>
```

### Level 4 — Advanced (Preformatted Text)
Browsers ignore extra whitespace. If you type 10 spaces in your HTML code, the browser collapses it into exactly 1 space when rendering. If you press "Enter" in your code, the browser ignores the line break.

If you are trying to display ASCII art or a massive block of formatted code, you must use the `<pre>` (Preformatted) tag. It tells the browser to respect every single space and line break exactly as typed.

```html
<!-- Usually combined with <code> for software documentation -->
<pre><code>
function hello() {
    console.log("The browser will respect this indentation!");
}
</code></pre>
```

## REAL_WORLD
**The `<span>` Tag**
What if you want to change the color of a single word in a paragraph, but you don't want to make it `<strong>` or `<em>`? 
You use a `<span>`. 
A `<span>` has absolutely zero semantic meaning and applies zero default styling. It is simply an empty container that allows you to target a specific piece of text with CSS or JavaScript.

```html
<p>
  The status of your order is <span class="status-green">Delivered</span>.
</p>
```

## INTERVIEW
**Q: What is the difference between a Block-level element and an Inline element?**
A: 
- **Block-level elements** (like `<h1>`, `<p>`, `<div>`) always start on a brand new line and stretch out as wide as possible to fill the entire width of their container.
- **Inline elements** (like `<span>`, `<strong>`, `<a>`) do NOT start on a new line. They only take up as much width as the text inside them, allowing them to sit side-by-side within a sentence.
