## WHY
Prop Drilling occurs when you pass data through 5 layers of components just so the 5th component can use it. It makes your code incredibly brittle.
Most developers immediately reach for the Context API or Redux to fix this. But 90% of the time, that is overkill. The official React documentation recommends solving Prop Drilling using a much simpler, more powerful pattern: **Component Composition** (passing JSX as props).

## THEORY
### Inversion of Control
Instead of Component A passing raw data to Component B, and Component B passing it to Component C to render it... 
Component A just renders Component C directly, and passes the *entire rendered JSX block* into Component B!
This "Inversion of Control" removes the need for intermediate components to know anything about the data.
