## WHY
Finding documents by an exact match (e.g., `{ age: 30 }`) is rarely enough for a real-world application. You need to find users older than 18, products that are either electronic or toys, or search for tags inside arrays. MongoDB provides a powerful set of **Query Operators** (always prefixed with a `$`) to perform complex filtering, mirroring the capabilities of SQL's `WHERE`, `IN`, and `> / <` clauses.

## THEORY
### Comparison vs Logical Operators
- **Comparison Operators** act on a single field to compare its value (e.g., Greater Than, In Array).
- **Logical Operators** combine multiple conditions together (e.g., AND, OR, NOR).

## CODE

### Level 1 — Beginner (Comparison Operators)
The most common operators are `$gt` (greater than), `$gte` (greater than or equal to), `$lt` (less than), and `$lte` (less than or equal to).

```javascript
// Find users exactly equal to 30 (No operator needed)
db.users.find({ age: 30 });

// Find users strictly older than 18
db.users.find({ age: { $gt: 18 } });

// Find products priced between 50 and 100
db.users.find({ 
    price: { $gte: 50, $lte: 100 } 
});

// Find users whose status is NOT equal to 'banned'
db.users.find({ status: { $ne: "banned" } });
```

### Level 2 — Building Blocks (The $in Operator)
What if you want to find users located in either New York, London, or Tokyo? You could use a logical `$or`, but the `$in` operator is much cleaner and more performant. It works exactly like the `IN` clause in SQL.

```javascript
// Find users in specific cities
db.users.find({ 
    city: { $in: ["New York", "London", "Tokyo"] } 
});

// Find users NOT in those cities
db.users.find({ 
    city: { $nin: ["New York", "London", "Tokyo"] } 
});
```

### Level 3 — Intermediate (Logical Operators)
MongoDB implicitly applies an `AND` operation if you pass multiple fields in a single query object. You only need the explicit `$and` operator in complex, overlapping scenarios. The `$or` operator is highly common.

Logical operators require an **Array** of query conditions!

```javascript
// IMPLICIT AND: Users older than 18 AND active
db.users.find({ age: { $gt: 18 }, status: "active" });

// EXPLICIT OR: Users who are EITHER older than 65 OR have a premium status
db.users.find({
    $or: [
        { age: { $gte: 65 } },
        { status: "premium" }
    ]
});

// COMPLEX COMBINATION: 
// (status == 'active') AND (age > 65 OR status == 'premium')
db.users.find({
    status: "active",
    $or: [
        { age: { $gt: 65 } },
        { status: "premium" }
    ]
});
```

### Level 4 — Advanced (Array Querying)
MongoDB's superpower is storing Arrays directly inside documents (e.g., `{ tags: ["tech", "sports"] }`). Querying these arrays requires special operators.

```javascript
// 1. Matches if the array contains EXACTLY these elements in this EXACT order
db.articles.find({ tags: ["tech", "sports"] });

// 2. Matches if the array contains 'tech' (Even if it has other tags too)
db.articles.find({ tags: "tech" });

// 3. $all: Matches if the array contains BOTH 'tech' and 'sports', regardless of order
db.articles.find({ 
    tags: { $all: ["tech", "sports"] } 
});

// 4. $size: Matches if the array has exactly 3 elements
db.articles.find({ tags: { $size: 3 } });
```

## REAL_WORLD
**The `$elemMatch` Operator**
If you have an array of *Objects* instead of simple strings, querying becomes dangerous.
Imagine: `{ results: [ { score: 80, item: "A" }, { score: 90, item: "B" } ] }`

If you query `db.tests.find({ "results.score": { $gt: 85 }, "results.item": "A" })`, you might expect it to return nothing. However, it **will match** the document above! Why? Because the document has *some* score > 85, and *some* item == "A", even though they aren't the *same* object in the array!

To fix this, you must use `$elemMatch` to force MongoDB to ensure both conditions are met by the *exact same object* inside the array:
```javascript
db.tests.find({ 
    results: { $elemMatch: { score: { $gt: 85 }, item: "A" } } 
});
```

## INTERVIEW
**Q: How do you search for a substring in MongoDB? (e.g., finding all users whose name contains 'John')**
A: You use the `$regex` (Regular Expression) operator. For example: `db.users.find({ name: { $regex: "John", $options: "i" } })`. However, you must be extremely careful in production. A regex that starts with a wildcard (e.g., searching for `%John%` instead of `John%`) cannot use standard B-Tree indexes. It will force MongoDB to execute a full collection scan (checking every single document on disk), which will instantly bottleneck and crash a large database. For heavy text searching, you should use MongoDB Atlas Search or Elasticsearch.
