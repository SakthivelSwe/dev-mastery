## WHY
HTML provides the structure of your document, but sometimes you need extra DOM nodes just for visual styling—like adding an icon, a decorative gradient overlay, or a tooltip triangle. 
Cluttering your HTML with empty `<div>` and `<span>` tags strictly for presentation violates the separation of concerns.

**Pseudo-elements** (`::before` and `::after`) act like virtual DOM nodes generated entirely by CSS. Mastering them allows you to create incredibly complex UI components (like custom checkboxes, tooltips, and ribbons) using only a single, clean HTML tag.

## THEORY
### The Phantom Nodes
When you use `::before` or `::after`, CSS injects a virtual inline element *inside* the target element, either right before the content or right after the content.

*The Golden Rule:* A pseudo-element **will not render** unless you provide it with a `content` property. Even if you just want a decorative square, you must write `content: "";`.

*(Note: Pseudo-classes use a single colon `:hover`, representing a State. Pseudo-elements use a double colon `::before`, representing a specific Part of the element).*

## CODE

### Level 1 — Beginner (Basic Injection)
Injecting text or simple shapes without touching the HTML.

```css
/* 1. Injecting Text */
/* Automatically append an asterisk to required form labels */
.required-label::after {
  content: " *";
  color: red;
}

/* 2. Injecting Quotes */
blockquote::before {
  content: "“";
  font-size: 3rem;
  color: #ccc;
}

/* 3. Decorative Shapes (Must have content: "") */
.bullet-list li::before {
  content: "";       /* MANDATORY */
  display: inline-block;
  width: 10px;
  height: 10px;
  background: blue;
  border-radius: 50%;
  margin-right: 10px;
}
```

### Level 2 — Building Blocks (Absolute Positioning Tricks)
Pseudo-elements truly shine when you combine them with `position: absolute`. They allow you to add complex overlays or badges to an element without adding wrapper divs.

```css
.card {
  position: relative; /* The pseudo-element will position itself relative to this! */
  padding: 20px;
  background: white;
}

/* A decorative ribbon hanging off the corner */
.card::before {
  content: "NEW";
  position: absolute;
  top: -10px;
  right: -10px;
  background: red;
  color: white;
  padding: 5px 15px;
  border-radius: 20px;
  font-weight: bold;
}

/* A gradient overlay that darkens an image on hover */
.image-container::after {
  content: "";
  position: absolute;
  inset: 0; /* top:0, left:0, right:0, bottom:0 */
  background: linear-gradient(to top, rgba(0,0,0,0.8), transparent);
  opacity: 0;
  transition: opacity 0.3s;
}

.image-container:hover::after {
  opacity: 1;
}
```

### Level 3 — Intermediate (Tooltips with Attributes)
You can use the `attr()` CSS function to extract data directly from HTML attributes and inject it into the `content` property! This is how pure-CSS tooltips are built.

```html
<!-- HTML -->
<button data-tooltip="Save your progress">Save</button>
```

```css
/* CSS */
button {
  position: relative;
}

button::after {
  /* Extract the text from the data-tooltip attribute! */
  content: attr(data-tooltip);
  
  position: absolute;
  bottom: 100%;
  left: 50%;
  transform: translateX(-50%);
  
  background: black;
  color: white;
  padding: 5px 10px;
  border-radius: 4px;
  font-size: 12px;
  white-space: nowrap; /* Prevent wrapping */
  
  /* Hide it by default */
  opacity: 0;
  pointer-events: none; /* Prevent tooltip from blocking mouse clicks */
}

button:hover::after {
  opacity: 1;
}
```

### Level 4 — Advanced (Custom Form Controls)
Native HTML checkboxes and radio buttons are notoriously difficult to style. The professional approach is to visually hide the native input, and draw a completely custom checkbox using `::before` on the `<label>`.

```html
<label class="custom-checkbox">
  <input type="checkbox">
  <span>I accept the terms</span>
</label>
```

```css
/* 1. Hide the native checkbox, but keep it accessible! */
.custom-checkbox input {
  position: absolute;
  opacity: 0;
  cursor: pointer;
}

/* 2. Draw our fake checkbox using ::before on the span */
.custom-checkbox span::before {
  content: "";
  display: inline-block;
  width: 20px;
  height: 20px;
  border: 2px solid #555;
  border-radius: 4px;
  margin-right: 10px;
  vertical-align: middle;
  background: white;
}

/* 3. When the invisible input is :checked, style the span's pseudo-element! */
.custom-checkbox input:checked + span::before {
  background-color: blue;
  border-color: blue;
  /* Use a background image or SVG for the checkmark */
  background-image: url('checkmark.svg');
  background-size: cover;
}

/* 4. Accessibility: Maintain focus rings! */
.custom-checkbox input:focus + span::before {
  box-shadow: 0 0 0 3px rgba(0, 0, 255, 0.3);
}
```

## REAL_WORLD
**CSS Art and Single-Div Components**
Browse CodePen, and you will see "CSS Art" where developers draw entire realistic cameras, characters, or landscapes using a single `<div class="camera"></div>`. How is this possible?
By using `box-shadow` to generate hundreds of distinct, positioned color blocks, and combining the main `<div>` with `::before` and `::after` (which can also each have hundreds of box-shadows), a single HTML node essentially becomes three distinct layered canvases.

## INTERVIEW
**Q: I added `::before` to an `<img>` tag to overlay a gradient, but nothing is showing up. What did I do wrong?**
A: You didn't do anything wrong; the specification forbids it. **Pseudo-elements (`::before` and `::after`) cannot be applied to Replaced Elements.** 
Replaced elements are tags whose contents are not defined by the document itself, but by an external resource (like `<img>`, `<video>`, `<iframe>`, or `<input>`). Because they do not have an internal formatting context or a closing tag in the DOM structure, CSS has nowhere to "inject" the virtual node. To overlay a gradient on an image, you must wrap the `<img>` in a `<div>`, and apply the pseudo-element to the `<div>`.
