## WHY
Static web pages feel cold and unresponsive. Animations transform a UI from a grid of information into a living, breathing experience that communicates state and guides attention. A button that responds to a click with a ripple effect, a menu that slides in smoothly, a loading skeleton that pulses — these micro-interactions are the difference between a product that feels cheap and one that feels premium.

CSS provides two mechanisms for animations: **Transitions** (for simple A→B state changes) and **Animations** (for complex, keyframe-driven sequences that can loop).

## THEORY
### CSS Transitions vs CSS Animations
- **Transition:** Smoothly animates between two states (hover on/off, active/inactive). Triggered by a CSS state change (`:hover`, class toggle). Cannot loop. Cannot define intermediate keyframes.
- **Animation:** Full, scriptable animation with multiple keyframes. Can loop indefinitely, run on page load, pause, and reverse. Defined with `@keyframes`.

## CODE

### Level 1 — Beginner (CSS Transitions)
```css
/* transition: property duration timing-function delay */
.btn {
    background-color: #6366f1;
    color: white;
    padding: 12px 24px;
    border-radius: 8px;
    
    /* Apply transition to specific properties */
    transition: background-color 200ms ease, transform 200ms ease, box-shadow 200ms ease;
    /* Or use 'all' (less efficient, but convenient) */
    /* transition: all 200ms ease; */
}

.btn:hover {
    background-color: #4f46e5;         /* Smoothly transitions! */
    transform: translateY(-3px);       /* Subtle lift */
    box-shadow: 0 8px 25px rgba(99, 102, 241, 0.4); /* Glow shadow */
}

.btn:active {
    transform: translateY(0px);        /* Press down on click */
    box-shadow: none;
}
```

### Level 2 — Building Blocks (Timing Functions)
```css
/* The timing-function controls the animation's acceleration curve */

.box {
    transition: transform 400ms ease;           /* Default: slow start, slow end */
}

/* Common timing functions */
transition: transform 400ms linear;         /* Constant speed throughout */
transition: transform 400ms ease-in;        /* Slow start, fast end (like falling) */
transition: transform 400ms ease-out;       /* Fast start, slow end (like decelerating) */
transition: transform 400ms ease-in-out;    /* Slow start, fast middle, slow end */

/* Custom Cubic Bezier — for custom spring-like feel */
transition: transform 400ms cubic-bezier(0.34, 1.56, 0.64, 1);
/* This creates an "overshoot" bounce effect! Values beyond 0-1 create spring physics. */

/* Modern: CSS Linear timing function (CSS 2023) */
transition: transform 400ms linear(0, 0.25, 0.55, 0.75, 0.92, 1);
```

### Level 3 — Intermediate (CSS @keyframes Animations)
```css
/* PULSE animation — used for loading states and notifications */
@keyframes pulse {
    0%, 100% { opacity: 1; transform: scale(1); }
    50%       { opacity: 0.7; transform: scale(0.97); }
}

.notification-badge {
    animation: pulse 1.5s ease-in-out infinite;
}

/* SLIDE IN animation */
@keyframes slideInFromLeft {
    from {
        opacity: 0;
        transform: translateX(-40px);
    }
    to {
        opacity: 1;
        transform: translateX(0);
    }
}

.sidebar {
    animation: slideInFromLeft 300ms ease-out forwards;
    /* 'forwards' keeps the final state after animation completes */
}

/* SKELETON SHIMMER loading effect */
@keyframes shimmer {
    from { background-position: -500px 0; }
    to   { background-position: 500px 0; }
}

.skeleton {
    background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
    background-size: 1000px 100%;
    animation: shimmer 1.5s linear infinite;
}
```

### Level 4 — Advanced (scroll-driven Animations — CSS 2024)
```css
/* NEW: Animations driven by scroll position — NO JavaScript required! */

@keyframes fadeInUp {
    from { opacity: 0; transform: translateY(30px); }
    to   { opacity: 1; transform: translateY(0); }
}

.section {
    /* Animate based on the element's scroll position in the viewport */
    animation: fadeInUp linear both;
    animation-timeline: view();
    /* Enter animation plays as the element enters the viewport from below */
    animation-range: entry 0% entry 40%;
}

/* Sticky header that changes on scroll */
@keyframes narrowHeader {
    from { padding: 24px 48px; box-shadow: none; }
    to   { padding: 12px 48px; box-shadow: 0 2px 20px rgba(0,0,0,0.1); }
}

header {
    animation: narrowHeader linear both;
    animation-timeline: scroll(root);
    animation-range: 0 200px;  /* Animate during the first 200px of page scroll */
}
```

## REAL_WORLD
**The Animation Performance Rule:**
Only animate `transform` and `opacity`. These are the ONLY two CSS properties that can be animated by the GPU (hardware acceleration), completely bypassing the CPU and avoiding layout recalculations.

Animating `width`, `height`, `top`, `left`, `margin`, or `padding` triggers a full browser Layout reflow (the browser recalculates the positions of ALL elements on the page), which causes janky, dropped frames. 

A 60fps animation (smooth) requires each frame to render in ≤16.67ms. A Layout reflow can take 100ms+, instantly dropping frames.

**The rule:** If you want to move an element, use `transform: translateX/Y()`. If you want to resize it, use `transform: scale()`. Never touch `width`, `height`, or position properties directly in an animation.

## INTERVIEW
**Q: What is `will-change: transform` and when should you use it?**
A: `will-change` is a performance hint that tells the browser to pre-promote the element to its own GPU compositing layer BEFORE the animation starts. This prevents the initial frame jank caused by the browser needing to create the layer on-the-fly. Only use it for elements that will DEFINITELY animate (too many compositing layers consume massive GPU memory). Add it, measure the performance, and remove it if there's no measurable improvement.

**Q: What is `prefers-reduced-motion` and why does it matter?**
A: Some users have vestibular disorders (inner ear conditions) where animations cause nausea, dizziness, or migraines. The `prefers-reduced-motion` CSS media query lets you respect the user's OS accessibility setting.
```css
@media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
        animation-duration: 0.01ms !important;
        transition-duration: 0.01ms !important;
    }
}
```
This is a legal accessibility requirement in many jurisdictions (WCAG 2.1 AAA).
