## WHY
If a button is blue, and its `:hover` state is red, moving the mouse over the button causes it to instantly snap to red in 0 milliseconds. It feels harsh, robotic, and cheap. 
**CSS Transitions** allow you to smooth out that state change, mathematically interpolating the colors over a set period of time (e.g., smoothly blending from blue to purple to red over 0.5 seconds). Transitions are the easiest and most performant way to make a website feel premium and interactive.

## THEORY
### Transitions vs Animations
- **Transitions** only occur when a property changes state (e.g., the user hovers over an element, or JavaScript adds an `.active` class). They run once from State A to State B.
- **Animations** (Keyframes) run independently. They can loop infinitely without any user interaction. (Covered in the CSS Animations module).

## CODE

### Level 1 — Beginner (The Shorthand Property)
You do NOT apply the transition to the `:hover` state! You apply it to the base element. It acts as an instruction: "If any of my properties ever change in the future, transition them smoothly."

```css
.button {
  background-color: blue;
  
  /* The Shorthand: [Property] [Duration] [Timing-Function] */
  /* Tells the browser: Animate the background color over half a second! */
  transition: background-color 0.5s ease;
}

.button:hover {
  background-color: red;
}
```

### Level 2 — Building Blocks (Transitioning Multiple Properties)
What if the button changes color *and* grows larger? You can transition multiple properties by separating them with a comma.

```css
.card {
  background-color: white;
  width: 200px;
  
  /* Color changes fast, but width expands slowly! */
  transition: background-color 0.2s ease, width 0.8s ease;
}

.card:hover {
  background-color: lightgray;
  width: 400px;
}
```
*(Lazy Trick: You can use `transition: all 0.5s ease;`. It works, but it is notoriously terrible for performance because the browser has to actively monitor every single CSS property in existence for changes. Always explicitly list the properties you want to transition!)*

### Level 3 — Intermediate (Timing Functions / Easing)
Things in the real world do not move at a perfectly constant speed. A car accelerates slowly, hits top speed, and decelerates as it approaches a stop sign.
The `transition-timing-function` dictates the acceleration curve of the animation.

- `linear`: Constant, robotic speed. (Usually looks terrible).
- `ease-in`: Starts slow, accelerates at the end. (Good for objects leaving the screen).
- `ease-out`: Starts fast, decelerates to a smooth stop. (Good for objects entering the screen).
- `ease-in-out`: Starts slow, fast in the middle, slow at the end.
- `cubic-bezier()`: A completely custom, mathematical acceleration curve!

```css
.menu-slider {
  left: -200px;
  /* Use a beautiful ease-out curve to make it slide in naturally */
  transition: left 0.4s ease-out;
}

.menu-slider.open {
  left: 0px;
}
```

### Level 4 — Advanced (Transition Delays)
You can force a transition to wait before executing using `transition-delay`. This is incredibly useful for staggering animations (e.g., dropping down a menu, waiting 0.2 seconds, and then fading in the text inside the menu).

```css
.tooltip-text {
  opacity: 0;
  
  /* [Property] [Duration] [Timing] [DELAY] */
  /* Wait a full second before fading in! */
  transition: opacity 0.3s ease 1s; 
}

.icon:hover .tooltip-text {
  opacity: 1;
}
```

## REAL_WORLD
**Discrete vs Continuous Properties**
You cannot transition a property that isn't mathematical. 
You can transition `height: 10px` to `height: 100px` because the browser can calculate the numbers in between (20, 30, 40). 
You **cannot** transition `display: none` to `display: block`. There are no mathematical steps between "does not exist" and "exists." If you try, it will instantly snap.

To smoothly fade in an element that is hidden, you must use `opacity` (0 to 1) instead of `display`! 
*(Note: As of late 2023, modern browsers are finally introducing a new `@starting-style` API to allow transitioning `display: none`, but it is brand new and not fully supported everywhere).*

## INTERVIEW
**Q: I wrote `transition: height 0.5s ease;` and tried to transition my element from `height: 0px` to `height: auto` so it naturally fits whatever text is inside it. But the transition didn't work; it instantly snapped open! Why?**
A: Browsers cannot calculate a transition to the word `auto`. The engine needs two strict numeric values (like `0px` to `500px`) to calculate the animation frames. Because `auto` is unknown until the browser finishes rendering the text inside, the math fails and defaults to an instant snap. A common industry workaround is to transition the `max-height` property instead (e.g., from `max-height: 0px` to `max-height: 1000px`), or to use CSS Grid (`grid-template-rows: 0fr` to `1fr`).
