## WHY
Flexbox excels at one-dimensional layouts — aligning items in a single row or column. But when you need to create a complex two-dimensional layout (a full page layout with a header, sidebar, main content, and footer, or a photo gallery with items of different sizes), Flexbox requires messy nesting and hacks.

**CSS Grid** is a two-dimensional layout system that lets you define explicit rows AND columns simultaneously, and place items at precise positions in that grid. It is the most powerful layout tool in the history of CSS.

## THEORY
### The Grid Vocabulary
- **Grid Container:** The element with `display: grid`.
- **Grid Lines:** The dividing lines that form the grid structure (numbered 1, 2, 3...).
- **Grid Track:** The space between two adjacent grid lines (a row or column).
- **Grid Cell:** The intersection of a row track and a column track (the smallest unit).
- **Grid Area:** A rectangular region spanning multiple cells, where you place an item.

## CODE

### Level 1 — Beginner (Defining Columns and Rows)
```css
/* A simple 3-column card grid */
.card-grid {
    display: grid;
    
    /* Define 3 equal-width columns */
    grid-template-columns: 1fr 1fr 1fr;  /* '1fr' = 1 Fraction of available space */
    /* Shorthand: */ 
    grid-template-columns: repeat(3, 1fr);
    
    /* Define row height (or just let it be automatic) */
    grid-template-rows: auto;  /* Height adjusts to content (default) */
    
    /* Space between cells */
    gap: 24px;
}

/* Items automatically fill the grid left-to-right, top-to-bottom */
.card { background: white; padding: 24px; border-radius: 8px; }
```

### Level 2 — Building Blocks (The fr Unit & auto-fill/auto-fit)
```css
/* The 'fr' (fraction) unit is revolutionary.
   It distributes the REMAINING space after fixed sizes are accounted for. */

.layout {
    display: grid;
    /* Sidebar: 280px fixed | Main: takes all remaining space | Right panel: 200px fixed */
    grid-template-columns: 280px 1fr 200px;
    height: 100vh;
}

/* Responsive grid WITHOUT media queries — auto-fill magic! */
.responsive-grid {
    display: grid;
    
    /* Create as many columns as fit, each AT LEAST 250px wide */
    /* auto-fill: creates columns even if empty (maintains grid lines) */
    /* auto-fit: collapses empty columns (items stretch to fill) */
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    
    gap: 24px;
}
/* On a 1400px wide screen: creates 5 columns of ~268px
   On a 768px wide screen: creates 3 columns of ~237px  
   On a 400px wide screen: creates 1 column of 400px
   All responsive! Zero @media queries! */
```

### Level 3 — Intermediate (Placing Items with Line Numbers)
```css
/* A full page layout using named grid lines */
.page-layout {
    display: grid;
    grid-template-columns: 260px 1fr;
    grid-template-rows: 64px 1fr 60px;
    min-height: 100vh;
}

/* Place items using grid-column and grid-row */
/* Syntax: grid-column: <start-line> / <end-line> */

.header {
    grid-column: 1 / -1;   /* Span from line 1 to the LAST line (-1) */
    grid-row: 1 / 2;
    background: #1e293b;
}

.sidebar {
    grid-column: 1 / 2;
    grid-row: 2 / 3;
    background: #0f172a;
}

.main {
    grid-column: 2 / -1;
    grid-row: 2 / 3;
    overflow-y: auto;
    padding: 32px;
}

.footer {
    grid-column: 1 / -1;
    grid-row: 3 / 4;
    background: #1e293b;
}
```

### Level 4 — Advanced (Named Grid Areas — The Most Readable Layout Tool)
```css
/* grid-template-areas lets you define your layout as ASCII art! */
.app-shell {
    display: grid;
    grid-template-columns: 260px 1fr;
    grid-template-rows: 64px 1fr 60px;
    min-height: 100vh;
    
    /* Define your layout visually! */
    grid-template-areas:
        "header  header"   /* Row 1: header spans both columns */
        "sidebar main  "   /* Row 2: sidebar left, main right */
        "footer  footer";  /* Row 3: footer spans both columns */
}

/* Assign elements to their named areas */
.header  { grid-area: header;  }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main;    }
.footer  { grid-area: footer;  }

/* RESPONSIVE: On mobile, stack everything vertically */
@media (max-width: 768px) {
    .app-shell {
        grid-template-columns: 1fr;
        grid-template-rows: auto auto 1fr auto;
        grid-template-areas:
            "header"
            "sidebar"
            "main"
            "footer";
    }
}
```

## REAL_WORLD
CSS Grid and Flexbox are the foundation of modern component libraries like **Chakra UI**, **Material UI**, and **shadcn/ui**. Their layout systems (stacks, grids, flex containers) are direct abstractions over these CSS features.

A powerful real-world pattern is the **"Holy Grail" layout** (Header + Left Sidebar + Main + Right Panel + Footer), which took hundreds of lines of CSS table hacks before Grid. With CSS Grid, it is 10 lines.

## INTERVIEW
**Q: When do you use CSS Grid vs Flexbox?**
A: **Grid** for the macro page layout (the overall scaffolding — where do the header, sidebar, and main content go?). **Flexbox** for the micro component layout (how are items arranged WITHIN a component — like aligning a button's icon and text in a row). They are complementary and are often used simultaneously: Grid for the page shell, Flexbox for the components within each grid area.

**Q: What is `grid-template-columns: repeat(auto-fill, minmax(200px, 1fr))`?**
A: It creates a responsive grid that automatically figures out how many columns fit based on the screen width. `minmax(200px, 1fr)` means each column is at least 200px wide and can grow to fill available space. `auto-fill` creates as many columns as physically fit. This creates a fully responsive multi-column layout with zero `@media` queries.

**Q: What is the difference between `auto-fill` and `auto-fit`?**
A: With 3 items in a 5-column grid: `auto-fill` maintains all 5 columns (the 2 empty ones still take up space). `auto-fit` collapses the 2 empty columns, allowing the 3 actual items to stretch and fill the entire container width.
