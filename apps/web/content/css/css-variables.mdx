## WHY
Before CSS Variables, if you wanted to use a specific brand blue (`#2563EB`) in 50 different places across your website, you had to hardcode that hex value 50 times. If the marketing team decided to change the brand color to purple, you had to perform a massive "Find and Replace", hoping you didn't accidentally change a completely different element that happened to use the same blue. 

Preprocessors like Sass introduced variables (`$primary-color: #2563EB;`), but they are compiled away at build time. **CSS Custom Properties (Variables)** are native to the browser, dynamic, and can be updated instantly with JavaScript or Media Queries without recompiling your CSS!

## THEORY
### Scope and Inheritance
CSS variables follow standard CSS inheritance rules. 
- Variables declared on the `:root` pseudo-class (which represents the `<html>` element) are **Global** — they can be accessed anywhere.
- Variables declared on a specific class (e.g., `.card`) are **Local** — they can only be accessed by the `.card` element and its children.

### Syntax
- **Declaration:** Start with two dashes: `--primary-color: blue;`
- **Usage:** Wrap the name in the `var()` function: `color: var(--primary-color);`

## CODE

### Level 1 — Beginner (Declaring and Using)
```css
/* 1. Declare variables in the global root scope */
:root {
  --primary: #4F46E5;
  --secondary: #10B981;
  --text-dark: #1F2937;
  --spacing-md: 16px;
  --border-radius: 8px;
}

/* 2. Use them throughout your CSS */
.button {
  background-color: var(--primary);
  color: white;
  padding: var(--spacing-md);
  border-radius: var(--border-radius);
}

.card {
  border: 1px solid var(--primary);
  padding: var(--spacing-md);
}
```

### Level 2 — Building Blocks (Fallbacks)
What happens if a variable isn't defined? The CSS property becomes invalid. You can provide a fallback value directly in the `var()` function.

```css
.alert {
  /* If --alert-bg is not defined, it will fall back to #FEE2E2 */
  background-color: var(--alert-bg, #FEE2E2);
  
  /* You can even nest fallbacks! */
  color: var(--alert-text, var(--primary, #000000));
}
```

### Level 3 — Intermediate (Local Scope for Theming Components)
You don't always want variables on the `:root`. Using local scope allows you to create highly reusable, themable components.

```css
/* Define the default component structure */
.badge {
  /* Use local variables that expect to be provided by modifier classes */
  background-color: var(--badge-bg, #E5E7EB); /* Default gray fallback */
  color: var(--badge-text, #374151);
  padding: 4px 8px;
  border-radius: 12px;
}

/* Create modifier classes that ONLY redefine the variables! */
.badge.success {
  --badge-bg: #D1FAE5;
  --badge-text: #065F46;
}

.badge.danger {
  --badge-bg: #FEE2E2;
  --badge-text: #991B1B;
}

/* Now your HTML is super clean: 
   <span class="badge success">Approved</span> 
*/
```

### Level 4 — Advanced (Dynamic Manipulation)
Because CSS variables live in the browser's DOM (unlike Sass variables), you can change them dynamically based on user interaction or media queries!

**1. Dark Mode using Media Queries (Zero JS!):**
```css
:root {
  --bg-color: #FFFFFF;
  --text-color: #000000;
}

/* When the user's OS is in Dark Mode, automatically redefine the variables! */
@media (prefers-color-scheme: dark) {
  :root {
    --bg-color: #121212;
    --text-color: #FFFFFF;
  }
}

body {
  background-color: var(--bg-color); /* Automatically switches! */
  color: var(--text-color);
}
```

**2. Updating variables with JavaScript:**
```javascript
// Imagine a user sliding a "Theme Color" slider in settings:
const slider = document.querySelector('#color-slider');

slider.addEventListener('input', (event) => {
  const newHue = event.target.value;
  
  // Instantly update the root CSS variable!
  // Every element using var(--theme-hue) will update instantly!
  document.documentElement.style.setProperty('--theme-hue', newHue);
});
```

## REAL_WORLD
Modern CSS frameworks rely heavily on CSS Variables.
**Tailwind CSS** uses CSS variables under the hood to handle arbitrary values and opacity modifiers. For example, `bg-blue-500/50` (blue with 50% opacity) works by using a CSS variable for the RGB color channels and inserting the opacity variable into an `rgba()` function.

## INTERVIEW
**Q: What is the difference between a Sass variable (`$color`) and a CSS variable (`--color`)?**
A: Sass variables are static and compiled at build-time. The browser never sees them; they are replaced with raw values in the final CSS file. Therefore, they cannot be changed via JavaScript, nor can they be updated dynamically inside an `@media` query. CSS variables are dynamic, live in the browser's DOM, inherit down the DOM tree, and can be updated at runtime.

**Q: Does using thousands of CSS variables hurt browser performance?**
A: Changing a CSS variable on the `:root` element can trigger a recalculation of styles for the entire page, which can be expensive if you are animating it rapidly (e.g., updating a variable 60 times a second tied to mouse movement). For static theming or infrequent changes, the performance impact is entirely negligible.
