## WHY
Content is the primary reason people visit your website. If your text is too small, the lines are too cramped, or you are using a generic, ugly default font (like Times New Roman), the user experience is ruined. **Typography** is the art of arranging text to make it legible, readable, and visually appealing. Mastering CSS typography is what separates an amateur-looking website from a premium, professional application.

## THEORY
### Serif vs Sans-Serif
Before you style text, you must choose the right family of fonts.
- **Serif Fonts** (e.g., Times New Roman, Georgia): Have small decorative strokes (feet) at the ends of the letters. They are traditionally used in printed books or long-form journalism (like The New York Times) because the "feet" help guide the eye along the line.
- **Sans-Serif Fonts** (e.g., Arial, Helvetica, Roboto): "Sans" means "without". These fonts do not have decorative feet. They are clean, modern, and drastically easier to read on low-resolution digital screens. 95% of modern web apps use Sans-Serif fonts.

## CODE

### Level 1 — Beginner (The Big Three)
The three most common properties you will use on almost every text element.

```css
p {
  /* 1. Font Size: How big is the text? */
  font-size: 16px;
  
  /* 2. Font Weight: How thick is the text? (normal, bold, or 100-900) */
  font-weight: 700; /* 700 is standard Bold */
  
  /* 3. Text Align: (left, right, center, justify) */
  text-align: center;
}
```

### Level 2 — Building Blocks (Readability & Line Height)
If your lines of text are crammed together, the user's eye will accidentally read the same line twice. You must provide vertical breathing room using `line-height`.

```css
article {
  /* Line Height acts as a multiplier. 
     1.5 means the line will be 1.5x taller than the font-size! */
  line-height: 1.5; 
  
  /* Letter Spacing pulls characters slightly apart (Tracking) */
  /* Excellent trick for making UPPERCASE headings look premium */
  letter-spacing: 2px;
  text-transform: uppercase;
}
```
*(Best Practice: Never use pixels for line-height (e.g., `line-height: 24px;`). If you later change the font-size to 30px, the lines will physically overlap and become unreadable. Using a unitless multiplier like `1.5` ensures the spacing scales dynamically!)*

### Level 3 — Intermediate (Font Families and Fallbacks)
When you tell the browser to use "Helvetica", it checks the user's physical hard drive. If they are on a Windows machine, they probably don't have Helvetica installed! The browser will fail and default to Times New Roman.

You must provide a **Font Stack** (a list of fallback fonts). The browser will try them in order until it finds one the user actually has installed.

```css
body {
  /* 1. Try Helvetica
     2. If not found, try Arial
     3. If all else fails, use whatever the operating system's default sans-serif is! */
  font-family: "Helvetica Neue", Arial, sans-serif;
}
```
*(Note: If a font name has a space in it, it MUST be wrapped in quotation marks).*

### Level 4 — Advanced (Web Fonts / Google Fonts)
Why rely on the fonts installed on the user's computer at all?
In 2010, **Web Fonts** changed everything. You can host the font file (`.woff2`) on your server (or use a CDN like Google Fonts) and force the browser to download the exact font you designed the site with!

1. **Import the font in your HTML `<head>`:**
```html
<link href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap" rel="stylesheet">
```

2. **Apply it in your CSS:**
```css
body {
  /* You now have guaranteed access to Roboto! */
  font-family: 'Roboto', sans-serif;
}
```

## REAL_WORLD
**The `font-display: swap` Trick**
Web fonts are massive files. If a user visits your site on a slow 3G connection, it might take 3 seconds to download the 'Roboto' font file. What happens during those 3 seconds? 

Historically, the browser would hide the text completely (Invisible Text). The user stared at a blank screen.
Today, Google Fonts appends `&display=swap` to the URL. This triggers the CSS `font-display: swap;` property. It tells the browser: "Immediately show the text using a fallback system font (like Arial). The millisecond the 'Roboto' file finishes downloading, instantly swap the font!" This ensures the user can start reading immediately, massively improving perceived performance.

## INTERVIEW
**Q: I want to center my text horizontally, so I used `text-align: center`. But I also want to center the text vertically inside its container. I tried `text-align-vertical: center`, but it didn't work. How do I vertically center text?**
A: `text-align` only works on the horizontal X-axis. There is no `text-align-vertical` property in CSS! To vertically center text inside a box, you historically had to use a hack: setting the `line-height` to be exactly equal to the `height` of the container (which only works for a single line of text). Today, the industry standard is to turn the container into a Flexbox (`display: flex`) and use `align-items: center; justify-content: center;` to perfectly center the text on both axes.
