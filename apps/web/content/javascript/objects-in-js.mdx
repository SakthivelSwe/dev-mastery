## WHY
If Arrays are for storing *lists* of data, Objects are for storing *descriptions* of data. 
If you want to represent a User in your application, an Array is a terrible choice (`["Alice", 25, "admin"]`). What does the number 25 represent? Age? Login count? 

An **Object** allows you to store data in explicit Key-Value pairs (`{ name: "Alice", age: 25 }`). Almost everything in JavaScript is fundamentally an Object under the hood. Furthermore, the syntax used to define JavaScript Objects was so clean and powerful that it was eventually standardized as **JSON** (JavaScript Object Notation)—which is now the undisputed universal language for transferring data across the entire internet.

## THEORY
### The Object Dictionary
In computer science, an Object is a "Hash Map" or "Dictionary". It maps a unique String key to a value in memory. 
- The **Key** (also called a Property Name) is always secretly converted into a String by the V8 Engine.
- The **Value** can be absolutely anything: a String, a Number, an Array, another Object, or even a Function (called a "Method").

### Dot Notation vs Bracket Notation
There are two ways to read data from an object:
- **Dot Notation (`user.name`)**: The industry standard. Fast, clean, and readable.
- **Bracket Notation (`user["name"]`)**: Used dynamically when the exact property name is stored inside a variable and is not known until the code actually runs.

## CODE

### Level 1 — Beginner (Creating and Accessing)
```javascript
// 1. The Object Literal Syntax
const user = {
  firstName: "Alice",
  age: 25,
  isActive: true
};

// 2. Reading Data (Dot Notation)
console.log(user.firstName); // "Alice"

// 3. Modifying Data
user.age = 26; // Objects are mutable!
user.role = "Admin"; // You can add BRAND NEW properties at any time!

console.log(user); // { firstName: "Alice", age: 26, isActive: true, role: "Admin" }

// 4. Deleting Data
// WARNING: This forces V8 to de-optimize the object in memory. Avoid if possible!
delete user.isActive; 
```

### Level 2 — Building Blocks (Methods and Bracket Notation)
```javascript
// 1. Methods (Functions attached to an Object)
const calculator = {
  brand: "Casio",
  add: function(a, b) {
    return a + b;
  },
  // ES6 Shorthand Syntax (Cleaner!):
  subtract(a, b) {
    return a - b;
  }
};

console.log(calculator.add(10, 5)); // 15

// 2. Bracket Notation (Dynamic Access)
const userProfile = { theme: "Dark", language: "en" };

// Imagine this string comes from a dropdown menu the user clicked:
const userSelectedSetting = "theme";

// DOT NOTATION FAILS: userProfile.userSelectedSetting looks for a key literally named "userSelectedSetting"!
// BRACKET NOTATION WINS: It evaluates the variable first, turning into userProfile["theme"]!
console.log(userProfile[userSelectedSetting]); // "Dark"
```

### Level 3 — Intermediate (JSON - The Language of the Web)
```javascript
// JSON (JavaScript Object Notation) is a strict, text-only format used to send data over APIs.
// DIFFERENCES FROM JS OBJECTS:
// 1. JSON Keys MUST be wrapped in double quotes!
// 2. JSON cannot contain Functions or undefined values!

const rawJSObject = { name: "Bob", age: 30, log() { console.log("Hi"); } };

// 1. Convert JS Object to a pure JSON String (Serialization)
// Perfect for sending over an HTTP Network or saving to a database!
const jsonString = JSON.stringify(rawJSObject);
console.log(jsonString); // '{"name":"Bob","age":30}' (Notice the function was stripped out!)

// 2. Convert JSON String back into a JS Object (Parsing)
// Used when receiving data from an API!
const parsedData = JSON.parse(jsonString);
console.log(parsedData.name); // "Bob"
```

### Level 4 — Advanced (Object Class Methods)
```javascript
// The global 'Object' class provides powerful utilities to manipulate other objects.

const settings = { volume: 10, graphics: "High" };

// 1. Object.keys() -> Returns an Array of all the property names
const keysArray = Object.keys(settings);
console.log(keysArray); // ["volume", "graphics"]

// 2. Object.values() -> Returns an Array of all the values
const valuesArray = Object.values(settings);
console.log(valuesArray); // [10, "High"]

// 3. Object.entries() -> Returns a 2D Array of [key, value] pairs!
// Incredible for looping over an object with for...of!
for (const [key, value] of Object.entries(settings)) {
  console.log(`${key} is set to ${value}`);
}

// 4. Object.freeze() -> Mathematically prevents mutation!
const config = { apiBase: "http://localhost:8080" };
Object.freeze(config);
// config.apiBase = "HACKED"; // Will fail silently, or throw error in Strict Mode!
```

## REAL_WORLD
When building applications, you will constantly need to "Merge" multiple objects together. 
Imagine a user has a `defaultSettings` object, but they just updated a few preferences in a `userSettings` object. You need to combine them to figure out the final application state.

In modern JavaScript (and especially Redux state management), developers use the **Object Spread Operator (`...`)** to elegantly merge objects.

```javascript
const defaultSettings = { theme: "Light", volume: 50, notifications: true };
const userChanges = { theme: "Dark", volume: 100 };

// We create a BRAND NEW object.
// We spread the defaults in first. 
// Then we spread the user changes in. The user changes OVERWRITE the defaults!
const finalSettings = { ...defaultSettings, ...userChanges };

console.log(finalSettings); 
// { theme: "Dark", volume: 100, notifications: true }
```

## INTERVIEW
**Q: Can you use a Number as an Object key?**
A: Technically yes (`const obj = { 1: "Apple" }`), but JavaScript silently coerces all Object keys into Strings behind the scenes. So your number `1` actually becomes the string `"1"`. If you want a Map that truly supports numbers, objects, or arrays as keys without coercing them to strings, you must use the ES6 `Map` data structure instead of a plain Object.

**Q: What is the difference between shallow copy and deep copy?**
A: If you copy an object using the spread operator (`const copy = { ...original }`), it performs a "Shallow Copy". If the `original` object had another object nested inside it, the copy does not clone the nested object; it simply copies the memory pointer. Mutating the nested object in the copy will accidentally mutate the original! A "Deep Copy" clones the object and every single nested child inside it. The easiest way to deep copy in JS is `JSON.parse(JSON.stringify(original))`, or using the modern `structuredClone(original)` API.

**Q: Why does `delete obj.property` negatively impact performance?**
A: V8 creates "Hidden Classes" to optimize object property access into O(1) memory offset lookups. When you use the `delete` keyword, V8 abandons the highly-optimized Hidden Class structure entirely and falls back to a slow "Dictionary Mode" Hash Map execution for that object, significantly degrading property access speeds.

## FEYNMAN CHECK

### Explain JS Objects Like I'm 10 Years Old
> A JS object is a DICTIONARY — a collection of key-value pairs. Keys are STRINGS (or Symbols); values can be anything. Three creation patterns: literal `{a:1}`, `Object.create(proto)` (with custom prototype), and constructor calls `new Thing()`. Every object has a hidden `[[Prototype]]` link (accessed via `Object.getPrototypeOf` or the deprecated `__proto__`) that points to another object — this is how method lookup works. The non-obvious power: property DESCRIPTORS. Every property has metadata: `writable`, `enumerable`, `configurable`, plus optional `get`/`set` accessors. `Object.defineProperty(obj, 'x', { value: 1, writable: false })` creates a read-only x. V8 optimises objects using HIDDEN CLASSES — adding properties in a CONSISTENT ORDER reuses the same hidden class (fast); inconsistent shape forces megamorphic dispatch (slow).

### 5 Deep Conceptual Questions

**Q1: When do you choose plain object vs Map vs Set?**
> **A:** Plain object `{}` for STRUCTURED records with KNOWN string keys (configs, DTOs, JSON). Map for arbitrary keys (incl. object keys), key insertion order preservation, frequent add/delete, and large key spaces. Set for uniqueness checks (`set.has(x)` is O(1)). Object property access is faster on tiny known-shape objects (V8 inline cache); Map is faster as the collection grows or keys are dynamic. Records & Tuples (TC39 stage 2) will provide deeply-immutable structural-equality alternatives someday.

**Q2: Mental model for property lookup?**
> **A:** When you read `obj.x`, the engine: (1) Checks obj's own properties (hash + hidden class). (2) If absent, walks the [[Prototype]] chain — checking each ancestor's own properties. (3) Returns undefined if not found anywhere. Writing `obj.x = 5` ALWAYS creates an own property on obj (it doesn't walk the chain to find where to write — except for accessors). This is why `Array.prototype.push = ...` adds push to ALL arrays.

**Q3: Most dangerous misconception?**
> **A:** Objects compared by `===` check value equality:
> ```js
> // ❌ Object equality is by REFERENCE, not contents
> { x: 1 } === { x: 1 }    // false — different objects
> [1, 2] === [1, 2]        // false
>
> // ✅ Deep equality requires explicit comparison
> JSON.stringify(a) === JSON.stringify(b)  // works for plain JSON, breaks on Date/Map/Set/cycles
> // Or use lodash isEqual, or write your own:
> function deepEq(a, b) {
>   if (a === b) return true;
>   if (typeof a !== 'object' || a === null) return false;
>   const keys = Object.keys(a);
>   if (keys.length !== Object.keys(b).length) return false;
>   return keys.every(k => deepEq(a[k], b[k]));
> }
> ```

**Q4: How do V8 hidden classes work?**
> **A:** When you create `{a:1}`, V8 assigns it Hidden Class C0. When you add b: `obj.b=2`, V8 creates Hidden Class C1 = C0 + b. Every object created the SAME WAY (same property order) shares the same hidden class. Property access compiles to "load from offset N" — blazing fast. Adding properties in DIFFERENT order (e.g. some objects get a-then-b, others b-then-a) creates DIFFERENT hidden classes — the inline cache becomes polymorphic, then megamorphic, 10-100× slower. Lesson: initialise all properties in the constructor in the same order.

**Q5: FAANG-grade definition?**
> **A:** "JavaScript objects are mutable dictionaries of string-keyed (and symbol-keyed) properties, each with descriptor metadata (value/getter/setter, writable, enumerable, configurable), linked via the [[Prototype]] chain to a single prototype object for method inheritance — created via literals, Object.create, constructor invocations, or class instantiation — and optimised by V8's hidden classes when property shape is initialised consistently across instances."

#### Step 2 — Reactive Object
```js
// reactive.js
export function reactive(initial) {
  const data = { ...initial };
  const subscribers = new Map();   // key -> Set<callback>
  const target = {};

  for (const key of Object.keys(initial)) {
    Object.defineProperty(target, key, {
      enumerable: true,
      configurable: false,
      get() { return data[key]; },
      set(newValue) {
        if (data[key] === newValue) return;        // no-op on same value
        const oldValue = data[key];
        data[key] = newValue;
        for (const cb of subscribers.get(key) ?? []) cb(newValue, oldValue);
      },
    });
  }

  target.$on = (key, cb) => {
    if (!subscribers.has(key)) subscribers.set(key, new Set());
    subscribers.get(key).add(cb);
    return () => subscribers.get(key).delete(cb);
  };

  return target;
}
```

#### Step 3 — Use The Reactive Object
```js
// demo.js
import { reactive } from './reactive.js';

const state = reactive({ count: 0, name: 'Ana' });

state.$on('count', (n, old) => console.log(`count: ${old} → ${n}`));
state.$on('name',  (n, old) => console.log(`name: ${old} → ${n}`));

state.count = 1;       // logs: count: 0 → 1
state.count = 1;       // no log (no change)
state.count++;         // logs: count: 1 → 2
state.name = 'Bob';    // logs: name: Ana → Bob

// Property descriptor introspection
console.log(Object.getOwnPropertyDescriptor(state, 'count'));
// { get, set, enumerable: true, configurable: false }

// Cannot delete
try { delete state.count; } catch (e) { console.log('delete failed:', e.message); }
```

// reactive.test.js
import { describe, it, expect, vi } from 'vitest';
import { reactive } from './reactive.js';

describe('reactive', () => {
  it('fires subscriber on change', () => {
    const s = reactive({ x: 0 });
    const spy = vi.fn();
    s.$on('x', spy);
    s.x = 5;
    expect(spy).toHaveBeenCalledWith(5, 0);
  });
  it('does not fire on same value', () => {

    const spy = vi.fn();
    s.$on('x', spy);
    s.x = 1;
    expect(spy).not.toHaveBeenCalled();

  it('disposer removes subscriber', () => {
    const s = reactive({ x: 0 });
    const spy = vi.fn();
    const off = s.$on('x', spy);
    off();
    s.x = 1;
    expect(spy).not.toHaveBeenCalled();
  });
  it('properties are not configurable', () => {
    const s = reactive({ x: 0 });
    expect(() => Object.defineProperty(s, 'x', { value: 99 })).toThrow();
  });
});
