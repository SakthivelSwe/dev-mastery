## WHY
You need to extract all email addresses from a 10,000-word block of text. Or, you need to validate that a user's password contains at least one uppercase letter, one number, and one special character.
Writing standard JavaScript `if` statements and `.split()` functions to do this would require hundreds of lines of complex, unmaintainable code.
**Regular Expressions (RegEx)** provide a hyper-condensed syntax for pattern matching. With a single line of RegEx, you can validate, extract, or replace incredibly complex text patterns instantly. It is considered a dark art, but mastering it gives you a massive advantage.

## THEORY
### The RegEx Engine
A Regular Expression is not just a string; it is a mini-program interpreted by the browser's underlying RegEx Engine (written in C++).
When you execute a RegEx, the engine scans the target text character by character from left to right, attempting to match the exact mathematical pattern you described.
In JavaScript, RegEx literals are wrapped in forward slashes: `/pattern/flags`.

## CODE

### Level 1 — Basic Matching & Metacharacters
Let's look at the basic building blocks.
- `\d` matches any digit (0-9).
- `\w` matches any word character (A-Z, a-z, 0-9, and underscore).
- `\s` matches any whitespace (space, tab, newline).
- `.` matches absolutely ANY character (except newline).

```javascript
// Let's validate a US Phone Number: exactly 3 digits, a dash, 4 digits.
// The pattern: \d\d\d-\d\d\d\d
const phoneRegex = /\d\d\d-\d\d\d\d/;

console.log(phoneRegex.test("My number is 555-1234")); // true!
console.log(phoneRegex.test("Call 55-1234 please")); // false!
```

### Level 2 — Quantifiers and Anchors
Writing `\d\d\d` is annoying. We use Quantifiers to specify *how many* times a pattern should occur.
- `+` means 1 or more times.
- `*` means 0 or more times.
- `{n,m}` means between n and m times.
- `^` anchors the match to the start of the string.
- `$` anchors the match to the end of the string.

```javascript
// Require exactly 3 digits, dash, exactly 4 digits.
// We use ^ and $ to ensure the ENTIRE string is the phone number, 
// not just a number buried inside a paragraph!
const strictPhoneRegex = /^\d{3}-\d{4}$/;

console.log(strictPhoneRegex.test("555-1234")); // true
console.log(strictPhoneRegex.test("My number is 555-1234")); // false (Because of ^ and $)

// Validating a username (letters and numbers only, 3 to 15 chars long)
const usernameRegex = /^\w{3,15}$/;
```

### Level 3 — Character Classes & Flags
What if we only want to accept lowercase vowels? We define custom Character Classes using `[]`.
What if we want to find ALL matches in a string, not just the first one? We use the `g` (global) and `i` (case-insensitive) flags at the end of the regex!

```javascript
// Find all vowels, ignoring case!
const vowelsRegex = /[aeiou]/gi;

const text = "JavaScript is Awesome!";

// .match() returns an array of all matches!
const result = text.match(vowelsRegex); 
console.log(result); // ['a', 'a', 'i', 'i', 'A', 'e', 'o', 'e']

// Find everything that is NOT a vowel! (The ^ inside brackets means NOT)
const consonantsRegex = /[^aeiou]/gi;
```

### Level 4 — Capture Groups and Replace
Capture Groups `()` are the most powerful feature of RegEx. They allow you to extract specific subsections of a match, or rearrange a string instantly.

```javascript
// Let's extract the Year, Month, and Day from a string!
// We wrap the parts we want to extract in parentheses ()
const dateRegex = /(\d{4})-(\d{2})-(\d{2})/;
const text = "The event is on 2025-12-31";

const match = text.match(dateRegex);
console.log(match[0]); // The full match: "2025-12-31"
console.log(match[1]); // Capture Group 1: "2025"
console.log(match[2]); // Capture Group 2: "12"
console.log(match[3]); // Capture Group 3: "31"

// THE MAGIC: Reformatting strings instantly!
// We use $1, $2, $3 to refer to the captured groups!
const reformatted = text.replace(dateRegex, "$2/$3/$1");
console.log(reformatted); // "The event is on 12/31/2025"
```

## REAL_WORLD
**Lookaheads (Password Validation)**
Password validation requires multiple overlapping rules: "At least 8 chars, 1 uppercase, 1 lowercase, 1 number."
If you use standard RegEx, it will scan from left to right and fail. You need a **Positive Lookahead** `(?=...)`, which tells the engine: *"Look ahead in the string to ensure this pattern exists, but don't actually consume the characters, just check!"*

```javascript
// The ultimate Password Regex
// (?=.*[a-z]) -> Asserts at least one lowercase exists anywhere ahead
// (?=.*[A-Z]) -> Asserts at least one uppercase exists anywhere ahead
// (?=.*\d)    -> Asserts at least one digit exists anywhere ahead
// .{8,}       -> Actually matches the password, ensuring it is 8+ chars
const passwordRegex = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$/;

console.log(passwordRegex.test("weakpass")); // false
console.log(passwordRegex.test("StrongPass1")); // true
```

## INTERVIEW
**Q: You write a regex to extract HTML tags: `const tagRegex = /<.*>/g;`. You run it on the string `<div>Content</div>`. Instead of returning `['<div>', '</div>']`, it returns one massive string: `['<div>Content</div>']`. Why did it match the entire string, and how do you fix it?**
A: **Greedy vs. Lazy Quantifiers.**
By default, the `*` and `+` quantifiers in RegEx are **Greedy**. They try to match as much text as absolutely possible before stopping.
In the pattern `/<.*>/`, the engine sees the first `<`. Then, the `.*` command says "match any character forever". It eats the entire string `div>Content</div>` all the way to the end. Then it realizes it still needs to match the final `>`. So it backtracks one character, finds the final `>`, and declares the entire massive string a match!
*The Fix:* You must make the quantifier **Lazy** by appending a question mark: `.*?`. 
The pattern `/<.*?>/g` tells the engine: *"Match any character, but STOP the exact moment you see the very first closing `>`."* This correctly extracts `['<div>', '</div>']`.
