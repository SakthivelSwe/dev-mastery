## WHY
Callbacks solved the problem of asynchronous programming, but they introduced massive architectural flaws: the "Pyramid of Doom" (nested unreadable code) and "Inversion of Control" (handing your callbacks over to third-party code and blindly trusting them).

To fix this, ES6 introduced the **Promise**. A Promise is a JavaScript Object that represents the *eventual* completion (or failure) of an asynchronous operation. Instead of passing a callback *into* a function, the function instantly returns a Promise object *back* to you. You maintain total control. 
In ES8, the JavaScript committee introduced **Async/Await**, which is purely syntactic sugar built on top of Promises. Async/Await allows you to write asynchronous code that visually looks exactly like synchronous code, completely eliminating callback hell and making complex data fetching incredibly easy to read.

## THEORY
### The 3 States of a Promise
When you are handed a Promise object, it exists in exactly one of three states:
1. **Pending**: The initial state. The background task (like a network request) is currently running.
2. **Fulfilled (Resolved)**: The background task succeeded! The Promise now permanently holds the requested data.
3. **Rejected**: The background task failed (e.g., Network Error). The Promise permanently holds the Error object.
*Note: Once a Promise is Fulfilled or Rejected, it is "Settled". It can never change its state again.*

### The Microtask Queue
When a Promise resolves, its `.then()` callbacks are NOT sent to the standard Task Queue (where `setTimeout` goes). They are sent to the **VIP Microtask Queue**. The Event Loop will forcefully pause all other operations (even UI rendering) to process resolved Promises first!

## CODE

### Level 1 — Beginner (Consuming a Promise with .then)
```javascript
// The Fetch API automatically returns a Promise!
const pizzaPromise = fetch('https://api.pizza.com/order');

console.log(pizzaPromise); // Output: Promise { <pending> }

// 1. We attach a .then() callback. 
// It ONLY executes if the Promise transitions to FULFILLED!
pizzaPromise.then((response) => {
  console.log("SUCCESS! Pizza has arrived:", response);
});

// 2. We attach a .catch() callback.
// It ONLY executes if the Promise transitions to REJECTED!
pizzaPromise.catch((error) => {
  console.error("FAILURE! The oven broke:", error);
});

// Note: The rest of the code keeps executing instantly! JS does not wait!
console.log("Waiting for the pizza..."); 
```

### Level 2 — Building Blocks (Chaining Promises)
```javascript
// The magic of Promises is that .then() ALWAYS returns a BRAND NEW Promise!
// This allows you to "Chain" them together, creating a flat, readable sequence 
// instead of a nested Callback Pyramid of Doom!

fetch('https://api.game.com/user/1')
  .then(response => {
    // 1. Get the raw network response, extract the JSON.
    // json() ALSO returns a Promise!
    return response.json(); 
  })
  .then(userData => {
    // 2. We got the JSON! Now fetch their inventory based on their ID.
    return fetch(`https://api.game.com/inventory/${userData.id}`);
  })
  .then(inventoryResponse => {
    // 3. Extract the inventory JSON.
    return inventoryResponse.json();
  })
  .then(inventoryData => {
    // 4. Finally, we have the final data!
    console.log("User's sword:", inventoryData.sword);
  })
  .catch(error => {
    // ONE CATCH TO RULE THEM ALL!
    // If ANY of the 4 steps above fail, the chain instantly short-circuits 
    // and teleports straight down to this single catch block!
    console.error("Something went wrong in the chain!", error);
  });
```

### Level 3 — Intermediate (Async / Await)
```javascript
// The chaining above is better than callbacks, but it's still messy.
// Welcome to Async/Await (ES8). The ultimate solution.

// 1. Put the 'async' keyword in front of the function.
// This magically wraps the function's return value in a Promise!
async function getSword() {
  
  try {
    // 2. Put the 'await' keyword in front of ANY Promise.
    // This literally PAUSES the function's execution until the Promise settles!
    // The main thread is NOT blocked (other code still runs), but THIS function waits.
    
    const response = await fetch('https://api.game.com/user/1');
    const userData = await response.json();
    
    const invResponse = await fetch(`https://api.game.com/inventory/${userData.id}`);
    const invData = await invResponse.json();
    
    console.log("User's sword:", invData.sword);
    
  } catch (error) {
    // We can use standard, synchronous try/catch blocks again!
    console.error("Network failed!", error);
  }
}

getSword();
```

### Level 4 — Advanced (Promise.all)
```javascript
// What if you need to fetch Data A, Data B, and Data C?
// If they don't depend on each other, awaiting them sequentially is a massive performance failure!
// It takes 2s + 2s + 2s = 6 seconds total!

async function fetchBad() {
  const users = await fetch('/api/users'); // Waits 2s
  const posts = await fetch('/api/posts'); // Waits 2s
  const comments = await fetch('/api/comments'); // Waits 2s
}

// THE FIX: Promise.all()
// This kicks off all 3 requests SIMULTANEOUSLY in parallel!
// It returns a single master Promise that resolves when ALL 3 are finished.
// Total time: 2 seconds!

async function fetchGood() {
  try {
    const [usersRes, postsRes, commentsRes] = await Promise.all([
      fetch('/api/users'),
      fetch('/api/posts'),
      fetch('/api/comments')
    ]);
    
    console.log("All three APIs finished at the exact same time!");
  } catch (error) {
    // WARNING: If even ONE of the 3 promises rejects, the entire Promise.all() 
    // instantly rejects and crashes! (Use Promise.allSettled() if you want to ignore failures).
    console.error("One of the APIs failed!");
  }
}
```

## REAL_WORLD
**Async/Await** is the absolute bedrock of modern web development. You will write `async` functions every single day of your career.

In a **React** component, if you want to fetch data when the screen loads, you place the async logic inside a `useEffect` hook. However, React strictly forbids passing an `async` function directly into `useEffect`. You must define the async function *inside* the effect, and then immediately invoke it:

```jsx
import { useState, useEffect } from 'react';

function UserProfile() {
  const [data, setData] = useState(null);

  useEffect(() => {
    // 1. Define the Async function
    const loadProfile = async () => {
      try {
        const response = await fetch('/api/profile');
        const json = await response.json();
        setData(json); // Update UI
      } catch (e) {
        console.error("Failed to load");
      }
    };
    
    // 2. Instantly call it!
    loadProfile();
  }, []);

  if (!data) return <p>Loading Spinner...</p>;
  return <h1>{data.name}</h1>;
}
```

## INTERVIEW
**Q: Can you use the `await` keyword in the global scope?**
A: Historically, no. You could only use `await` inside a function explicitly marked with `async`. However, Node.js v14 and modern browsers introduced "Top-Level Await" (ES2022). This allows you to write `const data = await fetch(...)` directly at the root level of a JS Module file. The engine simply treats the entire file as an invisible async function!

**Q: What is the exact difference between `Promise.all()` and `Promise.allSettled()`?**
A: `Promise.all()` uses "Fail-Fast" logic. If you give it an array of 10 Promises, and the 9th one Rejects, the entire master Promise instantly Rejects and throws an error, discarding the 8 successful results. `Promise.allSettled()` waits for all 10 Promises to finish completely, regardless of whether they succeed or fail. It returns an array of objects detailing the exact status (`fulfilled` or `rejected`) and value of each individual promise, ensuring you never lose successful data.

**Q: Does returning a value from an `async` function automatically create a Promise?**
A: Yes! The `async` keyword literally forces the function to return a Promise. If you write `async function getNumber() { return 5; }`, the function does not return the primitive number `5`. It returns a Fulfilled Promise object that *contains* the number `5`. If you call the function, you must do `getNumber().then(num => ...)` or `const num = await getNumber()`.
