## WHY
Programming languages are simply mathematical calculators capable of making logical decisions. Operators are the specific symbols (`+`, `>`, `&&`) that allow you to combine variables, perform math, and evaluate complex truths. 

While JavaScript shares most of its operators with C++ and Java, it introduces several unique and incredibly powerful operators (like Nullish Coalescing and Optional Chaining) that drastically reduce the amount of code required to safely read deeply nested data from external APIs.

## THEORY
### Categories of Operators
1. **Arithmetic**: Basic math operations (`+`, `-`, `*`, `/`, `%`, `**`).
2. **Assignment**: Storing values in variables (`=`, `+=`, `-=`).
3. **Comparison**: Asking questions that result in Booleans (`===`, `!==`, `>`, `<`).
4. **Logical**: Combining multiple questions together (`&&`, `||`, `!`).
5. **Modern JS Operators**: Handling `null` safely (`??`, `?.`).

### Short-Circuit Evaluation
Logical operators evaluate from left to right. JavaScript is optimized for speed; if it can figure out the final answer by only looking at the left side of the equation, it will completely "short-circuit" and instantly stop executing the right side. This is heavily exploited by React developers to conditionally render UI components!

## CODE

### Level 1 — Beginner (Math & Assignment)
```javascript
// Arithmetic
let score = 10 + 5; // 15
score = score - 2;  // 13
score = score * 2;  // 26
score = score / 2;  // 13

// Modulo (%) - Returns the Remainder of division!
console.log(10 % 3); // 1 (Because 3 goes into 10 three times, leaving 1 leftover)
// Pro-tip: Modulo is used to check if a number is Even or Odd!
// if (number % 2 === 0) { console.log("Even!") }

// Exponentiation (**)
console.log(2 ** 3); // 8 (2 to the power of 3)

// Assignment Shorthand
let x = 10;
x += 5; // Exact same as: x = x + 5
x++;    // Exact same as: x = x + 1
```

### Level 2 — Building Blocks (Comparison & Logical)
```javascript
// Comparison (Always use Strict Equality ===)
console.log(5 > 3);   // true
console.log(10 >= 10); // true
console.log(5 !== 4);  // true (Strict Not Equal)

// Logical AND (&&) - ALL sides must be true
console.log(true && true);  // true
console.log(true && false); // false

// Logical OR (||) - AT LEAST ONE side must be true
console.log(false || true); // true

// Logical NOT (!) - Flips the boolean
console.log(!true); // false
```

### Level 3 — Intermediate (Short-Circuiting)
```javascript
// The && and || operators don't actually return true/false!
// They return the ACTUAL VALUE that stopped the evaluation.

// 1. The OR (||) Short-Circuit (Often used for Default Values)
// JS looks at the left. Is "Alice" truthy? Yes! 
// It instantly STOPS and returns "Alice". It never even looks at "Guest".
const username = "Alice" || "Guest"; 
console.log(username); // "Alice"

// What if the left is Falsy?
const noName = "" || "Guest"; 
console.log(noName); // "Guest" (Because "" is falsy, it moves to the right)

// 2. The AND (&&) Short-Circuit (Often used to execute code safely)
let isLoaded = true;
// JS looks at the left. Is it truthy? Yes. 
// It MUST check the right side to be sure, so the console.log executes!
isLoaded && console.log("Data is ready!"); 
```

### Level 4 — Advanced (Optional Chaining & Nullish Coalescing)
```javascript
// When fetching API data, objects are often deeply nested.
const apiResponse = {
  user: {
    profile: {
      bio: "Software Engineer"
    }
  }
};

// THE OLD WAY: If 'profile' was missing, this would crash the entire app!
// "TypeError: Cannot read properties of undefined"
// const bio = apiResponse.user.profile.bio;

// 1. OPTIONAL CHAINING (?.) (ES2020)
// If any property before the `?.` is null/undefined, it instantly short-circuits
// and safely returns undefined instead of crashing!
const bio = apiResponse.user?.profile?.bio;

// 2. NULLISH COALESCING (??) (ES2020)
// The OR operator (||) has a massive flaw: it treats 0 and "" as falsy!
const score = 0 || 100; // Result: 100 (Wait, 0 is a valid score!)

// The ?? operator ONLY triggers if the left side is EXACTLY null or undefined!
const validScore = 0 ?? 100; // Result: 0 (Perfect!)
```

## REAL_WORLD
In **React**, short-circuiting with the `&&` operator is the primary way to conditionally render HTML elements on the screen.

```jsx
// A React Component
function Dashboard({ user }) {
  return (
    <div>
      <h1>Welcome</h1>
      
      {/* If user.isAdmin is false, React renders nothing.
          If user.isAdmin is true, React renders the Admin Panel! */}
      {user.isAdmin && <AdminPanel />}
      
      {/* Safe reading of nested data */}
      <p>Your subscription: {user?.billing?.plan ?? 'Free Tier'}</p>
    </div>
  );
}
```
This pattern is so ubiquitous in the frontend ecosystem that mastering `&&`, `||`, `?.`, and `??` is arguably more important than mastering standard `if` statements!

## INTERVIEW
**Q: What is the difference between `||` and `??`?**
A: `||` (Logical OR) checks if the left side is "Falsy" (which includes `0`, `""`, and `false`). `??` (Nullish Coalescing) strictly checks if the left side is exactly `null` or `undefined`. You should always use `??` when assigning default values for configurations or numbers, because `0` and `""` are often perfectly valid inputs that `||` would accidentally overwrite.

**Q: In the statement `const a = false && someFunction();`, does `someFunction()` execute?**
A: No. Because the `&&` operator requires BOTH sides to be true, the JavaScript engine sees that the left side is `false` and realizes the final equation can mathematically never be true. It immediately short-circuits and ignores the right side entirely.

**Q: Explain the Ternary Operator.**
A: The ternary operator (`condition ? trueResult : falseResult`) is the only operator in JavaScript that takes three operands. It is essentially an inline `if/else` statement that returns a value. It is heavily used in variable assignment: `const status = age >= 18 ? "Adult" : "Minor";`.
