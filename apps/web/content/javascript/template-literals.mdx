## WHY
Before 2015, working with text (Strings) in JavaScript was miserable. If you wanted to combine a variable with text, you had to use the clumsy `+` operator. If you wanted a string to span across multiple lines, you had to manually type `\n` carriage returns and stitch lines together. This resulted in incredibly messy, unreadable code, especially when trying to generate HTML dynamically.

ES6 introduced **Template Literals**, denoted by the backtick character (\`). This single feature completely revolutionized string manipulation. It brought multi-line strings, variable interpolation, and embedded expressions directly into the core language, allowing developers to construct massive, complex text blocks cleanly and elegantly.

## THEORY
### String Interpolation
"Interpolation" is the process of evaluating a variable or mathematical expression and injecting its result directly into a string. 
- Traditional Strings use Single Quotes (`' '`) or Double Quotes (`" "`). They do **not** support interpolation or multi-line formatting.
- Template Literals use Backticks (`` ` ` ``). They support everything. To inject JavaScript logic directly into a Template Literal, you wrap the logic inside a dollar sign and curly braces: `${ expression }`.

## CODE

### Level 1 — Beginner (The Nightmare of Concatenation)
```javascript
const user = "Alice";
const messages = 5;

// 1. THE OLD WAY (String Concatenation)
// Extremely error-prone. Missing a space is the most common bug!
const oldGreeting = "Hello " + user + ", you have " + messages + " unread messages.";
console.log(oldGreeting);

// 2. THE MODERN WAY (Template Literals)
// We use Backticks ` ` instead of Quotes " "
// We inject variables using ${ }
const modernGreeting = `Hello ${user}, you have ${messages} unread messages.`;
console.log(modernGreeting);
```

### Level 2 — Building Blocks (Multi-line Strings)
```javascript
// 1. THE OLD WAY
// You had to use \n to force a line break, and \ to escape the newline in the code editor.
const oldEmail = "Dear User,\n\n" +
  "Thank you for signing up for our service.\n" +
  "Please click the link below to verify your email.\n\n" +
  "Regards,\nThe Team";

// 2. THE MODERN WAY
// Template Literals respect literal line breaks! 
// You just press Enter on your keyboard.
const modernEmail = `Dear User,

Thank you for signing up for our service.
Please click the link below to verify your email.

Regards,
The Team`;

console.log(modernEmail);
```

### Level 3 — Intermediate (Embedded Expressions)
```javascript
// The ${ } syntax doesn't just accept variables. 
// It accepts ANY valid JavaScript Expression!

const price = 19.99;
const taxRate = 0.08;

// We can do raw Math inside the string!
const checkoutText = `Subtotal: $${price}
Tax: $${(price * taxRate).toFixed(2)}
Total: $${(price + (price * taxRate)).toFixed(2)}`;

console.log(checkoutText);

// We can use Ternary Operators inside the string!
const isPremium = true;
const welcomeMsg = `Welcome to your dashboard. ${isPremium ? "Enjoy your PRO features!" : "Upgrade to PRO today."}`;

console.log(welcomeMsg); // "Welcome to your dashboard. Enjoy your PRO features!"
```

### Level 4 — Advanced (Tagged Template Literals)
```javascript
// A highly advanced, rarely seen feature that powers massive libraries (like styled-components).
// You can actually attach a FUNCTION directly to a Template Literal!

function sanitizeHTML(strings, ...values) {
  // 'strings' is an array of the raw text: ["<h1>", "</h1>"]
  // 'values' is an array of the variables: ["<script>alert('HACKED')</script>"]
  
  let finalString = "";
  
  strings.forEach((stringChunk, i) => {
    let value = values[i] || "";
    
    // We intercept the variable and strip out dangerous HTML tags!
    if (typeof value === "string") {
      value = value.replace(/</g, "&lt;").replace(/>/g, "&gt;");
    }
    
    finalString += stringChunk + value;
  });
  
  return finalString;
}

const userInput = "<script>alert('HACKED')</script>";

// Notice there are NO parentheses () calling this function! 
// We just attach it directly to the Backticks!
const safeHTML = sanitizeHTML`<h1>${userInput}</h1>`;

console.log(safeHTML); 
// Output: <h1>&lt;script&gt;alert('HACKED')&lt;/script&gt;</h1>
// The dangerous script was successfully neutralized!
```

## REAL_WORLD
Template Literals are the backbone of modern Frontend Frameworks, but they are incredibly vital for Vanilla JavaScript DOM manipulation.

If you need to dynamically render an article onto a page, doing so with `document.createElement()` takes dozens of lines of code. With Template Literals, you can generate raw HTML strings with data injected instantly:

```javascript
const post = { title: "JS is Awesome", author: "Bob", likes: 42 };

// Generating HTML dynamically using Template Literals
const htmlCard = `
  <div class="card">
    <h2>${post.title}</h2>
    <p>Written by: <strong>${post.author}</strong></p>
    <span class="badge">Likes: ${post.likes}</span>
  </div>
`;

// Inject the entire HTML block into the browser DOM in one line
document.getElementById('feed').innerHTML = htmlCard;
```
*(Warning: Injecting raw HTML strings via `innerHTML` can expose you to XSS attacks if the data is from untrusted users. Modern frameworks like React use JSX to automatically protect against this!)*

## INTERVIEW
**Q: Can you put an `if/else` statement inside a Template Literal `${ }`?**
A: No! The `${ }` syntax strictly accepts JavaScript **Expressions**, not Statements. An Expression is a piece of code that evaluates to a single value (like Math `5+5`, a variable `name`, or a Ternary Operator `x ? y : z`). An `if` statement does not evaluate to a value, so it will trigger a Syntax Error. You must use the Ternary Operator inside `${ }`.

**Q: What is a Tagged Template Literal?**
A: It is an advanced ES6 feature where you place a function name immediately before a template literal (e.g., `myFunction\`Hello \${name}\``). Instead of returning a standard string, JavaScript parses the string into an array of text chunks and an array of evaluated variables, and passes them as arguments into the function. This allows the function to intercept, manipulate, or sanitize the string before it is finalized. It is famously used by the `styled-components` library in React.

**Q: Do Template Literals support Escape Characters?**
A: Yes, you can still use standard escape characters like `\n` or `\t` inside a Template Literal, but you rarely need to because Backticks naturally respect literal keyboard line breaks and whitespace formatting. If you actually want to print a literal Backtick character or Dollar sign inside the string, you must escape it with a backslash: `\`` or `\$`.

## FEYNMAN CHECK

### Explain Template Literals Like I'm 10 Years Old
> Template literals use BACKTICKS instead of quotes, and let you EMBED EXPRESSIONS inside `${...}`. They support multi-line strings without \n, and you can use ANY JS expression inside the interpolation — function calls, ternaries, math. The advanced feature is TAGGED TEMPLATES: `tag\`hello ${name}\`` calls `tag(strings, ...values)` where strings is an array of the static parts and values are the interpolated values. This is how styled-components writes CSS-in-JS, how SQL libraries safely parameterise queries, and how html template libraries (lit-html) work. The non-obvious detail: the strings array has a special `.raw` property with un-escaped versions (so `\n` stays as two chars), used by String.raw and useful for things like regex builders.

### 5 Deep Conceptual Questions

**Q1: When are template literals worth it vs '+' concatenation?**
> **A:** ALMOST ALWAYS for strings with variables. `\`Hello, ${name}! You have ${count} messages.\`` is far clearer than `'Hello, ' + name + '! You have ' + count + ' messages.'`. They also handle multi-line naturally (no need for '\\n' joins). The only case for concatenation: when one operand is undefined/null and you specifically want `'Hello, ' + null` → 'Hello, null' (template gives same result via String conversion — no real difference).

**Q2: Mental model for tagged templates?**
> **A:** A tag function receives the literal parts SEPARATED from the dynamic values. `tag\`a${b}c${d}e\`` becomes `tag(['a','c','e'], b, d)`. The tag can then process the array+values however it wants — escape HTML, inject SQL params, build a styled className. This is the foundation of safe DSLs inside JS: the static text is trusted (developer-written), the values are escaped automatically before being inserted.

**Q3: Most dangerous misconception?**
> **A:** Template literals automatically escape HTML/SQL:
> ```js
> // ❌ NO escaping — just string concatenation
> const name = '<script>alert(1)</script>';
> document.body.innerHTML = \`Welcome, ${name}!\`;   // XSS!
>
> // ❌ Same with SQL
> const id = "1' OR '1'='1";
> db.query(\`SELECT * FROM users WHERE id = '${id}'\`);   // SQL injection
>
> // ✅ Use a tagged template that escapes
> const safeHtml = (s, ...v) => s.reduce((acc, str, i) =>
>   acc + str + (i < v.length ? escapeHtml(v[i]) : ''), '');
> document.body.innerHTML = safeHtml\`Welcome, ${name}!\`;
> ```

**Q4: How does String.raw work?**
> **A:** String.raw is a built-in tag that returns the raw (un-escaped) version of the template. `String.raw\`Line 1\\nLine 2\`` returns `'Line 1\\nLine 2'` (literal backslash-n, two chars) instead of the actual newline. This is essential for: writing regex patterns (`String.raw\`\\d+\`` → `'\\d+'`), Windows paths (`String.raw\`C:\\Users\\Ana\``), and anywhere escape sequences must be preserved.

**Q5: FAANG-grade definition?**
> **A:** "Template literals are backtick-delimited string expressions supporting multi-line content and \${expression} interpolation — with tagged templates (\`tag\\\`...\\\`\`) invoking a function with the static-string array (plus a .raw property) and interpolated values as separate arguments, providing the foundation for DSL libraries (styled-components, lit-html, gql, parameterised SQL) to safely compose dynamic strings while escaping or transforming interpolated values."

#### Step 2 — Tagged Template Library
```js
// tags.js

// 1. html — escapes interpolated values
const HTML_ESCAPE = { '&': '&amp;', '<': '&lt;', '>': '&gt;', '"': '&quot;', "'": '&#39;' };
function escapeHtml(s) { return String(s).replace(/[&<>"']/g, c => HTML_ESCAPE[c]); }

export function html(strings, ...values) {
  return strings.reduce((acc, str, i) => {
    const value = i < values.length
      ? (Array.isArray(values[i]) ? values[i].join('') : escapeHtml(values[i]))
      : '';
    return acc + str + value;
  }, '');
}

// 2. sql — parameterises (compatible with pg / mysql2)
export function sql(strings, ...values) {
  const text = strings.reduce((acc, str, i) =>
    acc + str + (i < values.length ? `$${i + 1}` : ''), '');
  return { text, values };
}

// 3. css — vendor prefixing + minification
const VENDOR_PROPS = ['transform', 'transition', 'user-select', 'appearance'];
export function css(strings, ...values) {
  const raw = strings.reduce((acc, str, i) => acc + str + (i < values.length ? values[i] : ''), '');
  return raw.replace(/^(\s*)([a-z-]+)(\s*:\s*[^;]+;)/gm, (_, indent, prop, rest) => {
    if (VENDOR_PROPS.includes(prop)) {
      return `${indent}-webkit-${prop}${rest}\n${indent}-moz-${prop}${rest}\n${indent}${prop}${rest}`;
    }
    return _;
  });
}
```

#### Step 3 — Use Them
```js
// demo.js
import { html, sql, css } from './tags.js';

const name = '<script>alert(1)</script>';
const userIds = [1, 2, 3];

// HTML — escapes the script tag automatically
const greeting = html`<p>Hello, ${name}!</p>`;
console.log(greeting);
// <p>Hello, &lt;script&gt;alert(1)&lt;/script&gt;!</p>

// SQL — produces parameterised query, no injection possible
const query = sql`SELECT * FROM users WHERE id IN (${userIds[0]}, ${userIds[1]}, ${userIds[2]}) AND name = ${name}`;
console.log(query);
// { text: 'SELECT * FROM users WHERE id IN ($1, $2, $3) AND name = $4',
//   values: [1, 2, 3, '<script>alert(1)</script>'] }

// CSS — auto-prefixes
const styles = css`
  .btn {
    transform: rotate(45deg);
    transition: all 0.3s ease;
    color: blue;
  }
`;
console.log(styles);

// Array interpolation
const items = ['apple', 'banana', 'cherry'];
const list = html`<ul>${items.map(i => html`<li>${i}</li>`)}</ul>`;
console.log(list);
```

#### Step 4 — Error Handling: String.raw For Regex/Paths
```js
// rawDemo.js
// Without String.raw — backslash escapes are processed
const regex1 = /\d+/;   // works
const str1 = '\d+';     // 'd+' — backslash escapes the d!

// With String.raw — backslashes preserved as literals
const str2 = String.raw`\d+`;   // '\d+' — two chars
const regex2 = new RegExp(String.raw`\d{3}-\d{4}`);
console.log(regex2.test('555-1234'));   // true

// Windows paths
const path = String.raw`C:\Users\Ana\Documents`;
console.log(path);   // C:\Users\Ana\Documents (literal backslashes)
```

import { describe, it, expect } from 'vitest';
import { html, sql, css } from './tags.js';

describe('html tag', () => {
  it('escapes HTML entities in interpolated values', () => {
    const r = html`<p>${'<script>'}</p>`;

describe('sql tag', () => {
  it('produces parameterised query', () => {
    const q = sql`SELECT * WHERE id = ${42} AND name = ${'Ana'}`;
