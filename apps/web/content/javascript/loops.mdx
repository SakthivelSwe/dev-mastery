## WHY
Computers are designed to do repetitive, boring tasks at lightning speed. If you have an array of 10,000 usernames and you need to capitalize the first letter of each one, writing 10,000 lines of manual code is impossible. 

**Loops** allow you to write a block of code exactly once, and instruct the JavaScript engine to execute that exact block over and over again until a specific condition is met. Modern JavaScript provides several different loops tailored for specific data structures: standard `for` loops for basic numbers, `for...of` loops for Arrays, and `for...in` loops for Objects.

## THEORY
### The 4 primary Loops in JavaScript
1. **The classic `for` loop**: The most fundamental loop. You manually control the counter (`i`), the condition, and the increment step. Best for raw numbers.
2. **The `while` loop**: A loop that has no internal counter. It simply runs forever as long as a boolean condition remains true. (Dangerous if you forget to break!)
3. **The `for...of` loop**: Introduced in ES6. The absolute best way to iterate over Arrays, Strings, and Maps. It handles the counting and length checking automatically!
4. **The `for...in` loop**: Specifically designed to iterate over the "keys" (properties) of a JavaScript Object.

## CODE

### Level 1 — Beginner (The Classic For Loop & While Loop)
```javascript
// 1. THE CLASSIC FOR LOOP
// Syntax: for (Initialization; Condition; Step)
// Notice we MUST use 'let', because 'i' is reassigned every loop!
for (let i = 1; i <= 5; i++) {
  console.log(`Executing loop number: ${i}`);
}

// 2. THE WHILE LOOP
// Use this when you DO NOT know how many times the loop needs to run!
let powerLevel = 10;

while (powerLevel > 0) {
  console.log(`Power is at ${powerLevel}. Draining...`);
  powerLevel -= 3; // If you forget this line, the loop runs FOREVER and crashes the browser!
}
console.log("Battery Dead.");
```

### Level 2 — Building Blocks (Iterating Arrays with for...of)
```javascript
const frameworkList = ["React", "Angular", "Vue", "Svelte"];

// THE OLD, BAD WAY (Classic For Loop):
// You have to manually track the array length and the [i] index!
for (let i = 0; i < frameworkList.length; i++) {
  console.log("Old way:", frameworkList[i]);
}

// THE MODERN ES6 WAY (for...of):
// It automatically extracts the exact value into the 'framework' variable!
// It automatically stops when the array is finished.
for (const framework of frameworkList) {
  console.log("Modern way:", framework);
}

// Note: You can use 'const' here because a BRAND NEW 'framework' variable 
// is mathematically created in memory for every single iteration of the loop!
```

### Level 3 — Intermediate (Iterating Objects with for...in)
```javascript
const userProfile = {
  username: "Alice",
  role: "Admin",
  level: 42
};

// You CANNOT use 'for...of' on Objects! Objects are not mathematically "iterable".
// Instead, you use 'for...in' to grab the string KEYS of the object!

for (const key in userProfile) {
  // We get the key (e.g., "username")
  // We use Bracket Notation [] to get the actual value!
  const value = userProfile[key];
  
  console.log(`Key: ${key} | Value: ${value}`);
}

// Output:
// Key: username | Value: Alice
// Key: role | Value: Admin
// Key: level | Value: 42
```

### Level 4 — Advanced (Break and Continue)
```javascript
// Sometimes you need to forcefully manipulate a loop while it is running.

const databaseRecords = [100, 200, "CORRUPT_DATA", 400, 500];

// 1. BREAK: Immediately DESTROYS the loop entirely.
for (const record of databaseRecords) {
  if (record === "CORRUPT_DATA") {
    console.error("Critical error found! Stopping process entirely!");
    break; // The loop instantly dies. 400 and 500 will NEVER be checked.
  }
  console.log("Processing record:", record);
}

const networkPings = [10, 500, 12, 14];

// 2. CONTINUE: Skips the CURRENT iteration and jumps to the next one!
for (const ping of networkPings) {
  if (ping > 100) {
    console.warn("Ping too high, skipping this packet!");
    continue; // Instantly jumps back to the top of the loop! The log below is ignored.
  }
  console.log("Saving healthy ping:", ping); // Only 10, 12, and 14 get saved!
}
```

## REAL_WORLD
While loops are foundational, in modern Enterprise JavaScript and React development, developers almost *never* use `for`, `for...of`, or `while` loops! 

Why? Because traditional loops are **Imperative** (they tell the computer *how* to iterate). Modern JS heavily favors **Declarative / Functional** programming, where you tell the computer *what* you want.

Instead of writing a `for...of` loop to iterate an array, developers use built-in Array Higher-Order Methods like `.map()`, `.filter()`, and `.forEach()`. 

```jsx
// Real React Code:
// You CANNOT put a 'for' loop inside React JSX HTML! 
// You MUST use the Array .map() function to loop over data!
function UserList({ users }) {
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

## INTERVIEW
**Q: Can you use a `for...of` loop on an Object?**
A: No. A `for...of` loop requires the target data structure to implement the `[Symbol.iterator]` interface. Arrays, Strings, Maps, and Sets implement this interface natively. Plain Objects `{}` do not. If you attempt to `for...of` over an object, V8 will throw a `TypeError: obj is not iterable`. You must use `for...in` instead, or convert the object keys into an array using `Object.keys(obj)`.

**Q: What causes an Infinite Loop, and what happens to the browser?**
A: An infinite loop occurs when the condition in a `for` or `while` loop never evaluates to `false`. Because JavaScript is strictly single-threaded, the V8 engine becomes 100% permanently occupied running the loop. The browser's Event Loop is blocked, preventing the UI from rendering, responding to clicks, or closing tabs. The page entirely freezes ("locks up") until the browser's watchdog timer forcefully kills the tab with an "Aw, Snap" error.

**Q: When would you use a `while` loop over a `for` loop?**
A: You use a `for` loop when you know the exact boundary or length of the data before the loop starts (e.g., an array of 50 items). You use a `while` loop when you have absolutely no idea how many iterations are required, and the termination depends on an unpredictable external event (e.g., `while (database.isConnected() === false) { retry() }`).
