## WHY
If you are building an eCommerce store, you don't have one product; you have 10,000 products. You cannot create 10,000 individual variables (`const product1`, `const product2`). 

An **Array** is a data structure that holds a list of items in a specific, ordered sequence. In JavaScript, Arrays are incredibly powerful and flexible. They come with a massive built-in API of "Higher-Order Methods" (like `.map`, `.filter`, and `.reduce`) that allow you to transform, search, and aggregate massive lists of data with just a single line of functional code. Mastering Array Methods is the single most important skill for a frontend data engineer.

## THEORY
### The Nature of JS Arrays
In languages like Java or C, Arrays are rigid. You must specify their exact length when you create them, and they can only hold one type of data (e.g., an Array of solely Integers).

JavaScript Arrays are completely dynamic.
1. They automatically grow and shrink in memory.
2. They can hold mixed data types simultaneously (`[1, "Apple", true, { id: 5 }]`).
3. Internally, JS Arrays are actually just standard Objects with numbered keys (`{ "0": "Apple", "1": "Banana" }`) and a special `.length` property!

### Mutating vs Non-Mutating Methods
- **Mutating Methods**: Methods that permanently physically alter the original array in memory (e.g., `.push()`, `.pop()`, `.sort()`).
- **Non-Mutating Methods**: Methods that leave the original array completely untouched and return a *Brand New* array in memory (e.g., `.map()`, `.filter()`). **(Industry Standard for React)**

## CODE

### Level 1 — Beginner (Creating and Accessing)
```javascript
// 1. Creating an Array
const fruits = ["Apple", "Banana", "Cherry"];

// 2. Accessing by Index (Zero-Indexed!)
console.log(fruits[0]); // "Apple"
console.log(fruits[2]); // "Cherry"
console.log(fruits[99]); // undefined (Does not crash!)

// 3. Basic Mutating Methods
fruits.push("Date");    // Adds to the END
fruits.unshift("Kiwi"); // Adds to the START (Slow! Shifts all other elements)

console.log(fruits); // ["Kiwi", "Apple", "Banana", "Cherry", "Date"]

const lastFruit = fruits.pop();   // Removes and returns the LAST item ("Date")
const firstFruit = fruits.shift(); // Removes and returns the FIRST item ("Kiwi")
```

### Level 2 — Building Blocks (Searching and Finding)
```javascript
const users = [
  { id: 1, name: "Alice", active: true },
  { id: 2, name: "Bob", active: false },
  { id: 3, name: "Charlie", active: true }
];

// 1. .includes() (Great for arrays of primitives)
const numbers = [10, 20, 30];
console.log(numbers.includes(20)); // true

// 2. .find() (Finds the FIRST object that matches a condition)
// It accepts a callback function and returns the actual object!
const bob = users.find(user => user.name === "Bob");
console.log(bob); // { id: 2, name: "Bob", active: false }

// 3. .findIndex() (Returns the index number instead of the object)
const index = users.findIndex(user => user.id === 3);
console.log(index); // 2
```

### Level 3 — Intermediate (The Holy Trinity: Map & Filter)
```javascript
// These are Non-Mutating Higher-Order Functions. 
// They are the absolute backbone of functional JavaScript!

const products = [
  { name: "Laptop", price: 1000 },
  { name: "Mouse", price: 25 },
  { name: "Keyboard", price: 75 }
];

// 1. .filter()
// Returns a BRAND NEW array containing only items where the callback returned 'true'.
const cheapProducts = products.filter(product => product.price < 100);

// Original 'products' is untouched!
console.log(cheapProducts.length); // 2 (Mouse and Keyboard)

// 2. .map()
// Returns a BRAND NEW array where every item has been transformed by the callback!
// Example: We want an array of JUST the names!
const productNames = products.map(product => product.name);

console.log(productNames); // ["Laptop", "Mouse", "Keyboard"]

// 3. Chaining! (The true power)
// Because filter() returns an array, we can instantly call map() on it!
const cheapNames = products
  .filter(p => p.price < 100)
  .map(p => p.name.toUpperCase());

console.log(cheapNames); // ["MOUSE", "KEYBOARD"]
```

### Level 4 — Advanced (Reduce)
```javascript
// .reduce() is the most powerful (and most confusing) Array method.
// It takes an entire array and crushes it down into a SINGLE value 
// (a number, a string, or even a single massive object).

const cart = [
  { id: 1, price: 10, qty: 2 },
  { id: 2, price: 50, qty: 1 }
];

// Syntax: array.reduce((accumulator, currentItem) => { return newAccumulator }, initialValue)

const totalCost = cart.reduce((total, item) => {
  return total + (item.price * item.qty);
}, 0); // 0 is the starting value for 'total'

console.log(totalCost); // (10 * 2) + (50 * 1) = 70

// Reduce can also be used to convert an Array into an Object!
const userRoles = ["admin", "editor", "viewer"];

const rolesObject = userRoles.reduce((obj, role) => {
  obj[role] = true; // Add the key to the object
  return obj;
}, {}); // Start with an empty object {}

console.log(rolesObject); // { admin: true, editor: true, viewer: true }
```

## REAL_WORLD
In **React**, `.map()` and `.filter()` are used in almost every single component. 
Because React requires you to treat state as **Immutable** (unchangeable), you are mathematically forbidden from using `.push()`, `.pop()`, or `.splice()`. 

If you use `array.push()`, React will not detect the change and your UI will not update!
If you want to delete a user from a list in React, you do not use `.splice()`. You use `.filter()` to create a brand new array that excludes the user, and pass that new array into the state. If you want to render a list of 100 HTML `<li>` tags, you use `.map()` to iterate over the JSON data and return HTML nodes.

Mastering immutable array methods is the absolute prerequisite to learning modern frontend frameworks.

## INTERVIEW
**Q: What is the difference between `.forEach()` and `.map()`?**
A: `.map()` returns a brand new array containing the transformed items. It is used to generate new data. `.forEach()` returns absolutely nothing (`undefined`). It simply executes a callback function for every item. `.forEach()` is used purely for "Side Effects" (like saving data to a database or manipulating an external variable), while `.map()` is used for pure data transformation.

**Q: Does the `.sort()` method mutate the original array?**
A: Yes! This is a massive "Gotcha" in JavaScript. `array.sort()` permanently alters the original array in memory. If you want to sort an array safely without mutating it, you must copy it first: `const sorted = [...originalArray].sort()`. (Note: ES2023 introduced `.toSorted()` which does this automatically!).

**Q: How do you quickly empty an array?**
A: There are two common ways. The best way is simply reassigning it to a new empty array: `myArray = []` (requires the array to be declared with `let`). The "hack" way to empty a `const` array is to mutate its length directly: `myArray.length = 0;`. This forcefully deletes all elements inside it.

## FEYNMAN CHECK

### Explain JS Arrays Like I'm 10 Years Old
> JS arrays are special OBJECTS where the keys are numeric strings ('0', '1', '2') and there's a magic `length` property. They are NOT typed (unlike C arrays); a single array can hold numbers, strings, objects, functions mixed. The 30+ array methods divide into two camps: MUTATING (`push`, `pop`, `splice`, `sort`, `reverse`) and NON-MUTATING (`map`, `filter`, `reduce`, `slice`, `concat`, `flatMap`, the new `toSorted`, `toReversed`, `toSpliced`, `with`). Modern functional code prefers non-mutating. The non-obvious detail: `arr.sort()` is in-place AND defaults to LEXICOGRAPHIC string comparison — so `[10,1,2].sort()` returns `[1,10,2]` (sorted as strings). Always pass a comparator: `arr.sort((a,b) => a - b)`.

### 5 Deep Conceptual Questions

**Q1: When do you choose forEach vs for vs for-of vs map/filter/reduce?**
> **A:** Use `map/filter/reduce` when TRANSFORMING data into new arrays (declarative, composable). Use `for-of` when iterating with side effects or needing `break`/`continue`/early return. Use classic `for (let i = 0; ...)` only when you need the index AND mutating the array AND it's a perf-critical hot loop. `forEach` is rarely the right choice — it's the worst of both worlds (no return value, no break, harder to reason about than map/for-of).

**Q2: Mental model for reduce?**
> **A:** Reduce is a FOLD — it walks the array maintaining an ACCUMULATOR, applying a binary op to (acc, currentItem) → newAcc. Initial value matters: `reduce((a,b) => a+b)` on `[]` throws; `reduce((a,b) => a+b, 0)` returns 0. Almost every other array method can be implemented via reduce — map, filter, find, even forEach. But explicit map/filter is usually clearer; reach for reduce when reducing to a single value or grouping (groupBy implementations).

**Q3: Most dangerous misconception?**
> **A:** Mutating methods return the array (so chaining isn't always safe):
> ```js
> // ❌ sort() MUTATES the original AND returns it
> const arr = [3, 1, 2];
> const sorted = arr.sort();
> arr.push(99);
> console.log(sorted);   // [1, 2, 3, 99]  — sorted is the SAME array as arr!
>
> // ✅ Use toSorted (ES2023) or copy first
> const arr = [3, 1, 2];
> const sorted = arr.toSorted();          // new array
> // Or: const sorted = [...arr].sort();
> ```

**Q4: How do array holes interact with array methods?**
> **A:** Arrays can have HOLES: `const a = [1, , 3]` (notice the missing middle). Some methods SKIP holes (map, filter, forEach, reduce); others FILL them with undefined (Array.from, spread); others treat holes as undefined (find, includes). `[1, , 3].map(x => x*2)` gives `[2, <empty>, 6]` (hole preserved). Sparse arrays are a footgun — prefer dense arrays or use `Array(n).fill(undefined)`.

**Q5: FAANG-grade definition?**
> **A:** "JavaScript arrays are dense or sparse heterogeneous indexed collections — special objects with a numeric-string key space and an auto-managed length property — supporting both mutating in-place operations (push, pop, sort, splice, reverse) and non-mutating transformations (map, filter, reduce, slice, concat, flatMap), augmented in ES2023 by toSorted/toReversed/toSpliced/with for immutable variants — backed by V8's PACKED_SMI / PACKED / HOLEY internal element-kinds for performance."

```js
// functional.js
export function map(arr, fn) {
  const out = new Array(arr.length);   // pre-allocate for perf
  for (let i = 0; i < arr.length; i++) {
    if (i in arr) out[i] = fn(arr[i], i, arr);   // 'in' check respects holes
  }
  return out;
}

export function filter(arr, predicate) {
  const out = [];
  for (let i = 0; i < arr.length; i++) {
    if (i in arr && predicate(arr[i], i, arr)) out.push(arr[i]);
  }
  return out;
}

export function reduce(arr, reducer, initial) {
  let acc = initial;
  let start = 0;
  if (arguments.length < 3) {
    if (arr.length === 0) throw new TypeError('Reduce of empty array with no initial value');
    acc = arr[0];
    start = 1;
  }
  for (let i = start; i < arr.length; i++) {
    if (i in arr) acc = reducer(acc, arr[i], i, arr);
  }
  return acc;
}
```

#### Step 3 — Build groupBy and partition On Top
```js
// higher.js
import { reduce } from './functional.js';

// groupBy(arr, keyFn) -> { key: [items] }
export function groupBy(arr, keyFn) {
  return reduce(arr, (acc, item) => {
    const key = keyFn(item);
    (acc[key] ??= []).push(item);
    return acc;
  }, {});
}

// partition(arr, predicate) -> [matching, nonMatching]
export function partition(arr, predicate) {
  return reduce(arr, (acc, item) => {
    acc[predicate(item) ? 0 : 1].push(item);
    return acc;
  }, [[], []]);
}

// Demo
const users = [
  { name: 'Ana',  role: 'admin', age: 30 },
  { name: 'Bob',  role: 'user',  age: 25 },
  { name: 'Eve',  role: 'admin', age: 35 },
  { name: 'Tom',  role: 'user',  age: 40 },
];

console.log(groupBy(users, u => u.role));
// { admin: [Ana, Eve], user: [Bob, Tom] }

const [adults, minors] = partition(users, u => u.age >= 30);
console.log('adults:', adults.length, 'minors:', minors.length);
```

#### Step 4 — Error Handling: Empty Array + Holes
```js
// Edge cases
import { reduce, map, filter } from './functional.js';

try { reduce([], (a, b) => a + b); } catch (e) { console.log('OK:', e.message); }
```
